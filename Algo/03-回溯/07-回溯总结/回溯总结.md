# 回溯总结

> 回溯 = 暴力搜索 + 剪枝。掌握 4 种模型的区别和模板是核心。

---

## 1. 四种回溯模型对比

```
┌───────────┬───────────────────┬────────────────────┬──────────────────┐
│ 类型      │ 递归含义          │ 循环范围            │ 去重方式          │
├───────────┼───────────────────┼────────────────────┼──────────────────┤
│ 子集型    │ 每个元素选或不选  │ 从 i 开始 或        │ 排序+i>start     │
│           │                   │ 选/不选两个分支     │ &&同值→skip      │
├───────────┼───────────────────┼────────────────────┼──────────────────┤
│ 组合型    │ 从i开始选k个      │ for j in           │ 排序+j>i         │
│           │                   │ range(i, n)        │ &&同值→skip      │
├───────────┼───────────────────┼────────────────────┼──────────────────┤
│ 排列型    │ 每次选未选过的    │ for i in           │ 排序+not         │
│           │                   │ range(0, n)        │ used[i-1]→skip   │
├───────────┼───────────────────┼────────────────────┼──────────────────┤
│ 划分型    │ 在间隙处切割      │ for end in         │ 不常需去重        │
│           │                   │ range(start, n)    │ 取决于题目        │
└───────────┴───────────────────┴────────────────────┴──────────────────┘
```

## 2. 统一模板框架

```python
def backtrack(输入):
    res = []
    
    def dfs(状态参数):
        # 1. 终止条件
        if 满足条件:
            res.append(当前答案的拷贝)  # ⚠️ 必须拷贝 path[:]
            return
        
        # 2. 枚举选择
        for 选择 in 选择列表:
            # 3. 剪枝
            if 不合法:
                continue  # 或 break
            
            # 4. 做选择
            做选择(修改 path/used/状态)
            
            # 5. 递归
            dfs(新状态)
            
            # 6. 撤销选择
            撤销(恢复 path/used/状态)  # ⚠️ 与"做选择"完全对称
    
    dfs(初始状态)
    return res

# ⚠️ "做选择"和"撤销选择"必须一一对应:
#   path.append ↔ path.pop
#   used[i]=True ↔ used[i]=False
#   cols.add ↔ cols.remove
#   board[r][c]='#' ↔ board[r][c]=tmp
```

## 3. 递归树的两种视角

```
视角一: 选或不选 (输入视角)
  站在每个元素的角度: "我被选了" vs "我没被选"
  适合子集问题
  
  [1,2,3] 子集:
         root
        /    \
     选1     不选1
     / \      / \
   选2 不选2 选2 不选2
   ...

视角二: 选哪个 (答案视角)
  站在答案的每个位置: "这个位置选谁"
  适合组合/排列/划分问题
  
  [1,2,3] 排列:
         root
       / | \
      1  2  3     ← 第 1 个位置
     / \  ...     ← 第 2 个位置
    2   3
    |   |
    3   2         ← 第 3 个位置

⚠️ 两种视角经常可以互换
   子集问题用视角二也行 (枚举选哪些)
   排列问题很少用视角一 (不方便)
```

## 4. 去重方法速查

```
⚠️ 含重复元素时必须先排序! 否则去重逻辑失效

子集/组合去重:
  if i > start and nums[i] == nums[i-1]: continue
  含义: 同一层(同一递归层), 相同值只选第一个

排列去重:
  if i > 0 and nums[i] == nums[i-1] and not used[i-1]: continue
  含义: 相同值按原始顺序使用, 前一个没用就不能用后一个

⚠️ 区分:
  子集去重看 i > start → 同层
  排列去重看 not used[i-1] → 跨层(保持相对顺序)
```

## 5. 时间复杂度速查

```
┌─────────────┬────────────────┬────────────┐
│ 问题类型    │ 时间复杂度      │ 说明        │
├─────────────┼────────────────┼────────────┤
│ 子集        │ O(n × 2^n)     │ 2^n 个子集  │
│ 组合 C(n,k) │ O(C(n,k) × k) │ 每种方案花k │
│ 排列        │ O(n × n!)      │ n! 种排列   │
│ N皇后       │ O(n!)          │ 每行选一列  │
│ 数独        │ O(9^空格数)    │ 每格试9个数 │
│ 网格搜索    │ O(m×n×3^L)     │ L=路径长度  │
└─────────────┴────────────────┴────────────┘

⚠️ 剪枝不改变最坏时间复杂度, 但平均性能好很多
```

## 6. 常见易错点

```
❌ path 没拷贝
  错: res.append(path)      ← path 后续还会被修改!
  对: res.append(path[:])   ← 拷贝一份

❌ 回溯不对称
  做了 path.append + used[i]=True
  但只 pop 没 reset used
  → 后续递归状态错误

❌ 组合用了 range(0,n) 导致重复
  组合: range(start, n)  ← 只往后选
  排列: range(0, n)      ← 都可以选

❌ 去重条件写反
  子集: if i > start and ...   ← start, 不是 0
  排列: if not used[i-1] ...   ← not, 不是没有 not

❌ 划分型 dfs(end) 而非 dfs(end+1)
  dfs(end)   → 死循环(永远从同一位置开始)
  dfs(end+1) → 正确, 下一段从 end+1 开始

❌ break 和 continue 搞混
  排序后当前值太大 → 后面更大 → break
  当前不合法但后面可能合法 → continue
```

## 本节要点速查

```
✅ 4 种模型: 子集(选不选)/组合(从i选k)/排列(选谁)/划分(切哪)
✅ 撤销选择必须和做选择完全对称
✅ 含重复元素必须排序后去重
✅ path[:] 拷贝, 不能直接 append(path)
✅ 划分型 dfs(end+1), 不是 dfs(end)
✅ 排序剪枝用 break, 单个跳过用 continue
✅ 子集 O(2^n), 排列 O(n!), 组合 O(C(n,k))
```
