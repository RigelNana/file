# 组合型回溯

> 组合 = 子集 + 限定长度。在子集回溯的基础上加上长度约束和剪枝。

---

## 1. 组合

### LeetCode 77

```
输入: n=4, k=2
输出: [[1,2],[1,3],[1,4],[2,3],[2,4],[3,4]]

组合 vs 子集: 组合只要长度=k 的子集
```

```python
def combine(n, k):
    res = []
    
    def dfs(i, path):
        if len(path) == k:           # ⚠️ 长度到 k 就收集
            res.append(path[:])
            return                    # ⚠️ 到 k 就停, 不继续
        for j in range(i, n + 1):    # ⚠️ 1 到 n, 包含 n
            path.append(j)
            dfs(j + 1, path)
            path.pop()
    
    dfs(1, [])
    return res

# ⚠️ range(i, n+1): 题目说从 1 到 n, 是 1-indexed
# ⚠️ len(path) == k 时 return: 不继续选了

# ━━━━ 剪枝优化 ━━━━
def combine_pruned(n, k):
    res = []
    
    def dfs(i, path):
        d = k - len(path)            # 还需要选 d 个
        if d == 0:
            res.append(path[:])
            return
        # ⚠️ 剪枝: 剩余元素不够 d 个 → 不用继续
        # 从 j 到 n 有 n-j+1 个元素
        # 需要 n-j+1 >= d → j <= n-d+1
        for j in range(i, n - d + 2):  # ⚠️ n-d+2 而不是 n+1
            path.append(j)
            dfs(j + 1, path)
            path.pop()
    
    dfs(1, [])
    return res

# ⚠️ 剪枝公式推导:
#   当前还需要选 d = k - len(path) 个
#   从 j 开始到 n, 可选数量 = n - j + 1
#   需要 n - j + 1 >= d → j <= n - d + 1
#   所以 range 的上界改为 n - d + 2 (因为 range 不含右端点)
```

## 2. 组合总和（可重复选）

### LeetCode 39

```
输入: candidates=[2,3,6,7], target=7
输出: [[2,2,3],[7]]
每个数字可以重复使用

关键区别: dfs(j, ...) 而不是 dfs(j+1, ...)
  因为同一个元素可以再选一次
```

```python
def combinationSum(candidates, target):
    res = []
    candidates.sort()                 # ⚠️ 排序便于剪枝
    
    def dfs(i, path, remain):
        if remain == 0:               # ⚠️ 减到 0 → 找到一个组合
            res.append(path[:])
            return
        for j in range(i, len(candidates)):
            if candidates[j] > remain:  # ⚠️ 剪枝: 排序后, 后面的更大, 都不行
                break                    # ⚠️ break, 不是 continue!
            path.append(candidates[j])
            dfs(j, path, remain - candidates[j])  # ⚠️ j 不是 j+1!
            path.pop()
    
    dfs(0, [], target)
    return res

# ⚠️ dfs(j,...) vs dfs(j+1,...):
#   dfs(j):   可以重复选当前元素 (LC 39)
#   dfs(j+1): 每个元素只能选一次 (LC 40, 子集)
#
# ⚠️ break vs continue:
#   排序后, candidates[j] 太大 → 后面的也太大 → break 全部跳过
#   如果没排序, 只能 continue (不能排除后面更小的)
```

## 3. 组合总和（不可重复 + 有重复元素）

### LeetCode 40

```
输入: [10,1,2,7,6,1,5], target=8
输出: [[1,1,6],[1,2,5],[1,7],[2,6]]

每个数字只能用一次, 但数组中有重复元素
和子集 II 一样的去重方式: 排序 + 同层跳过
```

```python
def combinationSum2(candidates, target):
    candidates.sort()                 # ⚠️ 必须排序
    res = []
    
    def dfs(i, path, remain):
        if remain == 0:
            res.append(path[:])
            return
        for j in range(i, len(candidates)):
            if candidates[j] > remain:
                break
            if j > i and candidates[j] == candidates[j-1]:  # ⚠️ 同层去重
                continue
            path.append(candidates[j])
            dfs(j + 1, path, remain - candidates[j])  # ⚠️ j+1, 不可重复
            path.pop()
    
    dfs(0, [], target)
    return res

# ⚠️ 和 LC 39 的区别:
#   1. dfs(j+1): 不可重复选
#   2. 去重: j > i and nums[j] == nums[j-1] → continue
#   3. 排序 + 去重组合使用
```

## 4. 组合总和 III

### LeetCode 216

```
输入: k=3, n=7
输出: [[1,2,4]]
使用 1-9 的数字, 每个最多用一次, 选 k 个, 和为 n
```

```python
def combinationSum3(k, n):
    res = []
    
    def dfs(i, path, remain):
        if len(path) == k:
            if remain == 0:
                res.append(path[:])
            return                    # ⚠️ 不管是否为0都return, 长度到了
        for j in range(i, 10):        # ⚠️ 1到9
            if j > remain:            # 剪枝
                break
            path.append(j)
            dfs(j + 1, path, remain - j)
            path.pop()
    
    dfs(1, [], n)
    return res
```

## 本节要点速查

```
✅ 组合 = 子集 + len(path)==k 才收集
✅ 可重选: dfs(j); 不可重选: dfs(j+1)
✅ 有重复元素: 排序 + j>i && nums[j]==nums[j-1] 跳过
✅ 剪枝(长度): 剩余元素不够 → 缩小循环范围
✅ 剪枝(值): candidates[j] > remain → break
✅ target 递减法: remain -= nums[j], 到0收集
```
