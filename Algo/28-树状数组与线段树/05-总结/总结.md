# 树状数组与线段树总结

> BIT 和线段树的对比、选型策略和常见变种。

---

## 1. 数据结构对比

```
┌─────────────────┬──────────┬──────────┬──────────┬──────────┐
│ 数据结构        │ 单点改   │ 区间改   │ 区间查   │ 实现难度 │
├─────────────────┼──────────┼──────────┼──────────┼──────────┤
│ 前缀和          │ O(n)     │ O(n)     │ O(1)     │ ⭐       │
│ 差分            │ O(1)     │ O(1)     │ O(n)     │ ⭐       │
│ 树状数组 BIT    │ O(logn)  │ O(logn)* │ O(logn)  │ ⭐⭐     │
│ 线段树          │ O(logn)  │ O(logn)  │ O(logn)  │ ⭐⭐⭐   │
│ 动态开点线段树  │ O(logn)  │ O(logn)  │ O(logn)  │ ⭐⭐⭐⭐ │
└─────────────────┴──────────┴──────────┴──────────┴──────────┘
  * BIT 区间修改需要差分 BIT (双 BIT)

选型策略:
  只查前缀和 → 前缀和数组
  只做区间修改, 最后一次性查 → 差分
  单点改 + 区间查 → BIT (最简洁)
  区间改 + 区间查 → 线段树 (最通用)
  值域巨大 + 稀疏操作 → 动态开点线段树

  ⚠️ 能用 BIT 就用 BIT, 比线段树简洁得多
     BIT 不能做的: 区间最值 + 区间修改
```

---

## 2. BIT vs 线段树

```
BIT 能做的:
  ✅ 单点修改 + 前缀和查询
  ✅ 单点修改 + 区间和查询
  ✅ 区间修改 + 单点查询 (差分)
  ✅ 区间修改 + 区间查询 (双BIT)
  ✅ 逆序对 / 右侧小于计数
  ✅ 第 k 小 (利用树结构)

BIT 不能做的 (需要线段树):
  ❌ 区间最值 + 区间修改
  ❌ 区间赋值 (而非区间加)
  ❌ 维护复杂信息 (最长连续1, 最大子段和等)
  ❌ 线段树合并
  
线段树可以维护的信息:
  - 区间和 / 区间积
  - 区间最大值 / 最小值
  - 区间 GCD
  - 最长连续相同字符
  - 最大子段和
  - 历史最值
```

## 3. 模板速查

```python
# ─── BIT ───
class BIT:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)
    def update(self, i, d):
        while i <= self.n:
            self.tree[i] += d
            i += i & (-i)
    def query(self, i):
        s = 0
        while i > 0:
            s += self.tree[i]
            i -= i & (-i)
        return s

# ─── 线段树 (区间加+区间和) ───
class SegTree:
    def __init__(self, n):
        self.n = n
        self.t = [0]*(4*n)
        self.lz = [0]*(4*n)
    def _up(self, o):
        self.t[o] = self.t[2*o] + self.t[2*o+1]
    def _down(self, o, l, r):
        if self.lz[o]:
            m = (l+r)//2
            v = self.lz[o]
            self.t[2*o] += v*(m-l+1)
            self.lz[2*o] += v
            self.t[2*o+1] += v*(r-m)
            self.lz[2*o+1] += v
            self.lz[o] = 0
    def update(self, ql, qr, v, o=1, l=1, r=None):
        if r is None: r = self.n
        if ql<=l and r<=qr:
            self.t[o] += v*(r-l+1)
            self.lz[o] += v
            return
        self._down(o, l, r)
        m = (l+r)//2
        if ql<=m: self.update(ql,qr,v,2*o,l,m)
        if qr>m: self.update(ql,qr,v,2*o+1,m+1,r)
        self._up(o)
    def query(self, ql, qr, o=1, l=1, r=None):
        if r is None: r = self.n
        if ql<=l and r<=qr: return self.t[o]
        self._down(o, l, r)
        m = (l+r)//2
        s = 0
        if ql<=m: s += self.query(ql,qr,2*o,l,m)
        if qr>m: s += self.query(ql,qr,2*o+1,m+1,r)
        return s

# ─── 动态开点 ───
class DynSeg:
    def __init__(self):
        self.t = {}
        self.lz = {}
    def update(self, ql, qr, v, o=1, l=1, r=10**9):
        if ql<=l and r<=qr:
            self.t[o] = self.t.get(o,0)+v*(r-l+1)
            self.lz[o] = self.lz.get(o,0)+v
            return
        m = (l+r)//2
        self._down(o,l,r,m)
        if ql<=m: self.update(ql,qr,v,2*o,l,m)
        if qr>m: self.update(ql,qr,v,2*o+1,m+1,r)
        self.t[o] = self.t.get(2*o,0)+self.t.get(2*o+1,0)
    def query(self, ql, qr, o=1, l=1, r=10**9):
        if ql<=l and r<=qr: return self.t.get(o,0)
        m = (l+r)//2
        self._down(o,l,r,m)
        s = 0
        if ql<=m: s += self.query(ql,qr,2*o,l,m)
        if qr>m: s += self.query(ql,qr,2*o+1,m+1,r)
        return s
    def _down(self, o, l, r, m):
        v = self.lz.get(o,0)
        if v:
            self.t[2*o]=self.t.get(2*o,0)+v*(m-l+1)
            self.lz[2*o]=self.lz.get(2*o,0)+v
            self.t[2*o+1]=self.t.get(2*o+1,0)+v*(r-m)
            self.lz[2*o+1]=self.lz.get(2*o+1,0)+v
            self.lz[o]=0
```

## 4. 常见易错点

```
⚠️ BIT:
  - 1-indexed! 下标从 1 开始
  - update 中 i += lowbit(i), 不是 -= 
  - query 中 i -= lowbit(i)
  - 赋值要算差值 delta = new - old

⚠️ 线段树:
  - 数组开 4*n
  - push_down 在递归之前
  - push_up 在递归之后
  - ql <= mid 和 qr > mid 不能用 elif
  - lazy 下传后必须清零
  - 区间长度: r - l + 1 (l 和 r 都是闭区间)

⚠️ 动态开点:
  - .get(key, 0) 防止 KeyError
  - 值域 r 的默认值要足够大
```

## 本节要点速查

```
✅ 能用 BIT 就用 BIT (简洁)
✅ 区间最值 + 区间改 → 必须线段树
✅ 值域大稀疏操作 → 动态开点
✅ BIT: update加lowbit, query减lowbit
✅ 线段树: push_down先, push_up后, lazy清零
✅ 离散化 + 普通线段树 (离线, 更快)
✅ 动态开点 (在线, 更灵活)
```
