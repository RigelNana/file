# 最长递增子序列 (LIS)

> LIS 是线性 DP 中最经典的问题之一。O(n²) DP 和 O(n log n) 贪心+二分两种解法都要掌握。

---

## 1. 问题与可视化

```
问题: 找最长的 严格递增 子序列 (不要求连续)

  nums = [10, 9, 2, 5, 3, 7, 101, 18]
  
  LIS = [2, 3, 7, 101] 或 [2, 5, 7, 101]
  长度 = 4

  10  9  2  5  3  7  101  18
        ↗  ↗     ↗   ↗
      [2,  5,    7, 101]

⚠️ 严格递增: < 不是 ≤
   如果题目要非严格递增 (≤)，条件改为 nums[j] <= nums[i]
```

---

## 2. O(n²) DP 解法

```
状态: dp[i] = 以 nums[i] 结尾的 LIS 长度

转移: dp[i] = max(dp[j] + 1)  for all j < i where nums[j] < nums[i]
     如果没有这样的 j，dp[i] = 1 (自身)

初始化: dp[i] = 1 (每个元素自身就是长度 1 的递增子序列)

答案: max(dp)  ← 不是 dp[n-1]!

⚠️ 答案是 max(dp)，不是 dp[-1]
   因为最长子序列不一定以最后一个元素结尾

模拟: [10, 9, 2, 5, 3, 7, 101, 18]
  i=0 (10): 没有 j < 0 → dp[0] = 1
  i=1 (9):  没有 nums[j] < 9 → dp[1] = 1
  i=2 (2):  没有 nums[j] < 2 → dp[2] = 1
  i=3 (5):  nums[2]=2 < 5 → dp[3] = dp[2]+1 = 2
  i=4 (3):  nums[2]=2 < 3 → dp[4] = dp[2]+1 = 2
  i=5 (7):  nums[2]=2,nums[3]=5,nums[4]=3 < 7
            → max(dp[2],dp[3],dp[4])+1 = 2+1 = 3
  i=6 (101): 所有都 < 101 → max(dp[5])+1 = 4
  i=7 (18):  nums[2..5] < 18 → max(dp[5])+1 = 4
  
  dp = [1, 1, 1, 2, 2, 3, 4, 4]
  答案 = max(dp) = 4
```

```python
def lengthOfLIS(nums):
    n = len(nums)
    dp = [1] * n       # ⚠️ 初始化为 1，不是 0
    
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:   # ⚠️ 严格 <，不是 <=
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)     # ⚠️ 取全局 max，不是 dp[-1]

# 时间 O(n²), 空间 O(n)

# ⚠️ 三个常见错误:
#   ① dp 初始化为 0 → 应该是 1 (自身)
#   ② 条件 <= → 应该是 < (严格递增)
#   ③ return dp[-1] → 应该是 max(dp)
```

## 3. O(n log n) 贪心 + 二分 ⭐

```
思路: 维护一个 tails 数组
     tails[i] = 长度为 i+1 的所有递增子序列中，最小的末尾元素

性质: tails 始终是严格递增的 → 可以二分查找

对每个 nums[x]:
  如果 nums[x] > tails 最后一个 → append (LIS 延长)
  否则 → 二分找到第一个 ≥ nums[x] 的位置，替换

模拟: [10, 9, 2, 5, 3, 7, 101, 18]
  x=10:  tails = [10]
  x=9:   9 < 10 → 替换 tails[0] = 9    → [9]
  x=2:   2 < 9  → 替换 tails[0] = 2    → [2]
  x=5:   5 > 2  → append                → [2, 5]
  x=3:   3 > 2, 3 < 5 → 替换 tails[1]  → [2, 3]
  x=7:   7 > 3  → append                → [2, 3, 7]
  x=101: 101>7  → append                → [2, 3, 7, 101]
  x=18:  18>7, 18<101 → 替换 tails[3]   → [2, 3, 7, 18]
  
  len(tails) = 4

⚠️ tails 不是实际的 LIS！它的长度等于 LIS 长度，但元素不一定是
   例如上面 tails=[2,3,7,18]，但实际 LIS 可以是 [2,5,7,101]
```

```python
import bisect

def lengthOfLIS_fast(nums):
    tails = []
    for x in nums:
        # ⚠️ bisect_left: 找第一个 ≥ x 的位置
        pos = bisect.bisect_left(tails, x)
        if pos == len(tails):
            tails.append(x)    # 比所有都大 → 延长
        else:
            tails[pos] = x     # 替换 → 更小的末尾更优
    return len(tails)

# 时间 O(n log n), 空间 O(n)

# ⚠️ bisect_left vs bisect_right:
#    bisect_left:  严格递增 (<)    → 遇到相等替换
#    bisect_right: 非严格递增 (≤)  → 遇到相等跳过
```

## 4. 输出 LIS 本身 (回溯路径)

```python
def getLIS(nums):
    n = len(nums)
    dp = [1] * n
    prev = [-1] * n      # 记录前驱
    
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i] and dp[j] + 1 > dp[i]:
                dp[i] = dp[j] + 1
                prev[i] = j    # 记录从哪里转移来
    
    # 找到最大值的位置
    max_len = max(dp)
    idx = dp.index(max_len)
    
    # 回溯
    result = []
    while idx != -1:
        result.append(nums[idx])
        idx = prev[idx]
    
    return result[::-1]

# ⚠️ 需要 prev 数组记录前驱
#    空间优化后无法回溯
```

## 5. LeetCode 673: 最长递增子序列的个数

```
题目: 返回最长递增子序列的个数
示例: [1,3,5,4,7] → LIS 长度 2: [1,3,5,7] 和 [1,3,4,7] → 2 个

思路: 在 dp 基础上增加 cnt 数组
     dp[i]  = 以 i 结尾的 LIS 长度
     cnt[i] = 以 i 结尾的、长度为 dp[i] 的子序列个数
```

```python
def findNumberOfLIS(nums):
    n = len(nums)
    dp = [1] * n     # LIS 长度
    cnt = [1] * n    # LIS 个数
    
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                if dp[j] + 1 > dp[i]:
                    # 找到更长的 → 重置计数
                    dp[i] = dp[j] + 1
                    cnt[i] = cnt[j]
                elif dp[j] + 1 == dp[i]:
                    # 一样长 → 累加计数
                    cnt[i] += cnt[j]
    
    max_len = max(dp)
    # ⚠️ 统计所有 dp[i]==max_len 的 cnt[i] 之和
    return sum(cnt[i] for i in range(n) if dp[i] == max_len)

# ⚠️ 关键:
#   dp[j]+1 > dp[i]  → cnt[i] = cnt[j]   (更优方案, 重置)
#   dp[j]+1 == dp[i] → cnt[i] += cnt[j]   (同样长, 累加)
```

## 6. LeetCode 354: 俄罗斯套娃信封

```
题目: 信封 (w, h)，当 w1<w2 且 h1<h2 时可以嵌套
     最多嵌套几个?

思路: 按宽度升序排列
     宽度相同时，按高度降序排列 ← 关键!
     然后对高度求 LIS

为什么宽度相同按高度降序?
  宽度相同的信封不能互相嵌套
  降序排列保证宽度相同的信封在 LIS 中 最多选一个

示例:
  信封: (2,3) (5,4) (6,7) (6,4)
  排序: (2,3) (5,4) (6,7) (6,4)
  宽度6的两个按高度降序: (6,7) (6,4)
  高度序列: [3, 4, 7, 4]
  LIS: [3, 4, 7] → 3
```

```python
def maxEnvelopes(envelopes):
    # ⚠️ 宽度升序, 宽度相同按高度降序
    envelopes.sort(key=lambda x: (x[0], -x[1]))
    
    # 对高度求 LIS (O(n log n))
    import bisect
    tails = []
    for _, h in envelopes:
        pos = bisect.bisect_left(tails, h)
        if pos == len(tails):
            tails.append(h)
        else:
            tails[pos] = h
    
    return len(tails)

# ⚠️ 排序的 key: (x[0], -x[1])
#    宽度正序 (+x[0])
#    高度逆序 (-x[1])   ← 负号!
#    如果忘了逆序, 宽度相同的信封可能被同时选中
```

## 7. LIS 变体总结

```
┌──────────────────┬──────────────────────────────┐
│ 变体              │ 关键修改                      │
├──────────────────┼──────────────────────────────┤
│ 非严格递增 (≤)    │ bisect_right 代替 bisect_left│
│ 最长递减子序列    │ 反转数组后求 LIS              │
│ LIS 个数 (LC673) │ 增加 cnt 数组                 │
│ 二维(信封 LC354) │ 排序+降维后 LIS               │
│ 最长数对链(LC646)│ 按右端排序后贪心              │
│ 最长摆动子序列    │ 不是 LIS，用贪心             │
└──────────────────┴──────────────────────────────┘

⚠️ 严格递增用 bisect_left
   非严格递增用 bisect_right
   这是最常混淆的点
```

## 本节要点速查

```
✅ dp[i] = 以 nums[i] 结尾的 LIS 长度, 初始化 1
✅ 答案是 max(dp)，不是 dp[-1]
✅ O(n²): 双重循环, dp[i] = max(dp[j]+1)
✅ O(nlogn): tails 数组 + bisect_left
✅ tails 不是实际 LIS, 只是长度相等
✅ 严格 < → bisect_left; 非严格 ≤ → bisect_right
✅ 信封: 宽升序高降序，对高度求 LIS
✅ LIS 个数: dp[j]+1>dp[i] 重置, ==dp[i] 累加
```
