## 2. 循环方向速查

```
┌─────────────┬────────────────────────┬──────────────────┐
│ 背包类型     │ 内层循环               │ 原因              │
├─────────────┼────────────────────────┼──────────────────┤
│ 0-1 背包     │ for j in range(W,w-1,-1) │ 每个只用一次    │
│ 完全背包     │ for j in range(w, W+1)   │ 允许重复使用    │
│ 分组背包     │ for j in range(W,-1,-1)  │ 每组一个, 倒序  │
│              │   for w,v in group:      │ 组内枚举        │
└─────────────┴────────────────────────┴──────────────────┘

⚠️ 记忆技巧:
   "只用一次 → 倒序" (怕覆盖, 从后往前更新)
   "无限用 → 正序"   (就是要覆盖, 让新值传递)
```

## 3. 初始化速查

```
┌──────────────────┬────────────────────┬────────────────┐
│ 问题类型          │ dp 初始化           │ 转移方式        │
├──────────────────┼────────────────────┼────────────────┤
│ 最大价值          │ dp = [0]*(W+1)      │ max            │
│ 最小次数          │ dp = [inf]*(W+1)    │ min            │
│                  │ dp[0] = 0           │                │
│ 能否恰好装满      │ dp = [False]*(W+1)  │ or             │
│                  │ dp[0] = True        │                │
│ 方案数            │ dp = [0]*(W+1)      │ +=             │
│                  │ dp[0] = 1           │                │
└──────────────────┴────────────────────┴────────────────┘

⚠️ dp[0] 的含义:
   最大价值: 容量 0 → 价值 0
   最小次数: 凑出 0 → 0 次
   能否装满: 0 可以被凑出 → True
   方案数:   凑出 0 有一种方案 (空集) → 1
```

## 4. "恰好装满" vs "最多装"

```
最多装 W:
  dp 全初始化为 0
  答案 dp[W] 一定 ≥ 0

恰好装满 W:
  dp[0] = 0 (或 True/1)
  其余 dp = -inf (或 False/0)
  答案 dp[W], 如果仍为 -inf 则无解

原理:
  "恰好装满" → 只有从 dp[0] 转移过来的值才有效
  dp[j] = -inf 表示 "容量 j 从来没被合法地凑出过"
  如果转移后 dp[W] 还是 -inf → 无法恰好装满

示例: 物品 [w=3,v=5], W=4, 恰好装满
  dp = [0, -inf, -inf, -inf, -inf]
  j=4: dp[4] = max(-inf, dp[1]+5) = max(-inf, -inf+5) = -inf
  j=3: dp[3] = max(-inf, dp[0]+5) = max(-inf, 0+5) = 5
  结果: dp[4] = -inf → 不能恰好装满 4 ✓ (只能装 3)
```

## 5. 组合 vs 排列

```
组合数 (不同顺序算同一种):
  外层循环物品, 内层循环容量
  for coin in coins:
      for j in range(coin, amount+1):
          dp[j] += dp[j-coin]

排列数 (不同顺序算不同方案):
  外层循环容量, 内层循环物品
  for j in range(1, target+1):
      for x in nums:
          if j >= x:
              dp[j] += dp[j-x]

为什么?
  组合: 先固定 coin1 能贡献的，再加 coin2
        coins=[1,2], dp[3]: 先处理1,再处理2
        只会产生 [1,2] 不会产生 [2,1]
  
  排列: 每个 j 都考虑所有物品
        dp[3] = dp[2]+coins含1 + dp[1]+coins含2 + ...
        [1,2] 和 [2,1] 都能产生

⚠️ LC 518 (组合) vs LC 377 (排列) 的唯一区别就是循环顺序
```

## 6. 常见转化技巧

```
1. "分成两组使差最小" → 0-1 背包
   target = sum/2, 能装的最大值 → 差 = sum - 2*dp[target]
   LC 1049 最后一块石头的重量 II

2. "±号使得和为target" → 0-1 背包
   bag = (sum + target) / 2
   LC 494 目标和

3. "凑出金额" → 完全背包
   LC 322, 518

4. "选k个互斥物品" → 分组背包
   LC 1155

5. "字符串拆分" → 类完全背包
   LC 139 单词拆分

6. "二维限制" → 二维背包
   LC 474 一和零 (0的个数 + 1的个数)

⚠️ 关键判断:
   每个用一次 → 0-1
   每个用多次 → 完全
   分组互斥   → 分组

   求什么:
   最大/最小 → max/min
   方案数   → +=
   能否     → or
```

## 7. 边界陷阱清单

```
⚠️ 陷阱1: 物品数组 0 索引 vs dp 1 索引
   二维 dp: dp[i][j] 的 i 从 1 开始
   物品:     weights[i-1]，不是 weights[i]

⚠️ 陷阱2: target 的合法性
   (sum + target) % 2 != 0 → 无解
   abs(target) > sum → 无解
   target < 0 → 可能需要取绝对值

⚠️ 陷阱3: dp 数组大小
   dp = [0] * (W + 1)  不是  dp = [0] * W
   因为要访问 dp[W]

⚠️ 陷阱4: 空数组
   nums = [] → sum = 0
   如果 target = 0 → 返回 True/1
   否则返回 False/0   

⚠️ 陷阱5: 包含 0 的物品
   w[i] = 0 时，倒序循环 range(W, -1, -1) 会到 j=0
   可能导致死循环或重复计算
   需要特殊处理: 跳过 0 或单独计数
```

## 8. 背包问题的记忆化搜索

```python
# 0-1 背包的记忆化搜索写法
from functools import cache

def knapsack_memo(W, weights, values):
    n = len(weights)
    
    @cache
    def dfs(i, j):
        """考虑第 i~n-1 个物品, 剩余容量 j"""
        if i == n:
            return 0          # 没有物品可选
        # 不选
        res = dfs(i + 1, j)
        # 选
        if j >= weights[i]:
            res = max(res, dfs(i + 1, j - weights[i]) + values[i])
        return res
    
    return dfs(0, W)

# ⚠️ 记忆化搜索更直观, 适合理解和调试
#    递推更快 (无递归开销), 适合提交
#    先写记忆化, 确认正确后改递推
```

## 本节要点速查

```
✅ 0-1 倒序, 完全正序, 分组倒序+组内枚举
✅ max/min/or/+= 对应不同问题类型
✅ dp[0] 的初始值: 0/0/True/1 看问题类型
✅ 恰好装满: 其余初始化 -inf/False/0
✅ 组合: 外物品内容量; 排列: 外容量内物品
✅ 先判断 target 合法性 (奇偶、范围)
✅ 物品含 0 时需特殊处理
```
