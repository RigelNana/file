# 分组背包

> 物品分成若干组，每组最多选一个。注意三层循环的顺序。

---

## 1. 问题描述

```
给定:
  n 组物品，第 k 组有若干个物品
  每组最多选一个 (也可不选)
  背包容量 W

目标: 总重量 ≤ W，总价值最大

示例:
  组1: 物品A(w=2,v=3), 物品B(w=3,v=5)
  组2: 物品C(w=1,v=2), 物品D(w=4,v=7)
  W = 5

  最优: 选 B(w=3,v=5) + C(w=1,v=2) → 重量4, 价值7

与 0-1 背包的区别:
  0-1 背包: 每个物品独立决策
  分组背包: 每组内部互斥, 组间独立
```

---

## 2. 转移分析

```
状态: dp[j] = 容量为 j 时的最大价值

对于每组:
  要么不选 (dp[j] 不变)
  要么选组内的某一个物品

三层循环:
  外层: 枚举组
  中层: 倒序枚举容量 (每组最多选一个 → 类似 0-1 背包)
  内层: 枚举组内每个物品

⚠️ 中层必须在内层外面!
   如果 "内层容量, 最内层物品":
   每组内的多个物品会像独立物品一样处理 → 可能选多个 → 错!

正确:                            错误:
for group in groups:              for group in groups:
    for j in range(W, -1, -1):        for w, v in group:     ← 错
        for w, v in group:                for j in range(W, w-1, -1):
            if j >= w:                        dp[j] = max(...)
                dp[j] = max(...)
```

---

## 3. 代码模板

```python
def group_knapsack(W, groups):
    """
    groups[k] = [(w1,v1), (w2,v2), ...]  第 k 组的物品列表
    """
    dp = [0] * (W + 1)
    
    for group in groups:
        # ⚠️ 先遍历容量 (倒序)，再遍历组内物品
        for j in range(W, -1, -1):
            for w, v in group:
                if j >= w:
                    dp[j] = max(dp[j], dp[j - w] + v)
    
    return dp[W]

# ⚠️ 循环顺序至关重要:
#    外层: 组 (每组独立处理)
#    中层: 容量倒序 (保证每组只选一个)
#    内层: 组内物品 (选哪个最优)
```

## 4. 模拟示例

```
groups = [
    [(2,3), (3,5)],   # 组1
    [(1,2), (4,7)]    # 组2
]
W = 5

初始: dp = [0, 0, 0, 0, 0, 0]

处理组1: [(2,3), (3,5)]
  j=5: max(dp[5], dp[3]+3, dp[2]+5) = max(0, 0+3, 0+5) = 5
  j=4: max(dp[4], dp[2]+3, dp[1]+5) = max(0, 0+3, 0+5) = 5
  j=3: max(dp[3], dp[1]+3, dp[0]+5) = max(0, 0+3, 0+5) = 5
  j=2: max(dp[2], dp[0]+3) = 3  (只有物品A的重量≤2)
  j=1: 没有物品重量≤1 → dp[1] = 0
  j=0: dp[0] = 0
  dp = [0, 0, 3, 5, 5, 5]

处理组2: [(1,2), (4,7)]
  j=5: max(5, dp[4]+2, dp[1]+7) = max(5, 5+2, 0+7) = 7
  j=4: max(5, dp[3]+2, dp[0]+7) = max(5, 5+2, 0+7) = 7
  j=3: max(5, dp[2]+2) = max(5, 3+2) = 5
  j=2: max(3, dp[1]+2) = max(3, 0+2) = 3
  j=1: max(0, dp[0]+2) = 2
  dp = [0, 2, 3, 5, 7, 7]

答案: dp[5] = 7 (选组1的B + 组2的C，或组2的D + 组1的A)
```

## 5. 实际应用: LeetCode 1155 掷骰子

```
题目: d 个骰子, 每个 f 面 (1~f)
     掷出总和为 target 的方案数

转化: 分组背包!
     每个骰子 = 一组
     每组有 f 个物品 (面值 1~f)
     每组必须选恰好一个 (骰子一定要掷)
     求方案数

⚠️ 注意: 这是 "每组必选一个" (不是 "最多选一个")
```

```python
def numRollsToTarget(d, f, target):
    MOD = 10**9 + 7
    # dp[j] = 用当前已处理的骰子, 凑出和 j 的方案数
    dp = [0] * (target + 1)
    dp[0] = 1
    
    for _ in range(d):          # 每个骰子 = 一组
        new_dp = [0] * (target + 1)
        for j in range(1, target + 1):    # ⚠️ 从 1 开始 (必选一个)
            for face in range(1, f + 1):  # 组内: 面值 1~f
                if j >= face:
                    new_dp[j] = (new_dp[j] + dp[j - face]) % MOD
        dp = new_dp
    
    return dp[target]

# ⚠️ 这里用 new_dp 而不是原地更新:
#    因为"必选一个", 不能保留 dp[j] (不选的情况不允许)
#    如果用倒序原地更新, dp[j] 保留了"不选"的旧值 → 错

# ⚠️ 也可以用记忆化搜索:
#    dfs(dice_left, remaining_sum)
#    更直观, 但递推效率更高
```

## 6. 多重背包 (扩展)

```
多重背包: 第 i 种物品最多选 k[i] 个

朴素做法: 拆成 k[i] 个独立物品 → 0-1 背包
  时间 O(sum(k[i]) * W), 太慢

二进制优化: 
  k[i] 拆成 1, 2, 4, ..., 2^p, 剩余
  例如 k=13 → 1, 2, 4, 6 (1+2+4+6=13)
  每份作为独立物品做 0-1 背包
  时间 O(n * log(max_k) * W)

单调队列优化: O(n * W), 更复杂, 竞赛用

⚠️ 面试中多重背包较少考到
   0-1 背包和完全背包是重点
```

```python
# 多重背包 (二进制优化)
def multi_knapsack(W, items):
    """items[i] = (w, v, k)  重量, 价值, 数量"""
    # 二进制拆分
    new_items = []
    for w, v, k in items:
        t = 1
        while t <= k:
            new_items.append((w * t, v * t))
            k -= t
            t *= 2
        if k > 0:
            new_items.append((w * k, v * k))
    
    # 转化为 0-1 背包
    dp = [0] * (W + 1)
    for w, v in new_items:
        for j in range(W, w - 1, -1):
            dp[j] = max(dp[j], dp[j - w] + v)
    return dp[W]
```

## 7. 三种背包对比总结

```
┌─────────────┬──────────────────┬────────────────────────┐
│ 类型         │ 每个物品/每组     │ 内层循环                │
├─────────────┼──────────────────┼────────────────────────┤
│ 0-1 背包     │ 最多选 1 次       │ 倒序 W→w              │
│ 完全背包     │ 无限次            │ 正序 w→W              │
│ 分组背包     │ 每组最多 1 个     │ 倒序, 内层枚举组内物品  │
│ 多重背包     │ 最多 k[i] 次     │ 二进制拆分 → 0-1 背包   │
└─────────────┴──────────────────┴────────────────────────┘

⚠️ 分组背包的循环顺序: 组 → 容量(倒序) → 组内物品
   不能把容量放到最内层!
```

## 本节要点速查

```
✅ 分组背包: 每组最多选一个, 三层循环
✅ 循环顺序: 组 → 容量倒序 → 组内物品
✅ 容量在组内物品外面 (否则同组多个物品像独立物品)
✅ "必选一个" 场景用 new_dp 替代原地更新
✅ 多重背包用二进制拆分转化为 0-1 背包
✅ 面试重点: 0-1 > 完全 > 分组 > 多重
```
