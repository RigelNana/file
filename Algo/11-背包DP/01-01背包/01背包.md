# 0-1 背包

> 每个物品最多选一次，求最大价值。0-1 背包是所有背包问题的基础。

---

## 1. 问题描述

```
给定:
  n 个物品，每个有重量 w[i] 和价值 v[i]
  背包容量 W

目标: 选出若干物品(每个最多选一次)，使总重量 ≤ W，总价值最大

示例:
  物品 0: w=2, v=3
  物品 1: w=3, v=4
  物品 2: w=4, v=5
  背包容量 W=5

  最优: 选物品0 + 物品1 → 重量2+3=5, 价值3+4=7
```

---

## 2. 状态定义与转移

```
状态定义:
  dp[i][j] = 考虑前 i 个物品，背包容量为 j 时的最大价值

转移方程 (对于第 i 个物品):
  不选: dp[i][j] = dp[i-1][j]
  选:   dp[i][j] = dp[i-1][j - w[i]] + v[i]   (前提: j ≥ w[i])
  取 max

⚠️ "选"时用的是 dp[i-1][...]，不是 dp[i][...]
   因为每个物品只能选一次，选了 i 之后，
   剩余只能在前 i-1 个物品中选

填表过程:
       j:  0  1  2  3  4  5
  无物品:  0  0  0  0  0  0     ← base case
  物品0:   0  0  3  3  3  3     (w=2,v=3)
  物品1:   0  0  3  4  4  7     (w=3,v=4)
  物品2:   0  0  3  4  5  7     (w=4,v=5)

  dp[1][5] = max(dp[0][5], dp[0][5-3]+4) = max(3, 3+4) = 7
  dp[2][5] = max(dp[1][5], dp[1][5-4]+5) = max(7, 3+5) = max(7,8)? 
  
  等等，dp[1][1] = 0, dp[1][5-4=1] = 0, 0+5=5 < 7
  所以 dp[2][5] = 7 ✓
```

## 3. 二维 DP 代码

```python
def knapsack_01(W, weights, values):
    n = len(weights)
    # ⚠️ dp 大小是 (n+1) × (W+1)，第 0 行是 base case
    dp = [[0] * (W + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):       # 物品编号 1..n
        w, v = weights[i-1], values[i-1]  # ⚠️ i-1 因为物品数组是 0 索引
        for j in range(W + 1):
            dp[i][j] = dp[i-1][j]   # 不选第 i 个
            if j >= w:
                dp[i][j] = max(dp[i][j], dp[i-1][j-w] + v)
    
    return dp[n][W]

# ⚠️ 常见错误:
#   ① weights/values 是 0 索引，但 dp 的行是 1 索引
#      所以 weights[i-1] 不是 weights[i]
#   ② 忘记检查 j >= w[i]，导致下标为负
#   ③ dp 数组大小应该是 (n+1)×(W+1)，不是 n×W
```

## 4. 一维空间优化 ⭐

```
关键观察:
  dp[i][j] 只依赖 dp[i-1][j] 和 dp[i-1][j-w]
  即只依赖上一行 → 可以只用一维数组

  但必须 倒序遍历 j ！

为什么倒序?
  正序: 更新 dp[j] 时，dp[j-w] 已经被"当前物品"更新过
        → dp[j-w] 含有当前物品信息 → 物品被重复选择！
  
  倒序: 更新 dp[j] 时，dp[j-w] 还是"上一轮"的旧值
        → 每个物品只选一次 ✓

图示 (物品 w=2, v=3, 容量 W=5):
  正序 (错 ✗):
    dp = [0, 0, 0, 0, 0, 0]
    j=2: dp[2] = max(0, dp[0]+3) = 3  → dp = [0,0,3,0,0,0]
    j=3: dp[3] = max(0, dp[1]+3) = 3  → [0,0,3,3,0,0]
    j=4: dp[4] = max(0, dp[2]+3) = 6  ← dp[2]=3 已更新 ✗ 选了两次!

  倒序 (对 ✓):
    dp = [0, 0, 0, 0, 0, 0]
    j=5: dp[5] = max(0, dp[3]+3) = 3  → dp = [0,0,0,0,0,3]
    j=4: dp[4] = max(0, dp[2]+3) = 3  → [0,0,0,0,3,3]  
    j=3: dp[3] = max(0, dp[1]+3) = 3  → [0,0,0,3,3,3]
    j=2: dp[2] = max(0, dp[0]+3) = 3  → [0,0,3,3,3,3]  ✓ 每个值只选一次
```

```python
# 0-1 背包 一维优化 ⭐ 最常用
def knapsack_01_opt(W, weights, values):
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        w, v = weights[i], values[i]
        # ⚠️ 倒序! range(W, w-1, -1) 等价于 j = W, W-1, ..., w
        for j in range(W, w - 1, -1):
            dp[j] = max(dp[j], dp[j - w] + v)
    return dp[W]

# ⚠️ 倒序的范围: range(W, w-1, -1)
#    不是 range(W, -1, -1)
#    因为 j < w 时，dp[j-w] 下标为负，且肯定不选
#    所以直接从 w 开始即可
```

## 5. LeetCode 416: 分割等和子集

```
题目: 数组能否分成两个子集，使它们的和相等
示例: [1,5,11,5] → [1,5,5] 和 [11]，返回 True

转化: 总和 S, 找子集和 = S/2
     → 0-1 背包: 物品重量=nums[i], 容量=S/2, 能否恰好装满

⚠️ 如果 S 是奇数，直接返回 False
```

```python
def canPartition(nums):
    total = sum(nums)
    if total % 2:     # ⚠️ 奇数不可能平分
        return False
    target = total // 2
    
    dp = [False] * (target + 1)
    dp[0] = True       # ⚠️ 和为 0 是可以的 (什么都不选)

    for x in nums:
        # ⚠️ 倒序! 0-1 背包
        for j in range(target, x - 1, -1):
            dp[j] = dp[j] or dp[j - x]
    
    return dp[target]

# 模拟: nums = [1,5,11,5], target = 11
# 初始: dp = [T, F, F, F, F, F, F, F, F, F, F, F]
# x=1:  dp = [T, T, F, F, F, F, F, F, F, F, F, F]
# x=5:  dp = [T, T, F, F, F, T, T, F, F, F, F, F]
# x=11: dp = [T, T, F, F, F, T, T, F, F, F, F, T]  ← dp[11]=True
# 提前就找到了! (x=5 不再需要处理)
```

## 6. LeetCode 494: 目标和

```
题目: 数组中每个数前面加 + 或 -, 使得总和等于 target
     返回方案数
示例: [1,1,1,1,1], target=3 → 5 种

数学转化:
  设加 + 的数之和为 P，加 - 的数之和的绝对值为 Q
  P - Q = target
  P + Q = sum(nums)
  → P = (sum + target) / 2

  ⚠️ (sum + target) 必须是偶数且非负，否则返回 0
  ⚠️ |target| > sum 时也返回 0

  问题变成: 从 nums 中选若干个，凑出和 P 的方案数 → 0-1 背包
```

```python
def findTargetSumWays(nums, target):
    total = sum(nums)
    # ⚠️ 两个特判
    if (total + target) % 2 or abs(target) > total:
        return 0
    
    bag = (total + target) // 2
    dp = [0] * (bag + 1)
    dp[0] = 1   # ⚠️ 方案数: dp[0]=1 (空集也是一种方案)
    
    for x in nums:
        for j in range(bag, x - 1, -1):  # 倒序, 0-1 背包
            dp[j] += dp[j - x]   # ⚠️ 方案数用 +=，不是 max
    
    return dp[bag]

# ⚠️ 这里是 "方案数" 背包:
#    dp[j] += dp[j-x]  (不是 dp[j] = max(dp[j], dp[j-x]+v))
#    初始值 dp[0] = 1  (不是 dp[0] = 0)
```

## 7. LeetCode 474: 一和零 (二维背包)

```
题目: 字符串数组 strs，只含 '0' 和 '1'
     用 m 个 0 和 n 个 1 最多能组成几个字符串

本质: 二维 0-1 背包
     两个容量维度: 0 的个数 m, 1 的个数 n
     每个字符串是一个物品
```

```python
def findMaxForm(strs, m, n):
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for s in strs:
        zeros = s.count('0')
        ones = s.count('1')
        # ⚠️ 两个维度都要倒序 (0-1 背包)
        for i in range(m, zeros - 1, -1):
            for j in range(n, ones - 1, -1):
                dp[i][j] = max(dp[i][j], dp[i-zeros][j-ones] + 1)
    
    return dp[m][n]

# ⚠️ 二维背包只是多了一个维度，逻辑和一维完全一样
#    关键: 每个维度都要倒序遍历 (0-1 背包)
```

## 8. 背包变体总结

```
┌──────────────────┬───────────────────────┬───────────────┐
│ 问题类型          │ dp 定义               │ 初始值         │
├──────────────────┼───────────────────────┼───────────────┤
│ 最大价值          │ dp[j] = max value     │ dp 全 0        │
│ 能否恰好装满      │ dp[j] = True/False    │ dp[0]=True     │
│ 方案数            │ dp[j] = count         │ dp[0]=1        │
│ 最少物品数        │ dp[j] = min count     │ dp[0]=0, 余 inf│
└──────────────────┴───────────────────────┴───────────────┘

⚠️ "最多装 W" vs "恰好装满":
   最多装: dp 初始化为 0 (容量还没用完也算)
   恰好装: dp[0] = 0 或 True, 其余 -inf 或 False
          (只有从 dp[0] 转移过来的才有效)
```

## 本节要点速查

```
✅ dp[i][j] = max(dp[i-1][j], dp[i-1][j-w]+v)
✅ 一维优化: 倒序遍历 j (保证每个物品只选一次)
✅ 倒序范围: range(W, w-1, -1)，不是 range(W, -1, -1)
✅ 方案数用 +=，最值用 max/min
✅ dp[0] 的初始值取决于问题类型 (0/True/1)
✅ 二维背包: 每个维度都要倒序
✅ 奇偶性/|target|>sum 等边界条件先特判
```
