# 最近公共祖先 (LCA) ⭐

> LCA 是面试高频题，后序遍历的经典应用。理解"分叉即祖先"的思想，可以拓展到 BST、多节点、带父指针等变体。

---

## 1. 普通二叉树 LCA ⭐⭐

### LeetCode 236

### 理解与可视化

```
最近公共祖先(LCA): p 和 q 的最近公共祖先是从根到 p 和 q 路径上
                  最后分开的那个节点

         3
        / \
       5   1
      / \ / \
     6  2 0  8
       / \
      7   4

  LCA(5, 1) = 3    ← 5 在左子树, 1 在右子树, 分叉点=3
  LCA(5, 4) = 5    ← 4 在 5 的子树里, 5 本身就是 LCA
  LCA(7, 4) = 2    ← 7 在 2 的左, 4 在 2 的右, 分叉点=2

  后序递归思路:
    对于每个节点, 分别在左右子树中找 p 和 q
    
    情况 1: 左右子树各找到一个 → 当前节点就是 LCA
    情况 2: 只在一侧找到     → LCA 在那一侧
    情况 3: 当前节点就是 p/q → 直接返回自己
              (因为另一个一定在它的子树里, 它就是 LCA)
```

### 代码模板

```python
def lowestCommonAncestor(root, p, q):
    if not root or root == p or root == q:
        return root                   # ⚠️ 找到 p 或 q 就返回
    
    left = lowestCommonAncestor(root.left, p, q)   # 左子树找
    right = lowestCommonAncestor(root.right, p, q)  # 右子树找
    
    if left and right:                # ⚠️ 左右各找到一个 → 当前是 LCA
        return root
    return left or right              # ⚠️ 只有一侧找到 → 返回那一侧
                                      #    都没找到 → 返回 None

# ⚠️ 关键理解:
#   1. root == p → 不管 q 在哪, root 至少是 p, 也是往上走的必经之路
#      所以 root 就是 LCA
#   2. left and right → p 和 q 分布在左右两侧
#      当前节点是它们"分叉"的地方 = LCA
#   3. left or right → p 和 q 都在同一侧
#      那一侧返回的就是 LCA
#
# ⚠️ 时间 O(n), 空间 O(h), h 是树高
```

```
执行过程: LCA(7, 4), 树如上

  dfs(3):
    dfs(5):
      dfs(6): 不是7或4, 且没有子节点 → None
      dfs(2):
        dfs(7): root==7! → return 7
        dfs(4): root==4! → return 4
        left=7, right=4 → 都非空 → return 2 ← LCA!
      left=None, right=2 → return 2
    dfs(1):
      dfs(0): → None
      dfs(8): → None
      → return None
    left=2, right=None → return 2
  
  最终返回 2 ✓
```

## 2. BST 的 LCA

### LeetCode 235

```
BST 特点: 左 < 根 < 右
利用 BST 性质可以做到 O(h) 不用遍历整棵树

策略:
  如果 p, q 都小于 root → LCA 在左子树
  如果 p, q 都大于 root → LCA 在右子树
  否则 → root 就是 LCA（分叉点）
```

```python
def lowestCommonAncestor_BST(root, p, q):
    while root:
        if p.val < root.val and q.val < root.val:
            root = root.left          # 都比 root 小, 去左边
        elif p.val > root.val and q.val > root.val:
            root = root.right         # 都比 root 大, 去右边
        else:
            return root               # 分叉了! root 就是 LCA
    return None

# ⚠️ 迭代版, O(h) 时间, O(1) 空间
# ⚠️ "分叉"包括: p < root < q, 或 root == p/q
# ⚠️ 不需要判断 p.val == root.val 等特殊情况
#   因为 p==root 时不满足 p<root 也不满足 p>root → 走 else → 正确

# 递归版也很简洁:
def lowestCommonAncestor_BST_rec(root, p, q):
    if p.val < root.val and q.val < root.val:
        return lowestCommonAncestor_BST_rec(root.left, p, q)
    if p.val > root.val and q.val > root.val:
        return lowestCommonAncestor_BST_rec(root.right, p, q)
    return root
```

## 3. 多节点的 LCA

```
如果要找多个节点的 LCA? 

思路: 和两个节点的一样, 只要把 "root == p or root == q" 
     改成 "root in nodes" 即可
     只要左右子树都找到了至少一个 → 当前是 LCA
     只有一侧找到全部 → LCA 在那一侧
```

```python
def lca_multiple(root, nodes):
    """找 nodes 中所有节点的 LCA"""
    nodes_set = set(nodes)
    
    def dfs(node):
        if not node or node in nodes_set:
            return node
        left = dfs(node.left)
        right = dfs(node.right)
        if left and right:
            return node
        return left or right
    
    return dfs(root)

# ⚠️ 和标准 LCA 唯一区别: root == p or root == q → root in nodes_set
```

## 4. 带父指针的 LCA

```
如果每个节点有 parent 指针, 可以用相交链表的方法!

思路: p 和 q 到根的路径 = 两条链表
     找两条链表的交叉点 = LCA（和 LeetCode 160 完全一样!）
```

```python
def lca_with_parent(p, q):
    a, b = p, q
    while a != b:
        a = a.parent if a else q      # a 走完自己的路径后走 q 的
        b = b.parent if b else p      # b 走完自己的路径后走 p 的
    return a

# ⚠️ 和相交链表完全一样的双指针!
#   a 走: p → ... → root → q → ... → LCA
#   b 走: q → ... → root → p → ... → LCA
#   总步数相同, 在 LCA 相遇
```

```python
def is_in_subtree(root, target):
    """判断 target 是否在 root 的子树中"""
    if not root:
        return False
    if root == target:
        return True
    return is_in_subtree(root.left, target) or is_in_subtree(root.right, target)
```

## 易错点总结

```
┌──────────────────────────────────────────────────────┐
│                    LCA 易错点                         │
├──────────────────────────────────────────────────────┤
│ 1. root == p 时直接返回 root                          │
│    不需要继续找 q (q 一定在 root 子树里)              │
│                                                       │
│ 2. 返回 left or right, 不是 left and right            │
│    and 是判断是不是 LCA, or 是传递结果                 │
│                                                       │
│ 3. BST 版不需要 root==p/q 特判                        │
│    因为走到 else 分支时自然包含这种情况                 │
│                                                       │
│ 4. 函数的含义:                                        │
│    返回值 = "在以 root 为根的子树中, p 或 q 的节点"    │
│    如果返回 None = p 和 q 都不在这棵子树中             │
└──────────────────────────────────────────────────────┘
```
