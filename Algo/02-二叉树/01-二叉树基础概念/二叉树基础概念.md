# 二叉树基础概念

> 二叉树是算法面试中最重要的数据结构之一。理解基础概念和术语是学好所有树形算法的前提。

---

## 1. 什么是二叉树

### 理解与可视化

```
二叉树: 每个节点最多有两个孩子（左孩子、右孩子）

         1          ← 根节点 (root)，深度=0，高度=2
        / \
       2   3        ← 深度=1，节点2的高度=1
      / \   \
     4   5   6      ← 深度=2，叶子节点（没有孩子的节点），高度=0

  ┌───────────────────────────────────────────────┐
  │ 术语            │ 含义                          │
  ├───────────────────────────────────────────────┤
  │ 节点 (node)     │ 树中的元素                     │
  │ 根 (root)       │ 最顶部的节点                   │
  │ 叶子 (leaf)     │ 没有孩子的节点                 │
  │ 深度 (depth)    │ 从根到该节点的边数              │
  │ 高度 (height)   │ 从该节点到最远叶子的边数        │
  │ 层 (level)      │ 深度 + 1（有些定义 = 深度）     │
  │ 子树 (subtree)  │ 某节点及其所有后代构成的树      │
  │ 度 (degree)     │ 节点的孩子个数（二叉树最多2）   │
  └───────────────────────────────────────────────┘

  ⚠️ 深度 vs 高度:
     深度从上往下数（根=0）
     高度从下往上数（叶子=0）
     根的高度 = 整棵树的高度

  ⚠️ 有些题目的深度定义不同:
     LeetCode 104（最大深度）: 节点数, 不是边数
     即: 根的深度=1, 叶子多1
     一定要看题目定义!
```

## 2. 常见二叉树类型

### 理解与可视化

```
  ┌─────────── 满二叉树 ───────────┐
  │ 每个节点要么有 0 个孩子,       │
  │ 要么有 2 个孩子                 │
  │        1                        │
  │       / \                       │
  │      2   3                      │
  │     / \ / \                     │
  │    4  5 6  7                    │
  └────────────────────────────────┘

  ┌─────────── 完全二叉树 ─────────┐
  │ 除最后一层外全满,              │
  │ 最后一层从左往右连续            │
  │        1                        │
  │       / \                       │
  │      2   3                      │
  │     / \  /                      │
  │    4  5 6                       │
  │ ✓ 最后一层 4,5,6 从左到右连续  │
  │ ⚠️ 堆就是完全二叉树!           │
  └────────────────────────────────┘

  ┌─────────── 二叉搜索树 (BST) ──┐
  │ 左子树所有值 < 根 < 右子树所有值│
  │        8                        │
  │       / \                       │
  │      3   10                     │
  │     / \    \                    │
  │    1   6   14                   │
  │ 中序遍历是有序的: 1,3,6,8,10,14│
  └────────────────────────────────┘

  ┌─────────── 平衡二叉树 ─────────┐
  │ 任意节点左右子树高度差 ≤ 1      │
  │ AVL 树、红黑树都是平衡树        │
  │ 保证操作是 O(log n)            │
  └────────────────────────────────┘

  ┌─────────── 退化二叉树（链）────┐
  │ 每个节点只有一个孩子            │
  │    1                            │
  │     \                           │
  │      2                          │
  │       \                         │
  │        3    ← 本质就是链表!     │
  │ 此时树的操作退化为 O(n)        │
  └────────────────────────────────┘
```

## 3. 节点定义

### 代码模板

```python
# ━━━━ 基本二叉树节点 ━━━━
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# ⚠️ LeetCode 默认的 TreeNode 就是这个定义
# ⚠️ val 默认为 0, left 和 right 默认为 None

# ━━━━ 从列表构建二叉树（调试用）━━━━
from collections import deque

def build_tree_from_list(vals):
    """从 LeetCode 格式的列表构建二叉树
    
    [1, 2, 3, None, 5, None, 4]
    →      1
          / \
         2   3
          \   \
           5   4
    """
    if not vals or vals[0] is None:
        return None
    root = TreeNode(vals[0])
    queue = deque([root])
    i = 1
    while queue and i < len(vals):
        node = queue.popleft()
        # 左孩子
        if i < len(vals) and vals[i] is not None:
            node.left = TreeNode(vals[i])
            queue.append(node.left)
        i += 1
        # 右孩子
        if i < len(vals) and vals[i] is not None:
            node.right = TreeNode(vals[i])
            queue.append(node.right)
        i += 1
    return root

# ⚠️ LeetCode 列表格式是层序遍历, None 表示空节点
# ⚠️ 这个函数在本地调试时很有用
```

## 4. 二叉树的基本属性

### 代码模板

```python
# ━━━━ 节点个数 ━━━━
def count_nodes(root):
    if not root:
        return 0
    return 1 + count_nodes(root.left) + count_nodes(root.right)

# ━━━━ 最大深度（节点数定义）━━━━
def max_depth(root):
    if not root:
        return 0                  # ⚠️ 空树深度=0
    return max(max_depth(root.left), max_depth(root.right)) + 1

# ⚠️ LeetCode 104 的深度=节点数
#   单节点树: depth=1
#   空树: depth=0

# ━━━━ 最小深度 ━━━━
def min_depth(root):
    if not root:
        return 0
    if not root.left:             # ⚠️ 只有右子树时, 不算当前为叶子!
        return min_depth(root.right) + 1
    if not root.right:
        return min_depth(root.left) + 1
    return min(min_depth(root.left), min_depth(root.right)) + 1

# ⚠️ 最小深度的坑:
#      1
#       \
#        2
#         \
#          3
#   错误答案: min(0, 3)+1 = 1 ← 把根当叶子了
#   正确答案: 3 (从根到叶子3)
#   所以: 如果只有一边有子树, 必须往有子树的一边走

# ━━━━ 判断是否叶子 ━━━━
def is_leaf(node):
    return node and not node.left and not node.right

# ━━━━ 所有叶子节点 ━━━━
def get_leaves(root):
    if not root:
        return []
    if not root.left and not root.right:
        return [root.val]
    return get_leaves(root.left) + get_leaves(root.right)
```

## 5. 二叉树递归思维框架

### 理解与可视化

```
二叉树 90% 的题目都用递归解决!

递归三要素:
  1. 函数定义: 这个函数做什么?（明确输入输出）
  2. 终止条件: 什么时候停?（通常 if not root: return ...）
  3. 递归调用: 如何拆分成子问题?

两种递归思路:

  ┌──── 自顶向下（先序）────────────────────────┐
  │ def dfs(node, 参数):                         │
  │   if not node: return                        │
  │   ★ 处理当前节点（用参数）                    │
  │   dfs(node.left, 更新参数)                   │
  │   dfs(node.right, 更新参数)                  │
  │                                              │
  │ 特点: 把信息"带下去", 从根到叶子传递          │
  │ 典型: 求路径和、记录路径、计算深度(带depth参数)│
  └──────────────────────────────────────────────┘

  ┌──── 自底向上（后序）────────────────────────┐
  │ def dfs(node):                               │
  │   if not node: return 基本值                  │
  │   left = dfs(node.left)                      │
  │   right = dfs(node.right)                    │
  │   ★ return 利用left和right计算的结果          │
  │                                              │
  │ 特点: 信息"从下往上传", 子问题答案合成总答案   │
  │ 典型: 求深度、判平衡、求直径、翻转树          │
  └──────────────────────────────────────────────┘

  ⚠️ 如何选择?
    - 需要从根到叶子的信息（路径、深度参数）→ 自顶向下
    - 需要子树的信息来决定当前节点 → 自底向上
    - 很多题两种都行, 但自底向上通常更简洁
```

## 6. 易错点与边界

```
┌──────────────────────────────────────────────────────┐
│                  二叉树常见易错点                       │
├──────────────────────────────────────────────────────┤
│ 1. 空树: root=None 必须处理                           │
│ 2. 单节点: 只有根, left=right=None                    │
│ 3. 斜树: 只有左子树或只有右子树, 退化成链表            │
│ 4. 深度定义: 注意是"边数"还是"节点数"                  │
│ 5. 最小深度: 不是 min(左,右)+1, 要排除空子树           │
│ 6. BST: 是"所有"左子树的值 < 根, 不只是左孩子          │
│ 7. 递归返回值: None vs 0, 含义不同                     │
│ 8. 修改树: 注意递归后 root 要重新接上子树              │
└──────────────────────────────────────────────────────┘
```

## 本节要点速查

```
✅ 二叉树: 每个节点最多两个孩子
✅ 深度从上往下数, 高度从下往上数
✅ 满二叉树: 0或2个孩子; 完全二叉树: 最后一层左对齐; BST: 左<根<右
✅ TreeNode(val, left, right) 是标准定义
✅ 递归三要素: 函数定义、终止条件、递归调用
✅ 自顶向下(先序): 信息带下去; 自底向上(后序): 信息传上来
✅ 最小深度陷阱: 不能把只有单侧子树的节点当叶子
✅ 注意深度定义: LeetCode一般用节点数(根=1), 不是边数(根=0)
```
