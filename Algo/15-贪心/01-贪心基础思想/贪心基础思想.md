# 贪心基础思想

> 贪心的核心：每一步都做"当前看起来最好"的选择，最终得到全局最优。关键在于证明贪心策略正确。

---

## 1. 贪心 vs 动态规划

```
DP:   考虑所有子问题, 自底向上构建最优解
贪心: 每步选当前最优, 不回头

何时可以用贪心?
  1. 贪心选择性质: 局部最优能导出全局最优
  2. 最优子结构: 子问题的最优解包含在全局最优解中

例子:
  硬币找零 [1, 5, 10, 25]:
    50 分 → 贪心: 25+25 = 2枚 ✓ (正确)
  
  硬币找零 [1, 3, 4]:
    6 分 → 贪心: 4+1+1 = 3枚 ✗ (应该 3+3 = 2枚)

⚠️ 贪心不一定对! 需要证明或举反例
```

## 3. 跳跃游戏

### LeetCode 55

```
nums 中每个元素表示可以跳的最大步数
判断能否从 0 跳到末尾

贪心: 维护当前能到达的最远位置 farthest
  遍历 [0, farthest], 更新 farthest
  如果 farthest >= n-1 → 能到达

nums = [2, 3, 1, 1, 4]:
  i=0: farthest = max(0, 0+2) = 2
  i=1: farthest = max(2, 1+3) = 4 >= 4 ✓
```

```python
def canJump(nums):
    farthest = 0
    for i in range(len(nums)):
        if i > farthest:
            return False            # ⚠️ 当前位置超过最远可达 → 到不了
        farthest = max(farthest, i + nums[i])
    return True

# ⚠️ i > farthest: 说明这个位置不可达, 后面也不可达
# ⚠️ 不需要真的模拟跳跃, 只需要维护"最远可达"
```

### LeetCode 45: 跳跃游戏 II

```
保证能到达末尾, 求最少跳跃次数

贪心: 在当前跳跃范围内, 找下一次能跳最远的位置
  类似 BFS 的"层"

nums = [2, 3, 1, 1, 4]:
  第1跳: 从0出发, 最远到2, 选范围[1,2]中最远的 → 位置1(能到4)
  第2跳: 从1出发, 到4 → done
  答案: 2
```

```python
def jump(nums):
    n = len(nums)
    jumps = 0
    cur_end = 0              # 当前跳跃范围的右边界
    farthest = 0             # 下次跳跃能到的最远位置
    
    for i in range(n - 1):   # ⚠️ 不包括最后一个位置
        farthest = max(farthest, i + nums[i])
        if i == cur_end:     # 到达当前范围边界, 必须跳
            jumps += 1
            cur_end = farthest
    
    return jumps

# ⚠️ range(n-1): 不遍历最后一个元素
#    如果 cur_end 恰好等于 n-1, 在最后一个位置"跳"就多算了
#
# ⚠️ 本质是 BFS:
#    每一"层"是当前跳跃能覆盖的范围
#    层数 = 跳跃次数
```

## 4. 贪心证明方法

```
方法1: 交换论证
  假设最优解和贪心解不同
  证明把最优解的某步替换成贪心选择, 结果不变或更好

方法2: 反证法
  假设贪心不是最优, 推出矛盾

方法3: 归纳法
  证明每步贪心选择后, 剩余问题仍是同类子问题
  且贪心选择不影响后续最优

⚠️ 面试中: 如果不会证明, 至少要能说出贪心策略
   通常模拟几个例子确认即可
```
