# 区间贪心

> 区间问题是贪心最经典的应用场景，不同的问题需要不同的排序和选择策略。

---

## 1. 区间问题五大类型

```
┌───────────────────┬─────────────────────┬──────────────────┐
│ 问题类型          │ 排序方式            │ 贪心策略         │
├───────────────────┼─────────────────────┼──────────────────┤
│ 最多不相交区间    │ 按右端点排序        │ 选结束最早的     │
│ 最少箭/点覆盖     │ 按右端点排序        │ 选当前覆盖最多的 │
│ 区间分组(最少组)  │ 按左端点排序        │ 堆维护各组右端点 │
│ 最少区间覆盖目标  │ 按左端点排序        │ 选覆盖最远的     │
│ 合并区间          │ 按左端点排序        │ 重叠就合并       │
└───────────────────┴─────────────────────┴──────────────────┘
```

## 2. 最多不相交区间 / 无重叠区间

### LeetCode 435

```
给定区间集合, 求最少删除几个使剩余区间不重叠

等价于: 最多能选多少个不重叠区间, 删除数 = 总数 - 最多不重叠

贪心: 按右端点排序, 优先选结束最早的
  结束越早, 留给后面的空间越大

排序后:
  ──[  ]
  ─────[    ]      ← 和上一个重叠, 跳过
  ──────[ ]
  ──────────[   ]  ← 不重叠, 选
  ─────────────[  ] ← 不重叠, 选
```

```python
def eraseOverlapIntervals(intervals):
    intervals.sort(key=lambda x: x[1])   # ⚠️ 按右端点排序
    count = 0
    end = -float('inf')
    
    for s, e in intervals:
        if s >= end:           # ⚠️ >= 不重叠 (端点相同不算重叠)
            end = e
            count += 1         # 选这个区间
    
    return len(intervals) - count   # ⚠️ 删除数 = 总数 - 保留数

# ⚠️ 为什么按右端点排序?
#    贪心选结束最早的 → 右端点越小越早结束 → 按右端点排序
#
# ⚠️ s >= end (不是 s > end):
#    [1,2] 和 [2,3] 不重叠 (区间外的点)
#    如果题目说端点重叠也算, 改成 s > end
#
# ⚠️ 如果按左端点排序会怎样?
#    需要额外比较右端点, 更复杂
#    按右端点排序可以直接贪心
```

## 3. 用最少箭引爆气球

### LeetCode 452

```
二维平面上的气球, 用垂直箭射穿
区间 [start, end] 表示气球水平范围
一支箭 x 能射穿所有 start <= x <= end 的气球
求最少箭数

本质: 最多不相交区间的变体
  但这里 [1,2] 和 [2,3] 算相交 (共享点2)
```

```python
def findMinArrowShots(points):
    points.sort(key=lambda x: x[1])   # ⚠️ 按右端点排序
    arrows = 1
    end = points[0][1]
    
    for s, e in points[1:]:
        if s > end:              # ⚠️ > (不是 >=): 端点重叠算相交
            arrows += 1
            end = e
    
    return arrows

# ⚠️ 和 435 的区别:
#    435: s >= end → 端点相同不算重叠
#    452: s > end → 端点相同算重叠 (箭可以在端点射)
#
# ⚠️ 初始 arrows = 1: 至少需要一支箭
#    题目保证 points 非空
```

## 4. 区间分组 / 会议室 II

### LeetCode 253

```
给定会议时间 intervals, 求最少需要几个会议室

贪心: 按开始时间排序, 用最小堆维护各会议室的结束时间
  新会议开始时:
    如果堆顶(最早结束的会议室)已空闲 → 复用
    否则 → 开新会议室

intervals = [[0,30],[5,10],[15,20]]
排序: [[0,30],[5,10],[15,20]]

  [0,30]: heap=[] → push(30) → heap=[30]
  [5,10]: heap[0]=30 > 5 → 需新房间 → push(10) → heap=[10,30]
  [15,20]: heap[0]=10 <= 15 → 复用 → replace(20) → heap=[20,30]
  答案: len(heap) = 2
```

```python
import heapq

def minMeetingRooms(intervals):
    intervals.sort()                  # ⚠️ 按开始时间排序
    heap = []                         # 最小堆: 各会议室的结束时间
    
    for s, e in intervals:
        if heap and heap[0] <= s:     # ⚠️ <= 最早结束的已空闲
            heapq.heapreplace(heap, e)  # ⚠️ 复用, 更新结束时间
        else:
            heapq.heappush(heap, e)     # 开新会议室
    
    return len(heap)

# ⚠️ heap[0] <= s (不是 <):
#    [0,10] 和 [10,20] 可以用同一个房间
#    10点结束, 10点又开始 = 可以复用
#
# ⚠️ heapreplace vs heappush:
#    heapreplace: pop最小+push新值, O(log n), 一步完成
#    heappush(heap, e) 后 heappop(heap): 等效但多一次操作
#
# ⚠️ 等价做法: 差分/扫描线
#    每个区间 [s,e]: 在 s 处 +1, 在 e 处 -1
#    前缀和最大值 = 最少会议室
```

## 5. 最少区间覆盖目标 / 视频拼接

### LeetCode 1024

```
clips[i] = [start, end], 目标覆盖 [0, time]
选最少的片段覆盖整个范围, 无法覆盖返回 -1

贪心: 按左端点排序
  在当前覆盖范围能到的片段中, 选能覆盖最远的

clips = [[0,2],[4,6],[8,10],[1,9],[1,5],[5,9]]
排序: [[0,2],[1,5],[1,9],[4,6],[5,9],[8,10]], time=10

  cur_end=0:
    可选 [0,2],[1,5],[1,9] (start <= 0)
    最远 farthest=9 → 选 [1,9], res=1, cur_end=9
  cur_end=9:
    可选 [4,6],[5,9],[8,10] (start <= 9)
    最远 farthest=10 → 选 [8,10], res=2, cur_end=10
  cur_end=10 >= time=10 ✓, 答案 2
```

```python
def videoStitching(clips, time):
    clips.sort()
    res = 0
    cur_end = 0
    farthest = 0
    i = 0
    
    while cur_end < time:
        # 找所有起点 <= cur_end 的片段中, 终点最远的
        while i < len(clips) and clips[i][0] <= cur_end:
            farthest = max(farthest, clips[i][1])
            i += 1
        
        if farthest == cur_end:      # ⚠️ 无法前进, 有缺口
            return -1
        
        cur_end = farthest           # ⚠️ 跳到最远
        res += 1
    
    return res

# ⚠️ clips[i][0] <= cur_end (不是 <):
#    起点 = cur_end 可以接上
#
# ⚠️ farthest == cur_end:
#    所有能接上的片段都无法超过 cur_end → 有缺口
#
# ⚠️ 和跳跃游戏II非常像:
#    cur_end = 当前跳跃范围, farthest = 下次能到最远
```

## 6. 区间列表的交集

### LeetCode 986

```
两组已排序的区间, 求交集

双指针: 比较两个区间
  交集 = [max(a_start, b_start), min(a_end, b_end)]
  如果 start <= end → 有交集
  右端点小的那个指针前进
```

```python
def intervalIntersection(firstList, secondList):
    i = j = 0
    res = []
    
    while i < len(firstList) and j < len(secondList):
        lo = max(firstList[i][0], secondList[j][0])
        hi = min(firstList[i][1], secondList[j][1])
        
        if lo <= hi:                  # ⚠️ 有交集
            res.append([lo, hi])
        
        # ⚠️ 右端点小的先走
        if firstList[i][1] < secondList[j][1]:
            i += 1
        else:
            j += 1
    
    return res

# ⚠️ 这不算贪心, 但是区间问题的经典双指针
```

## 本节要点速查

```
✅ 不相交: 按右端点排, s>=end选, count=保留数

✅ 区间分组: 按左端点排, 堆存各组结束时间
✅ 覆盖目标: 按左端点排, 双while找最远
✅ 关键: >=还是>, <=还是<, 看端点是否算重叠
```
