> 贪心没有固定模板，关键是识别贪心策略并证明正确性。

## 2. 按排序维度分类

```
按右端点排序:
  · 最多不相交区间 (435)
  · 最少箭覆盖 (452)
  → 选"结束最早"的

按左端点排序:
  · 合并区间 (56)
  · 区间分组 (253)
  · 最少覆盖 (1024)
  → 选"覆盖最远"的

按截止时间排序:
  · 课程表III (630)
  → 反悔替换

按比率/某属性排序:
  · 按身高重建 (406: -h, k)
  · 雇佣工人 (857: wage/quality)

不需要排序:
  · 跳跃游戏 (55/45)
  · 加油站 (134)
  · 三元子序列 (334)
```

## 3. 常见边界条件

```
⚠️ 重叠判定: >= 还是 >
   [1,2] 和 [2,3]:
     s >= end → 不重叠 (435 无重叠区间)
     s > end → 重叠 (452 引爆气球)

⚠️ 左闭右闭 vs 左闭右开:
   看题意中端点是否被包含

⚠️ 排序稳定性:
   多维度排序时注意 key 的设计
   如 406: (-h, k), 身高降序+k升序

⚠️ 堆的正负:
   Python heapq 是最小堆
   最大堆 → 存负值
   heapreplace = pop最小 + push新值

⚠️ 特殊情况:
   空数组, 单元素, 全相同
   总量不够 → 提前返回
```

## 4. 贪心 vs DP 选择

```
贪心:
  ✅ 局部最优 = 全局最优
  ✅ 不需要记录子问题
  ✅ 时间更优 (通常 O(n log n) 或 O(n))
  ❌ 需要证明正确性
  ❌ 不一定能用

DP:
  ✅ 一定能得到最优解
  ✅ 不需要特殊证明
  ❌ 时间空间开销更大
  ❌ 实现可能更复杂

面试建议:
  先想贪心 → 如果能证明就用贪心
  证不出来 → 用 DP
  不确定 → 说出贪心思路, 提 DP 作为后备
```

---

## 5. 常见的贪心证明思路

```
1. 交换论证 (最常用):
   假设最优解 != 贪心解
   把最优解中某步替换成贪心选择
   证明结果不变或更好

2. 反证法:
   假设贪心不是最优 → 推出矛盾

3. 数学归纳:
   第k步贪心正确 → 第k+1步也正确

4. 直觉+举例:
   面试中通常够用
   模拟几个例子 → 找不到反例 → 大概率对
```
