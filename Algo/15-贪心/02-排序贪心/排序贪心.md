# 排序贪心

> 先按某个属性排序，然后贪心处理，是最常见的贪心模式。

---

## 1. 核心思想

```
排序贪心的套路:
  1. 按某个关键属性排序
  2. 按排序后的顺序, 贪心地做选择

关键: 选对排序的维度
  按左端点? 右端点? 大小? 截止时间?

常见排序策略:
  ┌──────────────────────┬──────────────────────┐
  │ 场景                 │ 排序方式             │
  ├──────────────────────┼──────────────────────┤
  │ 满足尽量多           │ 从小到大排序         │
  │ 合并区间             │ 按左端点排序         │
  │ 不相交区间           │ 按右端点排序         │
  │ 截止时间限制         │ 按截止时间排序       │
  │ 两个维度排序         │ 一维降序一维升序     │
  └──────────────────────┴──────────────────────┘
```

## 2. 分发饼干

### LeetCode 455

```
每个孩子有胃口值 g[i], 每块饼干有大小 s[j]
s[j] >= g[i] 才能满足孩子 i
每块饼干只能给一个孩子, 求最多满足几个孩子

贪心: 给胃口最小的孩子分最小的能满足的饼干
  排序 g 和 s, 双指针匹配
```

```python
def findContentChildren(g, s):
    g.sort()                     # ⚠️ 胃口从小到大
    s.sort()                     # ⚠️ 饼干从小到大
    child = cookie = 0
    
    while child < len(g) and cookie < len(s):
        if s[cookie] >= g[child]:
            child += 1           # ⚠️ 匹配成功, 孩子满足
        cookie += 1              # ⚠️ 饼干总是消耗掉
    
    return child

# ⚠️ 为什么贪心是对的?
#    如果一块大饼干给了胃口小的孩子, 可能浪费
#    如果一块小饼干不够, 跳过(不能满足任何更大胃口)
#    所以: 小饼干→小胃口, 大饼干→大胃口
```

## 3. 合并区间

### LeetCode 56

```
给定区间集合, 合并所有重叠的区间

intervals = [[1,3],[2,6],[8,10],[15,18]]
→ [[1,6],[8,10],[15,18]]

按左端点排序后, 逐个检查能否合并:
  [1,3] → merged = [[1,3]]
  [2,6] → 2 <= 3 → 合并 → [[1,6]]
  [8,10] → 8 > 6 → 新区间 → [[1,6],[8,10]]
  [15,18] → 15 > 10 → 新区间 → [[1,6],[8,10],[15,18]]
```

```python
def merge(intervals):
    intervals.sort()                  # ⚠️ 按左端点排序
    merged = [intervals[0]]
    
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:    # ⚠️ 有重叠 (<=不是<)
            merged[-1][1] = max(merged[-1][1], end)  # ⚠️ 右端点取max
        else:
            merged.append([start, end])
    
    return merged

# ⚠️ start <= merged[-1][1]: 
#    [1,3] 和 [3,5] 算重叠 (共享端点3)
#    如果题目说不算, 改成 <
#
# ⚠️ max(merged[-1][1], end): 不能直接赋值 end
#    因为可能 [1,6] 和 [2,4], end=4 < 6
```

## 4. 按身高重建队列

### LeetCode 406

```
people[i] = [h, k]
h = 身高, k = 前面有几个身高 >= h 的人
打乱后重建队列

贪心: 按身高降序(同身高按k升序)排序, 然后按k插入
  大个子先站好位, 小个子插入不影响大个子的计数

people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
排序: [[7,0],[7,1],[6,1],[5,0],[5,2],[4,4]]
插入:
  [7,0] → insert(0): [[7,0]]
  [7,1] → insert(1): [[7,0],[7,1]]
  [6,1] → insert(1): [[7,0],[6,1],[7,1]]
  [5,0] → insert(0): [[5,0],[7,0],[6,1],[7,1]]
  [5,2] → insert(2): [[5,0],[7,0],[5,2],[6,1],[7,1]]
  [4,4] → insert(4): [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
```

```python
def reconstructQueue(people):
    # ⚠️ 按身高降序, 同身高按 k 升序
    people.sort(key=lambda x: (-x[0], x[1]))
    result = []
    for p in people:
        result.insert(p[1], p)    # ⚠️ 按 k 值插入
    return result

# ⚠️ 排序关键: (-h, k)
#    身高降序: 先处理高的, 矮的插入不影响高的计数
#    同身高按k升序: k=0的先插, k=1的后插
#
# ⚠️ insert 操作: O(n), 总时间 O(n²)
#    面试中可以接受
```

## 5. 任务调度器

### LeetCode 621

```
CPU 任务列表 tasks, 冷却时间 n
相同任务之间至少间隔 n 个时间单位
求最少需要多少时间

贪心: 先算出现次数最多的任务
  最多次数 maxCount, 有 maxNum 种任务出现 maxCount 次

  最短时间 = max(len(tasks), (maxCount-1)*(n+1) + maxNum)

例: tasks=[A,A,A,B,B,B], n=2
  maxCount=3, maxNum=2 (A和B都是3次)
  (3-1)*(2+1)+2 = 8
  A B _ A B _ A B → 8 个时间单位
```

```python
from collections import Counter

def leastInterval(tasks, n):
    counts = Counter(tasks)
    max_count = max(counts.values())
    max_num = sum(1 for v in counts.values() if v == max_count)
    
    return max(len(tasks), (max_count - 1) * (n + 1) + max_num)

# ⚠️ 两种情况取 max:
#    1. (maxCount-1)*(n+1)+maxNum: 有足够的空闲时间
#    2. len(tasks): 任务种类多, 完全没有空闲
#
# ⚠️ 公式理解:
#    maxCount-1 组, 每组 n+1 个位置 (任务+冷却)
#    最后一组只需要 maxNum 个位置 (最后一批任务)
```
