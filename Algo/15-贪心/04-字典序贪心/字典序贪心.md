# 字典序贪心

> 字典序贪心常结合单调栈，在保证约束条件下逐位选最优字符/数字。

---

## 1. 核心思想

```
字典序最小/最大 → 每一位尽量选小/大的字符
约束: 剩余字符足够填满后续位置

工具: 单调栈
  维护一个尽量递增(最小)或递减(最大)的栈
  如果当前字符更优且栈顶后面还会出现 → 弹出栈顶

决策要素:
  1. 当前字符 vs 栈顶: 当前更小 → 想弹出栈顶
  2. 栈顶还会出现吗? 不会 → 不能弹
  3. 删除次数用完了吗? 用完 → 不能弹
```

## 2. 移掉K位数字

### LeetCode 402

```
给定数字字符串, 移掉 k 位使剩余数字最小

num = "1432219", k = 3

单调栈过程:
  1 → [1]
  4 → [1,4]       (4>1, 但不弹: 4更大没关系)
  3 → 4>3, pop4, k=2 → [1,3]
  2 → 3>2, pop3, k=1 → [1,2]
  2 → [1,2,2]
  1 → 2>1, pop2, k=0 → [1,2,1]
  9 → [1,2,1,9]   (k=0 不再弹)
  
  结果: "1219"
```

```python
def removeKdigits(num, k):
    stack = []
    for d in num:
        while k and stack and stack[-1] > d:
            stack.pop()
            k -= 1               # ⚠️ 每弹出一次消耗一次删除机会
        stack.append(d)
    
    # ⚠️ k 还有剩余, 从尾部删除
    stack = stack[:len(stack) - k]
    
    # ⚠️ 去除前导零
    return ''.join(stack).lstrip('0') or '0'

# ⚠️ while 三个条件:
#    k > 0: 还有删除次数
#    stack: 栈非空
#    stack[-1] > d: 栈顶比当前大 (弹出能让数变小)
#
# ⚠️ stack[:len-k]:
#    如果遍历完 k 还没用完 (递增序列)
#    例: "12345" k=2 → 栈=[1,2,3,4,5] → 截断后3个 → "123"
#
# ⚠️ lstrip('0'):
#    "00200" → "200"
#    如果全是0 → "" → or '0' → "0"
```

## 3. 去除重复字母 / 不同字符的最小子序列

### LeetCode 316 / 1081

```
给定字符串, 去除重复字母使每个字母只出现一次
结果必须是字典序最小的

s = "cbacdcbc" → "acdb"

比 402 多一个约束: 每个字母必须保留一个
  所以弹出栈顶前要检查"后面还有没有这个字母"
```

```python
def removeDuplicateLetters(s):
    last = {c: i for i, c in enumerate(s)}   # ⚠️ 每个字符最后出现位置
    stack = []
    in_stack = set()                           # ⚠️ 记录栈中已有的字符
    
    for i, c in enumerate(s):
        if c in in_stack:
            continue                           # ⚠️ 已在栈中, 跳过
        
        while stack and stack[-1] > c and last[stack[-1]] > i:
            in_stack.remove(stack.pop())        # ⚠️ 弹出并从集合移除
        
        stack.append(c)
        in_stack.add(c)
    
    return ''.join(stack)

# ⚠️ while 三个条件:
#    stack: 栈非空
#    stack[-1] > c: 栈顶字典序更大
#    last[stack[-1]] > i: 栈顶字符后面还会出现 (可以安全弹出)
#
# ⚠️ in_stack 集合:
#    如果字符已在栈中, 不需要再加入 (每个只保留一个)
#    跳过而不是替换
#
# ⚠️ 和 402 的区别:
#    402: 弹出条件是 k > 0 (有删除次数)
#    316: 弹出条件是 last[top] > i (后面还有这个字符)
```

## 4. 最小数字拼接

### LeetCode 179: 最大数 (反向问题)

```
给定非负整数数组, 拼接成最大的数字
[3, 30, 34, 5, 9] → "9534330"

贪心: 自定义排序
  比较 a+b 和 b+a, 取更大的排前面
  "3" + "30" = "330" > "303" = "30" + "3" → 3 排前面
```

```python
from functools import cmp_to_key

def largestNumber(nums):
    # ⚠️ 自定义比较: 用字符串拼接比较
    def compare(a, b):
        if a + b > b + a:
            return -1             # a 排前面
        elif a + b < b + a:
            return 1              # b 排前面
        else:
            return 0
    
    strs = [str(x) for x in nums]
    strs.sort(key=cmp_to_key(compare))
    
    result = ''.join(strs)
    return '0' if result[0] == '0' else result  # ⚠️ 全0的情况

# ⚠️ 为什么比较 a+b 和 b+a?
#    直接比较 a 和 b 不行: "3" > "30" 但 "303" < "330"
#    拼接后比较才是正确的字典序
#
# ⚠️ 结果 "000...0" → 输出 "0"
```

## 5. 拼接最大数

### LeetCode 321 (Hard)

```
从 nums1 和 nums2 中共选 k 个数, 组成最大的数
要求保持各数组内的相对顺序

思路: 枚举分配
  从 nums1 选 i 个 + 从 nums2 选 k-i 个
  每个数组内用单调栈选最大子序列
  两个子序列合并取最大
```

```python
def maxNumber(nums1, nums2, k):
    def max_subseq(nums, length):
        """单调栈: 从 nums 中选 length 个数组成最大子序列"""
        drop = len(nums) - length       # ⚠️ 可以扔掉的个数
        stack = []
        for x in nums:
            while drop and stack and stack[-1] < x:
                stack.pop()
                drop -= 1
            stack.append(x)
        return stack[:length]            # ⚠️ 截断到 length
    
    def merge(a, b):
        """合并两个序列, 每次选较大的前缀的第一个"""
        res = []
        while a or b:
            if a >= b:                   # ⚠️ 列表比较: 字典序
                res.append(a[0])
                a = a[1:]
            else:
                res.append(b[0])
                b = b[1:]
        return res
    
    best = []
    for i in range(k + 1):
        j = k - i
        if i > len(nums1) or j > len(nums2):
            continue
        candidate = merge(max_subseq(nums1, i), max_subseq(nums2, j))
        if candidate > best:
            best = candidate
    
    return best

# ⚠️ max_subseq 和 402 完全一样的结构
#    drop = 可删除次数, stack[-1] < x 时弹出 (这里求最大所以是 <)
#
# ⚠️ merge 时比较 a >= b (列表字典序):
#    不是只比较 a[0] vs b[0]
#    例: a=[5,3], b=[5,4] → b更大, 应该先选b的5
```

## 本节要点速查

```
✅ 移掉K位: 单调递增栈, stack[-1]>d且k>0就弹
✅ 去重最小: 单调栈+last位置+in_stack集合
✅ 最大数: cmp_to_key, 比较a+b vs b+a
✅ 弹出条件三要素: 栈非空, 栈顶更差, 有机会弹(k>0或后面还有)
```
