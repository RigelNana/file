## 1. 整数拆分

### LeetCode 343

```
把正整数 n 拆分成至少两个正整数之和, 使乘积最大

数学推导:
  拆出 1: 无用, 1*x < x+1 (x>1)
  拆出 2: OK, 2*(n-2)
  拆出 3: 更好, 3*(n-3) > 2*(n-2) 当 n >= 5
  拆出 >= 4: 等价于拆成更小的, 如 4 = 2+2

结论:
  尽量拆出 3, 余数特殊处理:
  余 0: 全是 3
  余 1: 最后的 3+1=4 拆成 2×2 更优
  余 2: 保留一个 2
```

```python
def integerBreak(n):
    if n <= 3:
        return n - 1           # ⚠️ 特殊: 2→1, 3→2
    
    q, r = divmod(n, 3)
    if r == 0:
        return 3 ** q
    elif r == 1:
        return 3 ** (q - 1) * 4   # ⚠️ 3+1=4, 拆成 2×2
    else:  # r == 2
        return 3 ** q * 2

# ⚠️ n<=3 特殊处理:
#    n=2: 必须拆成 1+1, 乘积=1 (不能不拆)
#    n=3: 必须拆成 1+2, 乘积=2 (不能直接用3)
#
# ⚠️ 为什么 3 最优?
#    AM-GM 不等式角度: n/e ≈ n/2.718
#    离 e 最近的整数是 3
#    6 = 3+3, 3×3=9 > 2+2+2, 2×2×2=8
```

## 2. 加油站

### LeetCode 134

```
环形路线上 n 个加油站
gas[i] = 第i站能加的油, cost[i] = 从i到i+1的耗油量
找到能走完一圈的起点, 无解返回 -1

贪心:
  如果总油量 < 总消耗 → 无解
  否则一定有解, 找起点:
    维护当前油量 tank
    如果 tank < 0, 说明从 start 到这里走不通
    → start 设为下一站, tank 重置

gas  = [1, 2, 3, 4, 5]
cost = [3, 4, 5, 1, 2]
diff = [-2,-2,-2, 3, 3]

  i=0: tank=-2 < 0 → start=1, tank=0
  i=1: tank=-2 < 0 → start=2, tank=0
  i=2: tank=-2 < 0 → start=3, tank=0
  i=3: tank=3
  i=4: tank=3+3=6 > 0 ✓
  sum(diff)=0 >= 0, 答案 start=3
```

```python
def canCompleteCircuit(gas, cost):
    if sum(gas) < sum(cost):
        return -1                    # ⚠️ 总油量不够, 无解
    
    start = 0
    tank = 0
    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:
            start = i + 1           # ⚠️ 从下一站重新开始
            tank = 0                # ⚠️ 重置油量
    
    return start

# ⚠️ sum(gas) >= sum(cost) → 一定有解
#    证明: 如果总油够, 一定存在某个起点能走完
#    (假设走到某站油不够, 那之前的站一定有多余的)
#
# ⚠️ 为什么 start = i+1 不用再验证?
#    因为 sum(gas) >= sum(cost) 保证了答案存在
#    从 start 开始的后半段一定够(否则总量会不够)
#
# ⚠️ tank < 0 的含义:
#    从 start 到 i 这一段总消耗 > 总加油
#    start 到 i 中的任何一站开始都不行
#    (因为 start 到那站的累计 >= 0, 后面只会更差)
```

## 3. 分发糖果

### LeetCode 135

```
n 个孩子站一排, ratings[i] 是评分
每个孩子至少 1 颗糖, 评分更高的比邻居多
求最少总糖果数

贪心: 两次遍历
  左→右: 如果 rating[i] > rating[i-1], candy[i] = candy[i-1]+1
  右→左: 如果 rating[i] > rating[i+1], candy[i] = max(candy[i], candy[i+1]+1)

ratings = [1, 0, 2]
左→右: [1, 1, 2]    (0≤1 不增, 2>0 加1)
右→左: [2, 1, 2]    (1>0 → max(1, 1+1)=2)
总计: 2+1+2=5
```

```python
def candy(ratings):
    n = len(ratings)
    candies = [1] * n
    
    # 左→右: 保证右边大的比左边多
    for i in range(1, n):
        if ratings[i] > ratings[i - 1]:
            candies[i] = candies[i - 1] + 1
    
    # 右→左: 保证左边大的比右边多
    for i in range(n - 2, -1, -1):
        if ratings[i] > ratings[i + 1]:
            candies[i] = max(candies[i], candies[i + 1] + 1)
    
    return sum(candies)

# ⚠️ 两次遍历各管一个方向:
#    左→右: 只保证了右邻居关系
#    右→左: 保证左邻居关系, 但不能覆盖右邻居的结果
#    所以取 max: 同时满足两个方向
#
# ⚠️ ratings[i] > (不是 >=):
#    评分相同的邻居不要求谁多
```

## 4. 递增的三元子序列

### LeetCode 334

```
判断数组中是否存在递增的三元子序列 i<j<k, nums[i]<nums[j]<nums[k]

贪心: 维护最小和次小
  first = 当前最小值
  second = 比 first 大的最小值
  如果出现 > second 的 → 找到三元组
```

```python
def increasingTriplet(nums):
    first = second = float('inf')
    
    for x in nums:
        if x <= first:
            first = x            # ⚠️ 更新最小值
        elif x <= second:
            second = x           # ⚠️ 更新次小值
        else:
            return True          # ⚠️ 第三个 > second > first
    
    return False

# ⚠️ x <= first (不是 <):
#    如果 x == first, 不更新 second
#    否则 first = second = 同一个值, 不是严格递增
#
# ⚠️ first 更新后 second 还有效吗?
#    有效! second 记录的是"存在某个更早的数 < second"
#    虽然 first 可能被覆盖了, 但 second 的存在性仍成立
```
