# 反悔贪心

> 反悔贪心允许"撤销"之前的选择，当发现更优方案时用堆来管理决策。

---

## 1. 核心思想

```
普通贪心: 每步选局部最优, 不回头
反悔贪心: 每步选局部最优, 但保留"反悔"的机会

什么时候需要反悔?
  当前贪心选择可能不是全局最优
  但如果后续发现更好的选择, 可以替换之前的

工具: 优先队列 (最大堆/最小堆)
  堆中存"之前的决策"
  新决策比堆顶更优 → 替换 (反悔)

模式:
  排序 → 遍历 → 尝试选入 → 如果不满足约束:
    检查堆顶是否可以被替换 → 替换 (反悔)
```

## 2. 课程表 III

### LeetCode 630

```
courses[i] = [duration, lastDay]
  duration: 课程持续天数
  lastDay: 必须在这天之前完成
求最多能上几门课

贪心: 按截止时间排序, 依次尝试选入
  如果选入后超时, 看能否替换之前最长的课

courses = [[100,200],[200,1300],[1000,1250],[2000,3200]]
排序: 已按 lastDay 排序

  [100,200]: total=100 <= 200 ✓, push(100), heap=[-100]
  [200,1300]: total=300 <= 1300 ✓, push(200), heap=[-200,-100]
  [1000,1250]: total=1300 > 1250 ✗
    堆顶 200 < 1000? 否 → 不反悔
    堆顶 200 > 1000? 否 → 不反悔
    等等, 堆顶是最大的(200), 当前是1000
    1000 > 200 → 替换后更差, 不反悔
  [2000,3200]: total=2300 > 3200? 300+2000=2300 <= 3200? 
    total=2300 <= 3200 ✓, push(2000), heap=[-2000,-200,-100]

  答案: len(heap) = 3
```

```python
import heapq

def scheduleCourse(courses):
    courses.sort(key=lambda x: x[1])    # ⚠️ 按截止时间排序
    heap = []                            # 最大堆 (存负的 duration)
    total = 0
    
    for duration, lastDay in courses:
        if total + duration <= lastDay:
            # 放得下, 直接选
            total += duration
            heapq.heappush(heap, -duration)
        elif heap and -heap[0] > duration:
            # 放不下, 但当前课比之前最长的课短 → 反悔!
            total += duration - (-heapq.heapreplace(heap, -duration))
            # ⚠️ 等价于: total = total - old_max + duration
    
    return len(heap)

# ⚠️ 反悔条件: -heap[0] > duration
#    当前课比之前最长的课短 → 替换后总时间更少
#    总时间更少 = 更有可能满足后续课程的截止时间
#
# ⚠️ 为什么按截止时间排序?
#    保证之前选的课的截止时间 <= 当前课
#    替换后, 新的 total 更小, 之前的截止时间约束仍满足
#
# ⚠️ heapreplace: pop最小(最大堆下是最大duration) + push新值
#    一步完成, O(log n)
#
# ⚠️ 如果 -heap[0] <= duration:
#    替换后时间不会减少, 没有好处, 不反悔
```

## 3. IPO

### LeetCode 502

```
初始资本 w, 最多做 k 个项目
profits[i] = 利润, capital[i] = 需要的最低资本
求最终最大资本

贪心: 每次选当前资本能做的项目中利润最大的
  用最大堆存"当前可做的项目利润"

  按 capital 排序
  每轮: 把 capital <= w 的项目利润入堆
        从堆中取最大利润
```

```python
import heapq

def findMaximizedCapital(k, w, profits, capital):
    projects = sorted(zip(capital, profits))   # ⚠️ 按所需资本排序
    heap = []                                   # 最大堆 (存负利润)
    i = 0
    
    for _ in range(k):
        # 把当前资本能做的项目利润入堆
        while i < len(projects) and projects[i][0] <= w:
            heapq.heappush(heap, -projects[i][1])
            i += 1
        
        if not heap:
            break                               # ⚠️ 没有可做的项目
        
        w += -heapq.heappop(heap)               # ⚠️ 选利润最大的
    
    return w

# ⚠️ 这更像是"堆+贪心"而非严格的反悔贪心
#    但思路类似: 在可选集合中选最优
#
# ⚠️ 注意 w 会增加:
#    做完一个项目后 w 增加, 可能解锁新的项目
#    所以每轮都要把新解锁的项目入堆
```

## 4. 雇佣K位工人的最低费用

### LeetCode 857

```
quality[i] = 工作质量, wage[i] = 最低期望工资
雇佣 k 人, 规则:
  1. 每人报酬 >= wage[i]
  2. 报酬与质量成正比

最低费用 = min( max(wage[i]/quality[i]) * sum(quality of k people) )
  按 wage/quality 比率排序
  用最大堆维护最小的 k 个 quality
```

```python
import heapq

def mincostToHireWorkers(quality, wage, k):
    # 按 wage/quality 比率排序
    workers = sorted(zip(quality, wage), key=lambda x: x[1] / x[0])
    
    heap = []              # 最大堆 (存负quality)
    total_q = 0
    ans = float('inf')
    
    for q, w in workers:
        heapq.heappush(heap, -q)
        total_q += q
        
        if len(heap) > k:
            total_q += heapq.heappop(heap)   # ⚠️ 弹出最大quality (减少总和)
        
        if len(heap) == k:
            # 当前比率 = w/q, 费用 = 比率 * total_q
            ans = min(ans, (w / q) * total_q)
    
    return ans

# ⚠️ 按比率排序后, 当前 w/q 是所有已遍历中最大的
#    所以总费用 = 当前比率 * sum(selected quality)
#    要最小化费用 → 最小化 total_q → 堆维护最小k个quality
#
# ⚠️ 弹出最大quality = 反悔:
#    之前选了quality大的, 现在发现它太"贵"(占比高), 换出去
```

## 5. 反悔贪心 vs 普通贪心

```
┌──────────────┬──────────────────┬──────────────────┐
│              │ 普通贪心         │ 反悔贪心         │
├──────────────┼──────────────────┼──────────────────┤
│ 决策         │ 一次性, 不可撤销 │ 可以撤销替换     │
│ 工具         │ 排序+遍历       │ 排序+堆           │
│ 正确性       │ 需要严格证明     │ 通过反悔保证最优 │
│ 适用         │ 局部最优=全局    │ 局部最优≠全局    │
│ 典型题       │ 区间贪心/跳跃   │ 课程表III/IPO    │
└──────────────┴──────────────────┴──────────────────┘

⚠️ 反悔贪心的识别特征:
   1. 有容量/资源限制
   2. 需要在有限选择中取最优
   3. 后来的选择可能比之前的更好
   4. 排序后, 用堆维护"已选集合"
```
