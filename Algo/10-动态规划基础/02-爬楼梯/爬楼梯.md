# 爬楼梯

> 爬楼梯是 DP 入门经典，变体丰富，本质是斐波那契。

---

## 1. 基础版

### LeetCode 70

```
每次爬 1 或 2 级台阶, 到第 n 阶有多少种方法?

阶梯:    1    2    3    4    5
方法数:  1    2    3    5    8
              ↗ ↗   ↗ ↗   ↗ ↗
        dp[i] = dp[i-1] + dp[i-2]

到第 i 阶: 从第 i-1 阶爬 1 步 + 从第 i-2 阶爬 2 步
```

```python
# 记忆化搜索
from functools import cache

def climbStairs_memo(n):
    @cache
    def dfs(i):
        if i <= 1:
            return 1              # ⚠️ 0阶和1阶都是1种
        return dfs(i - 1) + dfs(i - 2)
    return dfs(n)

# 递推
def climbStairs(n):
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2           # ⚠️ 1阶=1种, 2阶=2种
    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# 空间优化
def climbStairs_opt(n):
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b

# ⚠️ 初始值:
#    dp[1] = 1 (只有爬1步)
#    dp[2] = 2 (1+1 或 2)
#    也可以定义 dp[0] = 1 (空集, 站在地面算一种)
```

## 2. 最小花费爬楼梯

### LeetCode 746

```
cost[i] 是第 i 阶的费用, 可以从 0 或 1 开始
每次爬 1 或 2 阶, 求到顶的最小花费

dp[i] = 到第 i 阶的最小花费
dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])

⚠️ "到顶" = 到第 n 阶 (超越最后一个台阶)
   len(cost) = n, 顶在 n, 台阶编号 0 到 n-1

cost = [10, 15, 20]
  dp[0] = 0   (从0开始免费)
  dp[1] = 0   (从1开始免费)
  dp[2] = min(dp[1]+cost[1], dp[0]+cost[0]) = min(15, 10) = 10
  dp[3] = min(dp[2]+cost[2], dp[1]+cost[1]) = min(30, 15) = 15
```

```python
def minCostClimbingStairs(cost):
    n = len(cost)
    dp = [0] * (n + 1)
    # ⚠️ dp[0] = dp[1] = 0: 可以从0或1免费开始
    for i in range(2, n + 1):
        dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
    return dp[n]

# 空间优化
def minCostClimbingStairs_opt(cost):
    a, b = 0, 0
    for i in range(2, len(cost) + 1):
        a, b = b, min(b + cost[i-1], a + cost[i-2])
    return b

# ⚠️ 另一种定义: dp[i] = 从第 i 阶出发的最小花费
#    dp[i] = cost[i] + min(dp[i+1], dp[i+2])
#    从后往前算, 答案 = min(dp[0], dp[1])
#    两种都正确, 结果相同
```

## 3. 泰波那契数

### LeetCode 1137

```
T(0) = 0, T(1) = 1, T(2) = 1
T(n) = T(n-1) + T(n-2) + T(n-3)  (当 n >= 3)

和斐波那契类似, 但依赖前三个
```

```python
def tribonacci(n):
    if n == 0: return 0
    if n <= 2: return 1
    a, b, c = 0, 1, 1
    for _ in range(3, n + 1):
        a, b, c = b, c, a + b + c     # ⚠️ 三个变量滚动
    return c

# ⚠️ 依赖几个 → 几个变量:
#    dp[i] = dp[i-1] + dp[i-2] → 2 个变量
#    dp[i] = dp[i-1] + dp[i-2] + dp[i-3] → 3 个变量
```

## 4. 组合爬楼梯

### LeetCode 377: 组合总和 IV

```
给定数组 nums 和目标 target
每次可以从 nums 中选一个数, 求到达 target 的方案数
(不同顺序算不同方案)

和爬楼梯一样! nums 中的数就是每次能爬的台阶数

dp[i] = sum(dp[i - x] for x in nums if i >= x)

例: nums = [1,2,3], target = 4
  dp[0] = 1
  dp[1] = dp[0] = 1
  dp[2] = dp[1] + dp[0] = 2
  dp[3] = dp[2] + dp[1] + dp[0] = 4
  dp[4] = dp[3] + dp[2] + dp[1] = 7
```

```python
def combinationSum4(nums, target):
    dp = [0] * (target + 1)
    dp[0] = 1                          # ⚠️ 空组合(和为0)有1种
    for i in range(1, target + 1):
        for x in nums:
            if i >= x:
                dp[i] += dp[i - x]
    return dp[target]

# ⚠️ 外层循环 target, 内层循环 nums
#    这样计算的是"排列数" (1,2 和 2,1 算两种)
#    如果反过来 (外层nums, 内层target) → "组合数"
#    这是完全背包中排列 vs 组合的区别
```

## 5. 爬楼梯变体总结

```
┌──────────────────┬────────────────────────────┐
│ 变体             │ 转移方程                   │
├──────────────────┼────────────────────────────┤
│ 基础 (1或2步)    │ dp[i] = dp[i-1] + dp[i-2] │
│ 带权 (最小花费)  │ dp[i] = min(dp[i-1]+c,     │
│                  │         dp[i-2]+c)          │
│ 多步 (1到k步)    │ dp[i] = sum(dp[i-j])       │
│                  │         j=1..k              │
│ 组合 (任选nums)  │ dp[i] = sum(dp[i-x])       │
│                  │         x in nums           │
└──────────────────┴────────────────────────────┘

⚠️ 爬楼梯本质: "到达 i 的方案数/最优值"
   变化的是: 每步的选择范围和代价
```
