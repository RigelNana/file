# DP 方法论总结

> 从记忆化搜索到递推的完整转化方法，涵盖空间优化和调试技巧。

---

## 1. 记忆化搜索 → 递推 的转换

```
记忆化搜索                              递推
┌──────────────────────┐      ┌──────────────────────┐
│ from functools import │      │ dp = [0] * (n + 1)   │
│   cache               │      │                      │
│                       │      │ # base case          │
│ @cache                │  →   │ dp[0] = ...          │
│ def dfs(i):           │      │ dp[1] = ...          │
│   if i <= 1:          │      │                      │
│     return base       │      │ for i in range(2, n+1):│
│   return dfs(i-1)     │      │   dp[i] = dp[i-1]   │
│         + dfs(i-2)    │      │          + dp[i-2]   │
│                       │      │                      │
│ return dfs(n)         │      │ return dp[n]         │
└──────────────────────┘      └──────────────────────┘

转换规则:
  dfs(i) → dp[i]
  dfs 的参数 → dp 的维度 (下标)
  base case → dp 初始值
  递归调用 → 循环中的赋值
  return dfs(n) → return dp[n]

⚠️ 循环方向: 递推要保证计算 dp[i] 时, 依赖的值已经算好
   通常是 从小到大 (和记忆化搜索的调用顺序反过来)
```

## 2. 多维 DP 的转换

```python
# 二维记忆化搜索
@cache
def dfs(i, j):
    if base_case:
        return ...
    return min(dfs(i-1, j), dfs(i, j-1)) + cost

# 转换: dfs(i, j) → dp[i][j]
# 两个参数 → 二维数组, 两层循环

dp = [[0] * n for _ in range(m)]
# base case...
for i in range(m):
    for j in range(n):
        dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + cost[i][j]

# ⚠️ 多维 DP 的空间优化:
#    如果 dp[i][j] 只依赖 dp[i-1][...] (上一行)
#    → 可以用一维滚动数组
#    如果还依赖 dp[i][j-1] (同行左边)
#    → 也可以, 因为 j 从小到大, dp[j-1] 已更新
```

---

## 3. 空间优化三种方式

```
方式1: 滚动变量 (1D 只依赖相邻)
  dp[i] 只依赖 dp[i-1] 和 dp[i-2]
  → 两个变量 a, b

  a, b = base0, base1
  for i in range(2, n+1):
      a, b = b, func(a, b)

方式2: 滚动数组 (2D → 1D)
  dp[i][j] 只依赖 dp[i-1][j] 和 dp[i][j-1]
  → 一维数组, 每行更新

  dp = [0] * n
  for i in range(m):
      for j in range(n):
          dp[j] = func(dp[j], dp[j-1])  # dp[j]旧值=上一行

方式3: 奇偶滚动 (2D → 2×n)
  dp[i][j] 依赖 dp[i-1][j] 和 dp[i-2][j]
  → 只保留两行: dp[i%2][j]

  dp = [[0]*n for _ in range(2)]
  for i in range(m):
      for j in range(n):
          dp[i%2][j] = func(dp[(i-1)%2][j], ...)

⚠️ 空间优化后无法回溯路径
   如果需要输出方案 (不只是最优值), 保留完整 dp 表
```

## 5. DP 调试技巧

```
1. 先写暴力/记忆化, 和递推对拍
   跑小数据, 两种方法结果一样才对

2. 打印 DP 表
   for row in dp: print(row)
   看每个状态的值是否符合预期

3. 手动推导
   拿题目的 example, 在纸上手算 dp 表
   和代码的值逐一比较

4. 检查三个地方
   ① 初始值对不对?
   ② 转移方程对不对?
   ③ 循环范围对不对? (是否漏了边界)

5. 常见 bug
   ⚠️ dp 数组大小差 1 (n vs n+1)
   ⚠️ 初始值赋错 (0 vs 1, inf vs -inf)
   ⚠️ 循环方向错 (从小到大 vs 从大到小)
   ⚠️ 转移时下标越界 (i-1 当 i=0)
```

## 6. 从思路到代码的流程

```
Step 1: 读题 → 判断 DP 类型
  "最大/最小/方案数" + "子结构" → DP

Step 2: 定义 dfs(i, ...)
  参数 = 还没确定的信息
  返回值 = 题目要求的量

Step 3: 写 base case
  最小子问题, 递归终止条件

Step 4: 写转移
  "选或不选" 或 "选哪个"

Step 5: 测试记忆化搜索

Step 6: 转换为递推 (可选)
  dfs → dp, 改循环

Step 7: 空间优化 (可选)
  看依赖关系, 降维

⚠️ 不要一步到位写递推 + 空间优化
   先确保逻辑正确, 再优化
```

```
│ 状压 DP          │ O(2^n · n)   │ O(2^n)    │
└──────────────────┴───────────────┴───────────┘
```
