# DP 核心思想

> 动态规划的本质是"记住已解决的子问题"，避免重复计算。

---

## 1. 暴力递归的问题

```
斐波那契 fib(5) 的递归树:

              fib(5)
             /      \
          fib(4)   fib(3)       ← fib(3) 重复!
          /   \     /   \
       fib(3) fib(2) fib(2) fib(1)   ← fib(2) 重复!
       /  \
    fib(2) fib(1)

问题: fib(3) 被计算了 2 次, fib(2) 被计算了 3 次
时间复杂度: O(2^n) ← 指数级, 不可接受

解决: 把计算过的结果存起来
  第一次计算 fib(3) → 存下来
  第二次需要 fib(3) → 直接查表

优化后: O(n) ← 线性
```

## 2. 两种实现方式

```
┌─────────────────────────────────────────┐
│  自顶向下 (记忆化搜索 / Memoization)     │
│                                         │
│  从大问题出发, 递归分解为小问题           │
│  用 @cache 或 dict 记录已计算的结果       │
│  "需要什么就算什么"                      │
│                                         │
│  @cache                                │
│  def dfs(i):                            │
│      if i <= 1: return i                │
│      return dfs(i-1) + dfs(i-2)         │
├─────────────────────────────────────────┤
│  自底向上 (递推 / Tabulation)            │
│                                         │
│  从最小的子问题出发, 逐步推到大问题       │
│  用数组 dp[] 存储结果                    │
│  "从小到大依次算"                        │
│                                         │
│  dp[0] = 0                              │
│  dp[1] = 1                              │
│  for i in range(2, n+1):               │
│      dp[i] = dp[i-1] + dp[i-2]         │
└─────────────────────────────────────────┘

两者等价! 选哪个?
  记忆化: 代码更直观, 先写递归再加 @cache
  递推:   可以做空间优化 (滚动数组)
  建议:   先写记忆化, 再转递推, 最后优化空间
```

## 3. DP 三步走

```
解任何 DP 题的三步:

1. 定义状态
   dp[i] 表示什么?
   ( 或 dfs(i) 返回什么? )

   例: dp[i] = 到达第 i 阶台阶的方法数
       dp[i] = 前 i 个物品能获得的最大价值

2. 状态转移方程
   dp[i] 怎么从更小的子问题推导?

   例: dp[i] = dp[i-1] + dp[i-2]        (爬楼梯)
       dp[i] = max(dp[i-1], dp[i-2]+x)  (打家劫舍)

3. 初始条件 (Base Case)
   最小的子问题答案是什么?

   例: dp[0] = 1, dp[1] = 1             (爬楼梯)
       dp[0] = nums[0]                  (打家劫舍)

⚠️ 第一步最关键: 状态定义错了, 后面全白搭
   技巧: 想象"如果有人告诉你子问题的答案, 你能推出原问题吗?"
```

## 4. 记忆化搜索 vs 递推

```python
# ========== 记忆化搜索 ==========
from functools import cache

def solve_memo(n):
    @cache
    def dfs(i):
        if i <= 1:
            return i              # base case
        return dfs(i - 1) + dfs(i - 2)  # 递归
    return dfs(n)

# ========== 递推 ==========
def solve_dp(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[0], dp[1] = 0, 1          # base case
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]  # 转移
    return dp[n]

# ========== 空间优化 ==========
def solve_opt(n):
    if n <= 1:
        return n
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b          # 只保留两个变量
    return b

# ⚠️ 转换关系:
#    dfs(i) → dp[i]
#    dfs 参数 → dp 下标/维度
#    base case → dp 初始值
#    递归调用 → for 循环中的赋值
#
# ⚠️ 空间优化的前提:
#    dp[i] 只依赖 dp[i-1] 和 dp[i-2]
#    → 两个变量就够了
#    如果依赖 dp[i-k] → 需要 k 个变量或滚动数组
```

## 5. DP 的适用条件

```
什么问题可以用 DP?

1. 最优子结构
   大问题的最优解 包含 子问题的最优解
   "做了最优选择后, 剩下的问题也要最优解"

2. 重叠子问题
   子问题会被重复计算
   (如果没有重复 → 分治就够了, 不需要 DP)

3. 无后效性
   "当前状态确定了, 后续决策与之前如何到达无关"
   dp[i] 只取决于 dp[0..i-1], 不取决于如何到达 dp[i]

常见 DP 问题类型:
  ┌──────────────┬──────────────────────┐
  │ 类型         │ 例子                 │
  ├──────────────┼──────────────────────┤
  │ 计数         │ 爬楼梯、不同路径     │
  │ 最优化       │ 打家劫舍、最大子数组 │
  │ 判断         │ 能否凑出某个值       │
  │ 序列         │ LIS、LCS             │
  │ 区间         │ 戳气球、矩阵连乘     │
  │ 背包         │ 01背包、完全背包     │
  │ 状态机       │ 买卖股票             │
  └──────────────┴──────────────────────┘
```

## 本节要点速查

```
✅ DP = 记住子问题答案, 避免重复计算
✅ 三步: 定义状态 → 转移方程 → 初始条件
✅ 记忆化: @cache + 递归, 直观简单
✅ 递推: for循环 + dp数组, 可以优化空间
✅ 空间优化: dp[i]只依赖最近几个→用变量代替数组
✅ 先写记忆化, 再转递推, 最后优化空间
```
