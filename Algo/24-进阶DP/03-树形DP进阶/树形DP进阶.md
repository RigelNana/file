## 2. LeetCode 968: 监控二叉树

```
在二叉树节点上放摄像头, 每个摄像头可以覆盖 父/自身/子
求最少摄像头数量, 使所有节点被覆盖

状态设计:
  每个节点返回 3 种状态:
    0 — 未覆盖 (需要父节点的摄像头来覆盖)
    1 — 有摄像头
    2 — 已被覆盖 (被子节点的摄像头覆盖)

  贪心规则 (后序遍历, 自底向上):
    ┌─────────────────────────────────────────────┐
    │ 子返回 0 (未覆盖) → 必须在当前放摄像头 → 返1│
    │ 子返回 1 (有相机) → 当前已被覆盖 → 返回 2   │
    │ 子返回 2 (已覆盖) → 当前未覆盖 → 返回 0     │
    └─────────────────────────────────────────────┘

  ⚠️ null 节点返回 2 (已覆盖)
     这样叶子节点返回 0, 叶子的父亲放摄像头
     摄像头尽量放在叶子的父亲 → 贪心最优
```

```python
def minCameraCover(root):
    cameras = 0
    
    def dfs(node):
        nonlocal cameras
        if not node:
            return 2                       # ⚠️ null → 已覆盖
        
        left = dfs(node.left)
        right = dfs(node.right)
        
        if left == 0 or right == 0:        # ⚠️ 有子未覆盖 → 必须放
            cameras += 1
            return 1
        
        if left == 1 or right == 1:        # ⚠️ 有子有相机 → 已覆盖
            return 2
        
        return 0                           # ⚠️ 都是 2 → 自己未覆盖
    
    if dfs(root) == 0:                     # ⚠️ 根节点未覆盖 → 补一个
        cameras += 1
    
    return cameras

# ⚠️ 易错:
#  1. null返回2, 不是0! 
#     如果null返回0 → 所有叶子都放相机, 浪费
#  2. 根节点可能返回0, 需要额外检查
#  3. 优先级: 未覆盖(0) > 有相机(1) > 已覆盖(2)
#     只要一个子未覆盖, 就必须放, 不管另一个子的状态
```

## 3. LeetCode 124: 二叉树中的最大路径和

```
路径: 从某节点到另一节点, 经过的节点序列
       每个节点最多经过一次
       路径可以不经过根

求最大路径和

关键: 区分 "经过当前节点的路径" 和 "向上贡献的路径"

  ┌─────────────────────────────────────────────┐
  │ 节点 x 的两种视角:                           │
  │                                             │
  │ 1. 作为路径拐点 (最终答案候选):               │
  │    path = left贡献 + node.val + right贡献    │
  │    两条臂都可以用                             │
  │                                             │
  │ 2. 向父节点贡献 (供父节点使用):               │
  │    contribution = node.val + max(left, right)│
  │    只能选一条臂! (不能拐弯)                   │
  │                                             │
  │ ⚠️ 贡献为负数时, 取 0 (不选这条路径)          │
  └─────────────────────────────────────────────┘
```

```python
def maxPathSum(root):
    ans = float('-inf')
    
    def dfs(node):
        nonlocal ans
        if not node:
            return 0
        
        left = max(dfs(node.left), 0)      # ⚠️ 负贡献取0 (不选)
        right = max(dfs(node.right), 0)     # ⚠️ 同上
        
        # 当前节点作为拐点的路径和
        path_sum = left + node.val + right  # ⚠️ 两臂 + 自身
        ans = max(ans, path_sum)
        
        # 向父节点贡献: 只能选一条臂
        return node.val + max(left, right)  # ⚠️ 只走一边!
    
    dfs(root)
    return ans

# ⚠️ 易错:
#  1. 向上贡献只能选一条臂, 不能 left + right
#     因为路径不能分叉
#  2. 负贡献取 0, 意味着不走这条子路径
#     但 node.val 本身可以为负! ans 可以是负数
#  3. ans 初始化为 -inf, 不能是 0
#     因为所有节点值可能都是负数
```

## 4. LeetCode 979: 在二叉树中分配硬币

```
n 个节点, 共 n 枚硬币, 每个节点 node.val 枚
每步移动一枚到相邻节点
求最少移动次数, 使每个节点恰好 1 枚

DFS 时每个子树返回 "溢出量":
  excess = 子树总硬币 - 子树节点数
  正: 有多余硬币需要送出
  负: 缺硬币需要从外部获取
  移动次数 += |excess|
```

```python
def distributeCoins(root):
    moves = 0
    
    def dfs(node):
        nonlocal moves
        if not node:
            return 0
        
        left_excess = dfs(node.left)
        right_excess = dfs(node.right)
        
        moves += abs(left_excess) + abs(right_excess)  # ⚠️ 取绝对值
        
        return node.val + left_excess + right_excess - 1
        # ⚠️ 自己有 node.val 枚, 加上子树溢出, 减去自己需要的 1 枚
    
    dfs(root)
    return moves

# ⚠️ 为什么 |excess| 就是经过这条边的移动次数?
#    因为子树多出 k 枚就要送出 k 次 (k>0)
#    子树缺 k 枚就要送入 k 次 (k<0, |k| 次)
#    每次移动经过一条边 = 移动数 +1
```

## 5. 树形DP进阶通用模式

```
模式1: 多状态返回
  dfs(node) → 返回多个状态值 (元组)
  例: (不选, 选, 必选) 的最优值

模式2: 路径汇聚
  dfs(node) → 返回"向上的单链贡献"
  在 dfs 内部计算 "拐弯路径" 更新全局答案

模式3: 溢出/流量
  dfs(node) → 返回"净流量"
  经过边的流量 = |净流量| = 移动开销

模式4: 贪心状态机
  每个节点根据子节点状态决定自己的状态
  后序遍历天然支持自底向上
```

## 本节要点速查

```
✅ 监控: null→2, 子0→放相机(1), 子1→已覆盖(2), 都2→未覆盖(0)
✅ 路径和: 拐弯路径更新ans, 单链贡献向上返回
✅ 负贡献取0, ans初始化-inf
✅ 硬币分配: |excess| 就是边上移动次数
✅ 向上贡献只能选一条臂 (不能分叉)
✅ 根节点可能需要特殊处理 (968补相机, 124无需)
```
