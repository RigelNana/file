# 滑动窗口与双指针总结

> 选对模型是关键：定长/不定长/相向/同向，每种有固定模板。

---

## 1. 全类型决策树

```
问题涉及连续子数组/子串?
├── 是
│   ├── 窗口大小固定? → 定长滑动窗口
│   ├── 求最长?       → 不定长窗口 (不满足时收缩)
│   ├── 求最短?       → 不定长窗口 (满足时收缩)
│   ├── 求个数?       → atMost(k) - atMost(k-1)
│   └── 求最大面积?   → 相向双指针
├── 否, 涉及排序数组两数/三数之和?
│   └── 相向双指针
├── 否, 原地修改数组?
│   └── 同向双指针 (快慢指针)
└── 否, 判断子序列?
    └── 同向双指针
```

## 2. 模板速查

### 定长窗口

```python
for i in range(len(nums)):
    window += nums[i]
    if i < k - 1: continue           # 窗口没满
    ans = max(ans, window)
    window -= nums[i - k + 1]        # ⚠️ i-k+1 不是 i-k
```

### 不定长求最长

```python
for right in range(len(nums)):
    更新state(nums[right])
    while 不满足条件:                 # ⚠️ 不满足时收缩
        撤销state(nums[left])
        left += 1
    ans = max(ans, right - left + 1)  # ⚠️ while 外更新
```

### 不定长求最短

```python
for right in range(len(nums)):
    更新state(nums[right])
    while 满足条件:                   # ⚠️ 满足时收缩
        ans = min(ans, right - left + 1)  # ⚠️ while 内更新
        撤销state(nums[left])
        left += 1
```

### 求子数组个数

```python
# exactly(k) = atMost(k) - atMost(k-1)
def atMost(k):
    for right in range(len(nums)):
        更新state
        while state > k:
            撤销state(nums[left])
            left += 1
        ans += right - left + 1       # ⚠️ 加个数不是取max
```

### 相向双指针

```python
left, right = 0, len(nums) - 1
while left < right:                   # ⚠️ < 不是 <=
    if 条件满足: 记录/返回
    elif 需要增大: left += 1
    else: right -= 1
```

### 同向快慢指针

```python
slow = 0                             # 或 slow = 1 (保留第一个)
for fast in range(len(nums)):
    if 保留条件:
        nums[slow] = nums[fast]       # 或交换
        slow += 1
return slow                          # 结果长度
```

## 3. 常见易错点

```
❌ 定长窗口: i-k 而非 i-k+1
  移出左端下标 = i - k + 1 (当前窗口的左端)

❌ 求最长 while 条件写反
  求最长: while 不满足 → 收缩
  求最短: while 满足   → 收缩
  写反 → 要么不收缩, 要么永远收缩

❌ 求最短 ans 在 while 外更新
  求最长: while 外更新 (确保合法)
  求最短: while 内更新 (满足就记录)

❌ Counter 比较不删 0 值
  Counter({'a':1, 'b':0}) != Counter({'a':1})
  必须 del cnt[key] when cnt[key] == 0

❌ 子数组个数误用 max
  ans += right - left + 1 (不是 max!)
  是累加以 right 结尾的子数组个数

❌ 相向双指针用 <=
  while left < right, 不是 <=
  <= 时 left == right, 同一元素用两次

❌ 合并有序数组从前往后
  从后往前填! 否则覆盖未处理元素
```

## 4. 时间复杂度

```
┌──────────────────┬────────────┬───────────┐
│ 技巧             │ 时间       │ 空间       │
├──────────────────┼────────────┼───────────┤
│ 定长窗口         │ O(n)       │ O(1)~O(n) │
│ 不定长窗口       │ O(n)       │ O(1)~O(n) │
│ 子数组个数       │ O(n)       │ O(1)~O(n) │
│ 相向双指针       │ O(n)       │ O(1)      │
│ 三数之和         │ O(n²)      │ O(1)      │
│ 同向双指针       │ O(n)       │ O(1)      │
└──────────────────┴────────────┴───────────┘

⚠️ 滑窗为什么是 O(n)?
  left 和 right 都只向右移动, 各最多移 n 次
  总操作 ≤ 2n = O(n)
  即使 while 内 left 可能移很多次, 总共也只移 n 次!
```

## 5. 题型识别速查

```
"最长无重复子串"         → 不定长求最长 (LC3)
"最短覆盖子串"           → 不定长求最短 (LC76)
"和 ≥ target 的最短子数组" → 不定长求最短 (LC209)
"恰好 k 个不同整数"      → atMost(k)-atMost(k-1) (LC992)
"两数之和已排序"         → 相向双指针 (LC167)

"原地去重"               → 同向快慢指针 (LC26)
"移动零"                 → 同向快慢指针 (LC283)
"字母异位词"             → 定长窗口+频率 (LC438)
```

## 本节要点速查

```
✅ 定长: 加右减左, i-k+1是左端
✅ 求最长: while不满足→收缩, ans在while外
✅ 求最短: while满足→收缩, ans在while内
✅ 求个数: exactly(k)=atMost(k)-atMost(k-1)
✅ 相向: left<right, 移动"不够优"的那个指针
✅ 同向: fast遍历, slow写入, 最终slow=结果长度
✅ 滑窗O(n): left和right各最多移n次
```
