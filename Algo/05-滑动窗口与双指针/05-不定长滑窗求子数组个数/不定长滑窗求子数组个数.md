# 不定长滑动窗口 — 求子数组个数

> 核心技巧：恰好等于 k = 至多 k - 至多 (k-1)。直接求"恰好"很难用滑窗，但"至多"可以。

---

## 1. 核心转化

```
恰好等于 k ← 很难直接滑窗!

转化:
  exactly(k) = atMost(k) - atMost(k - 1)

atMost(k): 窗口内条件值 ≤ k 的所有子数组个数
  对于每个 right, 满足条件的子数组有:
  [left, right], [left+1, right], ..., [right, right]
  共 right - left + 1 个

  ⚠️ right - left + 1 是以 right 结尾的合法子数组个数
     不是窗口长度用来更新 max/min, 而是直接加到 ans 里!
```

### 代码模板

```python
def at_most_k(nums, k):
    left = 0
    ans = 0
    state = 0                         # (如不同元素个数)
    
    for right in range(len(nums)):
        更新state(nums[right])
        
        while state > k:             # ⚠️ > k 时收缩
            撤销state(nums[left])
            left += 1
        
        ans += right - left + 1       # ⚠️ 加个数, 不是取 max!
    
    return ans

def exactly_k(nums, k):
    return at_most_k(nums, k) - at_most_k(nums, k - 1)

# ⚠️ ans += right - left + 1 的含义:
#   以 right 结尾、左端在 [left, right] 的子数组
#   全部满足条件, 共 right - left + 1 个
#
# ⚠️ 如果直接求"恰好 k":
#   窗口内恰好有 k 种元素时, 无法用 while 收缩
#   因为收缩后可能变成 k-1, 也可能还是 k
```

## 2. K 个不同整数的子数组

### LeetCode 992

### 理解与可视化

```
输入: nums = [1,2,1,2,3], k = 2
输出: 7 → [1,2],[2,1],[1,2],[2,3],[1,2,1],[2,1,2],[1,2,1,2]

exactly(2) = atMost(2) - atMost(1)

atMost(2)= [1]+[1,2]+[2]+[2,1]+[1,2,1]+[1]+[1,2]+[2]+[2,3]+[1,2,3去掉左变成...
实际计算:
  right=0: [1] → +1          acc=1
  right=1: [1,2],[2] → +2    acc=3
  right=2: [1,2,1],[2,1],[1] → +3  acc=6
  right=3: [1,2,1,2],[2,1,2],[1,2],[2] → +4  acc=10
  right=4: 加入3, 种类=3>2, 收缩到left=3
           [2,3],[3] → +2    acc=12

atMost(1):
  只有连续相同元素的子数组
  [1],[2],[1],[2],[3] → acc=5

exactly(2) = 12 - 5 = 7 ✓
```

```python
def subarraysWithKDistinct(nums, k):
    def at_most(k):
        left = 0
        cnt = {}
        ans = 0
        for right in range(len(nums)):
            cnt[nums[right]] = cnt.get(nums[right], 0) + 1
            
            while len(cnt) > k:       # ⚠️ 不同种类 > k
                left_val = nums[left]
                cnt[left_val] -= 1
                if cnt[left_val] == 0:
                    del cnt[left_val] # ⚠️ 必须删除 0 值
                left += 1
            
            ans += right - left + 1
        return ans
    
    return at_most(k) - at_most(k - 1)

# ⚠️ del cnt[left_val] 必须做!
#   否则 len(cnt) 不会减少, while 永远执行
# ⚠️ at_most 函数调用两次: O(n) + O(n) = O(n)
```

## 3. 统计优美子数组

### LeetCode 1248

```
输入: nums = [1,1,2,1,1], k = 3
输出: 2 → 包含恰好3个奇数的子数组

转化: 把奇偶映射为 0/1
  奇数 → 1, 偶数 → 0
  问题变成: 恰好有 k 个 1 的子数组个数
  exactly(k) = atMost(k) - atMost(k-1)
```

```python
def numberOfSubarrays(nums, k):
    def at_most(k):
        left = 0
        odd_count = 0
        ans = 0
        for right in range(len(nums)):
            if nums[right] % 2 == 1:
                odd_count += 1
            while odd_count > k:
                if nums[left] % 2 == 1:
                    odd_count -= 1
                left += 1
            ans += right - left + 1
        return ans
    
    return at_most(k) - at_most(k - 1)

# ⚠️ 不需要真的把数组转成 0/1
#    只需在 state 中统计奇数个数
```

## 4. 元素和小于等于阈值的子数组数

```
问题: 求子数组和 ≤ threshold 的子数组个数
  (假设元素都是正数)

这就是 atMost(threshold), 不需要减!

  for right in range(n):
      cur_sum += nums[right]
      while cur_sum > threshold:
          cur_sum -= nums[left]
          left += 1
      ans += right - left + 1
```

## 5. 什么时候需要"至多"转化

```
需要转化 (恰好k = 至多k - 至多k-1):
  - 恰好 k 个不同整数 (LC992)
  - 恰好 k 个奇数 (LC1248)
  - 恰好 k 个某种元素

不需要转化 (直接用 atMost):
  - 子数组和 ≤ k
  - 最多 k 个不同字符
  - 所有满足条件的子数组个数

⚠️ 判断标准:
  如果条件是"≤ k", 直接 atMost(k) 就行
  如果条件是"恰好等于 k", 需要转化
```

## 本节要点速查

```
✅ 恰好k = atMost(k) - atMost(k-1)
✅ atMost: ans += right - left + 1 (不是max/min)  
✅ right-left+1 = 以right结尾的合法子数组个数
✅ 计数器为0时必须del, 否则len(cnt)不变
✅ 整个atMost函数O(n), 调用两次仍是O(n)
✅ 条件是"≤ k"直接用, "= k"需要转化
```
