# 相向双指针

> 两个指针从数组两端向中间移动。常用于排序数组上的搜索和面积/水量问题。

---

## 1. 核心思想

```
left 从左端开始, right 从右端开始
根据条件决定移动哪个指针

  ┌──────────────────────────────┐
  │ left →              ← right │
  │   条件不足 → left++         │
  │   条件过剩 → right--        │
  │   找到 → 记录/返回          │
  └──────────────────────────────┘

⚠️ 前提: 通常数组已排序 (或有某种单调性)
⚠️ 循环条件: while left < right
   不要用 left <= right (会重复处理或死循环)
```

## 2. 两数之和 II

### LeetCode 167

### 理解与可视化

```
排序数组: [2, 7, 11, 15], target = 9

  left=0, right=3: 2+15=17 > 9  → right--
  left=0, right=2: 2+11=13 > 9  → right--
  left=0, right=1: 2+7=9 = 9    → 找到! [1,2] (1-indexed)

为什么这样是对的?
  当 sum > target: right 右边的值更大, 与 left 配更大
    → right 固定时, 没有更小的 left 了 (left 已经最小)
    → 只能 right-- 来减小 sum
  当 sum < target: 同理只能 left++
```

```python
def twoSum(numbers, target):
    left, right = 0, len(numbers) - 1
    while left < right:               # ⚠️ < 不是 <=
        s = numbers[left] + numbers[right]
        if s == target:
            return [left + 1, right + 1]  # ⚠️ 1-indexed
        elif s < target:
            left += 1
        else:
            right -= 1
    return []

# ⚠️ left < right, 同一个元素不能用两次
# ⚠️ 题目要求 1-indexed → +1
# ⚠️ 时间 O(n), 空间 O(1)
#    vs 哈希表法 O(n) 时间 O(n) 空间
```

## 3. 三数之和

### LeetCode 15

### 理解与可视化

```
输入: [-1, 0, 1, 2, -1, -4]
排序: [-4, -1, -1, 0, 1, 2]
找所有不重复三元组使 a+b+c=0

固定 a=nums[i], 在 i+1~n-1 中用双指针找 b+c = -a

  i=0: a=-4, 在 [-1,-1,0,1,2] 中找和=4 → 没有
  i=1: a=-1, 在 [-1,0,1,2] 中找和=1
       l=2, r=5: -1+2=1  ✓ → [-1,-1,2]
       l=3, r=4: 0+1=1   ✓ → [-1,0,1]
  i=2: a=-1, 和 i=1 相同 → 跳过 (去重!)
  ...
```

```python
def threeSum(nums):
    nums.sort()
    res = []
    n = len(nums)
    
    for i in range(n - 2):
        if i > 0 and nums[i] == nums[i-1]:  # ⚠️ 去重: 跳过相同的 a
            continue
        if nums[i] + nums[i+1] + nums[i+2] > 0:  # ⚠️ 最小三个>0
            break
        if nums[i] + nums[n-1] + nums[n-2] < 0:  # ⚠️ 最大两个不够
            continue
        
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                # ⚠️ 去重: 跳过相同的 b 和 c
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1
            elif s < 0:
                left += 1
            else:
                right -= 1
    
    return res

# ⚠️ 三处去重:
#   1. i > 0 and nums[i] == nums[i-1] → 跳过相同的 a
#   2. nums[left] == nums[left+1] → 跳过相同的 b
#   3. nums[right] == nums[right-1] → 跳过相同的 c
#
# ⚠️ 两处剪枝:
#   最小三个 > 0 → 后面更大, 不可能 = 0 → break
#   当前 + 最大两个 < 0 → 当前太小 → continue (不是break!)
#
# ⚠️ 时间 O(n²), 空间 O(1) (不算输出)
```

## 4. 盛最多水的容器

### LeetCode 11

```
height = [1,8,6,2,5,4,8,3,7]

面积 = min(h[l], h[r]) × (r - l)

  l=0, r=8: min(1,7)×8=8      h[l]<h[r] → l++
  l=1, r=8: min(8,7)×7=49     h[l]>h[r] → r--
  l=1, r=7: min(8,3)×6=18     h[l]>h[r] → r--
  ...

⚠️ 为什么移动较短的那个?
  面积由短板决定
  移动长板: 宽度减1, 高度不会变高(还是受短板限制) → 面积一定减小
  移动短板: 宽度减1, 但高度可能变高 → 面积可能增大
```

```python
def maxArea(height):
    left, right = 0, len(height) - 1
    ans = 0
    
    while left < right:
        area = min(height[left], height[right]) * (right - left)
        ans = max(ans, area)
        
        if height[left] < height[right]:
            left += 1                 # ⚠️ 移动短板
        else:
            right -= 1               # ⚠️ 相等时移哪个都行
    
    return ans

# ⚠️ min(h[l], h[r]) 不是 max
#    水位由短板决定
# ⚠️ 相等时移哪个都行, 不影响结果
# ⚠️ 时间 O(n), 空间 O(1)
```

## 5. 接雨水

### LeetCode 42

```
height = [0,1,0,2,1,0,1,3,2,1,2,1]
答案: 6

每个位置能接的水 = min(左边最高, 右边最高) - 当前高度

双指针法: 维护 left_max 和 right_max
  哪边矮从哪边算 (矮的那边的水位是确定的)

  left_max < right_max:
    位置 left 的水位至少 left_max (右边有更高的)
    → water += left_max - height[left]
    left++
  
  否则:
    → water += right_max - height[right]
    right--
```

```python
def trap(height):
    left, right = 0, len(height) - 1
    left_max = right_max = 0
    ans = 0
    
    while left < right:               # ⚠️ < 不是 <=
        left_max = max(left_max, height[left])
        right_max = max(right_max, height[right])
        
        if left_max < right_max:
            ans += left_max - height[left]  # ⚠️ 一定 ≥ 0
            left += 1
        else:
            ans += right_max - height[right]
            right -= 1
    
    return ans

# ⚠️ left_max - height[left] 一定 ≥ 0:
#    因为 left_max = max(..., height[left])
#    所以最小值就是 height[left] 本身, 此时水=0
#
# ⚠️ 为什么 left_max < right_max 时可以确定 left 的水位?
#    left_max 是 left 左边(含自己)的最大值
#    right_max 是 right 右边(含自己)的最大值
#    但 right 右边可能还有更高的!
#    没关系: left_max < right_max, 瓶颈在左边
#    即使右边有更高的, 水位还是 left_max
#
# ⚠️ 时间 O(n), 空间 O(1) (优于前缀最大值法的 O(n) 空间)
```

```

```
