# 分治法

> 分治 = Divide + Conquer + Combine。把大问题拆成小问题，分别解决，再合并结果。

---

## 1. 分治三步骤

```
1. Divide (分):  把问题分成若干个规模更小的子问题
2. Conquer (治): 递归解决每个子问题
3. Combine (合): 合并子问题的解，得到原问题的解

分治模板:
  ┌──────────────────────────────┐
  │ def divide_conquer(problem): │
  │   if 足够小:                  │
  │       return 直接解决         │
  │   分割成子问题                │
  │   left = divide_conquer(左)  │
  │   right = divide_conquer(右) │
  │   return combine(left, right)│
  └──────────────────────────────┘

⚠️ 分治 vs 普通递归:
  普通递归: 缩小为 1 个子问题 (如阶乘)
  分治:     拆分为 多 个子问题, 再合并
```

## 2. 归并排序

### LeetCode 912

### 理解与可视化

```
[38, 27, 43, 3, 9, 82, 10]

分:  [38, 27, 43, 3] | [9, 82, 10]
     [38, 27] | [43, 3] | [9, 82] | [10]
     [38]|[27] [43]|[3] [9]|[82]  [10]

治:  [27,38] [3,43] [9,82] [10]

合:  [3, 27, 38, 43] | [9, 10, 82]

合:  [3, 9, 10, 27, 38, 43, 82]

合并过程 (merge):
  left = [3, 27, 38, 43]    right = [9, 10, 82]
  i=0, j=0
  
  比较 3 vs 9  → 取 3,  i=1     res=[3]
  比较 27 vs 9 → 取 9,  j=1     res=[3,9]
  比较 27 vs 10→ 取 10, j=2     res=[3,9,10]
  比较 27 vs 82→ 取 27, i=2     res=[3,9,10,27]
  比较 38 vs 82→ 取 38, i=3     res=[3,9,10,27,38]
  比较 43 vs 82→ 取 43, i=4     res=[3,9,10,27,38,43]
  left 耗尽, 追加 right 剩余    res=[3,9,10,27,38,43,82]
```

### 代码模板

```python
def merge_sort(arr):
    if len(arr) <= 1:                 # ⚠️ base case: 0或1个元素
        return arr
    mid = len(arr) // 2              # ⚠️ 分割点
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    res = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:      # ⚠️ <= 保证稳定排序
            res.append(left[i])
            i += 1
        else:
            res.append(right[j])
            j += 1
    res.extend(left[i:])             # ⚠️ 追加剩余
    res.extend(right[j:])
    return res

# ⚠️ 时间 O(n log n), 空间 O(n)
# ⚠️ 稳定排序: 相等元素保持原始顺序 (因为用 <=)
# ⚠️ merge 中用 <= 而非 <:
#   left[i]==right[j] 时先取 left → 保持稳定
#   如果用 <, 相等时取 right → 不稳定
```

### 原地归并排序（节省空间）

```python
def merge_sort_inplace(arr, left, right):
    if left >= right:                 # ⚠️ >= 不是 >
        return
    mid = (left + right) // 2
    merge_sort_inplace(arr, left, mid)
    merge_sort_inplace(arr, mid + 1, right)
    merge_inplace(arr, left, mid, right)

def merge_inplace(arr, left, mid, right):
    tmp = arr[left:right+1]           # ⚠️ 临时数组
    i, j = 0, mid - left + 1
    k = left
    while i <= mid - left and j <= right - left:
        if tmp[i] <= tmp[j]:
            arr[k] = tmp[i]
            i += 1
        else:
            arr[k] = tmp[j]
            j += 1
        k += 1
    while i <= mid - left:
        arr[k] = tmp[i]
        i += 1
        k += 1
    while j <= right - left:
        arr[k] = tmp[j]
        j += 1
        k += 1

# 调用: merge_sort_inplace(arr, 0, len(arr)-1)
# ⚠️ right 是包含的, 传 len(arr)-1 不是 len(arr)
```

## 3. 快速排序

### 理解与可视化

```
选一个 pivot, 把数组分成三部分:
  小于 pivot | 等于 pivot | 大于 pivot
  然后递归排序左右两部分

[3, 6, 8, 10, 1, 2, 1]  pivot=3
  small=[1,2,1]  equal=[3]  big=[6,8,10]
  递归 small → [1,1,2]
  递归 big   → [6,8,10]
  结果: [1,1,2] + [3] + [6,8,10] = [1,1,2,3,6,8,10]

⚠️ 快排的问题:
  如果每次选到最大/最小 → 退化为 O(n²)
  解决: 随机选 pivot
```

```python
import random

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = random.choice(arr)        # ⚠️ 随机选, 避免最坏情况
    small = [x for x in arr if x < pivot]
    equal = [x for x in arr if x == pivot]
    big   = [x for x in arr if x > pivot]
    return quick_sort(small) + equal + quick_sort(big)

# ⚠️ 三路分区 (Dutch National Flag):
#   等于 pivot 的不需要递归 → 大量重复元素时性能好
#
# ⚠️ 快排 vs 归并:
#   快排: 平均 O(n log n), 最坏 O(n²), 原地, 不稳定
#   归并: 始终 O(n log n), 需 O(n) 额外空间, 稳定
```

## 4. 逆序对（归并排序变体）

### LeetCode 剑指 Offer 51

```
逆序对: 对于 i < j, 如果 nums[i] > nums[j], 则 (i,j) 是逆序对
  [7, 5, 6, 4] → 5 个逆序对: (7,5)(7,6)(7,4)(5,4)(6,4)

思路: 在归并排序的 merge 过程中统计

  merge 时, 如果 left[i] > right[j]:
    说明 left[i], left[i+1], ..., left[末尾] 都 > right[j]
    逆序对数 += len(left) - i
```

```python
def count_inversions(arr):
    if len(arr) <= 1:
        return arr, 0
    mid = len(arr) // 2
    left, cnt_l = count_inversions(arr[:mid])
    right, cnt_r = count_inversions(arr[mid:])
    
    # merge + count
    res = []
    count = 0
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:       # ⚠️ <= 不是 <
            res.append(left[i])
            i += 1
        else:
            res.append(right[j])
            count += len(left) - i    # ⚠️ 关键: left[i:]都>right[j]
            j += 1
    res.extend(left[i:])
    res.extend(right[j:])
    return res, cnt_l + cnt_r + count

# ⚠️ 必须用 <= 而非 <:
#   left[i] == right[j] 不是逆序对!
#   如果用 <, 等于时也会加 count → 多算了
# ⚠️ count += len(left) - i 而非 count += 1
#   因为 left 已排序, left[i] > right[j] → left[i:]都 > right[j]
```

## 5. 分治的时间复杂度分析

```
主定理 (Master Theorem):

  T(n) = a × T(n/b) + O(n^d)

  a = 子问题个数
  b = 子问题规模缩小倍数
  d = 合并步骤的指数

  若 d > log_b(a): T(n) = O(n^d)
  若 d = log_b(a): T(n) = O(n^d × log n)
  若 d < log_b(a): T(n) = O(n^(log_b(a)))

常见例子:
  归并排序: a=2, b=2, d=1 → d=log_2(2)=1 → O(n log n) ✓
  二分查找: a=1, b=2, d=0 → d=log_2(1)=0 → O(log n) ✓
  快速幂:   a=1, b=2, d=0 → O(log n) ✓
```
