# 递归基础

> 递归 = 函数调用自身。理解递归的关键是：相信子问题已经解决，只关注当前层做什么。

---

## 1. 递归三要素

```
任何递归都需要:
  1. base case:  最小问题，直接返回，不再递归
  2. recursive case: 把问题缩小，调用自己
  3. 返回值: 当前层如何利用子问题的结果

⚠️ 没有 base case → 无限递归 → 栈溢出 (RecursionError)
⚠️ 问题没有缩小 → 也会无限递归
```

## 2. 阶乘

### 理解与可视化

```
factorial(5) = 5 × factorial(4)
             = 5 × 4 × factorial(3)
             = 5 × 4 × 3 × factorial(2)
             = 5 × 4 × 3 × 2 × factorial(1)
             = 5 × 4 × 3 × 2 × 1  ← base case
             = 120

调用栈:
  ┌────────────────────┐
  │ factorial(1) → 1   │ ← 栈顶 (最后进, 最先出)
  ├────────────────────┤
  │ factorial(2) → 2×1 │
  ├────────────────────┤
  │ factorial(3) → 3×2 │
  ├────────────────────┤
  │ factorial(4) → 4×6 │
  ├────────────────────┤
  │ factorial(5) → 5×24│ ← 栈底 (最先进, 最后出)
  └────────────────────┘
```

### 代码模板

```python
def factorial(n):
    if n <= 1:                        # ⚠️ base case
        return 1
    return n * factorial(n - 1)       # 问题缩小: n → n-1

# ⚠️ base case 用 n <= 1 而非 n == 1
#    防止传入 n=0 或负数时无限递归
# ⚠️ Python 默认递归深度 1000, 可用 sys.setrecursionlimit 修改
```

## 3. 斐波那契数

### LeetCode 509

### 理解与可视化

```
fib(5) 的递归树:
              fib(5)
            /        \
        fib(4)       fib(3)
       /     \       /    \
    fib(3)  fib(2) fib(2) fib(1)
    /   \    / \    / \
 fib(2) fib(1) ...  ...
  / \
fib(1) fib(0)

⚠️ 大量重复计算!
  fib(3) 算了 2 次, fib(2) 算了 3 次
  朴素递归时间: O(2^n) → 指数级!
```

### 代码（三种方式）

```python
# 方式一: 朴素递归 → O(2^n), 太慢
def fib_naive(n):
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# 方式二: 记忆化递归 → O(n)
from functools import cache

@cache                                # ⚠️ 自动缓存已计算的结果
def fib_memo(n):
    if n <= 1:
        return n
    return fib_memo(n - 1) + fib_memo(n - 2)

# 方式三: 迭代 → O(n) 时间, O(1) 空间
def fib_iter(n):
    if n <= 1:
        return n
    a, b = 0, 1
    for _ in range(2, n + 1):        # ⚠️ range(2, n+1), 从 2 开始
        a, b = b, a + b
    return b

# ⚠️ @cache 等价于手动写 memo 字典:
#   memo = {}
#   def fib(n):
#       if n in memo: return memo[n]
#       ...
#       memo[n] = result
#       return result
#
# ⚠️ 记忆化 = 自顶向下 DP
#    迭代   = 自底向上 DP
```

## 4. 快速幂

### LeetCode 50

```
求 x^n, 但 n 可能很大 (如 2^31)

朴素: x × x × ... × x (n次) → O(n)
快速幂: 每次把 n 减半 → O(log n)

原理:
  x^10 = x^5 × x^5
  x^5  = x^2 × x^2 × x
  x^2  = x × x

  只需 4 次乘法, 而非 10 次!
```

```python
def myPow(x, n):
    if n == 0:
        return 1
    if n < 0:                         # ⚠️ 负指数
        x = 1 / x
        n = -n
    
    half = myPow(x, n // 2)
    if n % 2 == 0:
        return half * half            # ⚠️ 偶数: x^n = (x^(n/2))^2
    else:
        return half * half * x        # ⚠️ 奇数: 多乘一个 x

# ⚠️ 只调用一次 myPow(x, n//2), 结果存在 half 中
#    不能写成 myPow(x, n//2) * myPow(x, n//2)
#    那样就是 O(n) 了, 失去了快速幂的意义!
#
# ⚠️ 迭代版 (位运算):
#   res = 1
#   while n > 0:
#       if n & 1: res *= x
#       x *= x
#       n >>= 1
```

## 5. 递归与迭代的转换

```
递归本质是用系统栈, 迭代可以用显式栈模拟

什么时候用递归:
  - 问题有天然的递归结构 (树、分治)
  - 代码更直观、更短

什么时候转迭代:
  - 递归深度太大 → 栈溢出风险
  - 尾递归优化 (Python 不支持, Go 也不支持)
  - 需要精确控制栈行为

⚠️ 树的遍历: 递归 → 迭代 (用栈)
⚠️ 斐波那契: 递归 → 迭代 (用变量)
⚠️ 快速排序: 递归 → 迭代 (用显式栈)
```

## 6. 常见递归错误

```
❌ base case 不完整
  fib(n): 只写了 if n==0 return 0
  忘了 if n==1 return 1 → fib(1) 调用 fib(0)+fib(-1) 无限递归

❌ 返回值没接住
  def foo(node):
      foo(node.left)        ← 没有 return!
      foo(node.right)

❌ 全局变量 vs 返回值混用
  该用返回值传递信息时用了全局变量
  或者该用全局变量时反复创建局部变量

❌ 重复计算子问题
  fib 不加 memo → O(2^n)
  加 @cache     → O(n)
```
