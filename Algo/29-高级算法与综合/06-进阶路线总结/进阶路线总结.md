## 6. 面试算法策略

```
解题四步法:
  ┌─────────────────────────────────────────────┐
  │ 1. 审题 (2min)                               │
  │    - 输入范围? n ≤ 10^5 → O(nlogn) 以内      │
  │    - 边界条件? 空数组, 单元素, 全相同          │
  │    - 在线还是离线?                            │
  ├─────────────────────────────────────────────┤
  │ 2. 选型 (2min)                               │
  │    - 对照 "问题→算法" 表选模板                 │
  │    - 先说暴力, 再优化                         │
  │    - 说清时间/空间复杂度                       │
  ├─────────────────────────────────────────────┤
  │ 3. 编码 (15min)                              │
  │    - 先写框架, 再填细节                       │
  │    - 边界条件在最后统一处理                    │
  │    - 变量命名清晰                              │
  ├─────────────────────────────────────────────┤
  │ 4. 验证 (5min)                               │
  │    - 用小例子手动模拟                         │
  │    - 检查边界: 空, 单元素, 最大                │
  │    - 检查 off-by-one: < vs <=, 0 vs 1 索引   │
  └─────────────────────────────────────────────┘

n 的量级与目标复杂度:
  n ≤ 10     → O(n!) 或 O(2^n)  回溯/状压
  n ≤ 20     → O(2^n)           状压DP/子集枚举
  n ≤ 500    → O(n^3)           区间DP/Floyd
  n ≤ 5000   → O(n^2)           普通DP
  n ≤ 10^5   → O(nlogn)        排序/BIT/线段树
  n ≤ 10^6   → O(n) 或 O(n√n)  线性/根号分解
  n ≤ 10^9   → O(logn)         二分/快速幂
```
