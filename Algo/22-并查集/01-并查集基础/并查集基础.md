# 并查集基础

> 并查集高效处理「动态连通性」问题：合并集合 O(α(n))、查询是否连通 O(α(n))。

---

## 1. 核心概念

```
并查集 (Union-Find / Disjoint Set):
  维护若干不相交集合, 支持:
    find(x):   查找 x 所在集合的代表元素 (根)
    union(x,y): 合并 x 和 y 所在的集合

  用 parent 数组实现:
    parent[x] = x 的父节点
    根节点: parent[x] == x

  初始: 每个元素自成一个集合
    parent: [0, 1, 2, 3, 4]
    5 个集合, 每个元素是自己的根

    union(0, 1): 1 的根指向 0
      parent: [0, 0, 2, 3, 4]
           0
           |
           1

    union(2, 3): 3 的根指向 2
    union(1, 3): find(1)=0, find(3)=2 → 2 指向 0
      parent: [0, 0, 0, 2, 4]
           0
          / \
         1   2
             |
             3
```

---

## 2. 两大优化

```
没有优化: find 最坏 O(n) (链状树)

优化1: 路径压缩
  find 时把路径上所有节点直接指向根
  一步: parent[x] = parent[parent[x]] (隔代压缩)
  或者: 递归让所有节点指向根 (完全压缩)

优化2: 按秩合并
  合并时, 矮树挂到高树上
  保持树高度较小

两者结合:
  均摊 O(α(n)) ≈ O(1)
  α 是反 Ackermann 函数, 实际不超过 5
```

## 3. 代码模板

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))   # ⚠️ 初始: 自己是自己的父
        self.rank = [0] * n            # ⚠️ 秩 (近似高度)
        self.count = n                 # ⚠️ 连通分量数
    
    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]  # ⚠️ 路径压缩(隔代)
            x = self.parent[x]
        return x
    
    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False               # ⚠️ 已连通, 合并失败
        # 按秩合并: 矮挂高
        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx            # ⚠️ 保证 rx 是高的
        self.parent[ry] = rx           # ⚠️ 矮的挂到高的下面
        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1         # ⚠️ 等高时高度+1
        self.count -= 1
        return True                    # ⚠️ 合并成功
    
    def connected(self, x, y):
        return self.find(x) == self.find(y)

# ⚠️ 路径压缩的两种写法:
#    隔代: parent[x] = parent[parent[x]] (推荐, 迭代)
#    递归: parent[x] = find(parent[x])   (完全压缩, 但可能栈溢出)
#
# ⚠️ union 返回值:
#    True: 合并成功 (之前不连通)
#    False: 已连通 (不需要合并)
#    返回值在 Kruskal、冗余连接等题中很有用
```

## 4. LeetCode 684: 冗余连接

```
树有 n 个节点, 加了一条多余的边形成了环
找到并返回这条多余的边 (最后一条导致环的边)
```

```python
def findRedundantConnection(edges):
    n = len(edges)
    uf = UnionFind(n + 1)             # ⚠️ 1-indexed
    
    for u, v in edges:
        if not uf.union(u, v):        # ⚠️ 合并失败 = 已连通 = 这条边多余
            return [u, v]

# ⚠️ n+1 大小: 节点编号 1~n
# ⚠️ 第一条让 union 返回 False 的边就是答案
```

## 5. LeetCode 547: 省份数量

```
n 个城市, isConnected[i][j]=1 表示 i 和 j 直接相连
求连通分量数 (省份数)
```

```python
def findCircleNum(isConnected):
    n = len(isConnected)
    uf = UnionFind(n)
    
    for i in range(n):
        for j in range(i + 1, n):    # ⚠️ 只遍历上三角
            if isConnected[i][j] == 1:
                uf.union(i, j)
    
    return uf.count                   # ⚠️ 连通分量数

# ⚠️ 也可用 DFS/BFS 求连通分量
#    并查集的优势: 动态加边时也能快速查询
```

## 6. LeetCode 990: 等式方程的可满足性

```
给定 ["a==b", "b!=c", "c==a"]
判断是否存在矛盾

方法: 先处理 ==, 再检查 !=
```

```python
def equationsPossible(equations):
    uf = UnionFind(26)                # ⚠️ 26个字母
    
    # 第1遍: 处理 == (合并)
    for eq in equations:
        if eq[1] == '=':
            uf.union(ord(eq[0]) - ord('a'), ord(eq[3]) - ord('a'))
    
    # 第2遍: 检查 != (矛盾)
    for eq in equations:
        if eq[1] == '!':
            if uf.connected(ord(eq[0]) - ord('a'), ord(eq[3]) - ord('a')):
                return False           # ⚠️ 不等但连通 = 矛盾
    
    return True

# ⚠️ 两遍遍历:
#    第1遍建立所有相等关系
#    第2遍检查不等关系是否与相等冲突
```

## 7. LeetCode 721: 账户合并

```
accounts[i] = [name, email1, email2, ...]
同一个人可能有多个账户
如果两个账户有相同邮箱, 合并为一个人

方法: 邮箱→账户id 映射 + 并查集合并
```

```python
from collections import defaultdict

def accountsMerge(accounts):
    n = len(accounts)
    uf = UnionFind(n)
    email_to_id = {}
    
    for i, account in enumerate(accounts):
        for email in account[1:]:
            if email in email_to_id:
                uf.union(i, email_to_id[email])  # ⚠️ 合并账户
            email_to_id[email] = i
    
    # 归类: 根 → 邮箱集合
    groups = defaultdict(set)
    for email, i in email_to_id.items():
        groups[uf.find(i)].add(email)
    
    return [[accounts[i][0]] + sorted(emails)
            for i, emails in groups.items()]

# ⚠️ email_to_id 第一次见到的邮箱记录账户id
#    再次见到时合并两个账户
#    最后用 find 归类到根
```

