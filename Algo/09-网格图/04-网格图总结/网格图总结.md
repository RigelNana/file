# 网格图总结

> 网格图问题的方法选择、常见坑和总结。

---

## 1. 方法选择决策树

```
网格图问题
│
├── 连通块/岛屿/区域 → DFS (洪水填充)
│   ├── 岛屿数量 (200)
│   ├── 最大面积 (695)
│   ├── 被围绕的区域 (130)
│   └── 太平洋大西洋 (417)
│
├── 最短路径 → 看边权
│   ├── 无权 (等权) → BFS
│   │   ├── 单源: 从一个起点
│   │   └── 多源: 多个起点同时入队
│   │
│   ├── 权 0/1 → 0-1 BFS (双端队列)
│   │
│   └── 非负权 → Dijkstra (优先队列)
│
├── 路径是否存在 → DFS 或 BFS 均可
│
└── 拓扑/依赖关系 → 拓扑排序 (BFS/Kahn)

⚠️ DFS 和 BFS 都能判连通性
   但求最短路只能用 BFS (或更高级算法)
   DFS 不保证最短!
```

## 2. 模板对比

```python
# ────────── DFS 模板 ──────────
def dfs(grid, r, c, rows, cols):
    if r < 0 or r >= rows or c < 0 or c >= cols:
        return
    if grid[r][c] != TARGET:
        return
    grid[r][c] = VISITED
    for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
        dfs(grid, r+dr, c+dc, rows, cols)

# ────────── BFS 模板 ──────────
from collections import deque

def bfs(grid, start):
    rows, cols = len(grid), len(grid[0])
    queue = deque([(*start, 0)])
    visited = {start}
    while queue:
        r, c, d = queue.popleft()
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r+dr, c+dc
            if 0<=nr<rows and 0<=nc<cols and (nr,nc) not in visited and grid[nr][nc]!=WALL:
                visited.add((nr,nc))
                queue.append((nr, nc, d+1))

# ────────── 0-1 BFS 模板 ──────────
def bfs_01(grid):
    rows, cols = len(grid), len(grid[0])
    dist = [[float('inf')]*cols for _ in range(rows)]
    dist[0][0] = 0
    dq = deque([(0,0)])
    while dq:
        r, c = dq.popleft()
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r+dr, c+dc
            if 0<=nr<rows and 0<=nc<cols:
                w = grid[nr][nc]  # 0 or 1
                if dist[r][c]+w < dist[nr][nc]:
                    dist[nr][nc] = dist[r][c]+w
                    if w==0: dq.appendleft((nr,nc))
                    else: dq.append((nr,nc))
```

## 3. 常见坑总结

```
⚠️ 1. 标记时机
   DFS: 进入时标记 (递归开头)
   BFS: 入队时标记 (不是出队时!)
   如果出队时标记, 同一格子可能多次入队, TLE!

⚠️ 2. 边界检查顺序
   先检查越界, 再检查格子值
   if grid[r][c] 放在越界检查前面会 IndexError

   正确: if r<0 or r>=rows or c<0 or c>=cols or grid[r][c] != '1'
   错误: if grid[r][c] != '1' or r<0 or r>=rows ...

⚠️ 3. 字符 vs 数字
   有些题 grid 是 ['1','0'] (字符)
   有些题 grid 是 [1, 0] (数字)
   注意 '1' != 1

⚠️ 4. 修改原数组
   grid[r][c] = '0' 会修改输入
   如果不允许修改, 用 visited 集合
   面试时可以问面试官

⚠️ 5. 多源 BFS 的初始时间
   腐烂橘子: 初始腐烂橘子在 t=0
   最后 minutes - 1: 因为最后一轮空转
   
⚠️ 6. 递归深度
   Python 默认递归限制 1000
   大网格可能栈溢出
   解决: sys.setrecursionlimit() 或用 BFS

⚠️ 7. 四方向 vs 八方向
   大部分题是四方向
   "国际象棋"等题可能是八方向
   仔细看题意
```

## 4. 复杂度

```
设网格大小 m × n:

┌──────────┬───────────┬───────────┐
│ 算法     │ 时间      │ 空间      │
├──────────┼───────────┼───────────┤
│ DFS      │ O(m·n)    │ O(m·n)    │ ← 递归栈
│ BFS      │ O(m·n)    │ O(m·n)    │ ← 队列
│ 0-1 BFS  │ O(m·n)    │ O(m·n)    │ ← 队列+dist
└──────────┴───────────┴───────────┘

⚠️ 空间是 O(m·n) 不是 O(min(m,n)):
   最坏情况队列/栈中可能有 O(m·n) 个元素
   但通常实际空间远小于理论最坏
```

## 5. 进阶技巧

```
1. 反向思维
   "所有被X包围的O" → 反过来找"连通到边界的O"
   "水能流到哪" → 反过来"从终点往高处走"
   正向难 → 反向简单

2. 多源 BFS
   "哪个源最近" → 所有源同时入队
   不需要对每个源单独 BFS

3. 并查集
   "动态判断连通性" → 并查集 (Union-Find)
   "岛屿数量可以用并查集做" (但 DFS 更简单)

4. 0-1 BFS
   "有些边免费有些收费" → 0-1 BFS
   比 Dijkstra 快一个 log
```
