# 前缀和与哈希表

> 前缀和+哈希表是解决"子数组和"类问题的终极方案，尤其适用于有负数的场景。

---

## 1. 核心模式

```
子数组 [l, r] 的某种"前缀统计量" = pre[r+1] - pre[l]
→ 固定 r, 需要找满足条件的 l
→ 用哈希表存储历史前缀和, O(1) 查找

模式:
  cnt = {0: 1}          ← 初始化空前缀
  pre = 0
  for x in nums:
      pre += x (或其他变换)
      ans += cnt[pre - target]   ← 查历史
      cnt[pre] += 1              ← 存当前

⚠️ 始终"先查后存":
  查 = 在历史中找匹配
  存 = 把当前加入历史
  先存后查 = 可能把当前和自己配对
```

## 2. 连续数组

### LeetCode 525

```
只含 0 和 1 的数组, 找最长的子数组使 0 和 1 个数相等

技巧: 把 0 当作 -1, 问题变成"找最长和为 0 的子数组"

nums = [0, 1, 0, 0, 1, 1, 0]
转换: [-1, 1,-1,-1, 1, 1,-1]
pre:  [0, -1, 0,-1,-2,-1, 0,-1]

和为 0 的子数组 [l, r]: pre[r+1] = pre[l]
→ 找相同前缀和的最远距离
```

```python
def findMaxLength(nums):
    first = {0: -1}                   # ⚠️ 前缀和0最早出现在-1
    pre = 0
    ans = 0
    
    for i, x in enumerate(nums):
        pre += 1 if x == 1 else -1    # ⚠️ 0→-1, 1→+1
        
        if pre in first:
            ans = max(ans, i - first[pre])  # ⚠️ 最远距离
        else:
            first[pre] = i            # ⚠️ 只记第一次出现的位置
    
    return ans

# ⚠️ 和 LC560 的区别:
#   LC560 (求个数): cnt[pre] += 1 (累计计数)
#   LC525 (求最长): first[pre] = i (只记第一次)
#
# ⚠️ first[0] = -1 的含义:
#   前缀和为 0 最早在下标 -1 (还没开始)
#   子数组 [0, i] 的和就是 pre[i+1] - pre[0] = pre[i+1]
#   如果 pre[i+1] = 0, 长度 = i - (-1) = i + 1
#
# ⚠️ 只记第一次: 要最长, 所以起点越早越好
```

## 3. 路径总和 III

### LeetCode 437

```
二叉树中找路径和 = targetSum 的路径数
路径方向向下, 不需要从根开始

思路: 在 DFS 路径上维护前缀和+哈希表
  和数组中的 LC560 完全一样!
```

```python
from collections import defaultdict

def pathSum(root, targetSum):
    cnt = defaultdict(int)
    cnt[0] = 1
    ans = 0
    
    def dfs(node, pre):
        nonlocal ans
        if not node:
            return
        pre += node.val
        ans += cnt[pre - targetSum]   # ⚠️ 和数组版一样
        cnt[pre] += 1
        dfs(node.left, pre)
        dfs(node.right, pre)
        cnt[pre] -= 1                 # ⚠️ 回溯! 离开这条路径

    dfs(root, 0)
    return ans

# ⚠️ cnt[pre] -= 1 是关键!
#   树的左子树和右子树是不同路径
#   离开左子树时要撤销, 否则右子树会算到左子树的前缀和
#   这和回溯的"撤销选择"一样
#
# ⚠️ 数组版不需要撤销: 因为只有一条路径(从左到右)
#    树版需要撤销: 因为有多条路径(不同分支)
```

## 4. 积为K的子数组

```
如果需要积为 k: 前缀积 + 哈希表
  pre_product[r+1] / pre_product[l] = k
  → pre_product[l] = pre_product[r+1] / k

但除法有精度问题, 通常用对数转化:
  log(积) = log(a) + log(b) + ... = 和
  积问题 → 转化为和问题 → 前缀和
```

## 5. 常见变体总结

```
┌──────────────────────┬─────────────────────────┬──────────┐
│ 问题                 │ 前缀和变换               │ 哈希表存  │
├──────────────────────┼─────────────────────────┼──────────┤
│ 和为k的子数组个数     │ pre += x                │ 计数 cnt │
│ 和可被k整除的子数组   │ pre = (pre+x) % k       │ 计数 cnt │
│ 0和1个数相等的最长    │ pre += (1 if x else -1) │ 首次位置 │
│ 和为k的最长子数组     │ pre += x                │ 首次位置 │
│ 和为k的最短子数组     │ pre += x                │ 最近位置 │
│ 树上路径和=k          │ pre += node.val         │ 计数+回溯│
└──────────────────────┴─────────────────────────┴──────────┘

⚠️ 求个数 → 哈希表存计数 (cnt[pre] += 1)
   求最长 → 哈希表存首次位置 (只记第一次)
   求最短 → 哈希表存最近位置 (每次覆盖)
   树上 → 需要回溯 (cnt[pre] -= 1)
```

## 本节要点速查

```
✅ 核心模式: cnt={0:1}, 枚举右端, 查cnt[pre-k]
✅ 先查后存, 避免自身配对
✅ 求个数→计数, 求最长→首次位置, 求最短→最近位置
✅ 连续数组: 0→-1, 找最远相同前缀和
✅ 树上路径: DFS+前缀和+回溯(cnt[pre]-=1)
✅ first[0]=-1: 空前缀假装在-1位置
```
