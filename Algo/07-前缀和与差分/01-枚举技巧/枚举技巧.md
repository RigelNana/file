# 枚举技巧

> 枚举是最基础的算法思想：穷举所有可能，选出满足条件的。关键是选好枚举对象和优化枚举范围。

---

## 1. 枚举的核心

```
三要素:
  1. 枚举什么? → 选择枚举对象 (元素/下标/对/子数组等)
  2. 枚举范围? → 确定边界 (避免越界和重复)
  3. 如何剪枝? → 提前排除不可能的情况

常见枚举对象:
  - 枚举一个元素: O(n)
  - 枚举一对: O(n²)
  - 枚举三元组: O(n³) → 通常需要优化
  - 枚举子数组: O(n²) → 配合前缀和到 O(n)
  - 枚举子集: O(2^n) → 回溯/位运算
```

## 2. 枚举右端点, 维护左端点信息

```
⚠️ 这是前缀和、滑窗、双指针的共同基础思想!

对于子数组问题:
  暴力: 枚举左端 i, 右端 j → O(n²)
  优化: 只枚举右端 j, 用数据结构维护左端信息

例: 两数之和 (LeetCode 1)
  暴力: 枚举 (i, j) 对 → O(n²)
  优化: 枚举 j, 用哈希表查 target - nums[j] 是否出现过 → O(n)
```

```python
# 两数之和 — 枚举右端点, 哈希表维护左端信息
def twoSum(nums, target):
    seen = {}                         # 值 → 下标
    for j, x in enumerate(nums):
        diff = target - x
        if diff in seen:              # ⚠️ 在左端信息中查找
            return [seen[diff], j]
        seen[x] = j                   # ⚠️ 先查后存, 避免自身配对
    return []

# ⚠️ 先查后存: if diff in seen → seen[x] = j
#    如果先存后查: 可能把自己和自己配对
#    例: nums=[3,3], target=6
#    先存 seen[3]=0, 再查 target-3=3 in seen → [0,0] 错!
```

## 3. 枚举中间值

```
三元组问题: 枚举中间值, 两端用预处理

例: 统计递增三元组 (a < b < c)
  枚举中间值 b = nums[j]
  左边有多少个 < b 的 → left_count
  右边有多少个 > b 的 → right_count
  贡献 = left_count × right_count
```

```python
def count_triplets(nums):
    n = len(nums)
    ans = 0
    for j in range(1, n - 1):
        left = sum(1 for i in range(j) if nums[i] < nums[j])
        right = sum(1 for k in range(j+1, n) if nums[k] > nums[j])
        ans += left * right
    return ans

# O(n²), 可以用树状数组优化到 O(n log n)
# 但思路上"枚举中间值"比枚举两端更高效
```

## 4. 枚举优化技巧

```
⚠️ 常见优化:
  1. 排序后枚举 → 二分/双指针代替内层循环
  2. 前缀和/后缀和 → O(1) 查询区间信息
  3. 哈希表 → O(1) 查询是否存在
  4. 单调栈/队列 → 维护窗口最值
  5. 提前终止 → break (排序后不可能更好就break)

选择技巧:
  看到"子数组和" → 前缀和
  看到"两数/三数之和" → 哈希表或排序+双指针
  看到"连续子数组最值" → 滑窗/单调队列
  看到"最近的更大/更小" → 单调栈
```
