## 2. 所有模板汇总

### lower_bound

```python
# >= target 的第一个位置
def lower_bound(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] >= target:       # 蓝色
            hi = mid - 1
        else:
            lo = mid + 1
    return lo
```

### 二分答案求最小

```python
# 第一个可行的答案
while lo <= hi:
    mid = (lo + hi) // 2
    if check(mid):    # 可行 → 试更小
        hi = mid - 1
    else:
        lo = mid + 1
return lo
```

### 二分答案求最大

```python
# 最后一个可行的答案
while lo <= hi:
    mid = (lo + hi) // 2
    if check(mid):    # 可行 → 试更大
        lo = mid + 1
    else:
        hi = mid - 1
return hi
```

## 3. 特殊二分场景

### 搜索旋转排序数组

### LeetCode 33

```
[4,5,6,7,0,1,2], target=0 → 4

旋转数组 = 两段有序
  [4,5,6,7 | 0,1,2]
  
  每次判断 mid 在哪段:
  如果 nums[mid] >= nums[lo] → mid 在左段
  否则 → mid 在右段
```

```python
def search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] == target:
            return mid
        
        if nums[lo] <= nums[mid]:     # ⚠️ 左半段有序
            if nums[lo] <= target < nums[mid]:  # ⚠️ target 在左半
                hi = mid - 1
            else:
                lo = mid + 1
        else:                         # 右半段有序
            if nums[mid] < target <= nums[hi]:  # ⚠️ target 在右半
                lo = mid + 1
            else:
                hi = mid - 1
    return -1

# ⚠️ nums[lo] <= nums[mid] 的 <= 很关键:
#   当 lo == mid 时 (只剩两个元素), 也要正确判断
# ⚠️ target 范围判断:
#   左半有序: nums[lo] <= target < nums[mid]
#   右半有序: nums[mid] < target <= nums[hi]
#   注意 < 和 <= 的位置
```

### 寻找峰值

### LeetCode 162

```
nums[i] != nums[i+1] (相邻不等)
找一个峰值 (比左右邻居都大)

nums[mid] > nums[mid+1] → 峰在左边(含mid)
nums[mid] < nums[mid+1] → 峰在右边
```

```python
def findPeakElement(nums):
    lo, hi = 0, len(nums) - 1
    while lo < hi:                    # ⚠️ lo < hi, 不是 lo <= hi
        mid = (lo + hi) // 2
        if nums[mid] > nums[mid+1]:
            hi = mid                  # ⚠️ mid 可能是峰, 不能跳过
        else:
            lo = mid + 1
    return lo

# ⚠️ 这里用 lo < hi (不是 lo <= hi):
#   因为 hi = mid (不是 mid-1), 用 <= 会死循环
# ⚠️ nums[mid] > nums[mid+1] 时 hi = mid (保留mid)
#   因为 mid 本身可能就是峰值
# ⚠️ 不使用 mid-1/mid+1 的写法, 需要 lo < hi
```

## 4. Python bisect 标准库

```python
import bisect

nums = [1, 3, 5, 5, 7, 9]

bisect.bisect_left(nums, 5)   # 2 → >= 5 的第一个位置 (lower_bound)
bisect.bisect_right(nums, 5)  # 4 → > 5 的第一个位置 (upper_bound)
bisect.bisect(nums, 5)        # 4 → 等同于 bisect_right

bisect.insort_left(nums, 5)   # 在 lower_bound 位置插入
bisect.insort_right(nums, 5)  # 在 upper_bound 位置插入

# ⚠️ bisect_left = lower_bound (>=)
# ⚠️ bisect_right = upper_bound (>)
# ⚠️ 底层用 C 实现, 比手写快
```

## 5. 常见易错点

```
❌ lo <= hi vs lo < hi
  闭区间 [lo,hi] + mid±1 → 用 lo <= hi
  保留 mid 的写法 (hi=mid) → 用 lo < hi
  搞混 → 死循环或遗漏

❌ mid 溢出 (其他语言)
  Java/C++: mid = lo + (hi - lo) / 2 (防溢出)
  Python: 不溢出 (大数支持), 直接 (lo+hi)//2 即可

❌ 答案范围设错
  lo 太大 → 漏掉正确答案
  hi 太小 → 漏掉正确答案
  建议: 宁可范围大一点, 多二分几次无伤大雅

❌ check 函数不单调
  二分的前提: 答案空间具有单调性
  如果 check(5)=True 但 check(6)=False 且 check(7)=True
  → 不能用二分!

❌ 旋转数组用 < 而非 <=
  nums[lo] <= nums[mid] 的 = 号不能丢
  lo==mid 时需要正确判断

❌ 浮点二分 ±1
  浮点数答案空间连续, 不能 mid+1/mid-1
  应该 lo=mid/hi=mid + 固定循环次数
```

## 6. 题型识别速查

```
"排序数组中查找"        → lower_bound / upper_bound
"查找插入位置"          → lower_bound (LC35)
"第一个和最后一个位置"   → lower_bound ×2 (LC34)
"旋转排序数组查找"      → 判断哪半有序 (LC33)
"寻找峰值"             → 比较 mid 和 mid+1 (LC162)
"最小化最大值"          → 二分答案求最小 (LC410)
"最大化最小值"          → 二分答案求最大 (LC1552)
"最少天数/速度/能力"    → 二分答案 + 贪心check
"平方根/n次方根"        → 二分答案 (整数或浮点)
```
