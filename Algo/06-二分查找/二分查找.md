# 二分查找

> 二分查找是将搜索空间每次缩小一半的高效算法。掌握二分查找的关键是确定搜索区间和判断条件。

---

## 1. 基础二分查找（红蓝染色法）

### 理解与可视化

```
核心思想: 每次排除一半的搜索空间

在有序数组 [1, 3, 5, 7, 9, 11] 中查找 7:

  [1, 3, 5, 7, 9, 11]
   l        m        r     mid=5, 5<7 → 去右半
         [7, 9, 11]
          l  m    r         mid=9, 9>7 → 去左半
         [7]
          l=m=r             mid=7 ✓ 找到！

时间复杂度: O(log n)
  n=100万 → 只需 ~20 次比较

红蓝染色法（最通用的理解方式）:
  把数组分成两部分: 满足条件的(蓝色) 和 不满足的(红色)
  二分就是找到红蓝的分界线

  [红, 红, 红, 红, 蓝, 蓝, 蓝]
                   ↑
                 分界点

  left 始终在红色区域，right 始终在蓝色区域
  最终 left 和 right 相邻，right 就是第一个蓝色位置
```

### 代码模板

```python
# 基础二分: 在有序数组中查找 target
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

# 红蓝染色法: 找第一个 ≥ target 的位置 (lower_bound)
# ⭐ 最通用模板，其他变体都从这里推导
def lower_bound(nums, target):
    left, right = 0, len(nums)  # 左闭右开 [left, right)
    while left < right:
        mid = (left + right) // 2
        if nums[mid] < target:
            left = mid + 1     # [mid+1, right)
        else:
            right = mid        # [left, mid)
    return left  # 第一个 ≥ target 的位置

# 四种常用变体（都基于 lower_bound）
def search_variants(nums, target):
    # ≥ target 的第一个位置
    pos_ge = lower_bound(nums, target)

    # > target 的第一个位置 = ≥ (target+1) 的第一个位置
    pos_gt = lower_bound(nums, target + 1)

    # ≤ target 的最后一个位置 = (> target 的第一个位置) - 1
    pos_le = lower_bound(nums, target + 1) - 1

    # < target 的最后一个位置 = (≥ target 的第一个位置) - 1
    pos_lt = lower_bound(nums, target) - 1

    return pos_ge, pos_gt, pos_le, pos_lt
```

## 2. 二分答案：求最小 ⭐

### 理解与可视化

```
二分答案: 不是在数组中二分，而是对「答案」二分！

问题: 使得某个条件成立的最小值是多少？

  答案空间:
  [不行, 不行, 不行, 可以, 可以, 可以]
                      ↑
                    最小可行答案

  本质: 答案空间是单调的（如果 x 可以，那 x+1 也可以）
  用二分找分界点

例: 在 D 天内运送完包裹的最低运载能力
  答案范围: [max(weights), sum(weights)]
  对运载能力 mid: 检查能否在 D 天内运完
  能 → 尝试更小的能力 → right = mid
  不能 → 需要更大的能力 → left = mid+1
```

### 代码模板

```python
# 二分答案模板：求最小（⭐ 核心模板）
def binary_search_min(lo, hi, check):
    """在 [lo, hi] 中找最小的满足 check 条件的值"""
    while lo < hi:
        mid = (lo + hi) // 2
        if check(mid):
            hi = mid       # mid 可以，尝试更小
        else:
            lo = mid + 1   # mid 不行，需要更大
    return lo

# 在 D 天内送达包裹的最低运载能力
def ship_within_days(weights, days):
    def check(cap):
        d, cur = 1, 0
        for w in weights:
            if cur + w > cap:
                d += 1
                cur = 0
            cur += w
        return d <= days
    return binary_search_min(max(weights), sum(weights), check)

# 爱吃香蕉的珂珂（最小速度）
def min_eating_speed(piles, h):
    def check(speed):
        return sum((p + speed - 1) // speed for p in piles) <= h
    return binary_search_min(1, max(piles), check)
```

## 3. 二分答案：求最大 ⭐

### 理解与可视化

```
问题: 使得某个条件成立的最大值是多少？

  答案空间:
  [可以, 可以, 可以, 不行, 不行, 不行]
                  ↑
               最大可行答案

  如果 x 可以，那 x-1 也可以（单调性）
```

### 代码模板

```python
# 二分答案模板：求最大
def binary_search_max(lo, hi, check):
    """在 [lo, hi] 中找最大的满足 check 条件的值"""
    while lo < hi:
        mid = (lo + hi + 1) // 2  # 注意 +1 防止死循环
        if check(mid):
            lo = mid       # mid 可以，尝试更大
        else:
            hi = mid - 1   # mid 不行，需要更小
    return lo

# 木材加工（切割出 k 段相同长度的木头，求最大长度）
def wood_cut(woods, k):
    def check(length):
        return sum(w // length for w in woods) >= k
    if not woods or max(woods) == 0:
        return 0
    return binary_search_max(1, max(woods), check)
```

## 4. 最小化最大值 / 最大化最小值

### 理解与可视化

```
最小化最大值 (Minimax):
  答案越大越容易满足 → 找最小的可行答案 → 和「求最小」一样

最大化最小值 (Maximin):
  答案越小越容易满足 → 找最大的可行答案 → 和「求最大」一样

例: 磁力最大（在 n 个位置放 m 个球，最大化最小距离）
  最小距离越小 → 越容易放下 → 单调性成立
  二分最小距离 → 检查能否放下 m 个球
```

### 代码模板

```python
# 磁力最大（最大化最小值）
def max_distance(position, m):
    position.sort()
    def check(min_dist):
        count = 1
        last = position[0]
        for p in position[1:]:
            if p - last >= min_dist:
                count += 1
                last = p
        return count >= m
    lo, hi = 1, (position[-1] - position[0]) // (m - 1)
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if check(mid):
            lo = mid
        else:
            hi = mid - 1
    return lo
```

## 5. 二分查找总结

```
判断使用哪种二分:

  ┌────────────────────────┬─────────────────────────┐
  │ 问题类型                │ 方法                     │
  ├────────────────────────┼─────────────────────────┤
  │ 有序数组查找            │ 基础二分                  │
  │ ≥ target 第一个位置     │ lower_bound              │
  │ 满足条件的最小值        │ check(mid) → hi=mid      │
  │ 满足条件的最大值        │ check(mid) → lo=mid (+1) │
  │ 最小化最大值            │ 同「求最小」              │
  │ 最大化最小值            │ 同「求最大」              │
  └────────────────────────┴─────────────────────────┘

关键: 确保答案空间的单调性 → 才能用二分
```
