# 最小化最大值 / 最大化最小值

> 看到"最小化最大值"或"最大化最小值"，立刻想到二分答案。

---

## 1. 题型识别

```
"最小化最大值" (Minimize the Maximum):
  有多种分配方案, 每种方案有一个"最大值"
  在所有方案中, 找最大值最小的那个
  → 二分答案: 猜一个最大值上限 mid, 检查是否可行
  → 本质: 求最小 (lower_bound)

"最大化最小值" (Maximize the Minimum):
  有多种分配方案, 每种方案有一个"最小值"
  在所有方案中, 找最小值最大的那个
  → 二分答案: 猜一个最小值下限 mid, 检查是否可行
  → 本质: 求最大

⚠️ 无论是哪种, 都是在答案空间上二分!
```

---

## 2. 最小化最大值

### LeetCode 410 — 分割数组的最大值

```
nums = [7,2,5,10,8], k = 2
分成 k 个连续子数组, 最小化子数组和的最大值

二分 mid = 子数组和的上限
  check(mid): 能否在每段和 ≤ mid 的限制下, 分成 ≤ k 段

  mid=18: [7,2,5] [10,8] → 2 段 ≤ 2 ✓
  mid=17: [7,2,5] [10] [8] → 3 段 > 2 ✗
  答案: 18
```

```python
def splitArray(nums, k):
    def check(limit):
        cnt = 1                       # ⚠️ 当前段数
        cur = 0                       # ⚠️ 当前段的和
        for x in nums:
            if cur + x > limit:       # ⚠️ 超过限制, 开新段
                cnt += 1
                cur = 0
            cur += x
        return cnt <= k               # ⚠️ 段数 ≤ k 就可行
    
    lo, hi = max(nums), sum(nums)
    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid):                # min化最大值 → 求最小
            hi = mid - 1
        else:
            lo = mid + 1
    return lo

# ⚠️ check 中 cur + x > limit 用 > 不是 >=
#   等于 limit 时还能装得下, 不用开新段
# ⚠️ "最小化最大值" → 可行时缩小 hi → 和求最小一样
```

## 3. 最大化最小值

### LeetCode 2517 — 礼物的最大甜蜜度

```
prices 中选 k 个, 最大化任意两个选中价格的最小差值

排序后, 二分 mid = 最小差值下限
check(mid): 贪心选, 每次选 ≥ 上一个 + mid 的价格, 能否选够 k 个
```

```python
def maximumTastiness(prices, k):
    prices.sort()
    
    def check(d):
        cnt = 1                       # ⚠️ 选第一个
        last = prices[0]
        for p in prices[1:]:
            if p - last >= d:         # ⚠️ 差值 ≥ d
                cnt += 1
                last = p
                if cnt >= k:
                    return True
        return False
    
    lo, hi = 0, prices[-1] - prices[0]
    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid):                # 最大化最小值 → 可行扩大 lo
            lo = mid + 1
        else:
            hi = mid - 1
    return hi                         # ⚠️ 返回 hi (最后一个可行)

# ⚠️ 排序后贪心: 每次选离上一个最近的 ≥ d 的
# ⚠️ 答案范围: [0, max-min]
# ⚠️ "最大化最小值" → 可行时扩大 lo → 和求最大一样
```

## 4. 磁力最大化

### LeetCode 1552

```
n 个位置放 m 个球, 最大化任意两球间的最小距离

排序后, 和 LC2517 思路几乎一样!
check(d): 贪心放球, 每次放 ≥ 上一个 + d 的位置, 能放 m 个
```

```python
def maxDistance(position, m):
    position.sort()
    
    def check(d):
        cnt = 1
        last = position[0]
        for p in position[1:]:
            if p - last >= d:
                cnt += 1
                last = p
                if cnt >= m:
                    return True
        return False
    
    lo, hi = 1, (position[-1] - position[0]) // (m - 1)
    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid):
            lo = mid + 1
        else:
            hi = mid - 1
    return hi

# ⚠️ hi = (max-min) // (m-1) 是最大可能间距
#   m 个球均匀分布的间距
# ⚠️ lo = 1 (最小间距至少为1, 因为位置各不同)
```

## 5. 模式总结

```
┌──────────────────────┬──────────────────┬──────────────┐
│ 类型                 │ 可行 → 动作       │ 返回         │
├──────────────────────┼──────────────────┼──────────────┤
│ 最小化最大值          │ hi = mid - 1     │ lo           │
│ (minimize max)       │ (试更小的limit)   │ (第一个可行)  │
├──────────────────────┼──────────────────┼──────────────┤
│ 最大化最小值          │ lo = mid + 1     │ hi           │
│ (maximize min)       │ (试更大的下限)    │ (最后一个可行)│
└──────────────────────┴──────────────────┴──────────────┘

check 函数通用模式:
  1. 贪心分段/放置: 依次处理, 超过限制就分段/跳过
  2. 统计段数/选中个数
  3. 判断是否满足约束 (≤ k 段, ≥ k 个)

⚠️ check 函数几乎都是贪心!
   二分猜答案, 贪心验证是否可行
   这是"二分+贪心"的经典组合
```

## 本节要点速查

```
✅ 最小化最大值 → 求最小 → check True → hi=mid-1, return lo
✅ 最大化最小值 → 求最大 → check True → lo=mid+1, return hi
✅ check 函数通常用贪心: 从左到右扫描, 贪心分段/选择
✅ 排序是前提: 贪心需要有序
✅ 答案范围: [max(元素)/1, sum/maxGap]
✅ "二分+贪心" 是最经典的二分答案套路
```
