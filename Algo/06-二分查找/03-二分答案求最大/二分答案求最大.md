# 二分答案 — 求最大

> 与求最小相反：找最后一个可行的答案。等价于找"最后一个红色"。

---

## 1. 核心思想

```
"求最大的满足条件的答案"
  [可以 可以 可以 | 不行 不行 不行]
              ↑ 找这个 (最后一个可以)

方法一: 转化为求最小
  "最大的可行" = "第一个不可行" - 1
  
方法二: 直接求最大 (upper_bound - 1 的思路)
  check(mid) 为 True → lo = mid + 1 (还能更大)
  check(mid) 为 False → hi = mid - 1 (太大了)
  return hi (或 lo - 1)

⚠️ 推荐方法一 (不容易搞混):
  反转 check 条件, 用 lower_bound 求第一个不可行的
  答案 = lower_bound 结果 - 1
```

### 代码模板

```python
# 方法: 直接求最大
def binary_search_max(lo, hi):
    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid):                # ⚠️ mid 可行 → 试更大
            lo = mid + 1
        else:                         # mid 不可行 → 太大了
            hi = mid - 1
    return hi                         # ⚠️ 返回 hi (最后一个可行)

# ⚠️ 和求最小的区别:
#   求最小: check True → hi=mid-1, return lo
#   求最大: check True → lo=mid+1, return hi
#   记忆: 求最小缩右边(hi), 求最大扩左边(lo)
```

## 2. 在线选举

### 概念示例

```
给一系列投票结果, 问某个时间点谁得票最多

答案范围: 时间轴上的位置
二分: 找 ≤ 查询时间 的最后一个投票 → 那时的领先者

upper_bound(query_time) - 1 = 最后一个 ≤ query_time 的位置
```

## 3. 木材切割

```
有 n 根木头, 长度各异, 要切成 k 段等长的
求最大段长

答案范围: [1, max(lengths)]
check(mid): sum(L // mid for L in lengths) >= k

  lengths = [232, 124, 456]  k = 7
  mid=100: 2+1+4 = 7 >= 7 ✓ → 试更大
  mid=114: 2+1+3 = 6 < 7  ✗ → 太大了
  答案: 114 不行, 往回找...
```

```python
def max_cut_length(lengths, k):
    def check(mid):
        return sum(L // mid for L in lengths) >= k
    
    lo, hi = 1, max(lengths)
    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid):
            lo = mid + 1              # ⚠️ 可行, 试更大
        else:
            hi = mid - 1              # ⚠️ 太大了
    return hi                         # ⚠️ 最后一个可行

# ⚠️ L // mid 是整除: 长232的木头切成100的段 → 2段
# ⚠️ check >= k (至少切 k 段)
# ⚠️ 返回 hi (最后一个使 check 为 True 的 mid)
```

## 4. 最大平均值 (二分浮点数)

```
有时答案不是整数, 需要二分浮点数

答案范围: [lo, hi] (浮点)
精度: while hi - lo > 1e-6

模板:
  lo, hi = 下界, 上界
  for _ in range(100):                # 或 while hi - lo > eps
      mid = (lo + hi) / 2
      if check(mid):
          lo = mid                    # ⚠️ 不是 mid + 1
      else:
          hi = mid                    # ⚠️ 不是 mid - 1
  return lo  # 或 hi, 差距 < eps

⚠️ 浮点二分不 ±1, 因为答案是连续的
⚠️ 用固定次数循环 (如100次) 比 eps 判断更安全
   每次区间缩小一半, 100次 → 精度 ~10^-30
```

## 5. 求最大 vs 求最小对比

```
┌──────────┬───────────────────┬───────────────────┐
│          │ 求最小 (first True)│ 求最大 (last True) │
├──────────┼───────────────────┼───────────────────┤
│ check T  │ hi = mid - 1      │ lo = mid + 1      │
│ check F  │ lo = mid + 1      │ hi = mid - 1      │
│ 返回     │ lo                │ hi                │
│ 含义     │ 第一个蓝色        │ 最后一个蓝色       │
│ 可行→    │ 试更小            │ 试更大             │
│ 不可行→  │ 试更大            │ 试更小             │
└──────────┴───────────────────┴───────────────────┘

⚠️ 记忆技巧:
  求最小: "可行→缩右边", 最终左边界 lo 就是答案
  求最大: "可行→扩左边", 最终右边界 hi 就是答案
```

## 本节要点速查

```
✅ 求最大: check True → lo=mid+1, return hi
✅ 求最小: check True → hi=mid-1, return lo
✅ 浮点二分: 不±1, 用 lo=mid/hi=mid, 循环100次
✅ 可以转化: 最大可行 = 第一个不可行 - 1
✅ 确认单调性: mid可行 → mid-1也可行 (向左单调)
```
