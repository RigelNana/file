# 红蓝染色法

> 二分查找的本质：将数组分成红蓝两段，找到分界点。不死记模板，理解染色即可推导一切。

---

## 1. 核心思想

### 理解与可视化

```
任何二分查找都可以看作：
  把数组元素分成"红色"和"蓝色"两段

  [红红红红红 | 蓝蓝蓝蓝蓝]
              ↑
          分界点 (我们要找的)

  红色: 不满足条件
  蓝色: 满足条件
  (也可以反过来，看具体问题)

例: 在 [1,3,5,7,9] 中找 ≥ 6 的第一个位置
  条件: nums[mid] >= 6
  
  [1, 3, 5, | 7, 9]
   红 红 红   蓝 蓝
              ↑ 答案 = 3 (下标)

⚠️ 红蓝染色法的好处:
  不需要记 left < right / left <= right
  不需要记 left = mid / left = mid + 1
  不需要记 return left / return right
  只需要理解: 找第一个蓝色 或 最后一个红色
```

### 代码模板

```python
def lower_bound(nums, target):
    """找 ≥ target 的第一个位置 (第一个蓝色)"""
    left, right = 0, len(nums) - 1
    
    while left <= right:              # ⚠️ <= 不是 <
        mid = (left + right) // 2
        if nums[mid] >= target:       # ⚠️ 蓝色条件
            right = mid - 1           # ⚠️ 蓝色, 答案在左边(含mid)
        else:
            left = mid + 1            # ⚠️ 红色, 答案在右边
    
    return left                       # ⚠️ left 就是第一个蓝色的位置

# ⚠️ while left <= right → 循环结束时 left = right + 1
#    left 指向第一个蓝色 (或 n, 表示全红)
#    right 指向最后一个红色 (或 -1, 表示全蓝)
#
# ⚠️ 上面等价于 bisect.bisect_left(nums, target)

def upper_bound(nums, target):
    """找 > target 的第一个位置"""
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] > target:        # ⚠️ 严格大于
            right = mid - 1
        else:
            left = mid + 1
    return left                       # 等价于 bisect.bisect_right

# ⚠️ lower_bound: >= target 的第一个位置
#    upper_bound: >  target 的第一个位置
#    二者唯一区别: 蓝色条件是 >= 还是 >
```

## 2. 四种基本查询

```
给定排序数组和 target:

1. >= target 的第一个位置 → lower_bound(target)
2. >  target 的第一个位置 → upper_bound(target) 或 lower_bound(target+1)
3. <  target 的最后一个位置 → lower_bound(target) - 1
4. <= target 的最后一个位置 → upper_bound(target) - 1

⚠️ 3 和 4 是 1 和 2 的补集!
  < target 的最后一个 = ≥ target 第一个的前一个
  ≤ target 的最后一个 = > target 第一个的前一个

只需要掌握 lower_bound, 其他都能推出来!
```

## 3. 在排序数组中查找元素

### LeetCode 34

```
输入: nums = [5,7,7,8,8,10], target = 8
输出: [3, 4] (第一个8和最后一个8的下标)

第一个 8: lower_bound(8) → 3
最后一个 8: upper_bound(8) - 1 → 5 - 1 = 4
```

```python
def searchRange(nums, target):
    def lower_bound(t):
        lo, hi = 0, len(nums) - 1
        while lo <= hi:
            mid = (lo + hi) // 2
            if nums[mid] >= t:
                hi = mid - 1
            else:
                lo = mid + 1
        return lo
    
    first = lower_bound(target)
    if first == len(nums) or nums[first] != target:
        return [-1, -1]              # ⚠️ target 不存在
    last = lower_bound(target + 1) - 1  # ⚠️ > target 的第一个 - 1
    return [first, last]

# ⚠️ 检查 first 合法性:
#   first == len(nums) → target 比所有元素大
#   nums[first] != target → target 不存在 (first 是 > target 的位置)
# ⚠️ 最后一个 target = lower_bound(target+1) - 1
#   比记两套模板简单!
```

## 4. 搜索插入位置

### LeetCode 35

```
排序数组, 找 target 的下标, 不存在则返回插入位置
→ 就是 lower_bound(target)!

[1,3,5,6], target=5 → 2
[1,3,5,6], target=2 → 1 (插入位置)
[1,3,5,6], target=7 → 4 (插入末尾)
```

```python
def searchInsert(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if nums[mid] >= target:
            hi = mid - 1
        else:
            lo = mid + 1
    return lo

# ⚠️ 就是 lower_bound, 一行都不用改
# ⚠️ 返回 lo, 范围 [0, len(nums)]
```

## 5. 开区间写法 (另一种风格)

```python
def lower_bound_open(nums, target):
    """开区间写法: (lo, hi) 表示答案在 lo 和 hi 之间"""
    lo, hi = -1, len(nums)           # ⚠️ 开区间!
    while lo + 1 < hi:               # ⚠️ lo + 1 < hi
        mid = (lo + hi) // 2
        if nums[mid] >= target:
            hi = mid                  # ⚠️ 不是 mid - 1
        else:
            lo = mid                  # ⚠️ 不是 mid + 1
    return hi                         # ⚠️ 返回 hi

# ⚠️ 闭区间 vs 开区间:
#   闭区间: [lo, hi], while lo <= hi, lo=mid+1/hi=mid-1, return lo
#   开区间: (lo, hi), while lo+1 < hi, lo=mid/hi=mid, return hi
#
# ⚠️ 两种写法效果完全相同, 选一种熟练掌握即可
#    推荐闭区间 (更常见, 和标准库一致)
```

## 本节要点速查

```
✅ 红蓝染色: 红=不满足, 蓝=满足, 找分界点
✅ lower_bound: >= target 第一个, 蓝色条件 >=
✅ upper_bound: > target 第一个, 蓝色条件 >
✅ 闭区间: [lo,hi], while lo<=hi, mid±1, return lo
✅ 四种查询都可归结为 lower_bound ± 1
✅ 先找 first, 再用 lower_bound(target+1)-1 找 last
```
