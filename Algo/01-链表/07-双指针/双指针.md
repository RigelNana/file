# 双指针技巧

> 链表双指针与快慢指针不同，更侧重两个指针分别遍历不同的链表或从不同起点出发，巧妙利用路程差来解题。

---

## 1. 相交链表 ⭐⭐

### 理解与可视化

```
LeetCode 160: 相交链表

题意: 找两个链表的交叉点 (物理交叉, 不是值相等)

  A:     4 → 1 ↘
                  8 → 4 → 5 → None
  B: 5 → 6 → 1 ↗

  交叉点 = 节点 8

核心思路: 
  设 A 独有部分长 a, B 独有部分长 b, 公共部分长 c
  指针 pA 走完 A 再走 B: 路程 = a + c + b
  指针 pB 走完 B 再走 A: 路程 = b + c + a
  a + c + b == b + c + a → 走的步数一样多!
  → 它们一定会在交叉点相遇!

  ⚠️ 如果不相交:
     A 走完 A 再走 B: a + b
     B 走完 B 再走 A: b + a
     最终都到 None → 返回 None ✓

  可视化:
    pA: [4 → 1 → 8 → 4 → 5] → [5 → 6 → 1 → 8] ← 到这里遇到 pB
    pB: [5 → 6 → 1 → 8 → 4 → 5] → [4 → 1 → 8] ← 到这里遇到 pA
                                            ↑ 相遇!
```

### 代码模板

```python
def getIntersectionNode(headA, headB):
    pA, pB = headA, headB
    while pA != pB:               # ⚠️ 比较的是节点, 不是值!
        pA = pA.next if pA else headB  # ⚠️ pA 走到 None 后跳到 headB
        pB = pB.next if pB else headA  # ⚠️ pB 走到 None 后跳到 headA
    return pA                     # 相遇点 or None(不相交时两者同时为None)

# ⚠️ 关键: pA = pA.next if pA else headB
#   不是 pA = pA.next if pA.next else headB  ← 这样会跳过 None, 不相交时死循环!
#   
#   正确流程: pA 走到尾部 → pA = 尾.next = None → 下一步 pA = headB
#   错误流程: pA 走到尾部 → pA.next = None → pA = headB (跳过了 None 这一步)
#
# ⚠️ 时间 O(a+b+c), 空间 O(1)
```

## 2. 链表的倒数第 k 个节点

### 理解与可视化

```
剑指 Offer 22 / LeetCode 19 前置

题意: 返回链表倒数第 k 个节点 (1-indexed, 即倒数第1个=最后一个)

  [1→2→3→4→5], k=2 → 返回节点 4

双指针思路:
  1. 快指针先走 k 步
  2. 然后快慢一起走
  3. 快到终点时, 慢就在倒数第 k 个

  fast 先走 k 步: fast 和 slow 差 k 步
  然后一起走, 保持差 k 步
  fast = None 时, slow 在倒数第 k 个  ← 因为 slow 和终点差 k 步

  可视化: k=2
    初始:           [1] [2] [3] [4] [5] None
    fast 先走2步:    s               f
    一起走1步:           s               f
    一起走2步:               s               f
    一起走3步:                   s               f=None
    返回 slow=4 ✓
```

### 代码模板

```python
def find_kth_from_end(head, k):
    fast = slow = head
    for _ in range(k):           # ⚠️ 1-indexed, 走 k 步
        fast = fast.next         # ⚠️ 如果 k > 链表长度, 这里会报错
    while fast:                  # ⚠️ while fast, 不是 while fast.next
        fast = fast.next
        slow = slow.next
    return slow

# ⚠️ 1-indexed vs 0-indexed:
#   倒数第 1 个 = 最后一个 (1-indexed)
#   先走 k 步 + while fast → 停在倒数第 k 个 ✓
#
# ⚠️ 如果要删除倒数第 k 个:
#   需要先走 k+1 步, 或者用 dummy 节点
#   这样 slow 停在倒数第 k 个的前驱

# ━━━━ 删除倒数第 k 个 (LC 19) ━━━━
def removeNthFromEnd(head, n):
    dummy = ListNode(next=head)
    fast = slow = dummy          # ⚠️ 从 dummy 开始, 不是 head
    for _ in range(n + 1):       # ⚠️ 走 n+1 步
        fast = fast.next
    while fast:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next
    return dummy.next
```

## 3. 合并两个有序链表 ⭐⭐

### LeetCode 21

```
思路: 双指针分别指向两个链表, 每次取较小的接到结果链表

  L1: 1→2→4
  L2: 1→3→4

  比较流程:
    p1=1, p2=1 → min=1(L1), 结果: 1
    p1=2, p2=1 → min=1(L2), 结果: 1→1
    p1=2, p2=3 → min=2(L1), 结果: 1→1→2
    p1=4, p2=3 → min=3(L2), 结果: 1→1→2→3
    p1=4, p2=4 → min=4(L1), 结果: 1→1→2→3→4
    p1=None     → 接上 L2 剩余: 1→1→2→3→4→4
```

### 代码模板

```python
def mergeTwoLists(l1, l2):
    dummy = ListNode()
    cur = dummy
    while l1 and l2:              # ⚠️ 两个都非空才比较
        if l1.val <= l2.val:      # ⚠️ <= 保持稳定性
            cur.next = l1
            l1 = l1.next
        else:
            cur.next = l2
            l2 = l2.next
        cur = cur.next
    cur.next = l1 or l2           # ⚠️ 接上剩余部分（一定只有一个非空）
    return dummy.next

# ⚠️ l1.val <= l2.val 用 <= 而不是 <
#   等于时取 l1 → 保持相对顺序（稳定排序特性）
#
# ⚠️ cur.next = l1 or l2
#   Python 中 None or l2 = l2, l1 or None = l1
#   简洁地处理了剩余部分
```

## 4. 合并 K 个有序链表 ⭐⭐

### LeetCode 23

```
三种方法:
  1. 逐个合并: O(kN) — 每次合并两个
  2. 分治合并: O(N log k) — 两两配对递归合并
  3. 堆合并:   O(N log k) — 最小堆取最小

推荐方法 2 和 3, 面试都要会
```

```python
# ━━━━ 方法 1: 分治合并 ━━━━
def mergeKLists(lists):
    if not lists:
        return None
    
    def merge(left, right):       # 合并 lists[left:right+1]
        if left == right:         # ⚠️ 只有一个链表, 直接返回
            return lists[left]
        mid = (left + right) // 2
        l1 = merge(left, mid)
        l2 = merge(mid + 1, right)
        return mergeTwoLists(l1, l2)
    
    return merge(0, len(lists) - 1)

# ━━━━ 方法 2: 最小堆 ━━━━
import heapq

def mergeKLists_heap(lists):
    dummy = ListNode()
    cur = dummy
    heap = []
    
    for i, head in enumerate(lists):
        if head:
            heapq.heappush(heap, (head.val, i, head))  # ⚠️ 用 i 打破值相同时的比较
    
    while heap:
        val, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    
    return dummy.next

# ⚠️ 堆的元组: (val, i, node)
#   val: 用于排序
#   i:   打破 val 相同时的 tie（Python 不能比较 ListNode）
#   node: 实际的链表节点
#   
# ⚠️ i 用初始的链表索引即可, 因为每个链表消费一个就补一个
#   同一条链表的节点不会同时在堆里
```

## 5. 分隔链表

### LeetCode 86

```
题意: 给定 x, 把 < x 的节点放前面, >= x 的放后面, 保持相对顺序
  [1,4,3,2,5,2], x=3 → [1,2,2,4,3,5]

思路: 用两个 dummy 头分别收集 < x 和 >= x 的节点, 最后连起来
```

```python
def partition(head, x):
    small = ListNode()            # < x 的链表
    big = ListNode()              # >= x 的链表
    sp, bp = small, big           # 两个尾指针
    
    cur = head
    while cur:
        if cur.val < x:           # ⚠️ 严格小于 x
            sp.next = cur
            sp = sp.next
        else:
            bp.next = cur
            bp = bp.next
        cur = cur.next
    
    bp.next = None                # ⚠️ 必须断尾! 否则可能形成环
    sp.next = big.next            # 小链表接大链表
    return small.next

# ⚠️ bp.next = None 非常重要!
#   如果大链表的最后一个节点在原链表中后面还有小节点
#   那它的 next 还指着那个小节点 → 形成环!
#
# 例: [1,4,3,2], x=3
#   小: 1→2
#   大: 4→3
#   但 3.next 原来指向 2 → 4→3→2→...→1→...环!
#   必须 bp.next = None 断开
```

## 6. 奇偶链表

### LeetCode 328

```
题意: 把奇数位节点放前面, 偶数位节点放后面（位置奇偶, 不是值奇偶）
  [1,2,3,4,5] → [1,3,5,2,4]
  [2,1,3,5,6,4,7] → [2,3,6,7,1,5,4]
```

```python
def oddEvenList(head):
    if not head:
        return None
    
    odd = head                    # 奇数位头
    even = head.next              # 偶数位头
    even_head = even              # ⚠️ 保存偶数位头, 最后连接用
    
    while even and even.next:     # ⚠️ 检查 even 和 even.next
        odd.next = even.next      # 奇数接下一个奇数
        odd = odd.next
        even.next = odd.next      # 偶数接下一个偶数
        even = even.next
    
    odd.next = even_head          # 奇数链表尾接偶数链表头
    return head

# ⚠️ while even and even.next
#   偶数长度: even 走到 None → 结束
#   奇数长度: even.next 走到 None → 结束
#
# ⚠️ 不需要额外断尾, 因为 even 本身就是链表尾
```

## 本节要点速查

```
✅ 相交链表: pA走完走B, pB走完走A, 总步数一样 → 在交点相遇
✅ 倒数第k个: 快先走k步, 再一起走, 快到终点慢在倒数第k
✅ 合并两个: dummy + while 两个都非空 + 接剩余
✅ 合并K个: 分治O(NlogK) 或 最小堆O(NlogK)
✅ 分隔链表: 两个dummy收集, 最后连接, 必须断尾!
✅ 奇偶链表: 保存even_head, 拆开后再连接
✅ 易错: 相交判断是节点相等不是值相等; 分隔必须断尾防环
```
