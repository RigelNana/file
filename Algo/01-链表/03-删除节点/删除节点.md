# 删除节点

> 删除是链表最高频的操作之一。核心就是「跳过」——让前一个节点的 next 绕过被删节点。

---

## 1. 删除的核心原理

### 理解与可视化

```
删除值为 3 的节点:

  之前: [1] → [2] → [3] → [4] → None
                prev   del

  操作: prev.next = prev.next.next    (也就是 prev.next = del.next)

  之后: [1] → [2] ─────────→ [4] → None
                      [3] ← 被跳过, Python 自动回收内存

⚠️ 删除操作的本质:
   你需要站在「被删节点的前一个节点」上操作
   所以遍历时通常检查 cur.next 的值而非 cur 的值

⚠️ 删除头节点的特殊性:
   头节点没有"前一个节点"
   → 要么特殊处理 head
   → 要么用 dummy 节点（推荐）
```

---

## 2. 用哨兵节点删除

### 代码模板

```python
# ━━━━━━━━━━━ 删除所有值为 val 的节点 (LeetCode 203) ━━━━━━━━━━━
def remove_elements(head, val):
    dummy = ListNode(next=head)      # ⚠️ 哨兵节点
    cur = dummy                       # ⚠️ 从 dummy 开始（不是 head）
    while cur.next:                   # ⚠️ 检查 cur.next 是否存在
        if cur.next.val == val:
            cur.next = cur.next.next  # 跳过被删节点
            # ⚠️ 这里不要 cur = cur.next!
            # 因为新的 cur.next 可能还是 val, 需要继续检查
        else:
            cur = cur.next            # 只有不删的时候才前进
    return dummy.next

# ⚠️ 关键细节:
#   删除后 cur 不动!  因为新的 cur.next 可能也要删
#   只在不删时才移动 cur
#
# 错误写法:
#   ❌ cur.next = cur.next.next; cur = cur.next  → 可能跳过连续重复
#   例: [1,2,2,3] 删 2 → 第一个 2 删了, cur 前进到第二个 2, 但它变成了 cur 而不是 cur.next, 跳过了
```

```
可视化执行过程:

  head = [1, 2, 6, 3, 4, 5, 6], val = 6

  dummy → [1] → [2] → [6] → [3] → [4] → [5] → [6] → None
  cur=dummy
  
  cur.next=1 ≠ 6 → cur 前进到 [1]
  cur.next=2 ≠ 6 → cur 前进到 [2]
  cur.next=6 == 6 → 删除! cur.next = [3]  (cur 不动, 还在 [2])
  cur.next=3 ≠ 6 → cur 前进到 [3]
  cur.next=4 ≠ 6 → cur 前进到 [4]
  cur.next=5 ≠ 6 → cur 前进到 [5]
  cur.next=6 == 6 → 删除! cur.next = None  (cur 不动)
  cur.next=None   → 循环结束

  结果: dummy → [1] → [2] → [3] → [4] → [5] → None ✓
```

## 3. 删除排序链表中的重复元素

### LeetCode 83（保留一个）

```
题意: 排序链表, 删除重复元素, 每个值只保留一个
  [1,1,2,3,3] → [1,2,3]

思路:
  因为是排序的, 重复元素一定相邻
  如果 cur.val == cur.next.val → 删除 cur.next
  否则前进

⚠️ 这题不需要 dummy 因为头节点不会被删（它是第一次出现）
```

```python
def delete_duplicates(head):
    cur = head
    while cur and cur.next:             # ⚠️ 两个条件都要
        if cur.val == cur.next.val:
            cur.next = cur.next.next    # 删除下一个
            # ⚠️ cur 不动! 新的 cur.next 可能还重复
        else:
            cur = cur.next
    return head                          # ⚠️ 头不变, 直接返回 head 即可

# ⚠️ while cur and cur.next 的含义:
#   cur 不是 None (链表没走完)
#   cur.next 不是 None (cur 不是最后一个节点)
#   两个都满足才能安全比较 cur.val 和 cur.next.val
```

### LeetCode 82（一个都不保留）⭐

```
题意: 排序链表, 删除所有出现重复的元素
  [1,2,3,3,4,4,5] → [1,2,5]      ← 3和4 全部删除
  [1,1,1,2,3]     → [2,3]         ← 1 全部删除

⚠️ 头节点可能被删! → 必须用 dummy

思路:
  如果 cur.next 和 cur.next.next 值相同
  → 找到所有相同值的节点, 全部跳过
  否则正常前进
```

```python
def delete_duplicates_ii(head):
    dummy = ListNode(next=head)
    cur = dummy                             # ⚠️ 从 dummy 开始
    while cur.next and cur.next.next:
        if cur.next.val == cur.next.next.val:
            dup_val = cur.next.val           # 记住重复的值
            while cur.next and cur.next.val == dup_val:
                cur.next = cur.next.next     # 逐个跳过所有相同值
            # ⚠️ 内层 while 结束后, cur.next 指向第一个不同值的节点(或 None)
        else:
            cur = cur.next
    return dummy.next

# ⚠️ 为什么外层要 cur.next and cur.next.next?
#   需要至少两个节点才能比较是否重复
#   只剩一个节点 → 它不可能是重复的 → 保留
```

```
可视化: [1,2,3,3,4,4,5]

  dummy → [1] → [2] → [3] → [3] → [4] → [4] → [5] → None
  cur = dummy

  cur.next=1, cur.next.next=2, 1≠2 → 前进, cur=[1]
  cur.next=2, cur.next.next=3, 2≠3 → 前进, cur=[2]
  cur.next=3, cur.next.next=3, 3==3 → 发现重复!
    dup_val = 3
    while: cur.next=[3], val==3 → 跳过 → cur.next=[3]
    while: cur.next=[3], val==3 → 跳过 → cur.next=[4]
    while: cur.next=[4], val≠3 → 停止
  cur.next=4, cur.next.next=4, 4==4 → 发现重复!
    dup_val = 4
    while: cur.next=[4] → 跳   → cur.next=[4]
    while: cur.next=[4] → 跳   → cur.next=[5]
    while: cur.next=[5], val≠4 → 停
  cur.next=5, cur.next.next=None → 外层 while 结束

  结果: dummy → [1] → [2] → [5] → None ✓
```

## 4. 删除倒数第 N 个节点 ⭐

### LeetCode 19

```
题意: 删除链表的倒数第 n 个节点 (n 是 1-indexed)

  [1,2,3,4,5], n=2  → 删除 4 → [1,2,3,5]
  [1], n=1           → 删除 1 → []

方法: 快慢指针
  让 fast 先走 n+1 步
  然后 fast 和 slow 一起走
  当 fast 到 None 时, slow 正好在倒数第 n+1 个节点
  删除 slow.next 即可

⚠️ 为什么是 n+1 步而不是 n 步?
  因为我们要站在「被删节点的前一个」来操作
  fast 和 slow 的间距是 n+1
  fast=None 时, slow 距离末尾还有 n 个节点
  → slow 就是倒数第 n+1 个 → slow.next 是倒数第 n 个
```

```python
def remove_nth_from_end(head, n):
    dummy = ListNode(next=head)
    fast = slow = dummy
    
    # fast 先走 n+1 步
    for _ in range(n + 1):          # ⚠️ 是 n+1 不是 n
        fast = fast.next
    
    # 一起走直到 fast 到 None
    while fast:                      # ⚠️ while fast 不是 while fast.next
        fast = fast.next
        slow = slow.next
    
    # slow.next 就是要删的节点
    slow.next = slow.next.next
    return dummy.next

# ⚠️ 为什么从 dummy 开始而不是 head?
#   如果删的是头节点 (n == 链表长度)
#   slow 需要在 dummy 位置 → slow.next = head → 删除 head
#   如果从 head 开始, 无法处理删头节点的情况
#
# ⚠️ 题目保证 n 合法 (1 ≤ n ≤ 链表长度)
#   如果不保证, 需要加边界检查
```

```
可视化: [1,2,3,4,5], n=2

  dummy → [1] → [2] → [3] → [4] → [5] → None
  fast=dummy, slow=dummy

  fast 先走 n+1=3 步:
    fast → [1] → [2] → [3]
  
  一起走:
    fast=[3]→[4], slow=dummy→[1]
    fast=[4]→[5], slow=[1]→  [2]
    fast=[5]→None, slow=[2]→ [3]
    fast=None → 停

  slow=[3], slow.next=[4] → 删除 [4]
  slow.next = [5]

  结果: [1] → [2] → [3] → [5] → None ✓
```

## 5. 删除节点（只给中间节点）

### LeetCode 237

```
题意: 给你链表中的一个节点 node（不是尾节点），请删除它。
     你无法访问前驱节点!

  [4,5,1,9], 给 node=5 → 删除后 [4,1,9]

⚠️ 骗局题! 既然访问不到前驱, 就不能传统删除
  → "偷天换日": 把下一个节点的值复制过来, 再删下一个节点

  before: 4 → [5] → 1 → 9 → None
                ↑ node

  step 1: node.val = node.next.val  → 4 → [1] → 1 → 9
  step 2: node.next = node.next.next → 4 → [1] ───→ 9

  等效于删了 5 ✓
```

```python
def delete_node(node):
    node.val = node.next.val          # ⚠️ 覆盖值
    node.next = node.next.next        # ⚠️ 跳过下一个

# ⚠️ 前提: node 不是尾节点（题目保证）
#   如果是尾节点, node.next 为 None, node.next.val 会报错
```

## 6. 删除操作核心对比

```
┌────────────────────────────────────────────────────────────────────┐
│ 场景                      │ 是否用 dummy │ 关键技巧               │
├────────────────────────────────────────────────────────────────────┤
│ 删除所有指定值 (203)       │ ✓           │ 删后 cur 不动           │
│ 删除排序重复-保留一个(83)  │ ✗           │ 头不会被删             │
│ 删除排序重复-不保留(82)    │ ✓           │ while 跳过所有相同值   │
│ 删除倒数第n个(19)          │ ✓           │ 快指针先走 n+1 步      │
│ 只给节点删自己(237)        │ ✗           │ 复制下一个值+删下一个  │
└────────────────────────────────────────────────────────────────────┘

⭐ 经验:
   能用 dummy 就用 dummy, 不会出错
   删除后重新检查 cur.next (可能连续删)
   while cur.next: 检查下一个, 永远站在前一个位置操作
```

## 本节要点速查

```
✅ 删除 = 让前驱的 next 跳过被删节点
✅ 站在「前一个节点」操作 → 检查 cur.next
✅ 删除后 cur 不前进（新 cur.next 可能也要删）
✅ 头可能被删 → 用 dummy
✅ 倒数第 n 个 → fast 先走 n+1 步
✅ 只给中间节点 → 复制 next 的值，删 next
```
