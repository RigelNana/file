# 排序链表

> 链表排序不能像数组那样随机访问，最适合归并排序。掌握链表归并排序是面试必备技能。

---

## 1. 归并排序（自顶向下）⭐⭐

### 理解与可视化

```
链表归并排序 = 找中点 + 分割 + 递归排序 + 合并

  [4] → [2] → [1] → [3] → None

  步骤1: 找中点, 分割
    左: [4] → [2] → None
    右: [1] → [3] → None

  步骤2: 递归排序左半
    左: [4] → [2] → None
      找中点: 左=[4], 右=[2]
      已是单节点,直接返回
      合并: [2] → [4] → None

  步骤3: 递归排序右半
    右: [1] → [3] → None
      找中点: 左=[1], 右=[3]
      合并: [1] → [3] → None

  步骤4: 合并两个有序链表
    merge([2→4], [1→3])
    比较: 1 < 2 → 取1
    比较: 2 < 3 → 取2
    比较: 3 < 4 → 取3
    剩余: 4
    结果: [1] → [2] → [3] → [4] → None ✓
```

### 代码模板

```python
# ━━━━ LeetCode 148: 排序链表 ━━━━
def sortList(head):
    # 递归终点: 空或只有一个节点
    if not head or not head.next:
        return head
    
    # 1. 找左中点并分割
    slow = fast = head
    while fast.next and fast.next.next:  # ⚠️ 左中点! 不是 fast and fast.next
        slow = slow.next
        fast = fast.next.next
    
    right_head = slow.next        # 右半的头
    slow.next = None              # ⚠️ 断开! 左半以 slow 结尾
    
    # 2. 递归排序两半
    left = sortList(head)
    right = sortList(right_head)
    
    # 3. 合并两个有序链表
    return merge(left, right)

def merge(l1, l2):
    dummy = ListNode()
    cur = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            cur.next = l1
            l1 = l1.next
        else:
            cur.next = l2
            l2 = l2.next
        cur = cur.next
    cur.next = l1 or l2
    return dummy.next

# ⚠️ 为什么用左中点?
#   [3→1] 如果用右中点: slow=1, 右半=None, 左半=[3→1] → 无限递归!
#   左中点: slow=3, 右半=[1], 左半=[3] → 正确分割
#
# ⚠️ 必须 slow.next = None 断开!
#   否则左半链表还连着右半, merge 时会出错
#
# ⚠️ 时间复杂度: O(n log n)  递归深度 log n, 每层合并 O(n)
# ⚠️ 空间复杂度: O(log n)  递归栈
```

## 2. 归并排序（自底向上）

### 理解与可视化

```
自底向上: 不用递归, 先两两合并, 再四四合并...

  [4] → [2] → [1] → [3] → None

  step=1: 每次取 1 个一组, 两两合并
    合并 [4] 和 [2] → [2,4]
    合并 [1] 和 [3] → [1,3]
    结果: [2] → [4] → [1] → [3]

  step=2: 每次取 2 个一组, 两两合并
    合并 [2,4] 和 [1,3] → [1,2,3,4]
    结果: [1] → [2] → [3] → [4] ✓

  step 翻倍直到 >= 链表长度
```

### 代码模板

```python
def sortList_bottom_up(head):
    if not head or not head.next:
        return head
    
    # 计算链表长度
    length = 0
    cur = head
    while cur:
        length += 1
        cur = cur.next
    
    dummy = ListNode(next=head)
    step = 1
    
    while step < length:
        prev = dummy
        cur = dummy.next
        
        while cur:
            # 取第一组 step 个节点
            left = cur
            for _ in range(step - 1):
                if cur.next:
                    cur = cur.next
                else:
                    break
            
            # 取第二组 step 个节点
            right = cur.next
            cur.next = None              # 断开第一组
            cur = right
            for _ in range(step - 1):
                if cur and cur.next:
                    cur = cur.next
                else:
                    break
            
            # 保存后续
            rest = None
            if cur:
                rest = cur.next
                cur.next = None          # 断开第二组
            
            # 合并两组
            merged = merge(left, right)
            prev.next = merged
            while prev.next:
                prev = prev.next
            prev.next = rest
            cur = rest
        
        step *= 2                        # ⚠️ 每轮 step 翻倍
    
    return dummy.next

# ⚠️ 自底向上空间 O(1) (不用递归栈)
# ⚠️ 面试中自顶向下更容易写, 推荐先写自顶向下
```

## 3. 插入排序

### LeetCode 147

```
题意: 对链表做插入排序

  [4,2,1,3]
  → sorted: [4]              unsorted: [2,1,3]
  → sorted: [2,4]            unsorted: [1,3]     (2插到4前面)
  → sorted: [1,2,4]          unsorted: [3]       (1插到最前面)
  → sorted: [1,2,3,4]        unsorted: []        (3插到2和4之间)
```

```python
def insertionSortList(head):
    dummy = ListNode(val=float('-inf'), next=head)  # ⚠️ 哨兵值极小
    cur = head
    
    while cur and cur.next:
        if cur.val <= cur.next.val:  # ⚠️ 已经有序, 直接前进
            cur = cur.next
            continue
        
        # 需要把 cur.next 插入到前面合适位置
        to_insert = cur.next
        cur.next = to_insert.next    # 从原位置摘出
        
        # 从头开始找插入位置
        prev = dummy
        while prev.next.val < to_insert.val:  # ⚠️ 严格小于, 保持稳定性
            prev = prev.next
        
        # 插入
        to_insert.next = prev.next
        prev.next = to_insert
    
    return dummy.next

# ⚠️ 时间 O(n²), 空间 O(1)
# ⚠️ 实际面试排序链表首选归并排序
# ⚠️ 优化: 如果 cur.val <= cur.next.val 就不需要插入, 直接跳过
```

## 4. 边界与易错总结

```
┌──────────────────────────────────────────────────────────┐
│                    链表排序易错点                          │
├──────────────────────────────────────────────────────────┤
│ 错误                    │ 后果          │ 修复           │
├────────────────────────┼───────────────┼───────────────┤
│ 找中点用右中点          │ [2,1] 无限递归 │ 用左中点       │
│ 忘记 slow.next=None    │ 左半没断开     │ 分割时断开     │
│ merge 里 < 而非 <=     │ 不稳定排序     │ 使用 <=        │
│ 排序后返回 head        │ head 已不是头  │ 返回 merge 结果│
│ 没有处理空/单节点       │ 递归无终止     │ 加 base case   │
└──────────────────────────────────────────────────────────┘
```

## 本节要点速查

```
✅ 链表排序首选归并排序: 时间 O(n log n)
✅ 三步走: 找左中点 → 断开 → 递归排序 → 合并
✅ 必须用左中点 (while fast.next and fast.next.next)
✅ 分割后必须 slow.next = None 断开
✅ merge 用 <= 保持稳定性
✅ 自底向上版本空间 O(1), 但代码更复杂
✅ 插入排序 O(n²), 了解即可, 面试不推荐
```
