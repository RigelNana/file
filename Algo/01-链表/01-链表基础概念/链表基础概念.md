# 链表基础概念

> 链表是所有数据结构的起点之一。理解链表就是理解「指针/引用」和「非连续存储」的思维方式。

---

## 1. 什么是链表？

### 理解与可视化

链表就像一列火车：每节车厢有两样东西 ——「货物」（数据）和「挂钩」（指向下一节的指针）。

```
单链表:
  ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ val: 1   │     │ val: 2   │     │ val: 3   │     │ val: 4   │
  │ next: ─────→   │ next: ─────→   │ next: ─────→   │ next:None│
  └──────────┘     └──────────┘     └──────────┘     └──────────┘
    head                                                 tail

  ● head 是「入口」，通过 head 可以沿着 next 走到任意节点
  ● tail.next == None 表示链表结尾
  ● 节点分散在内存各处，不像数组是连续存储
```

```
双链表:
  None ←── prev ┌──────┐ next ──→ prev ┌──────┐ next ──→ prev ┌──────┐ next → None
                │ val:1│              │ val:2│              │ val:3│
                └──────┘ ←── prev     └──────┘ ←── prev     └──────┘

  ● 双链表每个节点有 prev 和 next 两个指针
  ● 可以从前往后走，也可以从后往前走
  ● Python 的 collections.deque 底层就是双链表
```

## 2. 链表 vs 数组

### 理解与可视化

```
内存布局对比:

  数组 (连续):
  地址: 100  101  102  103  104
       [  1][  2][  3][  4][  5]
       ↑ 知道首地址就能算出任意元素地址: addr = base + index × size

  链表 (不连续):
  地址: 100       237       512       801
       [1|→237]  [2|→512]  [3|→801]  [4|None]
       ↑ 每个节点自带下一个的地址，不需要连续
```

| 操作 | 数组 | 链表 | 原因 |
|------|------|------|------|
| 按下标访问 | **O(1)** | O(n) | 数组连续，直接算地址；链表要从头走 |
| 头部插入 | O(n) | **O(1)** | 数组要搬移全部元素；链表改个指针 |
| 尾部插入 | O(1)* | O(n)/O(1)** | *摊还；**有尾指针时 |
| 中间插入 | O(n) | **O(1)*** | ***前提是已经定位到那个位置 |
| 中间删除 | O(n) | **O(1)*** | 同上 |
| 内存占用 | 紧凑 | 多（每节点多存指针） | 链表每节点额外 8 字节(64位) |
| 缓存友好 | ✓ | ✗ | 数组连续，CPU 缓存命中率高 |

**什么时候用链表？**
- 频繁在头部/中间 插入删除
- 不需要按下标随机访问
- 实现 LRU 缓存、任务队列等

## 3. 链表节点定义

### 代码模板

```python
# ━━━━━━━━━━━ 单链表节点 ━━━━━━━━━━━
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

# ⚠️ 注意: LeetCode 上链表节点就是这个定义
# ⚠️ next 默认 None，表示链表尾部
# ⚠️ val 默认 0，创建时一般会传入具体值
```

```python
# ━━━━━━━━━━━ 双链表节点 ━━━━━━━━━━━
class DListNode:
    def __init__(self, val=0, prev=None, next=None):
        self.val = val
        self.prev = prev
        self.next = next
```

## 4. 创建链表

### 代码模板

```python
# 从 Python 列表创建链表（刷题辅助函数）
def build_list(arr):
    """
    输入: [1, 2, 3, 4]
    输出: 1 → 2 → 3 → 4 → None 的头节点
    """
    dummy = ListNode()       # 哨兵节点，简化边界处理
    cur = dummy
    for x in arr:
        cur.next = ListNode(x)
        cur = cur.next
    return dummy.next        # dummy.next 才是真正的头节点
    #     ↑ 不是 dummy！dummy 只是辅助

# ⚠️ 易错点:
#   ❌ return dummy     → 返回了哨兵节点，多了一个 val=0 的头
#   ✓  return dummy.next → 返回真正的第一个节点
```

```python
# 打印链表（调试用）
def print_list(head):
    """输出: 1 → 2 → 3 → 4 → None"""
    parts = []
    while head:
        parts.append(str(head.val))
        head = head.next
    print(" → ".join(parts) + " → None")

# ⚠️ 如果链表有环，这个函数会死循环！
# 调试有环链表时要加计数器保护:
def print_list_safe(head, max_steps=100):
    parts = []
    step = 0
    while head and step < max_steps:
        parts.append(str(head.val))
        head = head.next
        step += 1
    if step == max_steps:
        parts.append("...(可能有环)")
    print(" → ".join(parts))
```

## 5. 哨兵节点（Dummy Head）⭐

### 理解与可视化

```
问题场景: 删除头节点时需要特殊处理

  不用哨兵:
    if head.val == target:
        head = head.next     ← 头节点要单独处理
    cur = head
    while cur.next:
        if cur.next.val == target:
            cur.next = cur.next.next
        else:
            cur = cur.next

  用哨兵:
    dummy = ListNode(next=head)   ← 加一个虚拟头
    cur = dummy                    ← 从 dummy 开始遍历
    while cur.next:                ← 统一处理，不用特殊判断头节点
        if cur.next.val == target:
            cur.next = cur.next.next
        else:
            cur = cur.next
    return dummy.next              ← 返回真正的头

    dummy → [1] → [2] → [3] → None
      ↑      ↑ 即使删除 1, dummy.next 自动指向 2
    虚拟节点   真正链表

⭐ 经验法则:
   凡是链表可能修改头节点（删除/插入/反转等），都用 dummy!
```

### 代码模板

```python
# 哨兵节点的标准用法
def some_linked_list_operation(head):
    dummy = ListNode(next=head)    # 关键: next=head 连接原链表
    
    cur = dummy
    # ... 操作 cur.next ...
    
    return dummy.next              # 关键: 返回 dummy.next 而非 dummy

# ⚠️ 常见错误:
#   ❌ dummy = ListNode()          → 没连 head!
#      dummy.next = head           → 虽然可以，但不如构造时传入简洁
#   ❌ return dummy                → 多返回了哨兵
#   ❌ return head                 → 如果 head 被删了就出错了
#   ✓  return dummy.next          → 永远正确
```

## 6. 链表常见错误总结

```
┌───────────────────────────────────────────────────────────────┐
│ 错误                          │ 后果                         │
├───────────────────────────────────────────────────────────────┤
│ 遍历时 cur 到 None 还访问     │ NoneType has no attr 'val'  │
│   cur.val                     │                              │
│                               │                              │
│ 插入顺序搞反:                  │ 丢失链表后半段               │
│   cur.next = new              │ new.next 还没设, 后面断了    │
│   new.next = cur.next         │                              │
│ 正确: new.next = cur.next     │                              │
│       cur.next = new          │                              │
│                               │                              │
│ 删除节点后 cur 还在用          │ 野指针（Python 不会崩,       │
│   deleted = cur.next          │ 但逻辑出错）                  │
│   cur.next = cur.next.next    │                              │
│   print(deleted.val)          │ 值还在但节点已脱链            │
│                               │                              │
│ 该用 dummy 没用               │ 特殊处理头节点容易写出 bug    │
│                               │                              │
│ while cur vs while cur.next   │ 前者遍历到最后一个节点        │
│                               │ 后者停在倒数第二个            │
│                               │ 要看你需要操作 cur 还是       │
│                               │ cur.next                     │
└───────────────────────────────────────────────────────────────┘
```

## 本节要点速查

```
✅ 链表 = 数据 + 指向下一个节点的指针
✅ 单链表只能从前往后走
✅ 插入/删除 O(1)（已知位置）, 访问 O(n)
✅ ListNode(val=0, next=None) 是标准定义
✅ dummy = ListNode(next=head) 是万能头技巧
✅ 返回 dummy.next 而非 dummy 或 head
✅ 插入时先连新节点的 next, 再改前节点的 next
```
