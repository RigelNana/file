# 快慢指针 ⭐

> 快慢指针是链表的经典技巧，用两个速度不同的指针遍历链表，可以解决找中点、判环、找环入口等问题。

---

## 1. 找中间节点 ⭐⭐

### 理解与可视化

```
核心思想: 快指针每次走 2 步, 慢指针每次走 1 步
         当快指针到终点时, 慢指针刚好在中间

  奇数长度 [1,2,3,4,5]:
    初始:   slow=1, fast=1
    第1步:  slow=2, fast=3
    第2步:  slow=3, fast=5
    fast.next=None → 停止, slow=3 ✓ (中间)
    
  偶数长度 [1,2,3,4]:
    初始:   slow=1, fast=1
    第1步:  slow=2, fast=3
    第2步:  slow=3, fast=5(None)
    fast=None → 停止, slow=3 (偏右的中间) ⚠️

  ⚠️ 偶数长度有两个中间节点:
     [1,2,3,4] → 中间是 2 或 3
     
  左中点 vs 右中点:
    while fast.next and fast.next.next   → 停在左中点 (2)
    while fast and fast.next             → 停在右中点 (3)
  
  记忆: "fast and fast.next" → 右(偏后)
        "fast.next and fast.next.next" → 左(偏前)
```

```
  可视化: 用 s 和 f 标记

  [1] → [2] → [3] → [4] → [5] → None
   sf

  [1] → [2] → [3] → [4] → [5] → None
          s           f

  [1] → [2] → [3] → [4] → [5] → None
                 s                  f
  f.next = None → 停止, 返回 s=3
```

### 代码模板

```python
# ━━━━ 右中点（LeetCode 876 标准）━━━━
def find_middle(head):
    slow = fast = head
    while fast and fast.next:     # ⚠️ fast 在前面! fast and fast.next
        slow = slow.next
        fast = fast.next.next
    return slow                   # 偶数时返回右中点

# ━━━━ 左中点（用于链表排序等场景）━━━━
def find_middle_left(head):
    slow = fast = head
    while fast.next and fast.next.next:  # ⚠️ fast.next 在前面
        slow = slow.next
        fast = fast.next.next
    return slow                   # 偶数时返回左中点

# ⚠️ 两种的区别:
#   [1,2,3,4]
#   右中点: while fast and fast.next → 返回 3
#   左中点: while fast.next and fast.next.next → 返回 2
#
# ⚠️ 什么时候用哪个:
#   找中间节点(LC 876): 右中点
#   链表排序(LC 148):   左中点 (需要在左半截尾处断开)
#   回文链表(LC 234):   左中点 (反转后半部分比较)
#
# ⚠️ 空链表处理:
#   右中点: head=None → while 不进入 → return None ✓
#   左中点: head=None → fast.next 报错! 需要提前判断
```

### LeetCode 876 链表的中间节点

```python
def middleNode(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

# 输入: [1,2,3,4,5] → 返回节点 3
# 输入: [1,2,3,4,5,6] → 返回节点 4（右中点）
```

## 2. 判断环形链表 ⭐⭐

### 理解与可视化

```
如果链表有环, 快指针一定会追上慢指针

  有环:
    1 → 2 → 3 → 4 → 5
              ↑         ↓
              ← ← ← ← ←

    slow 进入环后, fast 也在环里
    每一步 fast 比 slow 多走 1 步
    差距每次减少 1 → 一定能追上

  无环:
    1 → 2 → 3 → 4 → None
    fast 先到 None, 循环结束

  ⚠️ 为什么一定能追上?
     设环的长度为 C
     fast 每步多走 1, 即差距每步减少 1
     最多 C 步后差距从 C 减到 0 → 相遇
```

### 代码模板

```python
# ━━━━ LeetCode 141: 环形链表 ━━━━
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:     # ⚠️ 和找中点完全一样的循环条件
        slow = slow.next
        fast = fast.next.next
        if slow == fast:          # ⚠️ 走完再判断, 不是走之前判断
            return True
    return False

# ⚠️ 判断顺序:
#   先走 slow 和 fast
#   再判断 slow == fast
#   不能在循环开头判断（初始时 slow==fast==head, 会误判）
#
# ⚠️ 特殊情况:
#   空链表 → False ✓
#   单节点无环 → False ✓
#   单节点自环(head.next=head) → slow 和 fast 都走到 head → True ✓
```

## 3. 找环入口（关键推导）⭐⭐

### 理解与可视化

```
LeetCode 142: 环形链表 II

  设:
    a = 从 head 到环入口的距离
    b = 从环入口到相遇点的距离
    c = 从相遇点再回到环入口的距离
    环长 = b + c

  相遇时:
    slow 走了: a + b
    fast 走了: a + b + k(b+c)   (k 是在环里绕的整圈数, k≥1)
    fast = 2 * slow:
      a + b + k(b+c) = 2(a+b)
      k(b+c) = a + b
      a = k(b+c) - b = (k-1)(b+c) + c
      
  ⚠️ 结论: a = c + (k-1)(b+c)
     即: a 和 c 相差整数个环长
     
  所以: 一个指针从 head 出发, 一个从相遇点出发, 都每次走 1 步
        它们一定在环入口相遇!

  图示:
    head ─a─→ 入口 ─b─→ 相遇
               ↑          ↓
               ← ── c ── ←

    p1 从 head 走 a 步到入口
    p2 从相遇点走 c 步到入口（可能多绕几圈，但最终在入口相遇）
```

### 代码模板

```python
def detect_cycle(head):
    slow = fast = head
    
    # 第一阶段: 找相遇点
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            break
    else:                         # ⚠️ while-else: 循环正常结束(无环)
        return None
    
    # 第二阶段: 找入口
    p = head                      # 一个从头出发
    while p != slow:              # 一个从相遇点出发
        p = p.next
        slow = slow.next
    return p                      # 相遇处就是入口

# ⚠️ Python 的 while-else:
#   while 条件不满足正常退出 → 执行 else
#   while 中 break 退出 → 不执行 else
#   所以 else: return None 处理了无环情况
#
# ⚠️ 也可以不用 while-else:
def detect_cycle_v2(head):
    slow = fast = head
    has_cycle = False
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            has_cycle = True
            break
    if not has_cycle:
        return None
    p = head
    while p != slow:
        p = p.next
        slow = slow.next
    return p
```

## 4. 判断回文链表

### LeetCode 234

```
题意: 判断链表是否回文
  1→2→2→1 → True
  1→2     → False

思路: 
  1. 快慢指针找左中点
  2. 反转后半部分
  3. 逐一比较

图示:
  原始:    1 → 2 → 2 → 1 → None
  找中点:  slow=2(左中), cut: [1→2] 和 [2→1]
  反转后半: [1→2→None]  [1→2→None]
  比较: 1==1 ✓, 2==2 ✓ → 回文!
```

```python
def is_palindrome(head):
    if not head or not head.next:
        return True
    
    # 1. 快慢指针找左中点
    slow = fast = head
    while fast.next and fast.next.next:  # ⚠️ 左中点!
        slow = slow.next
        fast = fast.next.next
    
    # 2. 反转后半部分 (slow.next 开始)
    prev = None
    cur = slow.next
    slow.next = None                      # ⚠️ 断开前后两半
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    
    # 3. 逐一比较
    p1, p2 = head, prev
    while p1 and p2:
        if p1.val != p2.val:
            return False
        p1 = p1.next
        p2 = p2.next
    return True

# ⚠️ 必须用左中点! 
#   偶数 [1,2,2,1]: 左中=2(第一个), 后半从第二个2开始
#   如果用右中点, 后半少了一个节点
#
# ⚠️ 奇数 [1,2,1]: 左中=2, 后半=[1], 前半=[1,2]
#   比较: 1==1 ✓, p2=None → 停止 → True ✓
#   多出的中间节点不影响结果
```

## 5. 快慢指针其他应用

### 5.1 删除链表倒数第 N 个节点 (快慢指针法)

```python
# LeetCode 19 的另一种理解: 也属于快慢指针范畴
# 快指针先走 n 步, 然后两个一起走, 快到末尾时慢到倒数第 n 个的前驱

def removeNthFromEnd(head, n):
    dummy = ListNode(next=head)
    fast = slow = dummy
    for _ in range(n + 1):       # ⚠️ 走 n+1 步! 这样 slow 停在要删除节点的前驱
        fast = fast.next
    while fast:
        fast = fast.next
        slow = slow.next
    slow.next = slow.next.next   # 删除
    return dummy.next

# ⚠️ 为什么是 n+1 步?
#   fast 比 slow 多走 n+1 步
#   fast 从 dummy 走到 None = 总共走 len+1 步
#   slow 走了 len+1-(n+1) = len-n 步
#   slow 停在第 len-n 个位置(从 dummy 算起), 即倒数第 n 个的前驱
```

## 本节要点速查

```
✅ 找中点: 快2慢1，fast走完时slow在中间
✅ 右中点: while fast and fast.next → 偶数取靠后的
✅ 左中点: while fast.next and fast.next.next → 偶数取靠前的
✅ 判环: 快慢指针相遇则有环
✅ 找环入口: 相遇后一个从头走一个从相遇点走，再次相遇即入口
✅ 数学推导: a = c + (k-1)(b+c)
✅ 回文检测: 找左中点 → 反转后半 → 逐一比较
✅ 易错: 判环要先走再比较; 回文要用左中点; n+1步差距
```
