## 一、容量建模

### 1. 容量建模的主要方法有哪些？

**答：**

| 方法 | 原理 | 适用场景 | 精度 |
|------|------|----------|------|
| **线性回归** | QPS 与资源的线性关系 | 稳定增长服务 | 中 |
| **排队论** | Little's Law + M/M/c 模型 | 连接池/线程池 | 高 |
| **经验法则** | 历史数据 + 安全系数 | 快速估算 | 低 |
| **压测建模** | 实际负载测试数据 | 有压测条件 | 高 |
| **机器学习** | 时序预测模型 | 大量历史数据 | 高 |

### 2. 如何用线性回归做容量建模？

**答：**

```
线性容量模型：

Resource = a × QPS + b

从历史数据拟合：
  QPS    CPU_Cores
  100    0.5
  500    2.3
  1000   4.8
  2000   9.5
  
  拟合结果: CPU = 0.0048 × QPS + 0.05
  
  预测 5000 QPS: CPU = 0.0048 × 5000 + 0.05 = 24.05 cores

PromQL 做线性预测：
  # 预测 30 天后的 CPU 使用率
  predict_linear(
    avg_over_time(
      node_cpu_seconds_total{mode="idle"}[7d]
    )[30d:1d],
    30 * 86400
  )

注意事项：
  ├── 线性模型只在"线性区间"有效
  ├── 需要通过压测找到非线性拐点
  ├── 超过拐点需要非线性模型
  └── 定期用新数据校准模型
```

### 3. 排队论模型在容量规划中怎么用？

**答：**

```
Little's Law（最基础）：
  L = λ × W
  系统中请求数 = 到达率 × 平均停留时间

应用示例（数据库连接池）：
  λ = 1000 QPS
  W = 10ms = 0.01s（平均查询时间）
  L = 1000 × 0.01 = 10（并发连接数）
  
  加 Headroom: 10 × 2 = 20 连接
  加排队缓冲: max_pool = 50

M/M/c 排队模型：
  c = 服务器数量（Pod/线程数）
  λ = 到达率
  μ = 服务率（每服务器处理能力）
  ρ = λ / (c × μ)  利用率

  当 ρ > 0.7 时：
    排队延迟急剧增加
    → 需要扩容

  容量阈值：
    ρ < 0.5  安全（大量余裕）
    ρ = 0.7  阈值（开始规划扩容）
    ρ = 0.85 危险（尽快扩容）
    ρ > 0.95 临界（随时可能故障）
```

## 二、容量预测

### 4. 如何做流量预测？

**答：**

```
流量预测方法：

方法一：趋势外推
  观察过去 6-12 个月的增长趋势
  假设未来延续同样的增长率
  
  当前: 10000 QPS
  月增长: 15%
  3 个月后: 10000 × 1.15³ = 15209 QPS
  6 个月后: 10000 × 1.15⁶ = 23131 QPS

方法二：业务驱动
  产品计划 + 营销计划 → 用户增长 → 流量增长
  
  例：
  ├── 新功能上线 → 预计 +20% 用户
  ├── 大促活动 → 峰值 5x 日常
  ├── 新市场拓展 → +30% 流量
  └── 综合增长因子

方法三：时序分析（技术方案）
  ├── ARIMA/SARIMA：处理趋势+季节性
  ├── Prophet（Meta）：易用、自动季节性
  ├── Holt-Winters：指数平滑预测
  └── LSTM：深度学习时序预测
```

```python
# Prophet 流量预测示例
from prophet import Prophet
import pandas as pd

# 准备数据
df = pd.read_csv('daily_qps.csv')  # ds, y
df.columns = ['ds', 'y']

# 训练模型
model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=True,
)
model.fit(df)

# 预测未来 90 天
future = model.make_future_dataframe(periods=90)
forecast = model.predict(future)

# 输出预测值和置信区间
print(forecast[['ds', 'yhat', 'yhat_lower', 'yhat_upper']].tail())

# 可视化
model.plot(forecast)
model.plot_components(forecast)
```

### 5. 如何处理流量的季节性和周期性？

**答：**

```
常见的流量周期：

日周期（Daily）:
  QPS
  │    ╱╲
  │   ╱  ╲
  │··╱    ╲··
  │ ╱      ╲
  └──────────── 时间
  0  8  12 20 24

周周期（Weekly）:
  ├── 工作日流量高
  ├── 周末流量低（ToB）或高（ToC）

月/年周期（Seasonal）:
  ├── 电商：双11、618、圣诞大促
  ├── 教育：开学季、考试季
  ├── 金融：月末、季末、年末

容量规划应对策略：
  ┌─────────────────────────────────┐
  │ 1. 按预测峰值 × 1.3 规划       │
  │ 2. 大促按 peak × 2-3 规划      │
  │ 3. 利用弹性伸缩应对波动        │
  │ 4. 预留 Buffer 应对预测偏差    │
  └─────────────────────────────────┘
```

## 三、面试高频题

### 6. 面试题：如何预测双11流量？

**答：**

```
双11 容量规划方案：

1. 历史数据分析
   ├── 去年双11 峰值 vs 日常倍率
   ├── 去年活动前后增长趋势
   └── 历年倍率趋势（逐年递减？递增？）

2. 业务输入
   ├── 今年预期 GMV 增长
   ├── 新增品类/商家数
   ├── 营销投放计划
   └── 预计用户增长

3. 估算公式
   峰值 QPS = 日常峰值 × 活动倍率 × 业务增长系数

   例：
   日常峰值: 50000 QPS
   活动倍率: 8x (去年数据)
   业务增长: 1.3
   安全系数: 1.5
   
   目标容量: 50000 × 8 × 1.3 × 1.5 = 780000 QPS

4. 验证方案
   ├── 全链路压测验证
   ├── 按阶段扩容（T-30/T-7/T-1）
   └── 准备弹性扩容应急方案
```

### 7. 面试题：容量预测不准怎么办？

**答：**

```
应对预测不准的策略：

1. 防御性设计
   ├── 始终保持 30%+ Headroom
   ├── 弹性伸缩 + 限流保底
   └── 降级预案随时可用

2. 快速响应
   ├── 容量告警（利用率 > 70%）
   ├── 自动扩缩容（K8s HPA）
   ├── 云资源快速扩容能力
   └── CDN 缓存分流

3. 持续校准
   ├── 每月对比预测 vs 实际
   ├── 分析偏差原因
   ├── 调整模型参数
   └── 更新安全系数

偏差分析表：
  月份  预测QPS  实际QPS  偏差率  原因
  Jan   5000     4800     -4%    正常
  Feb   5500     6200     +13%   春节活动超预期
  Mar   6000     5900     -2%    正常
  → 需要加入节假日因子到预测模型
```
