## 学习建议

| 优先级 | 类别 | 说明 |

| ⭐⭐⭐⭐⭐ | SRE基础与文化 | SRE的核心理念，面试必考 |
| ⭐⭐⭐⭐⭐ | SLI/SLO/SLA | SRE最核心的方法论框架 |
| ⭐⭐⭐⭐⭐ | 可观测性体系 | 没有可观测性就没有可靠性 |
| ⭐⭐⭐⭐⭐ | 事件管理与On-Call | SRE日常核心工作 |
| ⭐⭐⭐⭐ | 故障排查与根因分析 | 实际工作中最常用的技能 |
| ⭐⭐⭐⭐ | 可靠性设计模式 | 系统设计面试高频考点 |
| ⭐⭐⭐⭐ | 变更管理与发布工程 | 大部分故障由变更引起 |
| ⭐⭐⭐⭐ | Linux系统与性能调优 | SRE的技术根基 |
| ⭐⭐⭐ | 容量规划与性能工程 | 保障系统长期稳定运行 |
| ⭐⭐⭐ | 混沌工程 | 主动发现系统弱点 |

| ⭐⭐ | SRE面试实战 | 面试冲刺阶段重点看 |

### SRE vs DevOps 学习侧重

| 维度 | SRE | DevOps |
|------|-----|--------|
| 核心目标 | **可靠性**（Reliability） | **交付速度**（Delivery） |
| 关键指标 | SLI/SLO/SLA、错误预算 | 部署频率、变更前置时间 |
| 日常工作 | On-Call、事件响应、容量规划 | CI/CD、自动化构建、基础设施 |
| 设计关注 | 故障域、降级、熔断 | 流水线、容器编排、IaC |
| 方法论 | 错误预算、Toil消除 | 持续集成/持续交付 |
| 推荐阅读 | 《SRE: How Google Runs Production Systems》 | 《The Phoenix Project》 |

### 推荐学习路径

1. **入门阶段**：01 SRE基础 → 02 SLI/SLO/SLA → 04 事件管理
2. **核心阶段**：03 可观测性 → 07 故障排查 → 08 可靠性模式
3. **进阶阶段**：05 容量规划 → 06 变更管理 → 09 混沌工程
4. **深入阶段**：10 自动化 → 11 Linux调优 → 12 分布式系统
5. **扩展阶段**：13 网络 → 14 K8s运维 → 15 面试实战
