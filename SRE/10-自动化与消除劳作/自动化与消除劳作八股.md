## 一、Toil 管理

### 1. 如何度量和管理 Toil？

**答：**

**Toil 度量维度：**
- **时间占比**：每周花在 Toil 上的时间百分比
- **工单量**：重复性运维工单的数量
- **中断频率**：On-Call 期间处理告警的次数
- **手动操作比例**：手动操作 vs 自动化操作的比例

**Toil 管理流程：**
1. **识别**：记录所有手动运维工作
2. **分类**：按频率和耗时排序
3. **评估**：计算自动化 ROI（投入产出比）
4. **消除**：从 ROI 最高的开始自动化
5. **持续监控**：定期审查 Toil 水平

### 2. 自动化的 ROI 如何计算？

**答：**

$$\text{ROI} = \frac{\text{节省时间} \times \text{频率} \times \text{时间窗口} - \text{开发成本}}{\text{开发成本}}$$

**决策参考：**
- 每天执行 → 投入几天开发也值得
- 每周执行 → 投入 1-2 天开发
- 每月执行 → 只在痛点很大时自动化
- 每年执行 → 通常不值得自动化（写好 Runbook 即可）

## 二、自动化实践

### 3. 自动化修复（Auto-remediation）是什么？

**答：** 自动化修复是指系统在检测到问题后**自动执行修复操作**，无需人工干预。

**自动化修复的层级：**

| 层级 | 描述 | 示例 |
|------|------|------|
| **L0** | 手动执行文档化的步骤 | 参考 Runbook 手动操作 |
| **L1** | 脚本化但需人工触发 | 点击按钮执行修复脚本 |
| **L2** | 告警触发，人工确认后执行 | ChatOps 中审批后执行 |
| **L3** | 全自动检测和修复 | 自动重启、自动扩容 |
| **L4** | 预测性修复 | 预测磁盘即将满，提前清理 |

### 4. 什么是 AIOps？

**答：** AIOps（Artificial Intelligence for IT Operations）是将 AI/ML 技术应用于运维的实践。

| 应用场景 | 传统方式 | AIOps 方式 |
|----------|----------|------------|
| 异常检测 | 静态阈值告警 | 动态基线、异常模式识别 |
| 根因分析 | 人工关联日志和指标 | 自动关联分析、拓扑推理 |
| 告警降噪 | 手动配置规则 | ML 聚类和去重 |
| 容量预测 | 人工分析趋势 | 时序预测模型 |
| 故障预测 | 无 | 基于历史模式预测故障 |

### 5. 什么是 ChatOps？

**答：** ChatOps 是将运维操作集成到**聊天工具**（Slack/Teams）中的实践。

**核心理念**：将操作、通知、协作集中到一个地方（Chat Room）。

**ChatOps 的价值：**
- **透明性**：所有操作在聊天频道中可见，团队共享上下文
- **协作**：多人实时协作处理事件
- **审计**：操作记录天然留痕
- **自动化入口**：通过 Bot 命令触发自动化操作
- **降低门槛**：新人无需记住复杂命令，通过 Bot 交互
