## 一、自运维概念

### 1. 什么是自运维系统（Self-Healing System）？

**答：**

```
自运维系统定义：
  系统能够自动检测异常、诊断问题、
  执行修复操作，无需人工干预。

自运维成熟度：
  Level 0: 告警通知人来处理
  Level 1: 提供诊断信息辅助人决策
  Level 2: 建议修复方案供人确认
  Level 3: 自动执行修复，人来审核
  Level 4: 完全自动化，人只看报告

自运维闭环：
  ┌── 检测 ──→ 诊断 ──→ 决策 ──→ 修复 ──→ 验证 ──┐
  │                                                │
  └────────────── 反馈学习 ←──────────────────────┘
```

### 2. 自运维系统的典型场景有哪些？

**答：**

| 场景 | 检测方式 | 自动修复 |
|------|----------|----------|
| Pod CrashLoop | K8s 健康检查 | 自动重启 + HPA |
| 磁盘空间不足 | 指标监控 | 自动清理日志/临时文件 |
| 证书即将过期 | 周期检查 | cert-manager 自动续期 |
| 流量突增 | QPS 监控 | HPA 自动扩容 |
| 节点不健康 | Node 状态 | 自动 Cordon + 迁移 Pod |
| 数据库连接池满 | 连接数监控 | 自动重启 + 告警 |
| OOM Kill | 内存监控 | 自动调整 Limit + 重启 |

### 3. Kubernetes 本身有哪些自愈机制？

**答：**

```
K8s 内置自愈：

1. Pod 自动重启
   restartPolicy: Always
   容器崩溃 → kubelet 自动重启

2. ReplicaSet 维持副本数
   Pod 被删除 → 自动创建新 Pod

3. Liveness Probe 健康检查
   检查失败 → 重启容器

4. Readiness Probe 就绪检查
   不就绪 → 从 Service 摘除

5. HPA 自动扩缩容
   负载增加 → 增加 Pod 数量

6. Node 自动驱逐
   节点 NotReady → Pod 迁移到健康节点

7. PDB 保障最小可用
   防止维护操作导致全部 Pod 下线

示例配置：
  livenessProbe:
    httpGet:
      path: /healthz
      port: 8080
    initialDelaySeconds: 10
    periodSeconds: 5
    failureThreshold: 3    # 3次失败重启

  readinessProbe:
    httpGet:
      path: /ready
      port: 8080
    periodSeconds: 5
    failureThreshold: 2
```

## 二、自运维设计模式

### 4. 如何设计自动化修复流程？

**答：**

```
自动修复设计模式：

模式一：重启修复（最简单）
  检测：Pod 内存使用持续超过 90%
  决策：可能内存泄漏
  动作：优雅重启 Pod
  验证：重启后内存恢复正常

模式二：扩容修复
  检测：QPS 超过阈值，延迟上升
  决策：容量不足
  动作：HPA 增加副本数
  验证：延迟恢复正常

模式三：回滚修复
  检测：发布后错误率突升
  决策：新版本有问题
  动作：自动回滚到上一版本
  验证：错误率恢复

模式四：切换修复
  检测：主实例不可用
  决策：需要故障转移
  动作：切换到备用实例
  验证：服务恢复可用

安全原则：
  ┌──────────────────────────────────────┐
  │ 1. 自动修复只做安全操作（重启/扩容）  │
  │ 2. 有限次重试（避免无限循环）          │
  │ 3. 修复失败自动升级给人               │
  │ 4. 所有操作有审计日志                  │
  └──────────────────────────────────────┘
```

### 5. 如何用 Kubernetes Operator 实现自运维？

**答：**

```go
// 简化的 Operator 自愈逻辑
func (r *AppReconciler) Reconcile(ctx context.Context, 
    req ctrl.Request) (ctrl.Result, error) {
    
    app := &myv1.MyApp{}
    r.Get(ctx, req.NamespacedName, app)
    
    // 检查健康状态
    if !isHealthy(app) {
        // 尝试自动修复
        switch detectIssue(app) {
        case HighMemory:
            // 重启 Pod
            restartPods(app)
            // 5分钟后重新检查
            return ctrl.Result{RequeueAfter: 5 * time.Minute}, nil
            
        case HighLatency:
            // 扩容
            scaleUp(app, app.Spec.Replicas + 2)
            return ctrl.Result{RequeueAfter: 3 * time.Minute}, nil
            
        case VersionRegression:
            // 回滚
            rollback(app, app.Status.LastGoodVersion)
            return ctrl.Result{RequeueAfter: 5 * time.Minute}, nil
        }
    }
    
    return ctrl.Result{RequeueAfter: 30 * time.Second}, nil
}
```

## 三、面试高频题

### 6. 面试题：自运维系统的风险是什么？

**答：**

```
主要风险及应对：

1. 误判导致错误修复
   风险：检测不准确导致不必要的重启/回滚
   应对：多条件验证 + 限制修复频率

2. 修复循环（Flapping）
   风险：自动扩容→缩容→扩容 反复循环
   应对：冷却期（Cooldown）+ 稳定窗口

3. 掩盖根因
   风险：自动重启掩盖了内存泄漏
   应对：自动修复后仍创建工单追踪根因

4. 级联修复
   风险：A 服务自动回滚引发 B 服务故障
   应对：修复操作的影响评估 + 限速

5. 权限风险
   风险：自动修复系统权限过大
   应对：最小权限 + 审计日志
```

### 7. 面试题：如何设计自动扩缩容？

**答：**

```yaml
# HPA 配置最佳实践
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: payment-api
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: payment-api
  minReplicas: 3
  maxReplicas: 50
  behavior:
    scaleUp:
      stabilizationWindowSeconds: 60   # 扩容冷静期
      policies:
        - type: Percent
          value: 50                     # 每次最多扩50%
          periodSeconds: 60
    scaleDown:
      stabilizationWindowSeconds: 300  # 缩容冷静期5分钟
      policies:
        - type: Percent
          value: 10                     # 每次最多缩10%
          periodSeconds: 60
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    - type: Pods
      pods:
        metric:
          name: http_requests_per_second
        target:
          type: AverageValue
          averageValue: 1000
```
