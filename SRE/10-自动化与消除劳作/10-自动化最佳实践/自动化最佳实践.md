## 一、自动化原则

### 1. Google SRE 的自动化原则是什么？

**答：**

```
Google SRE 自动化层级（由低到高）：

1. 无自动化
   人手动执行所有操作

2. 外部维护的系统特定自动化
   团队编写的一次性脚本

3. 外部维护的通用自动化
   通用自动化框架

4. 内部维护的系统特定自动化
   系统自身包含自动化逻辑

5. 不需要自动化的系统
   系统设计上不需要人工干预

核心原则：
  ┌──────────────────────────────────────┐
  │ 1. 消除 Toil 而非管理 Toil          │
  │ 2. 自动化应该像产品一样被维护        │
  │ 3. 如果做第三次，就自动化            │
  │ 4. 自动化必须有监控和告警            │
  │ 5. 自动化失败时必须安全降级          │
  │ 6. 代码 Review 自动化代码            │
  └──────────────────────────────────────┘

何时自动化 vs 何时不自动化：
  ✅ 自动化：重复 >3次、错误风险高、需快速响应
  ❌ 不自动化：一次性任务、需要复杂判断、ROI 很低
```

### 2. 自动化的常见反模式有哪些？

**答：**

| 反模式 | 描述 | 正确做法 |
|--------|------|----------|
| 脚本地狱 | 散落各处的脚本无人维护 | 统一自动化平台管理 |
| 过度自动化 | 一次性任务也要自动化 | 评估 ROI 后决定 |
| 只写不维护 | 自动化后不再关注 | 自动化代码也要 Review 和测试 |
| 无错误处理 | 脚本失败静默忽略 | 完善错误处理和告警 |
| 无文档 | 自动化逻辑无人理解 | 关键逻辑添加注释和文档 |
| 权限过大 | 自动化有 root 权限 | 最小权限原则 |
| 单点依赖 | 只有一个人懂这个脚本 | 知识共享、代码 Review |

## 二、自动化策略

### 3. 如何系统化消除 Toil？

**答：**

```
Toil 消除框架：

Step 1: 度量（Measure）
  - 记录所有手动操作
  - 分类：重复性/频率/耗时
  - 计算 Toil 占比

Step 2: 排序（Prioritize）
  使用 Impact/Effort 矩阵：
  
  高影响 │ ③深入投入  │ ①优先消除
         │            │
  低影响 │ ④暂不处理  │ ②快速解决
         └────────────┴──────────
           高成本       低成本

Step 3: 自动化（Automate）
  选择合适的自动化方案实施

Step 4: 验证（Validate）
  - 自动化是否真正减少了 Toil
  - Toil 占比是否下降

Step 5: 持续改进（Iterate）
  - 定期 Review Toil 列表
  - 新的 Toil 持续消除

Toil 预算管理：
  SRE 团队工作时间分配：
  ┌─────────────────────────────┐
  │ Toil（运维劳作）  ≤ 50%     │
  │ 工程工作          ≥ 50%     │
  │  ├── 自动化开发              │
  │  ├── 可靠性改进              │
  │  ├── 工具开发                │
  │  └── 架构优化                │
  └─────────────────────────────┘
```

## 三、自动化工程实践

### 5. 如何编写可维护的自动化代码？

**答：**

```python
# 自动化代码最佳实践示例

# ✅ 良好的自动化脚本结构
import logging
import sys
from datetime import datetime

logger = logging.getLogger(__name__)

class DiskCleanup:
    """磁盘清理自动化任务"""
    
    def __init__(self, threshold=85, dry_run=False):
        self.threshold = threshold
        self.dry_run = dry_run
    
    def check_preconditions(self):
        """预检：确认运行条件"""
        usage = get_disk_usage()
        if usage < self.threshold:
            logger.info(f"磁盘使用率 {usage}% 低于阈值，跳过清理")
            return False
        return True
    
    def execute(self):
        """执行清理"""
        if not self.check_preconditions():
            return
        
        # 按优先级清理
        cleaners = [
            self.clean_temp_files,
            self.clean_old_logs,
            self.clean_docker_images,
        ]
        
        for cleaner in cleaners:
            if self.dry_run:
                logger.info(f"[DRY RUN] 将执行: {cleaner.__name__}")
                continue
            
            freed = cleaner()
            logger.info(f"{cleaner.__name__} 释放 {freed}GB")
            
            # 每步后检查是否已足够
            if get_disk_usage() < self.threshold - 10:
                logger.info("磁盘空间已恢复，停止清理")
                break
    
    def verify(self):
        """验证清理效果"""
        usage = get_disk_usage()
        if usage > self.threshold:
            logger.warning(f"清理后磁盘使用率仍为 {usage}%")
            return False
        logger.info(f"清理完成，磁盘使用率: {usage}%")
        return True

# 关键实践：
# 1. 有 dry_run 模式（安全测试）
# 2. 有预检条件（避免误操作）
# 3. 有日志记录（可审计）
# 4. 有验证步骤（确认效果）
# 5. 可增量执行（不过度清理）
```

### 6. 自动化任务如何监控？

**答：**

```
自动化任务监控体系：

1. 执行监控
   - 任务是否按时执行
   - 执行时长是否正常
   - 成功/失败率

2. 效果监控
   - 自动化是否真正解决了问题
   - 有无副作用

3. 关键指标
   ┌──────────────────┬──────────────────┐
   │ 指标             │ 告警条件          │
   ├──────────────────┼──────────────────┤
   │ 任务执行成功率   │ < 95%            │
   │ 任务执行延迟     │ > 基线 2 倍      │
   │ 未执行任务数     │ > 0              │
   │ 重试次数         │ > 3              │
   │ 自动化覆盖率     │ 持续跟踪         │
   │ Toil 占比        │ > 50%            │
   └──────────────────┴──────────────────┘

Prometheus 指标设计：
  # 任务执行计数
  automation_task_total{task="disk_cleanup",status="success"}
  automation_task_total{task="disk_cleanup",status="failure"}

  # 执行时长
  automation_task_duration_seconds{task="disk_cleanup"}

  # Toil 时间
  toil_hours_total{team="sre",type="manual_deploy"}
```
