## 一、ROI 分析基础

### 1. 如何计算自动化项目的 ROI？

**答：**

```
基本公式：
  ROI = (收益 - 成本) / 成本 × 100%

自动化收益计算：
  年节省时间 = 频率 × 单次耗时 × 12
  年节省成本 = 年节省时间 × 工程师时薪
  风险避免收益 = 人为错误概率 × 错误成本

自动化成本计算：
  开发成本 = 开发时间 × 工程师时薪
  维护成本 = 年度维护时间 × 工程师时薪
  基础设施成本 = 运行所需资源费用

示例计算：
  手动扩容任务：
  ├── 频率：8 次/月
  ├── 单次耗时：15 分钟
  ├── 年手动成本：8 × 0.25h × 12 × ¥300/h = ¥7,200
  ├── 人为出错风险：年均1次 × ¥50,000 = ¥50,000
  ├── 总年收益：¥57,200
  │
  ├── 开发成本：40h × ¥300/h = ¥12,000
  ├── 年维护成本：10h × ¥300/h = ¥3,000
  ├── 总成本：¥15,000
  │
  └── ROI = (57,200 - 15,000) / 15,000 = 281%
      回本周期 = 15,000 / (57,200/12) ≈ 3.1 个月
```

### 2. xkcd 的"值不值得自动化"图表怎么理解？

**答：**

```
著名的 xkcd #1205 "Is It Worth the Time?"

如果你每次节省时间... 且每天/周/月做这么多次...
那么 5 年内你总共节省的时间是：

              每天    每周    每月    每年
  1秒         1天     2小时   25分   4分
  5秒         5天     10小时  2小时  21分
  30秒        4周     2.5天   12小时 2小时
  1分钟       8周     5天     1天    4小时
  5分钟       9月     3.5周   5天    1天
  30分钟      *       5月     4周    5天
  1小时       *       10月    8周    10天

  * = 可能不值得，因为自动化本身也要 5 年维护

实际应用原则：
  ┌──────────────────────────────────────┐
  │ 1. 高频低耗时 → 自动化收益大         │
  │ 2. 低频高耗时 → 要看风险收益         │
  │ 3. 低频低耗时 → 可能不值得自动化     │
  │ 4. 别忘了算维护成本                   │
  └──────────────────────────────────────┘
```

### 3. 自动化的隐性收益有哪些？

**答：**

| 隐性收益 | 说明 | 难以量化但很重要 |
|----------|------|-----------------|
| **减少人为错误** | 自动化不会手抖 | 避免 P0 事件 |
| **提升一致性** | 每次执行结果相同 | 避免配置漂移 |
| **可审计** | 自动化有日志记录 | 合规需求 |
| **知识固化** | 流程变成代码 | 降低人员依赖 |
| **团队士气** | 消除无聊重复工作 | 降低离职率 |
| **加速交付** | 自动化比人快 | 提升业务响应 |

## 二、决策框架

### 4. 何时应该自动化，何时不应该？

**答：**

```
自动化决策矩阵：

             高频率
              │
    ┌─────────┤─────────┐
    │ 必须     │ 优先     │
    │ 自动化   │ 自动化   │
    │         ├─────────┤
    │ (高频+   │ (高频+   │
    │  高风险) │  低风险) │
    ├─────────┤─────────┤
    │ 考虑     │ 不值得   │
    │ 自动化   │ 自动化   │
    │         ├─────────┤
    │ (低频+   │ (低频+   │
    │  高风险) │  低风险) │
    └─────────┤─────────┘
              │
             低频率
    高风险 ←─────────→ 低风险

不应该自动化的场景：
  ├── 一次性任务
  ├── 频繁变化的流程（自动化跟不上变化）
  ├── 需要复杂判断的操作
  └── 自动化成本 >> 手动成本（5年内）
```

### 5. 面试题：如何说服团队投入时间做自动化？

**答：**

```
说服策略：

1. 用数据说话
   "我们每月花 X 小时在这项任务上"
   "自动化后 Y 个月回本"

2. 用风险说话
   "手动操作导致了 N 次事故"
   "自动化可以消除人为错误"

3. 用对标说话
   "行业标准 Toil 率 < 50%，我们是 70%"

4. 找速赢项目
   选一个小且高 ROI 的项目先做
   用成果推动更多自动化投入

5. 量化机会成本
   "如果 SRE 把 Toil 时间做工程"
   "可以完成 N 个提升可靠性的项目"
```
