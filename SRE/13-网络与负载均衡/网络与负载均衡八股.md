## 一、网络基础

### 1. TCP 三次握手和四次挥手的过程？

**答：**

**三次握手：**
```
Client          Server
  │── SYN ──────→│     1. 客户端发送 SYN
  │←── SYN+ACK ──│     2. 服务端回复 SYN+ACK
  │── ACK ──────→│     3. 客户端发送 ACK
```

**四次挥手：**
```
Client          Server
  │── FIN ──────→│     1. 客户端发送 FIN（我不再发送）
  │←── ACK ──────│     2. 服务端确认
  │←── FIN ──────│     3. 服务端发送 FIN（我也不再发送）
  │── ACK ──────→│     4. 客户端确认，进入 TIME_WAIT
```

**SRE 关注点：**
- TIME_WAIT 过多 → 调整 `net.ipv4.tcp_tw_reuse`
- 连接建立慢 → 检查 SYN 队列和 Accept 队列
- `ss -s` 查看各状态连接统计

## 二、DNS

### 2. DNS 解析过程及 SRE 最佳实践？

**答：**

**解析过程**（递归+迭代）：
```
客户端 → 本地DNS缓存 → 递归解析器 → 根DNS → TLD DNS → 权威DNS
```

**SRE DNS 最佳实践：**
- 合理设置 TTL（权衡缓存和更新速度）
- 使用 DNS 做全局负载均衡（GeoDNS）
- 配置健康检查剔除不健康后端
- 监控 DNS 解析延迟和失败率
- 考虑 DNS 缓存导致的切换延迟

## 三、负载均衡

### 3. 四层和七层负载均衡的区别？

**答：**

| 维度 | 四层（L4） | 七层（L7） |
|------|-----------|-----------|
| 工作层次 | 传输层（TCP/UDP） | 应用层（HTTP/gRPC） |
| 转发依据 | IP + Port | URL、Header、Cookie |
| 性能 | 高（不解析应用层） | 较低（需解析 HTTP） |
| 功能 | 简单转发 | 路由、重写、认证等 |
| 代表 | LVS, NLB | Nginx, Envoy, ALB |

### 4. 常见的负载均衡算法有哪些？

**答：**

| 算法 | 原理 | 适用场景 |
|------|------|----------|
| **轮询** | 按顺序分配 | 后端性能一致 |
| **加权轮询** | 按权重比例分配 | 后端性能不一致 |
| **最少连接** | 分配给连接数最少的 | 长连接场景 |
| **IP Hash** | 按源 IP 哈希分配 | 有状态会话保持 |
| **一致性哈希** | 哈希环分配 | 缓存层、减少重分配 |
| **随机** | 随机选择 | 大规模集群 |

### 5. 什么是 GSLB（全局负载均衡）？

**答：** GSLB 基于 DNS 实现跨地域的流量调度。

**GSLB 策略：**
- **地理位置**：将用户路由到最近的数据中心
- **健康检查**：自动剔除不健康的站点
- **权重分配**：按比例分配流量到不同站点
- **延迟优化**：基于实时延迟选择最优站点
- **灾备切换**：主站点故障时自动切换到备站点
