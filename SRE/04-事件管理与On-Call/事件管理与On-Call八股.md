# 事件管理与 On-Call 八股文

---

## 一、事件响应

### 1. 什么是事件管理（Incident Management）生命周期？

**答：** 事件管理的完整生命周期：

```
检测 → 响应 → 缓解 → 恢复 → 复盘 → 改进
```

| 阶段 | 说明 | 关键动作 |
|------|------|----------|
| **检测** | 发现问题 | 监控告警、用户报告 |
| **响应** | 组织应急 | 确认事件、分配角色、拉起 War Room |
| **缓解** | 止血恢复 | 执行应急预案，降低影响范围 |
| **恢复** | 彻底修复 | 部署修复、验证恢复 |
| **复盘** | 总结教训 | 编写 Postmortem，分析根因 |
| **改进** | 预防再发 | 落实改进项（Action Items） |

### 2. 事件严重级别如何定义？

**答：**

| 级别 | 名称 | 影响范围 | 响应时间 | 示例 |
|------|------|----------|----------|------|
| **P0/SEV1** | 严重 | 核心服务完全不可用 | 5 分钟内响应 | 全站宕机、数据丢失 |
| **P1/SEV2** | 高 | 核心功能部分受损 | 15 分钟内响应 | 支付失败率 > 10% |
| **P2/SEV3** | 中 | 非核心功能受影响 | 1 小时内响应 | 报表延迟、非关键告警 |
| **P3/SEV4** | 低 | 边缘问题/美观问题 | 下个工作日 | UI 显示异常 |

### 3. 事件指挥官（Incident Commander）的职责是什么？

**答：** IC 是事件响应中的**总指挥**，负责协调整个事件处理流程：

1. **评估和声明事件严重程度**
2. **组建响应团队**：召集相关人员
3. **分配角色**：通信官、运维执行、技术负责人
4. **协调行动**：确保各方有序工作，避免混乱
5. **决策升级**：决定是否升级事件严重程度
6. **对外沟通**：更新状态页面、通知利益相关者
7. **宣布事件结束**：确认恢复后关闭事件

## 二、On-Call 实践

### 4. On-Call 轮值有哪些最佳实践？

**答：**

| 方面 | 最佳实践 |
|------|----------|
| **轮值周期** | 1-2 周一轮，避免过短（频繁交接）或过长（疲劳） |
| **人员配置** | 至少 8 人轮值队伍（保证足够的轮换间隔） |
| **交接** | 结构化交接文档，包含当前问题和注意事项 |
| **补偿** | On-Call 补贴、调休制度 |
| **支持** | 二线 Escalation 机制，不要让一个人孤军奋战 |
| **工具** | 自动化 On-Call 日历、告警路由、Runbook |
| **质量** | 定期审查 On-Call 负载，每次轮值不超过 2 个 Page |

### 5. 什么是 Postmortem（事后复盘）？如何写好一份 Postmortem？

**答：** Postmortem 是事件结束后的**结构化复盘文档**，目标是从故障中学习，而非追责。

**Postmortem 模板：**

```markdown
## 事件标题
**日期**：2024-XX-XX
**严重程度**：P1
**影响时长**：2 小时 15 分钟
**影响范围**：30% 的用户支付失败

## 摘要
一句话描述发生了什么。

## 影响
- 受影响用户数：50,000
- 收入损失估算：¥XX 万

## 时间线
- 14:00 告警触发
- 14:05 On-Call 响应
- 14:15 拉起 War Room
- 15:00 定位根因
- 16:15 服务恢复

## 根因分析
配置变更导致数据库连接池耗尽...

## 改进项（Action Items）
| 优先级 | 改进项 | 负责人 | 截止日期 |
|--------|--------|--------|----------|
| P0 | 添加连接池监控告警 | @张三 | 2024-XX-XX |
| P1 | 配置变更增加灰度发布 | @李四 | 2024-XX-XX |
```

### 6. 什么是无责文化（Blameless Culture）？

**答：** 无责文化是 SRE 事后复盘的核心原则：

- **不追究个人责任**：聚焦于系统和流程的改进
- **鼓励透明报告**：让人们敢于说出真实情况
- **关注根因而非替罪羊**：问"系统为什么允许这个错误发生"，而不是"谁犯了这个错误"
- **分享教训**：将 Postmortem 公开，让全组织学习
