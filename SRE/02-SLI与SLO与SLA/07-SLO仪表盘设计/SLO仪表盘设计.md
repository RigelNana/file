## 一、SLO 仪表盘核心组件

### 1. SLO 仪表盘应该包含哪些核心面板？

**答：**

| 面板 | 展示内容 | 目的 |
|------|----------|------|
| **SLO 状态概览** | 各服务 SLO 达标/违约 | 一目了然全局状态 |
| **错误预算剩余** | 预算消耗百分比和时间趋势 | 判断是否需要行动 |
| **SLI 时序图** | SLI 随时间变化的曲线 | 观察趋势和异常 |
| **燃烧率** | 当前各窗口的燃烧率 | 判断消耗速度 |
| **事件标注** | 发布和事故事件标记 | 关联变更与影响 |
| **预算预测** | 按当前消耗速率预测剩余天数 | 提前预警 |

### 2. 错误预算面板如何设计？

**答：**

### 3. Grafana 中如何实现 SLO 概览面板？

**答：**

```promql
# SLI 值（30天可用性）
sum(rate(http_requests_total{code!~"5.."}[30d]))
/
sum(rate(http_requests_total[30d]))
```

```promql
# 错误预算剩余百分比
(1 - (
  (1 - (
    sum(rate(http_requests_total{code!~"5.."}[30d]))
    / sum(rate(http_requests_total[30d]))
  ))
  / (1 - 0.999)
)) * 100
```

```promql
# 错误预算剩余分钟数
(1 - (
  (1 - (
    sum(rate(http_requests_total{code!~"5.."}[30d]))
    / sum(rate(http_requests_total[30d]))
  ))
  / (1 - 0.999)
)) * 43200
```

## 二、仪表盘分层设计

### 4. SLO 仪表盘的分层架构是什么？

**答：**

```
L0 - 全局概览（管理层/值班长）
  所有服务的 SLO 状态、红绿灯
  错误预算热力图
  关键事件数

L1 - 服务视图（SRE/开发负责人）
  单个服务的 SLI 趋势
  错误预算消耗详情
  燃烧率和告警状态

L2 - 排查视图（On-Call 工程师）
  按接口/端点的 SLI 拆分
  错误类型分布
  延迟直方图
  关联的基础设施指标
```

### 5. L0 全局概览面板如何设计？

**答：**

```
SLO 全局概览

┌────────┬──────────┬──────────┬──────────┐
│ 服务    │ SLO      │ 当前SLI  │ 预算剩余  │
├────────┼──────────┼──────────┼──────────┤
│ 🔴 支付 │ 99.99%  │ 99.97%  │ 12%      │
│ 🟡 订单 │ 99.95%  │ 99.93%  │ 45%      │
│ 🟢 搜索 │ 99.9%   │ 99.97%  │ 78%      │
│ 🟢 用户 │ 99.5%   │ 99.89%  │ 92%      │
│ 🟢 推荐 │ 99.9%   │ 99.95%  │ 85%      │
└────────┴──────────┴──────────┴──────────┘

颜色规则：
🟢 预算 > 50%
🟡 预算 25%-50%
🔴 预算 < 25%
```

### 6. 如何在仪表盘中标注事件和发布？

**答：**

## 三、仪表盘最佳实践

### 7. SLO 仪表盘设计有哪些最佳实践？

**答：**

| 实践 | 描述 |
|------|------|
| **颜色编码** | 红/黄/绿直观展示状态 |
| **阈值线** | 在图表上标注 SLO 目标线 |
| **时间对比** | 支持与上周/上月同期对比 |
| **下钻链接** | 从概览可以点击进入详情 |
| **自动刷新** | 5-30 秒自动更新 |
| **移动适配** | 支持手机查看关键面板 |
| **事件标注** | 标记发布、事故等事件 |

### 8. SLO 仪表盘中常见的错误？

**答：**

| 错误 | 问题 | 改进 |
|------|------|------|
| 信息过载 | 一个面板塞 50 个指标 | 分层设计，每层聚焦 |
| 缺少上下文 | 只有数字没有趋势 | 加入时序图和同期对比 |
| 没有行动指引 | 看到红灯不知道做什么 | 加链接跳转到 Runbook |
| 未标注事件 | 曲线跳变不知道原因 | 集成发布和事故标注 |
| 只看成功率 | 忽略延迟和吞吐量 | 多维 SLI 综合展示 |

### 10. 面试题：如何让非技术人员也能看懂 SLO 仪表盘？

**答：**

1. **用交通灯颜色**——🟢🟡🔴 不需要解释
2. **用百分比而非原始数字**——"剩余 67%"比"剩余 29.1 分钟"直观
3. **加入自然语言摘要**——"支付服务本月状态良好，预算充足"
4. **隐藏技术细节**——不展示 PromQL、燃烧率公式
5. **提供趋势而非瞬时值**——曲线比单个数字更有信息量

```
管理层视图示例：

╔═══════════════════════════════════╗
║  本月服务健康度                     ║
║                                   ║
║  🟢 支付服务    状态良好  预算充足   ║
║  🟡 订单服务    需要关注  预算偏紧   ║
║  🟢 用户服务    状态良好  预算充足   ║
║                                   ║
║  本月事件：P1×1  P2×3              ║
║  MTTR：平均 18 分钟（↓ 上月 25 分） ║
╚═══════════════════════════════════╝
```
