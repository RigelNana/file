# 模块详解

---

## 1. Ansible 模块是什么？分类？

**回答：**

模块 (Module) 是 Ansible 执行任务的最小单元，每个模块完成一个特定功能。

```
模块分类:
  包管理       → apt, yum, dnf, pip, npm, gem
  文件操作     → copy, template, file, lineinfile, blockinfile, synchronize
  服务管理     → service, systemd
  用户管理     → user, group, authorized_key
  命令执行     → command, shell, raw, script, expect
  系统管理     → sysctl, cron, hostname, timezone, mount
  网络         → uri, get_url, wait_for, firewalld, iptables
  容器         → docker_container, docker_image, docker_network, docker_compose
  云平台       → aws_*, azure_*, gcp_*
  数据库       → mysql_db, mysql_user, postgresql_db, postgresql_user
  信息收集     → setup, debug, assert, fail, meta

模块调用方式:
  FQCN (推荐): ansible.builtin.copy
  短名称:      copy
```

## 2. 包管理模块 (apt / yum / pip)？

**回答：**

```yaml
# apt — Debian/Ubuntu
- name: Update cache and install
  ansible.builtin.apt:
    update_cache: yes
    cache_valid_time: 3600        # 缓存有效期 (秒)
    name:
      - nginx
      - python3
      - git
    state: present                # present/absent/latest

- name: Install specific version
  apt:
    name: nginx=1.24.0-1~jammy
    state: present

- name: Remove package
  apt:
    name: apache2
    state: absent
    purge: yes                    # 同时删除配置文件
    autoremove: yes

- name: Upgrade all packages
  apt:
    upgrade: dist                 # yes/safe/full/dist

# yum/dnf — RHEL/CentOS
- name: Install package
  ansible.builtin.yum:            # CentOS 7
    name: nginx
    state: present
    enablerepo: epel              # 启用特定仓库

- name: Install via dnf
  ansible.builtin.dnf:            # CentOS 8+ / Fedora
    name:
      - nginx
      - python3
    state: latest

# pip — Python
- name: Install Python packages
  ansible.builtin.pip:
    name:
      - flask==2.3.0
      - gunicorn
    virtualenv: /opt/app/venv
    virtualenv_python: python3

# npm — Node.js
- name: Install Node packages
  community.general.npm:
    name: pm2
    global: yes
    state: present
```

## 3. 文件操作模块 (copy / template / file)？

**回答：**

```yaml
# copy — 复制文件到远程
- name: Copy file
  ansible.builtin.copy:
    src: files/app.conf              # 本地路径 (相对 role/files/)
    dest: /etc/app/app.conf          # 远程路径
    owner: root
    group: root
    mode: "0644"
    backup: yes                      # 修改前备份

- name: Copy content directly
  copy:
    content: |
      server_name={{ server_name }}
      port={{ app_port }}
    dest: /etc/app/env.conf

# template — Jinja2 模板渲染
- name: Deploy template
  ansible.builtin.template:
    src: templates/nginx.conf.j2     # Jinja2 模板
    dest: /etc/nginx/nginx.conf
    owner: root
    mode: "0644"
    validate: "nginx -t -c %s"      # 部署前验证

# file — 文件/目录属性管理
- name: Create directory
  ansible.builtin.file:
    path: /opt/app/data
    state: directory                 # directory/file/link/absent/touch
    owner: app
    group: app
    mode: "0755"
    recurse: yes                     # 递归设置权限

- name: Create symlink
  file:
    src: /opt/app/releases/v1.2.3
    dest: /opt/app/current
    state: link

- name: Delete file
  file:
    path: /tmp/old_file
    state: absent

# lineinfile — 修改文件单行
- name: Ensure line in file
  ansible.builtin.lineinfile:
    path: /etc/ssh/sshd_config
    regexp: "^PermitRootLogin"       # 匹配行 (正则)
    line: "PermitRootLogin no"       # 替换为
    backup: yes

- name: Add line if not exists
  lineinfile:
    path: /etc/hosts
    line: "192.168.1.100 app.local"
    state: present

# blockinfile — 管理文件块
- name: Add config block
  ansible.builtin.blockinfile:
    path: /etc/nginx/conf.d/app.conf
    marker: "# {mark} ANSIBLE MANAGED - APP CONFIG"
    block: |
      location /api {
          proxy_pass http://127.0.0.1:8080;
          proxy_set_header Host $host;
      }

# synchronize — rsync 同步
- name: Sync directory
  ansible.posix.synchronize:
    src: /opt/app/static/
    dest: /var/www/static/
    delete: yes                      # 删除目标多余文件
    rsync_opts:
      - "--exclude=*.tmp"
```

## 4. 服务管理模块 (service / systemd)？

**回答：**

```yaml
# service — 通用服务管理
- name: Start and enable service
  ansible.builtin.service:
    name: nginx
    state: started                   # started/stopped/restarted/reloaded
    enabled: yes                     # 开机自启

# systemd — systemd 专用 (更多功能)
- name: Start service
  ansible.builtin.systemd:
    name: myapp
    state: started
    enabled: yes
    daemon_reload: yes               # 重新加载 unit 文件

- name: Reload systemd daemon
  systemd:
    daemon_reload: yes               # systemctl daemon-reload

# 部署自定义 systemd unit
- name: Deploy service unit
  copy:
    content: |
      [Unit]
      Description=My Application
      After=network.target

      [Service]
      Type=simple
      User=app
      WorkingDirectory=/opt/app
      ExecStart=/opt/app/venv/bin/gunicorn -w 4 -b 0.0.0.0:8080 app:app
      Restart=always
      RestartSec=5
      Environment=APP_ENV=production

      [Install]
      WantedBy=multi-user.target
    dest: /etc/systemd/system/myapp.service
  notify:
    - reload systemd
    - restart myapp
```

## 5. 命令执行模块 (command / shell / raw / script)？

**回答：**

```yaml
# command — 执行命令 (不经过 Shell, 更安全)
- name: Run command
  ansible.builtin.command:
    cmd: /opt/app/bin/migrate
    chdir: /opt/app                  # 工作目录
    creates: /opt/app/.migrated      # 文件存在则跳过
    removes: /opt/app/pending.lock   # 文件不存在则跳过
  register: migrate_result

# shell — 通过 Shell 执行 (支持管道、重定向)
- name: Run shell command
  ansible.builtin.shell: |
    cd /opt/app && \
    source venv/bin/activate && \
    python manage.py collectstatic --noinput 2>&1 | tee /var/log/collect.log
  args:
    executable: /bin/bash

# raw — 原始命令 (不依赖 Python, 用于初始化)
- name: Install Python on minimal host
  ansible.builtin.raw: apt-get install -y python3
  become: true
  # 用于目标主机没有 Python 的场景

# script — 传输并执行本地脚本
- name: Run local script on remote
  ansible.builtin.script:
    cmd: scripts/setup.sh --env production
    executable: /bin/bash
    creates: /opt/app/.setup_done
```

```
选择指南:
  command   → 默认首选, 简单命令, 更安全 (无Shell注入)
  shell     → 需要管道 (|) / 重定向 (>) / 环境变量 / 通配符
  raw       → 目标无 Python / 网络设备 / 初始引导
  script    → 传输并运行本地脚本

安全注意:
  ✗ 避免: shell: "rm -rf {{ user_input }}"   # 注入风险
  ✓ 推荐: 尽量用专用模块代替 command/shell
  ✓ 优先: apt 模块 > command: apt install
```

## 6. 用户与权限模块？

**回答：**

```yaml
# user — 用户管理
- name: Create user
  ansible.builtin.user:
    name: deploy
    uid: 1050
    group: deploy
    groups: sudo,docker              # 附加组
    shell: /bin/bash
    home: /home/deploy
    create_home: yes
    comment: "Deployment User"
    password: "{{ 'password123' | password_hash('sha512') }}"
    state: present

- name: Remove user
  user:
    name: olduser
    state: absent
    remove: yes                      # 同时删除 home 目录

# group — 组管理
- name: Create group
  ansible.builtin.group:
    name: deploy
    gid: 1050
    state: present

# authorized_key — SSH 公钥管理
- name: Add SSH key
  ansible.posix.authorized_key:
    user: deploy
    key: "{{ lookup('file', 'files/deploy_key.pub') }}"
    state: present
    exclusive: no                    # 不删除其他 key

- name: Add multiple keys
  authorized_key:
    user: deploy
    key: "{{ item }}"
  loop:
    - "ssh-ed25519 AAAA... user1@laptop"
    - "ssh-ed25519 AAAA... user2@workstation"

# cron — 定时任务
- name: Add cron job
  ansible.builtin.cron:
    name: "backup database"          # 任务标识
    minute: "0"
    hour: "2"
    job: "/opt/scripts/backup.sh >> /var/log/backup.log 2>&1"
    user: root
```

## 7. 网络相关模块 (uri / get_url / wait_for)？

**回答：**

```yaml
# uri — HTTP 请求
- name: Health check
  ansible.builtin.uri:
    url: "http://localhost:8080/health"
    method: GET
    return_content: yes
    status_code: 200
    timeout: 10
  register: health_result

- name: Call API
  uri:
    url: "https://api.example.com/deploy"
    method: POST
    body_format: json
    body:
      version: "{{ app_version }}"
      environment: "{{ env }}"
    headers:
      Authorization: "Bearer {{ api_token }}"
    status_code: [200, 201]

# get_url — 下载文件
- name: Download binary
  ansible.builtin.get_url:
    url: "https://releases.example.com/app-{{ version }}.tar.gz"
    dest: /tmp/app.tar.gz
    checksum: "sha256:{{ expected_checksum }}"
    mode: "0644"
    timeout: 30

# wait_for — 等待条件满足
- name: Wait for port to be available
  ansible.builtin.wait_for:
    host: "{{ inventory_hostname }}"
    port: 8080
    delay: 5                         # 开始等待前延迟
    timeout: 60                      # 最长等待

- name: Wait for file to appear
  wait_for:
    path: /opt/app/ready.flag
    state: present
    timeout: 120

- name: Wait for log line
  wait_for:
    path: /var/log/app.log
    search_regex: "Application started"
    timeout: 60

# firewalld — 防火墙
- name: Open port
  ansible.posix.firewalld:
    port: 8080/tcp
    permanent: yes
    state: enabled
    immediate: yes
```

## 8. 容器模块 (docker_container / docker_image)？

**回答：**

```yaml
# 需要安装: ansible-galaxy collection install community.docker

# docker_image — 镜像管理
- name: Pull image
  community.docker.docker_image:
    name: myapp
    tag: "{{ version }}"
    source: pull

- name: Build image
  community.docker.docker_image:
    name: myapp
    tag: latest
    build:
      path: /opt/app
      dockerfile: Dockerfile
    source: build

# docker_container — 容器管理
- name: Run container
  community.docker.docker_container:
    name: myapp
    image: "myapp:{{ version }}"
    state: started                   # started/stopped/absent/present
    restart_policy: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - "/opt/app/data:/app/data:rw"
      - "/opt/app/config:/app/config:ro"
    env:
      APP_ENV: production
      DB_HOST: "{{ db_host }}"
    networks:
      - name: app_network
    memory: "512m"
    cpus: 1.0
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 10s
      retries: 3

# docker_network — 网络管理
- name: Create network
  community.docker.docker_network:
    name: app_network
    driver: bridge
    ipam_config:
      - subnet: 172.20.0.0/16

# docker_compose — Docker Compose
- name: Deploy with compose
  community.docker.docker_compose_v2:
    project_src: /opt/app
    state: present                   # present/absent
    pull: always
```

## 9. 调试与信息模块 (debug / assert / setup)？

**回答：**

```yaml
# debug — 打印调试信息
- name: Print variable
  ansible.builtin.debug:
    msg: "Server {{ inventory_hostname }} has IP {{ ansible_default_ipv4.address }}"

- name: Print variable value
  debug:
    var: my_complex_variable         # 直接输出变量结构

- name: Verbose only debug
  debug:
    msg: "Detailed info: {{ result }}"
    verbosity: 2                     # 只在 -vv 或更高时显示

# assert — 断言检查
- name: Verify prerequisites
  ansible.builtin.assert:
    that:
      - ansible_memtotal_mb >= 2048
      - ansible_distribution == "Ubuntu"
      - app_version is defined
    fail_msg: "Server does not meet requirements"
    success_msg: "All checks passed"

# setup — 收集系统信息 (Facts)
- name: Gather facts
  ansible.builtin.setup:
    filter: "ansible_*"              # 过滤

- name: Display OS info
  debug:
    msg: |
      OS: {{ ansible_distribution }} {{ ansible_distribution_version }}
      Kernel: {{ ansible_kernel }}
      CPU: {{ ansible_processor_cores }} cores
      RAM: {{ ansible_memtotal_mb }} MB
      IP: {{ ansible_default_ipv4.address }}

# fail — 主动失败
- name: Fail if wrong environment
  ansible.builtin.fail:
    msg: "Cannot run on production without approval!"
  when:
    - env == "production"
    - approved is not defined

# pause — 暂停等待
- name: Wait for confirmation
  ansible.builtin.pause:
    prompt: "Press Enter to continue deployment"

- name: Wait 30 seconds
  pause:
    seconds: 30
```

## 10. 自定义模块怎么写？

**回答：**

```python
#!/usr/bin/python3
# library/my_app_config.py — 自定义模块

from ansible.module_utils.basic import AnsibleModule
import json
import os

DOCUMENTATION = r'''
---
module: my_app_config
short_description: Manage application configuration
description:
    - Create or update application configuration files
options:
    path:
        description: Path to config file
        required: true
        type: str
    settings:
        description: Dict of settings to apply
        required: true
        type: dict
    backup:
        description: Create backup before changes
        required: false
        type: bool
        default: true
'''

def main():
    module = AnsibleModule(
        argument_spec=dict(
            path=dict(type='str', required=True),
            settings=dict(type='dict', required=True),
            backup=dict(type='bool', default=True)
        ),
        supports_check_mode=True
    )

    path = module.params['path']
    settings = module.params['settings']
    backup = module.params['backup']
    changed = False

    # 读取现有配置
    current = {}
    if os.path.exists(path):
        with open(path, 'r') as f:
            current = json.load(f)

    # 检查是否需要变更
    if current != settings:
        changed = True

        if not module.check_mode:
            if backup and os.path.exists(path):
                module.backup_local(path)

            with open(path, 'w') as f:
                json.dump(settings, f, indent=2)

    module.exit_json(
        changed=changed,
        path=path,
        settings=settings
    )

if __name__ == '__main__':
    main()
```

```yaml
# 使用自定义模块
- name: Apply app config
  my_app_config:
    path: /etc/myapp/config.json
    settings:
      database_url: "{{ db_url }}"
      log_level: info
      workers: 4
    backup: true
```

```
自定义模块位置:
  ./library/                       # Playbook 同级目录
  roles/myrole/library/            # Role 内
  ~/.ansible/plugins/modules/      # 用户级
  /usr/share/ansible/plugins/modules/  # 全局

关键点:
  ✓ 使用 AnsibleModule 处理参数
  ✓ 支持 check_mode (幂等)
  ✓ 返回 changed 状态
  ✓ 用 module.fail_json() 报错
  ✓ 用 module.exit_json() 返回成功
```
