## 1. Ansible 变量的 22 级优先级？

**回答：**

```
优先级从低到高:
  1.  role defaults (roles/x/defaults/main.yml)
  2.  inventory file group_vars
  3.  inventory group_vars/all
  4.  playbook group_vars/all
  5.  inventory group_vars/*
  6.  playbook group_vars/*
  7.  inventory file host_vars
  8.  inventory host_vars/*
  9.  playbook host_vars/*
  10. host facts / cached set_facts
  11. play vars
  12. play vars_prompt
  13. play vars_files
  14. role vars (roles/x/vars/main.yml)
  15. block vars
  16. task vars
  17. include_vars
  18. set_facts / registered vars
  19. role params
  20. include params
  21. extra vars (-e) ← 最高优先级

记忆口诀:
  role defaults → inventory → playbook → play → role vars → task → set_fact → extra vars
  越具体 → 优先级越高
  -e 命令行最高, role defaults 最低
```

## 2. 变量定义的各种方式？

**回答：**

```yaml
# 1. Play 变量
- hosts: webservers
  vars:
    http_port: 80
    app_name: myapp

# 2. 变量文件
- hosts: webservers
  vars_files:
    - vars/common.yml
    - "vars/{{ env }}.yml"

# 3. 交互输入
- hosts: webservers
  vars_prompt:
    - name: deploy_version
      prompt: "Version to deploy?"
      private: false
      default: "latest"
    - name: db_password
      prompt: "Database password?"
      private: true    # 不回显
      encrypt: sha512_crypt
      confirm: true

# 4. Inventory 变量
# group_vars/webservers.yml
http_port: 80

# host_vars/web1.yml
http_port: 8080

# 5. 注册变量
- name: Get disk usage
  command: df -h /
  register: disk_result

- debug:
    msg: "Disk: {{ disk_result.stdout_lines[1] }}"

# 6. set_fact (运行时设置)
- set_fact:
    deploy_timestamp: "{{ ansible_date_time.iso8601 }}"
    app_url: "https://{{ domain }}:{{ https_port }}"

# 7. include_vars (动态加载)
- include_vars:
    file: "{{ ansible_os_family }}.yml"

- include_vars:
    dir: vars/
    extensions: [yml, yaml]
    ignore_unknown_extensions: true

# 8. 命令行 extra vars (最高优先级)
# ansible-playbook site.yml -e "env=prod version=1.2.3"
# ansible-playbook site.yml -e "@vars/override.yml"
```

## 3. Facts (系统信息) 详解？

**回答：**

```yaml
# Facts = Ansible 自动收集的系统信息
# 执行时通过 setup 模块收集

# 常用 Facts
ansible_hostname              # 主机名
ansible_fqdn                  # 完整域名
ansible_distribution          # 发行版 (Ubuntu/CentOS)
ansible_distribution_version  # 版本号 (22.04)
ansible_os_family             # 系统族 (Debian/RedHat)
ansible_kernel                # 内核版本
ansible_architecture          # 架构 (x86_64)
ansible_processor_cores       # CPU 核心数
ansible_processor_count       # CPU 数量
ansible_memtotal_mb           # 总内存 (MB)
ansible_default_ipv4.address  # 默认 IPv4 地址
ansible_default_ipv4.gateway  # 默认网关
ansible_all_ipv4_addresses    # 所有 IPv4
ansible_interfaces            # 网络接口列表
ansible_mounts                # 挂载点信息
ansible_date_time             # 日期时间
ansible_env                   # 环境变量
ansible_pkg_mgr               # 包管理器 (apt/yum)
ansible_service_mgr           # 服务管理器 (systemd)
ansible_python_version        # Python 版本
```

```yaml
# 控制 Facts 收集
- hosts: webservers
  gather_facts: false          # 禁用 (加速执行)

  tasks:
    - name: Manual gather
      setup:
        filter: "ansible_distribution*"  # 只收集部分

# 自定义 Facts
# 在目标主机创建: /etc/ansible/facts.d/app.fact
# INI 格式:
# [app]
# version=1.2.3
# env=production

# JSON 格式:
# {"app": {"version": "1.2.3", "env": "production"}}

# 使用自定义 Facts:
- debug:
    msg: "App version: {{ ansible_local.app.app.version }}"

# Facts 缓存
# ansible.cfg
[defaults]
gathering = smart                # smart/implicit/explicit
fact_caching = jsonfile
fact_caching_connection = /tmp/ansible_facts
fact_caching_timeout = 86400     # 秒
```

## 4. Jinja2 模板基础语法？

**回答：**

```jinja2
{# 注释 #}
{# 这是 Jinja2 注释, 不会出现在渲染结果中 #}

{# 变量输出 #}
{{ variable }}
{{ dict.key }}
{{ list[0] }}
{{ ansible_default_ipv4.address }}

{# 条件判断 #}
{% if env == "production" %}
log_level = warn
{% elif env == "staging" %}
log_level = info
{% else %}
log_level = debug
{% endif %}

{# for 循环 #}
{% for server in backend_servers %}
server {{ server.host }}:{{ server.port }};
{% endfor %}

{# 循环 + 条件 #}
{% for user in users if user.active %}
{{ user.name }}
{% endfor %}

{# 循环变量 #}
{% for item in items %}
  {{ loop.index }}      {# 1-based 索引 #}
  {{ loop.index0 }}     {# 0-based 索引 #}
  {{ loop.first }}      {# 是否第一个 #}
  {{ loop.last }}       {# 是否最后一个 #}
  {{ loop.length }}     {# 总长度 #}
{% endfor %}

{# 空白控制 #}
{% for item in list -%}       {# 去除右侧空白 #}
{%- if item.active -%}        {# 去除两侧空白 #}
{{ item.name }}
{%- endif %}
{%- endfor %}

{# 设置局部变量 #}
{% set my_var = "hello" %}
{% set my_list = [1, 2, 3] %}
```

## 5. Jinja2 过滤器 (Filters)？

**回答：**

```yaml
# 字符串过滤器
"{{ name | upper }}"                    # 大写 → HELLO
"{{ name | lower }}"                    # 小写 → hello
"{{ name | capitalize }}"              # 首字母大写
"{{ name | title }}"                   # 每词首字母大写
"{{ name | replace('old', 'new') }}"   # 替换
"{{ name | trim }}"                    # 去除首尾空白
"{{ name | regex_replace('\\d+', '') }}" # 正则替换
"{{ path | basename }}"                # 文件名
"{{ path | dirname }}"                 # 目录名
"{{ text | b64encode }}"              # Base64 编码
"{{ encoded | b64decode }}"           # Base64 解码
"{{ password | password_hash('sha512') }}"  # 密码哈希

# 数字过滤器
"{{ value | int }}"                    # 转整数
"{{ value | float }}"                  # 转浮点
"{{ value | abs }}"                    # 绝对值
"{{ values | max }}"                   # 最大值
"{{ values | min }}"                   # 最小值
"{{ values | sum }}"                   # 求和

# 列表过滤器
"{{ list | length }}"                  # 长度
"{{ list | first }}"                   # 第一个元素
"{{ list | last }}"                    # 最后一个
"{{ list | sort }}"                    # 排序
"{{ list | unique }}"                  # 去重
"{{ list | flatten }}"                # 展平
"{{ list | join(', ') }}"             # 连接为字符串
"{{ list | select('match', '^web') | list }}"  # 筛选
"{{ list | map('upper') | list }}"    # 映射

# 字典过滤器
"{{ dict | combine(other_dict) }}"     # 合并字典
"{{ dict | dict2items }}"            # 转为列表 [{key:k, value:v}]
"{{ items | items2dict }}"           # 列表转字典

# 默认值
"{{ undefined_var | default('fallback') }}"      # 未定义时的默认值
"{{ empty_var | default('fallback', true) }}"    # 空值也用默认

# 类型测试
"{{ var is defined }}"
"{{ var is undefined }}"
"{{ var is string }}"
"{{ var is number }}"
"{{ var is mapping }}"                # 是否字典
"{{ var is iterable }}"               # 是否可迭代
"{{ path is file }}"                  # 路径是否是文件 (远程)
"{{ path is directory }}"             # 路径是否是目录

# JSON/YAML
"{{ data | to_json }}"               # 转 JSON
"{{ data | to_nice_json(indent=2) }}" # 格式化 JSON
"{{ data | to_yaml }}"               # 转 YAML
"{{ json_str | from_json }}"         # 解析 JSON
"{{ yaml_str | from_yaml }}"         # 解析 YAML
```

## 6. Jinja2 模板实战 — Nginx 配置？

**回答：**

```jinja2
{# templates/nginx.conf.j2 #}
# Managed by Ansible - DO NOT EDIT MANUALLY
user {{ nginx_user | default('www-data') }};
worker_processes {{ ansible_processor_cores }};
pid /run/nginx.pid;

events {
    worker_connections {{ nginx_worker_connections | default(1024) }};
}

http {
    sendfile on;
    tcp_nopush on;
    keepalive_timeout 65;

    # Logging
    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log {{ 'warn' if env == 'production' else 'info' }};

    # Upstream backends
{% if backend_servers is defined and backend_servers | length > 0 %}
    upstream app_backend {
{% for server in backend_servers %}
        server {{ server.host }}:{{ server.port }}{{ ' weight=' ~ server.weight if server.weight is defined else '' }};
{% endfor %}
    }
{% endif %}

    # Virtual Hosts
{% for vhost in virtual_hosts | default([]) %}
    server {
        listen {{ vhost.port | default(80) }};
        server_name {{ vhost.server_name }};

{% if vhost.ssl | default(false) %}
        listen 443 ssl;
        ssl_certificate {{ vhost.ssl_cert }};
        ssl_certificate_key {{ vhost.ssl_key }};
{% endif %}

        root {{ vhost.root | default('/var/www/html') }};

{% for location in vhost.locations | default([]) %}
        location {{ location.path }} {
{% if location.proxy_pass is defined %}
            proxy_pass {{ location.proxy_pass }};
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
{% endif %}
{% if location.static is defined and location.static %}
            expires 30d;
            add_header Cache-Control "public, immutable";
{% endif %}
        }
{% endfor %}
    }
{% endfor %}
}
```

```yaml
# group_vars/webservers.yml
nginx_user: www-data
nginx_worker_connections: 2048

backend_servers:
  - host: 127.0.0.1
    port: 8080
    weight: 3
  - host: 127.0.0.1
    port: 8081
    weight: 1

virtual_hosts:
  - server_name: app.example.com
    port: 80
    ssl: true
    ssl_cert: /etc/ssl/certs/app.crt
    ssl_key: /etc/ssl/private/app.key
    root: /var/www/app
    locations:
      - path: /
        proxy_pass: http://app_backend
      - path: /static
        static: true
```

## 7. lookup 插件？

**回答：**

```yaml
# lookup — 从外部数据源获取数据 (在控制节点执行)

# 文件内容
"{{ lookup('file', '/path/to/file') }}"

# 环境变量
"{{ lookup('env', 'HOME') }}"
"{{ lookup('env', 'AWS_ACCESS_KEY_ID') }}"

# 模板渲染
"{{ lookup('template', 'templates/snippet.j2') }}"

# 密码生成
"{{ lookup('password', '/tmp/pw length=20 chars=ascii_letters,digits') }}"

# pipe — 执行命令
"{{ lookup('pipe', 'date +%Y%m%d') }}"
"{{ lookup('pipe', 'git rev-parse HEAD') }}"

# URL
"{{ lookup('url', 'https://api.example.com/config') }}"

# 列表
"{{ lookup('sequence', 'start=1 end=10') }}"
"{{ lookup('random_choice', 'a', 'b', 'c') }}"

# fileglob — 文件通配
"{{ lookup('fileglob', 'files/*.conf') }}"

# csvfile
"{{ lookup('csvfile', 'user1 file=users.csv delimiter=, col=2') }}"

# dict — 遍历字典
"{{ lookup('dict', my_dict) }}"

# first_found — 第一个存在的文件
"{{ lookup('first_found', params) }}"
vars:
  params:
    files:
      - "{{ ansible_distribution }}.yml"
      - "{{ ansible_os_family }}.yml"
      - default.yml
    paths:
      - vars/

# 组合使用
- name: Read all config files
  template:
    src: "{{ item }}"
    dest: "/etc/app/{{ item | basename | regex_replace('\\.j2$', '') }}"
  loop: "{{ lookup('fileglob', 'templates/conf.d/*.j2', wantlist=True) }}"
```

## 8. 特殊变量（Magic Variables）？

**回答：**

```yaml
# 主机相关
inventory_hostname        # Inventory 中定义的主机名
inventory_hostname_short  # 短主机名
ansible_host              # 实际连接地址
ansible_play_hosts        # 当前 Play 的所有主机列表
ansible_play_hosts_all    # 当前 Play 的所有主机 (含失败)
ansible_play_batch        # 当前批次的主机列表 (serial)
play_hosts                # 同 ansible_play_hosts

# 组相关
groups                    # 所有组及其主机
groups['webservers']      # webservers 组的主机列表
group_names               # 当前主机所属的组列表

# 变量访问
hostvars                  # 所有主机的变量
hostvars['web1']          # 获取 web1 的变量
hostvars[hostname].ansible_default_ipv4.address  # 跨主机获取 IP

# 角色/任务相关
role_name                 # 当前 Role 名称
role_path                 # 当前 Role 路径
ansible_play_name         # 当前 Play 名称
ansible_check_mode        # 是否 check 模式

# 路径相关
playbook_dir              # Playbook 文件所在目录
inventory_dir             # Inventory 文件所在目录
```

```yaml
# 实用示例: 跨主机引用变量
- name: Configure app with DB host
  template:
    src: app.conf.j2
    dest: /etc/app/app.conf
  vars:
    db_ip: "{{ hostvars[groups['dbservers'][0]].ansible_default_ipv4.address }}"

# 动态生成配置
- name: Generate hosts file
  lineinfile:
    path: /etc/hosts
    line: "{{ hostvars[item].ansible_default_ipv4.address }} {{ item }}"
  loop: "{{ groups['all'] }}"
```

## 9. 变量加密与安全？

**回答：**

```yaml
# 敏感变量加密存储 (Vault)
# group_vars/production/vault.yml (已加密)
vault_db_password: "s3cretP@ss"
vault_api_key: "abc123def456"

# group_vars/production/vars.yml (明文引用)
db_password: "{{ vault_db_password }}"
api_key: "{{ vault_api_key }}"

# 内联加密变量
ansible-vault encrypt_string 's3cretP@ss' --name 'db_password'
# 输出:
# db_password: !vault |
#   $ANSIBLE_VAULT;1.1;AES256
#   61626364...

# no_log — 隐藏任务输出
- name: Set database password
  mysql_user:
    name: app
    password: "{{ db_password }}"
  no_log: true                    # 不在日志中显示参数和结果

# 避免在调试中输出敏感信息
- name: Check result
  debug:
    msg: "Connection status: {{ result.rc }}"   # 只输出需要的字段
  # 而不是: debug: var: result  (可能泄露密码)
```

## 10. 变量与模板最佳实践？

**回答：**

```
变量命名:
  ✓ 使用小写 + 下划线: app_port, db_host
  ✗ 避免: appPort, APP-PORT
  ✓ Role 变量加前缀: nginx_port, postgres_max_connections
  ✓ 敏感变量加 vault_ 前缀: vault_db_password

变量组织:
  ✓ group_vars/all.yml        → 全局默认值
  ✓ group_vars/<group>.yml    → 组级覆盖
  ✓ host_vars/<host>.yml      → 主机级覆盖
  ✓ roles/x/defaults/main.yml → Role 默认值 (可覆盖)
  ✓ roles/x/vars/main.yml     → Role 内部值 (不应覆盖)

模板:
  ✓ 模板文件加 .j2 后缀
  ✓ 添加 "Managed by Ansible" 头注释
  ✓ 使用 validate 参数验证生成结果
  ✓ 使用 default() 过滤器避免未定义错误
  ✓ 复杂逻辑放到变量中, 模板保持简洁

安全:
  ✓ 敏感变量用 Vault 加密
  ✓ 引用密文时用间接引用 (vault_ 前缀)
  ✓ 任务中用 no_log: true
  ✓ 不把密码硬编码在 Playbook 中

调试:
  ansible -m debug -a "var=hostvars[inventory_hostname]" web1
  ansible-playbook site.yml -e "debug=true" --check --diff
```
