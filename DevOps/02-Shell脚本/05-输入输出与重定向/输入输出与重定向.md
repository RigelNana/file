## 1. 文件描述符（File Descriptor）是什么？标准的三个 fd 分别是什么？

**回答：**

文件描述符是内核为每个进程维护的一个**非负整数索引**，指向内核中打开文件表的条目。

### 标准文件描述符

| fd | 名称 | 缩写 | 默认指向 | 用途 |
|----|------|------|----------|------|
| 0 | 标准输入 | stdin | 键盘/终端 | 程序读取输入 |
| 1 | 标准输出 | stdout | 终端 | 正常输出 |
| 2 | 标准错误 | stderr | 终端 | 错误信息 |

```bash
# 查看进程的文件描述符
ls -la /proc/$$/fd
# lrwx------ 0 -> /dev/pts/0    (stdin)
# lrwx------ 1 -> /dev/pts/0    (stdout)
# lrwx------ 2 -> /dev/pts/0    (stderr)
# lr-x------ 255 -> /dev/pts/0  (Bash 保留)

# 查看当前 Shell 打开的 fd 数量
ls /proc/$$/fd | wc -l
```

### fd 的本质

```
进程                    内核文件表              文件/设备
┌─────────┐          ┌──────────┐          ┌──────────┐
│ fd 0 ────┼────────→│ 打开文件1 │────────→│ /dev/pts/0│
│ fd 1 ────┼────────→│ 打开文件2 │────────→│ /dev/pts/0│
│ fd 2 ────┼────────→│ 打开文件3 │────────→│ /dev/pts/0│
│ fd 3 ────┼────────→│ 打开文件4 │────────→│ output.log│
└─────────┘          └──────────┘          └──────────┘
```

### 自定义文件描述符

```bash
# 打开 fd 3 用于写入
exec 3> /tmp/output.log

# 打开 fd 4 用于读取
exec 4< /tmp/input.txt

# 读写模式
exec 5<> /tmp/data.txt

# 使用自定义 fd
echo "写入 fd3" >&3
read -r line <&4

# 关闭 fd
exec 3>&-   # 关闭输出 fd
exec 4<&-   # 关闭输入 fd
```

## 2. 重定向操作符有哪些？分别是什么含义？

**回答：**

### 输出重定向

```bash
cmd > file          # stdout 重定向到文件（覆盖）
cmd >> file         # stdout 追加到文件
cmd 2> file         # stderr 重定向到文件
cmd 2>> file        # stderr 追加到文件
cmd &> file         # stdout 和 stderr 都重定向（Bash 扩展，覆盖）
cmd &>> file        # stdout 和 stderr 都追加（Bash 扩展）
cmd > file 2>&1     # stdout 到文件，stderr 合并到 stdout（POSIX 标准）
cmd >> file 2>&1    # 追加版

# 注意顺序很重要！
cmd 2>&1 > file     # 错误！stderr 指向原 stdout（终端），只有 stdout 到文件
cmd > file 2>&1     # 正确！先 stdout 到文件，再 stderr 到同一个文件
```

### 输入重定向

```bash
cmd < file          # 从文件读取 stdin
cmd << EOF          # Here Document
line1
line2
EOF
cmd <<< "string"    # Here String（Bash 扩展）
cmd < <(other_cmd)  # 进程替换作为输入
```

### fd 操作

```bash
cmd n> file         # fd n 重定向到文件
cmd n>> file        # fd n 追加到文件
cmd n>&m            # fd n 复制到 fd m（输出）
cmd n<&m            # fd n 复制到 fd m（输入）
cmd n>&-            # 关闭输出 fd n
cmd n<&-            # 关闭输入 fd n
cmd n<> file        # fd n 以读写模式打开文件
```

### 特殊重定向

```bash
# 丢弃输出
cmd > /dev/null 2>&1    # 丢弃所有输出
cmd &> /dev/null        # 同上（Bash 简写）

# 丢弃 stdin
cmd < /dev/null         # 关闭标准输入

# 从 /dev/zero 读取（二进制零填充）
dd if=/dev/zero of=empty.img bs=1M count=100

# 组合重定向
{
    echo "stdout 信息"
    echo "stderr 信息" >&2
} > stdout.log 2> stderr.log
```

## 3. exec 在重定向中有什么特殊用途？

**回答：**

`exec` 用于**永久性**重定向当前 Shell 的文件描述符（持续到关闭或脚本结束）。

### 永久重定向

```bash
# 将所有后续 stdout 写入文件
exec > /var/log/script.log
echo "这行会写入文件"        # 后续所有 echo 都到文件
echo "这也是" 

# 将 stdout 和 stderr 都重定向
exec > /var/log/script.log 2>&1

# 恢复 stdout 到终端
exec 3>&1              # 先用 fd3 保存原 stdout
exec > /tmp/output.log # 重定向 stdout
echo "写入文件"
exec 1>&3              # 恢复 stdout
exec 3>&-              # 关闭 fd3
echo "回到终端"
```

### 日志脚本常用模式

```bash
#!/bin/bash
# 所有输出同时写终端和日志文件
LOG_FILE="/var/log/deploy.log"

# 方式1：使用 exec + tee
exec > >(tee -a "$LOG_FILE") 2>&1

echo "这行同时出现在终端和日志文件"
echo "错误信息也是" >&2

# 方式2：分离 stdout 和 stderr 日志
exec 3>&1 4>&2                          # 保存原始 fd
exec 1> >(tee -a "$LOG_FILE") 2> >(tee -a "$LOG_FILE" >&2)
```

### 打开和管理文件描述符

```bash
# 打开文件用于读取
exec 3< /etc/hosts
while IFS= read -r line <&3; do
    echo "$line"
done
exec 3<&-   # 关闭

# 打开文件用于写入
exec 4> /tmp/output.txt
echo "行1" >&4
echo "行2" >&4
exec 4>&-   # 关闭

# 读写模式
exec 5<> /tmp/data.txt
echo "写入" >&5
read -r line <&5
exec 5>&-
```

### exec 替换当前进程

```bash
# exec 还可以不接重定向，用于替换当前 Shell 进程
exec /usr/bin/python3 script.py  # 替换当前 Shell，不会返回
echo "这行永远不会执行"

# 常见用途：wrapper 脚本
#!/bin/bash
export JAVA_HOME="/usr/lib/jvm/java-17"
export APP_OPTS="-Xmx2g"
exec java $APP_OPTS -jar /opt/app/app.jar "$@"
```

## 4. Here Document 和 Here String 的用法和场景？

**回答：**

### Here Document（`<<`）

```bash
# 基本语法
cat << EOF
Hello, $USER
当前目录: $PWD
今天是: $(date)
EOF

# 带引号的定界符：禁止变量展开
cat << 'EOF'
这里的 $USER 不会被展开
$(date) 也不会执行
EOF

# 缩进的 Here Document：<<-（去掉前导 Tab）
if true; then
    cat <<-EOF
		这一行前面的 Tab 会被去掉
		但空格不会
	EOF
fi

# 重定向到文件
cat > /tmp/config.ini << EOF
[database]
host=localhost
port=3306
user=$DB_USER
password=$DB_PASS
EOF
```

### Here String（`<<<`，Bash 扩展）

```bash
# 将字符串作为 stdin 传给命令
grep "pattern" <<< "search in this string"

# 变量内容作为 stdin
data="hello world"
read -r first second <<< "$data"
echo "first=$first second=$second"

# 替代 echo | cmd 模式
# 不推荐（创建子进程）
echo "$var" | tr 'a-z' 'A-Z'
# 推荐（无子进程）
tr 'a-z' 'A-Z' <<< "$var"

# 分割字符串
IFS=: read -r user _ uid gid _ home shell <<< "root:x:0:0:root:/root:/bin/bash"
```

### 实际应用场景

```bash
# 1. 远程执行命令
ssh user@server << 'REMOTE'
    cd /opt/app
    git pull
    systemctl restart app
REMOTE

# 2. 生成配置文件
generate_nginx_conf() {
    local server_name=$1
    local port=$2
    cat << EOF
server {
    listen 80;
    server_name $server_name;

    location / {
        proxy_pass http://127.0.0.1:$port;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
    }
}
EOF
}
generate_nginx_conf "example.com" 8080 > /etc/nginx/conf.d/example.conf

# 3. SQL 查询
mysql -u root -p"$MYSQL_PWD" << SQL
USE mydb;
SELECT COUNT(*) FROM users WHERE active = 1;
SQL

# 4. 多行变量赋值
readme=$(cat << 'EOF'
# My Project
这是一个示例项目
## 用法
./run.sh
EOF
)
```

## 5. 管道（Pipeline）的工作原理和注意事项？

**回答：**

### 工作原理

```
cmd1 | cmd2 | cmd3

cmd1 的 stdout ──→ cmd2 的 stdin ──→ cmd3 的 stdin
cmd1 的 stderr ──→ 终端
cmd2 的 stderr ──→ 终端

每个命令在 独立的子进程 中运行
所有命令 同时启动、并行执行
通过内核管道缓冲区传递数据（默认 64KB）
```

### 管道状态码

```bash
# $? 只返回最后一个命令的退出码
false | true
echo $?      # 0（只看 true 的状态）

# PIPESTATUS 数组保存每个命令的状态
false | true | false
echo "${PIPESTATUS[@]}"   # 1 0 1

# set -o pipefail：任一命令失败则管道失败
set -o pipefail
false | true
echo $?      # 1（false 失败了）
```

### 常见管道组合

```bash
# 过滤 + 排序 + 去重 + 统计
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -10

# 更高效的写法（避免无用 cat）
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# 多级过滤
ps aux | grep nginx | grep -v grep | awk '{print $2}'

# 替代方案（避免 grep -v grep）
pgrep -a nginx
```

### 管道中的子 Shell 问题

```bash
# 管道最后一个命令在子 Shell 中运行（Bash 默认）
count=0
echo -e "a\nb\nc" | while read -r line; do
    (( count++ ))
done
echo "$count"   # 0！while 在子 Shell 中

# 解决方案1：进程替换
count=0
while read -r line; do
    (( count++ ))
done < <(echo -e "a\nb\nc")
echo "$count"   # 3

# 解决方案2：lastpipe（Bash 4.2+）
shopt -s lastpipe
set +m   # 需要关闭 job control
count=0
echo -e "a\nb\nc" | while read -r line; do
    (( count++ ))
done
echo "$count"   # 3

# 解决方案3：命名管道（FIFO）
mkfifo /tmp/mypipe
producer > /tmp/mypipe &
while read -r line; do
    (( count++ ))
done < /tmp/mypipe
rm /tmp/mypipe
```

### 管道与 stderr

```bash
# 管道只传递 stdout，不传递 stderr
cmd1 2>&1 | cmd2             # stdout + stderr 都传给 cmd2
cmd1 |& cmd2                 # Bash 简写，等价上面

# 只传 stderr，不传 stdout
cmd1 3>&1 1>&2 2>&3 | cmd2   # 交换 stdout 和 stderr
# 更清晰的写法
cmd1 2>&1 1>/dev/null | cmd2 # stdout 丢弃，stderr 通过管道
```

## 6. tee 命令的用法和场景？

**回答：**

`tee` 从 stdin 读取，同时写入 stdout **和**文件（T 型分流）。

```bash
# 基本用法
echo "hello" | tee output.txt         # 终端显示 + 写入文件
echo "hello" | tee -a output.txt      # -a 追加模式

# 同时写入多个文件
echo "log" | tee file1.txt file2.txt file3.txt

# 管道中间记录
cat data.csv | tee /tmp/raw.csv | awk -F, '{print $1}' | sort | tee /tmp/sorted.txt | head

# 结合 sudo 写入受保护的文件
# echo "text" > /etc/config  # 失败！重定向由当前用户的 Shell 处理
echo "text" | sudo tee /etc/config > /dev/null   # 正确
echo "text" | sudo tee -a /etc/config > /dev/null # 追加

# 记录日志同时显示
deploy.sh 2>&1 | tee -a /var/log/deploy.log

# 只记录不显示
echo "secret" | tee /tmp/file > /dev/null
```

### 进程替换 + tee 分流

```bash
# 将 stdout 分流到多个处理程序
echo "data" | tee >(gzip > data.gz) >(sha256sum > data.sha256) > data.txt
# 同时：写入 data.txt、压缩为 data.gz、计算哈希

# 日志同时发送到文件和 syslog
app_command 2>&1 | tee /var/log/app.log | logger -t myapp
```

## 7. 进程替换（Process Substitution）是什么？怎么用？

**回答：**

进程替换将命令的输出/输入绑定到一个临时文件描述符路径（`/dev/fd/N`），让命令输出看起来像一个文件。

### 语法

```bash
<(command)   # 将 command 的 stdout 作为文件来读取
>(command)   # 将写入的内容作为 command 的 stdin
```

### 常见用法

```bash
# 1. 比较两个命令的输出
diff <(sort file1.txt) <(sort file2.txt)

# 2. 比较远程和本地文件
diff <(ssh server cat /etc/nginx/nginx.conf) /etc/nginx/nginx.conf

# 3. 替代临时文件
# 不用进程替换（需要临时文件）
sort file1 > /tmp/sorted1
sort file2 > /tmp/sorted2
comm /tmp/sorted1 /tmp/sorted2
rm /tmp/sorted1 /tmp/sorted2

# 用进程替换（无临时文件）
comm <(sort file1) <(sort file2)

# 4. 同时处理多个输出
paste <(cut -f1 data.tsv) <(cut -f3 data.tsv)

# 5. while 读取不创建子 Shell（替代管道）
while IFS= read -r line; do
    (( count++ ))
done < <(find . -name "*.log")
echo "$count"   # 变量正常可用

# 6. 写入多个目标
echo "log entry" | tee >(logger -t myapp) >(mail -s "Alert" admin@co.com)
```

### 进程替换 vs 管道

```bash
# 管道：需要支持 stdin
echo "data" | wc -l

# 进程替换：用于需要文件名参数的命令
wc -l <(echo "data")

# 进程替换不创建子 Shell（对于读取端）
# 管道的 while 在子 Shell 中，进程替换的 while 不在
```

### 底层原理

```bash
# 进程替换创建 /dev/fd/N 路径
echo <(echo hello)
# 输出类似：/dev/fd/63

# 本质上是一个命名管道或 /dev/fd 文件
# Bash 创建管道，启动子进程执行命令，将管道的 fd 路径传给主命令
```

## 8. 如何实现输出的格式化（printf、column、格式化输出）？

**回答：**

### printf（推荐用于格式化）

```bash
# 基本格式
printf "格式字符串" 参数...

# 常用格式说明符
printf "%-20s %5d %8.2f\n" "Name" 42 3.14
#        ↑左对齐20宽  ↑右对齐5宽  ↑8宽2位小数

# 格式说明符
# %s — 字符串         %d — 十进制整数
# %f — 浮点数         %e — 科学计数法
# %x — 十六进制       %o — 八进制
# %% — 字面 %         %b — 解释反斜杠转义
# %-Ns — 左对齐N宽    %Ns — 右对齐N宽

# 表格输出
printf "%-15s %-10s %8s\n" "服务" "状态" "端口"
printf "%-15s %-10s %8d\n" "nginx" "running" 80
printf "%-15s %-10s %8d\n" "mysql" "running" 3306
printf "%-15s %-10s %8d\n" "redis" "stopped" 6379

# 输出：
# 服务            状态       端口
# nginx          running        80
# mysql          running      3306
# redis          stopped      6379

# 颜色输出
RED='\033[0;31m'
GREEN='\033[0;32m'
NC='\033[0m'
printf "${GREEN}✓${NC} %s 部署成功\n" "nginx"
printf "${RED}✗${NC} %s 部署失败\n" "mysql"

# printf -v 赋值给变量（不输出）
printf -v formatted "%.2f" 3.14159
echo "$formatted"   # 3.14
```

### echo vs printf

```bash
# echo 的问题
echo -e "hello\tworld"   # 有些系统不支持 -e
echo -n "no newline"     # 有些系统把 -n 当文本

# printf 更可靠
printf "hello\tworld\n"
printf "no newline"
```

### column 命令

```bash
# 表格格式化
echo -e "Name\tAge\tCity\nAlice\t30\tBeijing\nBob\t25\tShanghai" | column -t
# Name   Age  City
# Alice  30   Beijing
# Bob    25   Shanghai

# 指定分隔符
cat /etc/passwd | column -t -s ':'

# 多列显示
ls /usr/bin | column -c 80
```

## 9. 如何用 read 命令实现交互式输入？

**回答：**

```bash
# 基本用法
read -r name
echo "你好, $name"

# -p 提示符
read -rp "请输入用户名: " username

# -s 隐藏输入（密码）
read -rsp "请输入密码: " password
echo    # read -s 不换行，手动换行

# -t 超时
if read -rt 10 -p "10秒内输入 (y/n): " answer; then
    echo "你输入了: $answer"
else
    echo "超时！"
fi

# -n 限制字符数
read -rn 1 -p "按任意键继续..."
echo

# -a 读入数组
echo "输入多个值（空格分隔）："
read -ra items
echo "第一个: ${items[0]}"

# -d 指定定界符（替代换行符）
read -rd ',' field <<< "hello,world"
echo "$field"   # hello

# -e 启用 readline 编辑（支持方向键、Tab 补全）
read -rep "输入文件路径: " filepath

# -i 设置默认值（配合 -e）
read -rep "端口号: " -i "8080" port
```

### 读取密码的安全模式

```bash
read_password() {
    local prompt=${1:-"Password: "}
    local password

    # 关闭终端回显
    if [ -t 0 ]; then
        read -rsp "$prompt" password
        echo >&2
    else
        # 非交互模式直接读取
        IFS= read -r password
    fi

    echo "$password"
}

pass=$(read_password "请输入数据库密码: ")
```

### 确认提示

```bash
confirm() {
    local prompt=${1:-"确认执行？"}
    local answer

    read -rp "$prompt [y/N] " answer
    case "$answer" in
        [yY]|[yY][eE][sS]) return 0 ;;
        *) return 1 ;;
    esac
}

if confirm "是否部署到生产环境？"; then
    deploy_prod
else
    echo "已取消"
fi
```

## 10. 命名管道（FIFO）是什么？怎么使用？

**回答：**

命名管道是文件系统中的特殊文件，允许不相关的进程之间通过文件路径进行通信。

```bash
# 创建命名管道
mkfifo /tmp/mypipe
ls -l /tmp/mypipe
# prw-r--r-- 1 user user 0 ... /tmp/mypipe
#  ↑ p 表示管道文件

# 基本使用（需要两个终端或后台进程）
# 终端1（生产者）
echo "Hello from producer" > /tmp/mypipe   # 阻塞直到有读取者

# 终端2（消费者）
cat < /tmp/mypipe                           # 阻塞直到有数据

# 清理
rm /tmp/mypipe
```

### 实际应用

```bash
# 进程间通信
mkfifo /tmp/job_queue

# 生产者（后台持续写入）
(
    for task in build test deploy; do
        echo "$task"
        sleep 1
    done
) > /tmp/job_queue &

# 消费者
while IFS= read -r task; do
    echo "处理任务: $task"
done < /tmp/job_queue

rm /tmp/job_queue
```

### 日志聚合

```bash
# 多个进程写入同一个 FIFO
mkfifo /tmp/log_pipe

# 日志处理器（后台）
while IFS= read -r line; do
    echo "[$(date '+%H:%M:%S')] $line" >> /var/log/combined.log
done < /tmp/log_pipe &
LOG_PID=$!

# 多个进程写入
echo "Worker 1 started" > /tmp/log_pipe
echo "Worker 2 started" > /tmp/log_pipe

# 清理
kill $LOG_PID
rm /tmp/log_pipe
```

## 11. /dev/null、/dev/zero、/dev/urandom 等特殊文件有什么用？

**回答：**

| 设备文件 | 读取行为 | 写入行为 | 用途 |
|----------|----------|----------|------|
| `/dev/null` | 立即 EOF | 丢弃所有数据 | 黑洞，丢弃输出 |
| `/dev/zero` | 无限零字节 | 丢弃 | 创建空文件/清零 |
| `/dev/urandom` | 随机数据 | 丢弃 | 生成随机数据 |
| `/dev/random` | 高质量随机（可能阻塞） | 丢弃 | 密码学 |
| `/dev/full` | 零字节 | 返回 ENOSPC | 测试磁盘满 |
| `/dev/stdin` | 链接到 fd 0 | - | 显式引用 stdin |
| `/dev/stdout` | - | 链接到 fd 1 | 显式引用 stdout |
| `/dev/stderr` | - | 链接到 fd 2 | 显式引用 stderr |
| `/dev/fd/N` | 链接到 fd N | 链接到 fd N | 访问任意 fd |
| `/dev/tcp/host/port` | TCP 连接 | TCP 连接 | Bash 网络（非真实文件） |

```bash
# /dev/null：丢弃输出
command > /dev/null 2>&1    # 不看任何输出
: > file.log                # 清空文件（: 是空命令）

# /dev/zero：创建文件
dd if=/dev/zero of=swap.img bs=1M count=1024  # 创建 1GB 空文件

# /dev/urandom：生成随机数据
head -c 32 /dev/urandom | base64     # 随机 base64 字符串
od -An -tx1 -N16 /dev/urandom | tr -d ' '  # 随机十六进制

# /dev/tcp：Bash 网络功能（非 POSIX）
exec 3<>/dev/tcp/example.com/80
echo -e "GET / HTTP/1.0\r\nHost: example.com\r\n\r\n" >&3
cat <&3
exec 3>&-

# /dev/full：测试磁盘满场景
echo "test" > /dev/full 2>&1
# bash: echo: write error: No space left on device
```

## 12. 如何同时捕获 stdout 和 stderr 到不同变量？

**回答：**

```bash
# 方法1：临时文件
tmpfile=$(mktemp)
stdout=$(command 2>"$tmpfile")
stderr=$(cat "$tmpfile")
rm "$tmpfile"

# 方法2：fd 交换技巧
{
    result=$( { command; } 2>&1 1>&3 )
} 3>&1
# result 包含 stderr，stdout 正常输出到终端

# 方法3：完整捕获两者
capture_both() {
    local tmp_err
    tmp_err=$(mktemp)
    local stdout
    local exit_code

    stdout=$("$@" 2>"$tmp_err")
    exit_code=$?

    CAPTURED_STDOUT="$stdout"
    CAPTURED_STDERR="$(cat "$tmp_err")"
    CAPTURED_EXIT=$exit_code

    rm -f "$tmp_err"
    return $exit_code
}

capture_both ls /nonexistent /tmp
echo "stdout: $CAPTURED_STDOUT"
echo "stderr: $CAPTURED_STDERR"
echo "exit: $CAPTURED_EXIT"

# 方法4：子 Shell + eval（无临时文件，复杂但高效）
eval "$( (command) 2> >(sed 's/^/STDERR:/') > >(sed 's/^/STDOUT:/') )"
```

## 13. 重定向的执行顺序和常见陷阱有哪些？

**回答：**

### 执行顺序

重定向按**从左到右**的顺序应用：

```bash
# 示例分析
cmd > file 2>&1
# 步骤1: stdout(fd1) → file
# 步骤2: stderr(fd2) → fd1 当前指向的位置（file）
# 结果：stdout 和 stderr 都到 file

cmd 2>&1 > file
# 步骤1: stderr(fd2) → fd1 当前指向的位置（终端）
# 步骤2: stdout(fd1) → file
# 结果：stdout 到 file，stderr 到终端！

# 可视化
# 初始状态：  fd1→终端  fd2→终端
# > file:      fd1→file fd2→终端
# 2>&1:        fd1→file fd2→file    ← stdout 和 stderr 都到 file

# 初始状态：  fd1→终端  fd2→终端
# 2>&1:        fd1→终端  fd2→终端    ← stderr 指向 fd1（此刻是终端）
# > file:      fd1→file  fd2→终端    ← 只有 stdout 到 file
```

### 常见陷阱

```bash
# 陷阱1：重定向覆盖输入文件
sort < file > file    # file 被清空！重定向在命令执行前生效
# 正确做法
sort file > file.sorted && mv file.sorted file
sort -o file file     # sort 的 -o 选项可以原地排序

# 陷阱2：循环中的重定向性能
# 慢（每次迭代打开关闭文件）
for i in {1..1000}; do
    echo "$i" >> output.txt
done
# 快（只打开一次）
for i in {1..1000}; do
    echo "$i"
done > output.txt

# 陷阱3：noclobber 保护
set -o noclobber            # 开启覆盖保护
echo "test" > existing.txt  # 错误！文件已存在
echo "test" >| existing.txt # 强制覆盖（使用 >|）

# 陷阱4：子 Shell 中的重定向不影响父 Shell
(exec > /tmp/log) 
echo "这行不会到 /tmp/log"  # 父 Shell 未受影响
```
