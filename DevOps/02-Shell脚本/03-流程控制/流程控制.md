## 1. if 条件判断的完整语法是什么？有哪些注意事项？

**回答：**

```bash
# 基本语法
if condition; then
    commands
elif condition; then
    commands
else
    commands
fi

# condition 可以是：
# 1. 命令的退出状态（0 为真）
if grep -q "pattern" file; then
    echo "找到了"
fi

# 2. test 表达式
if [ -f /etc/passwd ]; then
    echo "文件存在"
fi

# 3. [[ ]] 扩展测试
if [[ "$str" =~ ^[0-9]+$ ]]; then
    echo "是数字"
fi

# 4. (( )) 算术表达式
if (( count > 10 )); then
    echo "大于10"
fi

# 5. 命令组合
if cmd1 && cmd2; then
    echo "两个都成功"
fi
```

**注意事项：**
- `then` 必须在新行或 `;then`
- `[ ]` 内部两侧**必须有空格**：`[ "$a" = "b" ]`
- 变量引用加双引号避免分词问题
- `if` 判断的本质是命令的**退出状态码**（0=真，非0=假）

## 2. `[ ]`（test）和 `[[ ]]` 有什么区别？什么时候用哪个？

**回答：**

| 特性 | `[ ]` (test) | `[[ ]]` |
|------|-------------|---------|
| 标准 | POSIX 标准 | Bash/Ksh 扩展 |
| 字符串比较 | `=`, `!=` | `=`, `!=`, `<`, `>` |
| 正则匹配 | 不支持 | `=~` 支持 |
| 模式匹配 | 不支持 | `==` 支持 glob |
| 逻辑运算 | `-a`, `-o`（不推荐） | `&&`, `||` |
| 变量分词 | 会分词，必须加引号 | 不分词，引号可选 |
| 文件名展开 | 会展开 | 不展开 |
| 可移植性 | 所有 Shell | 仅 Bash/Zsh/Ksh |

```bash
# [ ] 的陷阱
var=""
[ $var = "hello" ]        # 错误！变量为空导致语法错误
[ "$var" = "hello" ]      # 正确

# [[ ]] 更安全
[[ $var = "hello" ]]      # 即使 var 为空也不报错

# [[ ]] 支持正则
[[ "abc123" =~ ^[a-z]+[0-9]+$ ]] && echo "匹配"

# [[ ]] 支持模式匹配
[[ "hello.txt" == *.txt ]] && echo "是txt文件"

# [[ ]] 逻辑运算更直观
[[ -f file && -r file ]]  # 文件存在且可读

# [ ] 逻辑运算（不推荐 -a/-o）
[ -f file ] && [ -r file ]  # 推荐这种写法
```

**最佳实践：**
- 写 Bash 脚本优先用 `[[ ]]`
- 需要 POSIX 兼容（如 `#!/bin/sh`）则用 `[ ]`
- 数值比较用 `(( ))`

## 3. test 命令有哪些常用的文件测试、字符串测试和数值测试操作符？

**回答：**

### 文件测试

```bash
[ -e file ]    # 文件/目录存在
[ -f file ]    # 是普通文件
[ -d file ]    # 是目录
[ -L file ]    # 是符号链接
[ -r file ]    # 可读
[ -w file ]    # 可写
[ -x file ]    # 可执行
[ -s file ]    # 非空文件（大小 > 0）
[ -b file ]    # 块设备
[ -c file ]    # 字符设备
[ -p file ]    # 命名管道
[ -S file ]    # Socket 文件
[ -g file ]    # 设置了 SGID
[ -u file ]    # 设置了 SUID
[ -k file ]    # 设置了 Sticky Bit
[ -O file ]    # 当前用户拥有
[ -G file ]    # 当前用户组拥有
[ -N file ]    # 自上次读取后被修改过
[ f1 -nt f2 ]  # f1 比 f2 新（newer than）
[ f1 -ot f2 ]  # f1 比 f2 旧（older than）
[ f1 -ef f2 ]  # f1 和 f2 是同一个文件（硬链接）
```

### 字符串测试

```bash
[ -z "$str" ]           # 字符串长度为 0
[ -n "$str" ]           # 字符串长度非 0
[ "$str1" = "$str2" ]   # 字符串相等
[ "$str1" != "$str2" ]  # 字符串不等
[[ "$str1" < "$str2" ]] # 字典序小于（仅 [[ ]]）
[[ "$str1" > "$str2" ]] # 字典序大于（仅 [[ ]]）
[[ "$str" =~ regex ]]   # 正则匹配（仅 [[ ]]）
[[ "$str" == pattern ]] # 模式匹配（仅 [[ ]]）
```

### 数值测试

```bash
[ "$a" -eq "$b" ]   # 等于（equal）
[ "$a" -ne "$b" ]   # 不等于（not equal）
[ "$a" -gt "$b" ]   # 大于（greater than）
[ "$a" -ge "$b" ]   # 大于等于（greater or equal）
[ "$a" -lt "$b" ]   # 小于（less than）
[ "$a" -le "$b" ]   # 小于等于（less or equal）

# (( )) 中可以用 C 风格
(( a == b ))
(( a != b ))
(( a > b ))
(( a >= b ))
(( a < b ))
(( a <= b ))
```

## 4. case 语句的用法和高级模式匹配？

**回答：**

```bash
# 基本语法
case "$variable" in
    pattern1)
        commands
        ;;      # 等价于 break
    pattern2 | pattern3)  # 多模式用 | 分隔
        commands
        ;;
    *)          # 默认匹配（类似 default）
        commands
        ;;
esac
```

### 模式匹配类型

```bash
case "$input" in
    # 精确匹配
    "start")  echo "启动" ;;
    "stop")   echo "停止" ;;

    # 通配符
    *.txt)    echo "文本文件" ;;
    *.tar.*)  echo "压缩包" ;;

    # 字符类
    [0-9]*)   echo "以数字开头" ;;
    [a-zA-Z]*) echo "以字母开头" ;;

    # 多选一
    yes|y|Y)  echo "确认" ;;
    no|n|N)   echo "取消" ;;
esac
```

### Bash 4.0+ 高级终止符

```bash
case "$value" in
    pattern1)
        echo "匹配1"
        ;;      # 终止（默认 break）
    pattern2)
        echo "匹配2"
        ;&      # 穿透：直接执行下一个分支（不检查模式）
    pattern3)
        echo "匹配3"
        ;;&     # 继续匹配：检查后续分支的模式
    pattern4)
        echo "匹配4"
        ;;
esac
```

### 实际用法：服务管理脚本

```bash
case "$1" in
    start)
        start_service
        ;;
    stop)
        stop_service
        ;;
    restart)
        stop_service
        sleep 1
        start_service
        ;;
    status)
        check_status
        ;;
    reload|force-reload)
        reload_config
        ;;
    *)
        echo "Usage: $0 {start|stop|restart|status|reload}" >&2
        exit 1
        ;;
esac
```

## 5. for 循环有哪些形式？各适用于什么场景？

**回答：**

### 列表型 for 循环

```bash
# 直接列表
for fruit in apple banana cherry; do
    echo "$fruit"
done

# 变量列表
items="a b c d e"
for item in $items; do
    echo "$item"
done

# 命令替换
for file in $(ls *.log); do   # 不推荐：有分词和展开问题
    echo "$file"
done

# Glob 模式（推荐）
for file in /var/log/*.log; do
    [ -f "$file" ] || continue  # 防止没有匹配时 glob 不展开
    echo "$file"
done

# 数组
arr=(one two three)
for item in "${arr[@]}"; do
    echo "$item"
done

# 序列（Brace Expansion）
for i in {1..10}; do
    echo "$i"
done

for i in {0..100..5}; do   # 步长为 5
    echo "$i"
done

# seq 命令
for i in $(seq 1 2 20); do  # 从 1 到 20，步长 2
    echo "$i"
done
```

### C 风格 for 循环

```bash
for (( i = 0; i < 10; i++ )); do
    echo "$i"
done

# 多个变量
for (( i = 0, j = 10; i < j; i++, j-- )); do
    echo "i=$i j=$j"
done

# 无限循环
for (( ;; )); do
    echo "无限"
    break
done
```

### 遍历文件行

```bash
# 推荐方式：while + read
while IFS= read -r line; do
    echo "$line"
done < file.txt

# for 不适合逐行读文件（会按空白分词）
# 如果一定要用 for：
IFS=$'\n'
for line in $(cat file.txt); do
    echo "$line"
done
unset IFS
```

### 常见陷阱

```bash
# 陷阱1：文件名含空格
for f in $(ls); do          # 错误！"my file.txt" 被分成两个词
for f in *; do              # 正确：使用 glob

# 陷阱2：{1..$n} 不工作（Brace Expansion 先于变量展开）
n=10
for i in {1..$n}; do        # 错误！不展开
for i in $(seq 1 "$n"); do  # 正确
for (( i = 1; i <= n; i++ )); do  # 正确
```

## 6. while 和 until 循环的用法及典型应用？

**回答：**

### while 循环（条件为真时继续）

```bash
# 基本语法
count=0
while [ "$count" -lt 10 ]; do
    echo "$count"
    (( count++ ))
done

# 无限循环
while true; do
    echo "运行中..."
    sleep 1
done

# 读取文件（最常见用法）
while IFS= read -r line; do
    echo "行内容: $line"
done < input.txt

# 读取命令输出
ps aux | while IFS= read -r line; do
    echo "$line"
done
# 注意：管道创建子 Shell，while 里的变量修改对外不可见

# 解决子 Shell 问题：进程替换
while IFS= read -r line; do
    (( count++ ))
done < <(ps aux)
echo "总行数: $count"   # 这里 count 是正确的
```

### until 循环（条件为假时继续）

```bash
# 等待服务启动
until curl -sf http://localhost:8080/health > /dev/null 2>&1; do
    echo "等待服务启动..."
    sleep 2
done
echo "服务已启动"

# 等待文件出现
until [ -f /tmp/ready.flag ]; do
    sleep 1
done

# 带超时的等待
timeout=30
elapsed=0
until curl -sf http://localhost:8080/health > /dev/null 2>&1; do
    if (( elapsed >= timeout )); then
        echo "超时！" >&2
        exit 1
    fi
    sleep 1
    (( elapsed++ ))
done
```

### 多条件读取

```bash
# 同时读取多个变量
while IFS=: read -r user _ uid gid _ home shell; do
    echo "用户: $user, UID: $uid, Shell: $shell"
done < /etc/passwd

# 按分隔符读取 CSV
while IFS=, read -r name age city; do
    echo "$name 在 $city，$age 岁"
done < data.csv

# 读取带行号
linenum=0
while IFS= read -r line; do
    (( linenum++ ))
    printf "%4d: %s\n" "$linenum" "$line"
done < file.txt
```

### while 读取的注意事项

```bash
# 保留前导/尾随空白
while IFS= read -r line; do   # IFS= 防止去掉空白
    echo "$line"
done

# -r 防止反斜杠转义
while read -r line; do   # -r 很重要！
    echo "$line"
done

# 处理最后一行没有换行的情况
while IFS= read -r line || [ -n "$line" ]; do
    echo "$line"
done < file.txt
```

## 7. select 菜单循环如何使用？

**回答：**

```bash
# 基本语法
PS3="请选择操作: "   # select 使用 PS3 作为提示符
select opt in "查看日志" "重启服务" "退出"; do
    case "$opt" in
        "查看日志")
            tail -100 /var/log/syslog
            ;;
        "重启服务")
            systemctl restart myapp
            ;;
        "退出")
            echo "再见"
            break
            ;;
        *)
            echo "无效选择，请重试"
            ;;
    esac
done
```

**输出效果：**
```
1) 查看日志
2) 重启服务
3) 退出
请选择操作: 
```

### 进阶用法

```bash
# 动态菜单项
services=(nginx mysql redis postgresql)
PS3="选择要管理的服务: "
select svc in "${services[@]}" "全部重启" "退出"; do
    case "$svc" in
        "全部重启")
            for s in "${services[@]}"; do
                systemctl restart "$s"
            done
            ;;
        "退出") break ;;
        "")
            echo "无效选择"
            ;;
        *)
            echo "管理 $svc..."
            systemctl status "$svc"
            ;;
    esac
done

# select 中 REPLY 保存用户原始输入
select opt in a b c; do
    echo "选项: $opt"     # 选中的值
    echo "输入: $REPLY"   # 用户输入的数字
    break
done
```

**特点：**
- 自动编号显示菜单
- 使用 `PS3` 变量作为提示符
- 无效输入时 `opt` 为空，需要自行处理
- 需要 `break` 退出，否则会一直循环

## 8. break 和 continue 的用法？支持多层嵌套控制吗？

**回答：**

```bash
# break — 跳出循环
for i in {1..10}; do
    if (( i == 5 )); then
        break     # 跳出整个 for 循环
    fi
    echo "$i"
done
# 输出：1 2 3 4

# continue — 跳过本次迭代
for i in {1..10}; do
    if (( i % 2 == 0 )); then
        continue  # 跳过偶数
    fi
    echo "$i"
done
# 输出：1 3 5 7 9
```

### 多层嵌套控制

```bash
# break N — 跳出 N 层循环
for i in {1..3}; do
    for j in {1..3}; do
        if (( i == 2 && j == 2 )); then
            break 2     # 跳出两层循环
        fi
        echo "i=$i j=$j"
    done
done
# i=1 j=1, i=1 j=2, i=1 j=3, i=2 j=1 后直接结束

# continue N — 跳转到外层第 N 层循环的下一次迭代
for i in {1..3}; do
    for j in {1..3}; do
        if (( j == 2 )); then
            continue 2   # 跳到外层 for 的下一次迭代
        fi
        echo "i=$i j=$j"
    done
done
# 输出：i=1 j=1, i=2 j=1, i=3 j=1
```

### 实用场景

```bash
# 检索服务列表，找到第一个运行的
for host in server{1..5}; do
    for svc in nginx httpd apache2; do
        if ssh "$host" "systemctl is-active $svc" 2>/dev/null; then
            echo "找到: $host 运行 $svc"
            break 2
        fi
    done
done

# 批处理中跳过错误项
for file in *.csv; do
    head -1 "$file" | grep -q "^id," || continue   # 格式不对就跳过
    process_csv "$file"
done
```

## 9. 条件表达式中的逻辑运算怎么写？短路求值是怎么回事？

**回答：**

### 逻辑运算

```bash
# 在 [[ ]] 中
[[ cond1 && cond2 ]]   # 逻辑与
[[ cond1 || cond2 ]]   # 逻辑或
[[ ! cond ]]            # 逻辑非

# 在 [ ] 中（不推荐用 -a/-o）
[ cond1 ] && [ cond2 ]  # 推荐
[ cond1 -a cond2 ]      # 不推荐：有解析歧义
[ cond1 ] || [ cond2 ]  # 推荐
[ cond1 -o cond2 ]      # 不推荐

# 命令级逻辑
cmd1 && cmd2   # cmd1 成功则执行 cmd2
cmd1 || cmd2   # cmd1 失败则执行 cmd2
! cmd          # 取反退出状态
```

### 短路求值

```bash
# && 短路：左边为假，右边不执行
[ -f file ] && echo "文件存在"    # 文件不存在就不打印

# || 短路：左边为真，右边不执行
[ -f file ] || echo "文件不存在"  # 文件存在就不打印

# 常见的条件执行模式
[ -d "$dir" ] || mkdir -p "$dir"       # 目录不存在则创建
command -v docker > /dev/null || { echo "需要安装 Docker"; exit 1; }

# && 和 || 组合（模拟三元运算符）
# 注意：这不完全等价于三元，有坑！
[ -f file ] && echo "存在" || echo "不存在"
# 如果 echo "存在" 执行失败（几乎不会），会执行 echo "不存在"
# 安全写法：
if [ -f file ]; then echo "存在"; else echo "不存在"; fi
```

### 复合条件

```bash
# 检查文件可读且非空
if [[ -f "$file" && -r "$file" && -s "$file" ]]; then
    echo "文件存在、可读且非空"
fi

# 嵌套条件
if [[ ( "$env" == "prod" || "$env" == "staging" ) && -n "$token" ]]; then
    deploy
fi

# 算术条件
if (( (age >= 18 && age <= 65) || vip == 1 )); then
    echo "允许访问"
fi
```

## 10. 如何实现类似其他语言的 switch-case 穿透和多条件匹配？

**回答：**

### Bash 4.0+ 穿透机制

```bash
# ;; — 终止（默认，不穿透）
# ;& — 无条件穿透（Fall Through，不检查下一个模式）
# ;;& — 条件穿透（Continue Matching，检查下一个模式）

# ;& 示例：无条件穿透
case "$level" in
    critical)
        send_page
        ;&        # 穿透到下一个
    error)
        send_email
        ;&        # 继续穿透
    warning)
        log_warning
        ;;
    info)
        log_info
        ;;
esac
# 如果 level=critical，依次执行 send_page, send_email, log_warning

# ;;& 示例：条件穿透
case "$filename" in
    *.tar.gz | *.tgz)
        echo "是 gzip 压缩包"
        ;;&    # 继续检查
    *.tar.*)
        echo "是 tar 包"
        ;;&    # 继续检查
    *.gz)
        echo "是 gzip 文件"
        ;;
esac
# "file.tar.gz" 输出：是 gzip 压缩包, 是 tar 包, 是 gzip 文件
```

### 用函数数组模拟 switch

```bash
# 当 case 分支太多时用关联数组
declare -A handlers=(
    [start]=do_start
    [stop]=do_stop
    [restart]=do_restart
    [status]=do_status
)

if [[ -n "${handlers[$1]+set}" ]]; then
    "${handlers[$1]}"
else
    echo "未知命令: $1"
fi
```

## 11. 如何处理命令行参数和选项解析（getopts/getopt）？

**回答：**

### 手动解析

```bash
while [ $# -gt 0 ]; do
    case "$1" in
        -v|--verbose) VERBOSE=1; shift ;;
        -f|--file)    FILE="$2"; shift 2 ;;
        -h|--help)    usage; exit 0 ;;
        --)           shift; break ;;    # -- 后面都是位置参数
        -*)           echo "未知选项: $1" >&2; exit 1 ;;
        *)            break ;;           # 非选项参数，结束解析
    esac
done
# $@ 中剩余的是位置参数
```

### 内置 getopts（POSIX，仅支持短选项）

```bash
# 用法：getopts optstring name
# 选项后跟 : 表示需要参数

while getopts ":vf:o:h" opt; do
    case "$opt" in
        v) VERBOSE=1 ;;
        f) FILE="$OPTARG" ;;       # OPTARG 是选项的参数
        o) OUTPUT="$OPTARG" ;;
        h) usage; exit 0 ;;
        :) echo "选项 -$OPTARG 缺少参数" >&2; exit 1 ;;
        \?) echo "未知选项 -$OPTARG" >&2; exit 1 ;;
    esac
done
shift $((OPTIND - 1))   # 移除已解析的选项，剩余是位置参数
echo "剩余参数: $*"
```

**getopts 要点：**
- optstring 开头的 `:` 启用静默模式（自己处理错误）
- 不支持长选项 `--file`
- `OPTIND` 跟踪当前解析位置

### 外部 getopt（GNU，支持长选项）

```bash
# 需要 util-linux 的 getopt（不是旧版）
OPTS=$(getopt -o vf:o:h --long verbose,file:,output:,help \
    -n "$0" -- "$@") || exit 1

eval set -- "$OPTS"

while true; do
    case "$1" in
        -v|--verbose) VERBOSE=1; shift ;;
        -f|--file)    FILE="$2"; shift 2 ;;
        -o|--output)  OUTPUT="$2"; shift 2 ;;
        -h|--help)    usage; exit 0 ;;
        --)           shift; break ;;
        *)            echo "内部错误"; exit 1 ;;
    esac
done
```

## 12. 流程控制中有哪些常见的陷阱和最佳实践？

**回答：**

### 常见陷阱

```bash
# 陷阱1：未引用变量导致错误
var=""
if [ $var = "hello" ]; then   # 展开成 [ = "hello" ]，语法错误
if [ "$var" = "hello" ]; then # 正确

# 陷阱2：= 和 == 的区别
[ "$a" = "$b" ]     # POSIX 标准，[ ] 中只能用 =
[[ "$a" == "$b" ]]  # Bash 扩展，[[ ]] 中 = 和 == 都可以

# 陷阱3：数值比较用错运算符
[ "$a" > "$b" ]     # 这是重定向！不是比较
[ "$a" -gt "$b" ]   # 正确做法
(( a > b ))          # 更好

# 陷阱4：管道中的 while 创建子 Shell
count=0
cat file | while read -r line; do
    (( count++ ))
done
echo "$count"  # 输出 0！子 Shell 中的修改不影响父进程

# 解决：
count=0
while read -r line; do
    (( count++ ))
done < file
echo "$count"  # 正确

# 或使用 lastpipe（Bash 4.2+）
shopt -s lastpipe
count=0
cat file | while read -r line; do
    (( count++ ))
done
echo "$count"  # 正确

# 陷阱5：for 循环中的命令替换分词
for f in $(find . -name "*.txt"); do  # 文件名有空格会出错
    echo "$f"
done

# 正确做法
while IFS= read -r -d '' f; do
    echo "$f"
done < <(find . -name "*.txt" -print0)

# 或使用 glob
shopt -s globstar
for f in **/*.txt; do
    [ -f "$f" ] || continue
    echo "$f"
done
```

### 最佳实践

```bash
# 1. 始终引用变量
if [[ "$status" == "running" ]]; then ...

# 2. 使用 (( )) 做数值比较
if (( retries >= max_retries )); then
    echo "重试次数用尽"
fi

# 3. 优先用 [[ ]] 而非 [ ]
[[ -f "$file" && -n "$content" ]]

# 4. case 比多个 if-elif 更清晰
case "$env" in
    dev|development)  config="dev.conf" ;;
    staging|stg)      config="stg.conf" ;;
    prod|production)  config="prod.conf" ;;
esac

# 5. 用函数封装复杂条件
is_running() {
    systemctl is-active --quiet "$1"
}
if is_running nginx; then
    echo "Nginx 运行中"
fi

# 6. 防御性编程
for f in /path/to/files/*; do
    [ -e "$f" ] || break   # 没有匹配时 glob 不展开
    process "$f"
done
```

## 13. 如何在 Shell 中实现类似 try-catch 的异常处理流程？

**回答：**

Shell 没有原生 try-catch，但可以通过多种方式模拟：

### 方式1：&& / || 链式处理

```bash
command1 && command2 || {
    echo "执行失败" >&2
    exit 1
}
```

### 方式2：set -e + trap ERR

```bash
#!/bin/bash
set -euo pipefail

cleanup() {
    echo "清理资源..."
    rm -f "$tmpfile"
}
trap cleanup EXIT    # 无论如何退出都会执行

error_handler() {
    local line=$1
    local cmd=$2
    echo "错误: 命令 '$cmd' 在第 $line 行失败" >&2
}
trap 'error_handler $LINENO "$BASH_COMMAND"' ERR

# 以下任何命令失败都会触发 error_handler 然后 cleanup
tmpfile=$(mktemp)
curl -sSf http://example.com/data > "$tmpfile"
process_data "$tmpfile"
deploy_result
```

### 方式3：子 Shell 封装 try-catch

```bash
try() {
    [[ $- = *e* ]]
    SAVED_OPT_E=$?
    set +e
}

catch() {
    EXIT_CODE=$?
    if [ $SAVED_OPT_E -ne 0 ]; then
        set +e
    else
        set -e
    fi
    return $EXIT_CODE
}

# 使用
try
(
    set -e
    risky_command_1
    risky_command_2
    risky_command_3
)
catch && {
    echo "全部成功"
} || {
    echo "捕获到错误，退出码: $?" >&2
    recovery_action
}
```

### 方式4：利用子 Shell 隔离错误

```bash
# 子 Shell 中的错误不影响父进程
if output=$(
    set -e
    step1
    step2
    step3
); then
    echo "成功: $output"
else
    echo "失败，退出码: $?" >&2
fi
```

### 方式5：封装重试逻辑

```bash
retry() {
    local max_attempts=$1
    local delay=$2
    shift 2
    local attempt=1

    while (( attempt <= max_attempts )); do
        if "$@"; then
            return 0
        fi
        echo "第 $attempt 次失败，${delay}s 后重试..." >&2
        sleep "$delay"
        (( attempt++ ))
    done

    echo "重试 $max_attempts 次后仍失败" >&2
    return 1
}

# 使用
retry 3 5 curl -sSf http://api.example.com/health
```
