## 1. Git 初始化和克隆？

**回答：**

```bash
# 初始化新仓库
git init                       # 在当前目录创建 .git
git init my-project            # 创建目录并初始化
git init --bare my-repo.git    # 创建裸仓库（无工作区，用于服务器端）

# 克隆仓库
git clone <url>                         # 克隆到同名目录
git clone <url> my-dir                  # 克隆到指定目录
git clone --depth 1 <url>               # 浅克隆（只克隆最近1次提交）
git clone --branch v1.0 <url>           # 克隆指定分支/标签
git clone --single-branch <url>         # 只克隆默认分支
git clone --recurse-submodules <url>    # 递归克隆子模块

# 克隆协议
git clone https://github.com/user/repo.git    # HTTPS（需要凭据）
git clone git@github.com:user/repo.git        # SSH（推荐，用密钥认证）
git clone /path/to/local/repo                 # 本地路径

# 浅克隆 → 完整克隆
git fetch --unshallow
```

## 2. git add 的各种用法？

**回答：**

```bash
# 添加指定文件
git add file.txt

# 添加指定目录
git add src/

# 添加所有修改（新文件 + 修改 + 删除）
git add -A
git add --all

# 添加所有修改（当前目录及子目录）
git add .

# 只添加已追踪文件的修改（不包含新文件）
git add -u

# 交互式暂存（选择性暂存文件的部分内容）
git add -p
# 或
git add --patch
# 会逐个显示 hunk，选择 y(暂存) / n(跳过) / s(拆分更小) / e(手动编辑)

# 交互式暂存示例：
# @@ -1,3 +1,4 @@
#  line 1
# +new line       ← 只暂存这一块
#  line 2
# Stage this hunk [y,n,q,a,d,s,e,?]?

# 强制添加被 .gitignore 忽略的文件
git add -f ignored_file.log

# 模拟添加（不实际暂存）
git add --dry-run .
```

## 3. git commit 的最佳实践？

**回答：**

```bash
# 基本提交
git commit -m "feat: add user login"

# 多行提交信息
git commit -m "feat: add user login" -m "Implement OAuth2 authentication"

# 使用编辑器编写提交信息
git commit                     # 打开配置的编辑器

# 暂存并提交已追踪的文件（跳过 git add）
git commit -am "fix: typo"
# 注意：不会添加新文件（untracked）

# 修改最近一次提交
git commit --amend                     # 修改提交信息
git commit --amend -m "new message"
git commit --amend --no-edit           # 追加文件但不改信息

# 空提交（用于触发 CI）
git commit --allow-empty -m "trigger CI"

# 指定作者
git commit --author="Name <email>"

# 指定日期
git commit --date="2024-01-01T00:00:00"

# 签名提交（GPG）
git commit -S -m "signed commit"
```

### commit 的原子性原则

```
好的提交习惯：
  ✅ 一个提交只做一件事（原子提交）
  ✅ 提交前 git diff --staged 检查
  ✅ 使用 git add -p 精细暂存
  ✅ 提交信息清晰描述 "做了什么" 和 "为什么"
  
  ❌ 一个大提交包含多个不相关修改
  ❌ 提交信息写 "fix" "update" "修改"
  ❌ 提交编译不过或测试失败的代码
```

## 4. git log 查看历史的各种姿势？

**回答：**

```bash
# 基本查看
git log                          # 详细历史
git log --oneline                # 单行简洁
git log --oneline --graph        # 图形化分支
git log --oneline --graph --all  # 所有分支的图形化

# 限制数量
git log -5                       # 最近 5 条
git log --since="2024-01-01"     # 指定日期之后
git log --until="2024-06-01"     # 指定日期之前
git log --after="2 weeks ago"    # 相对日期

# 按作者过滤
git log --author="Alice"
git log --author="Alice\|Bob"    # 多个作者

# 按内容搜索
git log --grep="fix"             # 搜索提交信息
git log -S "functionName"        # 搜索代码变更（pickaxe）
git log -G "regex_pattern"       # 正则搜索代码变更

# 按文件过滤
git log -- path/to/file          # 查看文件的修改历史
git log --follow -- file.txt     # 跟踪重命名

# 格式化输出
git log --pretty=format:"%h %an %ar %s"
# %h  短hash
# %H  完整hash
# %an 作者名
# %ae 作者邮箱
# %ar 相对日期
# %ad 绝对日期
# %s  提交信息主题
# %b  提交信息正文

# 查看每个提交的统计
git log --stat                   # 文件变更统计
git log --shortstat              # 简短统计
git log --name-only              # 只显示变更文件名
git log --name-status            # 文件名 + 状态（A/M/D）

# 查看某个提交的详情
git show <commit>
git show <commit>:<file>         # 查看某次提交时的文件内容
```

## 5. git diff 对比差异？

**回答：**

```bash
# 工作区 vs 暂存区（未暂存的修改）
git diff

# 暂存区 vs 最近提交（已暂存待提交的修改）
git diff --staged
git diff --cached              # 等同于 --staged

# 工作区 vs 最近提交
git diff HEAD

# 两个提交之间的差异
git diff <commit1> <commit2>
git diff HEAD~3 HEAD

# 两个分支之间的差异
git diff main..feature
git diff main...feature        # feature 从 main 分出后的变更

# 只看特定文件
git diff -- path/to/file
git diff main..feature -- src/

# 只看文件名
git diff --name-only
git diff --name-status         # 名字 + 状态

# 统计信息
git diff --stat

# 单词级别的差异（而非行级别）
git diff --word-diff

# 忽略空白变化
git diff -w
git diff --ignore-all-space

# 生成 patch
git diff > changes.patch
git apply changes.patch        # 应用 patch
```

## 6. git status 和 git clean？

**回答：**

```bash
# 查看状态
git status                     # 详细状态
git status -s                  # 简短格式
git status -sb                 # 简短 + 分支信息

# 简短格式解读
# ?? file.txt    → Untracked（新文件）
#  M file.txt    → Modified（工作区已修改）
# M  file.txt    → Modified（已暂存）
# MM file.txt    → 暂存后又修改了
# A  file.txt    → 新文件已暂存
# D  file.txt    → 已删除
# R  file.txt    → 已重命名

# git clean - 清理未追踪的文件
git clean -n               # 预览将要删除的文件（dry run）
git clean -f               # 删除未追踪的文件
git clean -fd              # 删除未追踪的文件和目录
git clean -fX              # 只删除被 .gitignore 忽略的文件
git clean -fx              # 删除所有未追踪（含被忽略的）
git clean -fdx             # 删除所有未追踪的文件和目录（最彻底）

# 注意：git clean 是不可逆的！务必先 -n 预览
```

## 7. git blame 查看文件历史？

**回答：**

```bash
# 查看文件每行的最后修改者
git blame file.txt

# 输出格式：
# abc1234 (Alice 2024-01-15 10:30:00 +0800  1) line content
# def5678 (Bob   2024-01-16 14:20:00 +0800  2) another line

# 只查看特定行范围
git blame -L 10,20 file.txt     # 第10-20行
git blame -L 10,+5 file.txt     # 从第10行开始5行

# 忽略空白变化
git blame -w file.txt

# 检测行移动（从同一文件）
git blame -M file.txt

# 检测行移动（从其他文件，检测代码复制）
git blame -C file.txt
git blame -CCC file.txt         # 更积极的检测

# 显示邮箱
git blame -e file.txt

# 只显示 commit hash
git blame --porcelain file.txt

# 场景：
#   找出某行代码是谁在什么时候为什么改的
#   追踪 bug 引入点
#   代码 review 时了解历史上下文
```

## 8. git stash 储藏操作？

**回答：**

```bash
# 储藏当前修改（已追踪文件的修改 + 暂存区）
git stash
git stash save "work in progress"    # 带说明（老语法）
git stash push -m "work in progress" # 带说明（新语法）

# 储藏包含未追踪文件
git stash -u
git stash --include-untracked

# 储藏所有（包含被忽略的文件）
git stash -a
git stash --all

# 只储藏暂存区的内容
git stash --staged                   # Git 2.35+

# 查看储藏列表
git stash list
# stash@{0}: On main: work in progress
# stash@{1}: WIP on feature: abc1234 previous commit

# 恢复储藏
git stash pop                  # 恢复最近的储藏并删除
git stash apply                # 恢复但不删除（可以应用到其他分支）
git stash apply stash@{1}     # 恢复指定的储藏

# 查看储藏的内容
git stash show                 # 简要统计
git stash show -p              # 详细 diff
git stash show stash@{1} -p

# 删除储藏
git stash drop                 # 删除最近的
git stash drop stash@{1}      # 删除指定的
git stash clear                # 清空所有储藏

# 从储藏创建分支（避免冲突）
git stash branch new-branch stash@{0}

# 交互式储藏（选择性储藏部分修改）
git stash push -p
```

## 9. git tag 标签管理？

**回答：**

```bash
# 创建标签
git tag v1.0                           # 轻量标签
git tag -a v1.0 -m "Release 1.0"      # 注释标签（推荐）
git tag -a v1.0 abc1234                # 给历史提交打标签
git tag -s v1.0 -m "Signed release"   # GPG 签名标签

# 查看标签
git tag                                # 列出所有标签
git tag -l "v1.*"                      # 按模式筛选
git show v1.0                          # 查看标签详情

# 推送标签到远程
git push origin v1.0                   # 推送指定标签
git push origin --tags                 # 推送所有标签
git push origin --follow-tags          # 只推送注释标签

# 删除标签
git tag -d v1.0                        # 删除本地标签
git push origin --delete v1.0          # 删除远程标签
git push origin :refs/tags/v1.0        # 另一种删除远程标签方式

# 切换到标签（Detached HEAD）
git checkout v1.0
git switch --detach v1.0               # 明确进入 detached HEAD

# 基于标签创建分支
git checkout -b hotfix v1.0

# 语义化版本 (SemVer): vMAJOR.MINOR.PATCH
#   MAJOR: 不兼容的 API 修改
#   MINOR: 向后兼容的功能新增
#   PATCH: 向后兼容的 bug 修复
```

## 10. git grep 搜索代码？

**回答：**

```bash
# 在工作区搜索
git grep "searchTerm"

# 在特定提交中搜索
git grep "searchTerm" HEAD
git grep "searchTerm" v1.0

# 显示行号
git grep -n "searchTerm"

# 只显示文件名
git grep -l "searchTerm"

# 统计每个文件的匹配数
git grep -c "searchTerm"

# 正则表达式
git grep -e "pattern" --and -e "another"   # 同一行匹配两个模式
git grep -e "pattern" --or -e "another"    # 匹配任一模式

# 搜索特定文件类型
git grep "TODO" -- "*.py"
git grep "TODO" -- "*.go" "*.py"

# 忽略大小写
git grep -i "searchterm"

# 显示匹配行的上下文
git grep -C 3 "searchTerm"       # 上下文各3行
git grep -B 2 -A 2 "searchTerm"  # 前2行后2行

# git grep vs grep
#   git grep: 只搜索 Git 追踪的文件，更快
#   grep:     搜索所有文件
#   git grep 可以搜索历史版本中的代码
```

## 11. 文件操作命令？

**回答：**

```bash
# 移动/重命名文件
git mv old_name.txt new_name.txt
# 等价于:
# mv old_name.txt new_name.txt
# git add new_name.txt
# git rm old_name.txt

# 删除文件
git rm file.txt                  # 从工作区和暂存区删除
git rm --cached file.txt         # 只从暂存区删除（保留工作区文件）
git rm -r --cached dir/          # 递归删除目录（取消追踪）
git rm -f file.txt               # 强制删除（已修改或已暂存的文件）

# 查看追踪的文件
git ls-files                     # 已追踪的文件
git ls-files -o                  # 未追踪的文件
git ls-files -i --exclude-standard  # 被忽略的文件

# 假设某文件不变（不再跟踪修改）
git update-index --assume-unchanged file.txt
# 恢复
git update-index --no-assume-unchanged file.txt

# 跳过工作区（类似，但更适合本地配置文件）
git update-index --skip-worktree file.txt
git update-index --no-skip-worktree file.txt
```
