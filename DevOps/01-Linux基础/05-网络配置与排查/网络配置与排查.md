### 2. TCP 三次握手和四次挥手？（详细版）

**答：**

```
三次握手 (Three-Way Handshake):
┌──────────┐                              ┌──────────┐
│  Client   │                              │  Server   │
│  CLOSED   │                              │  LISTEN   │
└────┬─────┘                              └────┬─────┘
     │                                         │
     │  1. SYN (seq=x)                         │
     │ ──────────────────────────────────────→  │  Client → SYN_SENT
     │                                         │
     │  2. SYN+ACK (seq=y, ack=x+1)           │
     │ ←──────────────────────────────────────  │  Server → SYN_RCVD
     │                                         │
     │  3. ACK (ack=y+1)                       │
     │ ──────────────────────────────────────→  │
     │                                         │
     │       ✅ 连接建立 (ESTABLISHED)          │
     │         可以开始传输数据                   │

为什么是三次？
- 两次不够：Server 无法确认 Client 收到了 SYN+ACK
- 防止已失效的连接请求突然到达 Server，导致资源浪费
- 双方确认各自的发送和接收能力

四次挥手 (Four-Way Handshake):
┌──────────┐                              ┌──────────┐
│  Client   │                              │  Server   │
│ESTABLISHED│                              │ESTABLISHED│
└────┬─────┘                              └────┬─────┘
     │                                         │
     │  1. FIN (seq=u)                         │
     │ ──────────────────────────────────────→  │  Client → FIN_WAIT_1
     │                                         │
     │  2. ACK (ack=u+1)                       │
     │ ←──────────────────────────────────────  │  Server → CLOSE_WAIT
     │       Client → FIN_WAIT_2               │  (Server 可能还有数据要发)
     │                                         │
     │  3. FIN (seq=w)                         │
     │ ←──────────────────────────────────────  │  Server → LAST_ACK
     │                                         │
     │  4. ACK (ack=w+1)                       │
     │ ──────────────────────────────────────→  │  Server → CLOSED
     │       Client → TIME_WAIT                │
     │       (等待 2MSL 后关闭)                 │
     │       Client → CLOSED                   │

为什么是四次？
- TCP 全双工，每个方向独立关闭
- Server 收到 FIN 后可能还有数据未发完，先 ACK 再发 FIN

TIME_WAIT:
- 持续 2MSL (Maximum Segment Lifetime)，Linux 默认 60 秒
- 确保最后一个 ACK 到达 Server
- 防止旧连接的延迟数据包干扰新连接
```

### 3. TCP vs UDP 的区别？

**答：**

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接（三次握手） | 无连接 |
| 可靠性 | 可靠（确认、重传、排序） | 不可靠（尽力交付） |
| 流量控制 | 有（滑动窗口） | 无 |
| 拥塞控制 | 有（慢启动、拥塞避免等） | 无 |
| 数据边界 | 字节流（无边界） | 数据报（有边界） |
| 传输效率 | 较低（头部开销大） | 高（头部8字节） |
| 适用场景 | HTTP, SSH, FTP, 数据库 | DNS, DHCP, 视频流, 游戏 |

## 二、网络配置

### 5. IP 地址配置（临时和永久）？

**答：**

```bash
# ===== 查看网络信息 =====
ip addr show                   # 所有接口 IP
ip addr show eth0              # 特定接口
ip -4 addr                     # 只看 IPv4
ip -6 addr                     # 只看 IPv6
ip link show                   # 链路状态
hostname -I                    # 简单查看 IP

# ===== 临时配置 =====
# 添加 IP
ip addr add 192.168.1.100/24 dev eth0
# 删除 IP
ip addr del 192.168.1.100/24 dev eth0
# 启用/禁用接口
ip link set eth0 up
ip link set eth0 down

# ===== 永久配置 - Ubuntu/Debian (Netplan) =====
# /etc/netplan/01-netcfg.yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: false
      addresses:
        - 192.168.1.100/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]

# netplan apply    # 应用配置

# ===== 永久配置 - RHEL/CentOS (NetworkManager) =====
# /etc/sysconfig/network-scripts/ifcfg-eth0
TYPE=Ethernet
BOOTPROTO=static
IPADDR=192.168.1.100
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
DNS1=8.8.8.8
DNS2=8.8.4.4
ONBOOT=yes

# systemctl restart NetworkManager
# 或使用 nmcli
nmcli connection modify eth0 ipv4.addresses 192.168.1.100/24
nmcli connection modify eth0 ipv4.gateway 192.168.1.1
nmcli connection modify eth0 ipv4.dns "8.8.8.8 8.8.4.4"
nmcli connection modify eth0 ipv4.method manual
nmcli connection up eth0
```

## 三、网络排查工具

### 8. 网络连通性排查流程？

**答：**

```
排查流程:
1. 检查本机网络配置
   ip addr show / ip route show

2. 检查本机连通性
   ping 127.0.0.1              # 回环
   ping <本机IP>               # 本机网卡

3. 检查网关连通性
   ping <网关IP>               # 默认网关
   
4. 检查外网连通性
   ping 8.8.8.8                # 外网 IP

5. 检查 DNS 解析
   ping example.com            # 域名
   dig example.com             # DNS 查询

6. 检查端口可达
   telnet <host> <port>
   nc -zv <host> <port>
   curl -v http://<host>:<port>

7. 路由追踪
   traceroute <host>
   mtr <host>

8. 抓包分析
   tcpdump -i eth0 host <target>
```

### 9. 网络监控和诊断命令大全？

**答：**

```bash
# ===== 连通性 =====
ping -c 4 8.8.8.8                    # ping 4次
ping -i 0.2 8.8.8.8                  # 间隔0.2秒
ping -s 1472 -M do 8.8.8.8          # 测试 MTU（1472+28=1500）

# ===== 路由追踪 =====
traceroute 8.8.8.8                    # UDP 模式（默认）
traceroute -T 8.8.8.8                 # TCP 模式
traceroute -I 8.8.8.8                 # ICMP 模式
mtr 8.8.8.8                          # 综合 ping+traceroute（持续监控）
mtr -r -c 100 8.8.8.8                # 报告模式，100次

# ===== 端口和连接 =====
ss -tulnp                             # TCP/UDP 监听端口
# -t TCP  -u UDP  -l 监听  -n 数字  -p 进程

ss -s                                  # 连接统计摘要
ss -tn state established              # 已建立的 TCP 连接
ss -tn state time-wait | wc -l        # TIME_WAIT 数量
ss -tn dst :80                         # 目标端口 80 的连接
ss -tn src :22                         # 源端口 22 的连接

netstat -tunlp                         # 等价于 ss -tulnp
lsof -i :80                           # 使用 80 端口的进程
lsof -i -P -n                         # 所有网络连接
fuser 80/tcp                          # 使用端口的进程 PID

# ===== 网络流量 =====
iftop -i eth0                          # 接口实时流量（按连接）
nload eth0                             # 接口实时流量（带宽图）
nethogs                                # 按进程查看流量
bmon                                   # 带宽监控
sar -n DEV 1                           # 网卡收发统计

# ===== HTTP 调试 =====
curl -v https://example.com            # 详细输出
curl -s -o /dev/null -w "\
  HTTP Code: %{http_code}\n\
  DNS Time: %{time_namelookup}s\n\
  Connect Time: %{time_connect}s\n\
  TLS Time: %{time_appconnect}s\n\
  Total Time: %{time_total}s\n" \
  https://example.com                  # 性能分析

curl -I https://example.com            # 只看响应头
curl --connect-timeout 5 http://host   # 连接超时
curl -x http://proxy:8080 http://host  # 通过代理

wget --spider https://example.com      # 检查 URL 是否可达
```

### 10. tcpdump 抓包详解？

**答：**

```bash
# 基本用法
tcpdump -i eth0                         # 抓 eth0 上的所有包
tcpdump -i any                          # 抓所有接口
tcpdump -i eth0 -c 100                  # 只抓 100 个包
tcpdump -i eth0 -w capture.pcap         # 保存到文件
tcpdump -r capture.pcap                 # 读取文件

# 过滤表达式
tcpdump -i eth0 host 192.168.1.100             # 指定主机
tcpdump -i eth0 src host 192.168.1.100         # 源主机
tcpdump -i eth0 dst host 192.168.1.100         # 目标主机
tcpdump -i eth0 port 80                        # 指定端口
tcpdump -i eth0 src port 80                    # 源端口
tcpdump -i eth0 dst port 443                   # 目标端口
tcpdump -i eth0 net 192.168.1.0/24             # 网段
tcpdump -i eth0 tcp                            # 只看 TCP
tcpdump -i eth0 udp                            # 只看 UDP
tcpdump -i eth0 icmp                           # 只看 ICMP

# 组合过滤
tcpdump -i eth0 'host 192.168.1.100 and port 80'
tcpdump -i eth0 'src 192.168.1.100 and (dst port 80 or dst port 443)'
tcpdump -i eth0 'tcp[tcpflags] & tcp-syn != 0'    # 只看 SYN 包
tcpdump -i eth0 'tcp[tcpflags] & tcp-rst != 0'    # 只看 RST 包

# 输出格式
tcpdump -i eth0 -n                      # 不解析域名
tcpdump -i eth0 -nn                     # 不解析域名和端口名
tcpdump -i eth0 -X                      # 显示包内容（hex+ASCII）
tcpdump -i eth0 -A                      # 只显示 ASCII
tcpdump -i eth0 -v                      # 详细输出
tcpdump -i eth0 -vvv                    # 最详细

# 实用场景
# 抓 DNS 查询
tcpdump -i eth0 port 53

# 抓 HTTP 请求
tcpdump -i eth0 -A -s 0 'tcp port 80 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)'

# 抓 SYN 洪水
tcpdump -i eth0 'tcp[tcpflags] == tcp-syn' -c 1000
```

## 四、网络内核参数调优

### 11. 常见网络内核参数及调优？

**答：**

```bash
# /etc/sysctl.d/99-network.conf

# ===== TCP 连接优化 =====
# SYN 队列
net.ipv4.tcp_max_syn_backlog = 65535      # SYN 半连接队列大小
net.core.somaxconn = 65535                 # 全连接队列大小（listen backlog）

# TIME_WAIT 优化
net.ipv4.tcp_tw_reuse = 1                 # 允许 TIME_WAIT 复用
net.ipv4.tcp_fin_timeout = 15             # FIN_WAIT_2 超时（默认60）
net.ipv4.tcp_max_tw_buckets = 65535       # TIME_WAIT 最大数量

# keepalive
net.ipv4.tcp_keepalive_time = 600         # 空闲连接保活探测间隔（默认7200）
net.ipv4.tcp_keepalive_intvl = 30         # 探测间隔
net.ipv4.tcp_keepalive_probes = 3         # 探测次数

# 端口范围
net.ipv4.ip_local_port_range = 1024 65535 # 本地端口范围

# ===== 缓冲区优化 =====
net.core.rmem_max = 16777216              # 接收缓冲区最大值 (16MB)
net.core.wmem_max = 16777216              # 发送缓冲区最大值
net.core.rmem_default = 262144            # 接收缓冲区默认值
net.core.wmem_default = 262144            # 发送缓冲区默认值
net.ipv4.tcp_rmem = 4096 87380 16777216   # TCP 接收缓冲 (min default max)
net.ipv4.tcp_wmem = 4096 65536 16777216   # TCP 发送缓冲

# ===== 安全 =====
net.ipv4.tcp_syncookies = 1               # 防 SYN 洪水攻击
net.ipv4.conf.all.rp_filter = 1           # 反向路径过滤（防 IP 欺骗）
net.ipv4.icmp_echo_ignore_broadcasts = 1  # 忽略广播 ping
net.ipv4.conf.all.accept_redirects = 0    # 不接受 ICMP 重定向
net.ipv4.conf.all.send_redirects = 0      # 不发送 ICMP 重定向
net.ipv4.ip_forward = 0                   # 禁止 IP 转发（除非做路由/NAT）

# 设备队列
net.core.netdev_max_backlog = 65535       # 网卡接收队列

# 应用
sysctl -p /etc/sysctl.d/99-network.conf
```

### 12. TIME_WAIT 过多如何处理？

**答：**

```bash
# 查看 TIME_WAIT 数量
ss -s
ss -tn state time-wait | wc -l

# TIME_WAIT 过多的原因
# - 短连接高并发（每次请求新建/断开连接）
# - 客户端主动关闭连接
# - 连接池未使用或配置不当

# 解决方案
# 1. 使用连接池/长连接（根本方案）
#    Nginx: keepalive
#    数据库: 连接池
#    HTTP: Keep-Alive

# 2. 内核参数调优
net.ipv4.tcp_tw_reuse = 1        # 允许复用 TIME_WAIT
net.ipv4.tcp_fin_timeout = 15    # 缩短 FIN_WAIT_2 超时
net.ipv4.tcp_max_tw_buckets = 20000  # 限制最大数量

# 3. 扩大端口范围
net.ipv4.ip_local_port_range = 1024 65535

# ⚠️ 注意
# 不要使用 tcp_tw_recycle（已从 Linux 4.12 移除，NAT 环境下有问题）
```

### 13. Linux 网桥和 VLAN 配置？

**答：**

```bash
# ===== 网桥 (Bridge) =====

ip link set eth0.100 up

# 查看 VLAN
cat /proc/net/vlan/config

# 删除 VLAN
ip link delete eth0.100
```

### 14. Linux 网络命名空间 (Network Namespace)？

**答：**

网络命名空间提供完全隔离的网络栈（接口、路由、iptables 等），Docker 底层使用此特性。

```bash
# 创建命名空间
ip netns add ns1
ip netns add ns2

# 列出命名空间
ip netns list

# 在命名空间中执行命令
ip netns exec ns1 ip addr show
ip netns exec ns1 bash    # 进入命名空间的 shell

# 创建 veth pair（虚拟以太网对）并连接两个命名空间
ip link add veth0 type veth peer name veth1
ip link set veth0 netns ns1
ip link set veth1 netns ns2

ip netns exec ns1 ip addr add 10.0.0.1/24 dev veth0
ip netns exec ns1 ip link set veth0 up
ip netns exec ns1 ip link set lo up

ip netns exec ns2 ip addr add 10.0.0.2/24 dev veth1
ip netns exec ns2 ip link set veth1 up
ip netns exec ns2 ip link set lo up

# 测试连通性
ip netns exec ns1 ping 10.0.0.2

# 这就是 Docker 容器网络的基本原理：
# 每个容器一个 netns + veth pair + bridge
```

## 六、高级网络场景

### 15. Linux 作为路由器/NAT 网关如何配置？

**答：**

```bash
# 1. 启用 IP 转发
echo 1 > /proc/sys/net/ipv4/ip_forward           # 临时
echo "net.ipv4.ip_forward = 1" >> /etc/sysctl.d/99-forward.conf  # 永久
sysctl -p

# 2. SNAT（源地址转换 - 内网访问外网）
iptables -t nat -A POSTROUTING -s 10.0.0.0/8 -o eth0 -j MASQUERADE
# MASQUERADE 自动使用出口的 IP

# 3. DNAT（目标地址转换 - 外网访问内网服务）
iptables -t nat -A PREROUTING -i eth0 -p tcp --dport 80 \
  -j DNAT --to-destination 10.0.0.10:8080

# 4. 端口转发
iptables -t nat -A PREROUTING -p tcp --dport 2222 \
  -j DNAT --to-destination 10.0.0.20:22

# 允许转发
iptables -A FORWARD -s 10.0.0.0/8 -j ACCEPT
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT
```

### 16. 网络Bond（链路聚合）和Team配置？

**答：**

```bash
# Bond（链路聚合）- 多网卡绑定
# 模式:
# mode 0 (balance-rr)     轮询
# mode 1 (active-backup)  主备切换（最常用）
# mode 2 (balance-xor)    XOR 负载
# mode 4 (802.3ad)        LACP（需交换机支持）
# mode 5 (balance-tlb)    自适应发送负载
# mode 6 (balance-alb)    自适应收发负载

# 配置 Bond（nmcli 方式）
```
