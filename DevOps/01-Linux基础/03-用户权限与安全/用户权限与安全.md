## 一、用户与组管理

### 1. Linux 用户类型有哪些？

**答：**

| 类型 | UID 范围 | 说明 |
|------|---------|------|
| root | 0 | 超级管理员，权限无限制 |
| 系统用户 | 1-999 (Debian) / 1-499 (RHEL) | 服务专用账户（nginx, mysql 等），通常不可登录 |
| 普通用户 | ≥1000 (Debian) / ≥500 (RHEL) | 正常使用者 |
| nobody | 65534 | 最低权限用户，NFS/服务降权使用 |

### 2. /etc/passwd、/etc/shadow、/etc/group 详解？

**答：**

```bash
# /etc/passwd（所有人可读）
# 格式: 用户名:密码占位:UID:GID:描述:家目录:登录Shell
root:x:0:0:root:/root:/bin/bash
nginx:x:101:101:nginx user:/var/lib/nginx:/usr/sbin/nologin
zhangsan:x:1000:1000:Zhang San:/home/zhangsan:/bin/bash

# /etc/shadow（仅 root 可读，存储密码哈希）
# 格式: 用户名:密码哈希:最后修改:最小间隔:最大有效期:警告天数:失效天数:过期日期:保留
root:$6$salt$hash...:19000:0:99999:7:::
zhangsan:$y$j9T$xx...:19500:0:90:7:30::

# 密码哈希算法标识
# $1$  → MD5        (不安全，已弃用)
# $5$  → SHA-256
# $6$  → SHA-512    (推荐)
# $y$  → yescrypt   (最新，Ubuntu 22.04+ 默认)
# !!   → 账号锁定/无密码
# *    → 不可通过密码登录

# /etc/group
# 格式: 组名:密码:GID:组成员列表
root:x:0:
docker:x:998:zhangsan,lisi
devops:x:1001:zhangsan

# /etc/gshadow（组密码，较少使用）
```

### 3. 用户和组的完整管理命令？

**答：**

```bash
# ===== 用户管理 =====
# 创建用户
useradd -m -s /bin/bash -c "Zhang San" -G docker,sudo zhangsan
# -m  创建家目录
# -M  不创建家目录
# -s  指定登录 Shell
# -c  注释/描述
# -G  附加组（逗号分隔）
# -g  主组
# -u  指定 UID
# -d  指定家目录路径
# -e  账户过期日期 (YYYY-MM-DD)
# -r  创建系统用户（不创建家目录、UID < 1000）

# 创建系统用户（服务专用）
useradd -r -s /usr/sbin/nologin -d /var/lib/myapp myapp

# 修改用户
usermod -aG docker zhangsan    # 添加到附加组（-a 追加，不加会覆盖！）
usermod -s /bin/zsh zhangsan   # 更改 Shell
usermod -L zhangsan            # 锁定账户
usermod -U zhangsan            # 解锁账户
usermod -e 2026-12-31 zhangsan # 设置过期日期

# 删除用户
userdel zhangsan               # 删除用户（保留家目录）
userdel -r zhangsan            # 删除用户和家目录

# 查看用户信息
id zhangsan                    # UID, GID, 组
whoami                         # 当前用户
who / w                        # 在线用户
last                           # 登录历史
lastlog                        # 所有用户最后登录
finger zhangsan                # 详细用户信息

# ===== 密码管理 =====
passwd zhangsan                # 修改密码
passwd -l zhangsan             # 锁定密码
passwd -u zhangsan             # 解锁密码
passwd -e zhangsan             # 强制下次登录改密码
chage -l zhangsan              # 查看密码策略
chage -M 90 zhangsan           # 密码最大有效期 90 天
chage -m 7 zhangsan            # 密码最少使用 7 天
chage -W 14 zhangsan           # 过期前 14 天提醒

# ===== 组管理 =====
groupadd devops                # 创建组
groupadd -g 2000 devops        # 指定 GID
groupmod -n newname oldname    # 重命名组
groupdel devops                # 删除组
gpasswd -a zhangsan devops     # 添加用户到组
gpasswd -d zhangsan devops     # 从组中移除用户
gpasswd -A zhangsan devops     # 设置组管理员
groups zhangsan                # 查看用户所属的组
```

## 二、文件权限

### 4. UGO 权限模型详解？

**答：**

```
文件权限解读:
-rwxr-xr-- 1 zhangsan devops 4096 Mar 10 12:00 deploy.sh

│ │││ │││ │││  │      │        │
│ │││ │││ │││  │      │        └── 文件大小
│ │││ │││ │││  │      └─────────── 所属组
│ │││ │││ │││  └────────────────── 所有者
│ │││ │││ └┤└── Other: r-- (4)  只读
│ │││ └┤└───── Group: r-x (5)  读+执行
│ └┤└──────── User:  rwx (7)   读+写+执行
└──────────── 文件类型: - 普通文件

权限含义（文件 vs 目录）:
┌──────┬─────────────────────────────┬───────────────────────────┐
│ 权限 │         文件                │        目录               │
├──────┼─────────────────────────────┼───────────────────────────┤
│  r   │ 可以读取文件内容            │ 可以列出目录内容 (ls)     │
│  w   │ 可以修改文件内容            │ 可以创建/删除目录中的文件  │
│  x   │ 可以执行文件               │ 可以进入目录 (cd)         │
└──────┴─────────────────────────────┴───────────────────────────┘
```

```bash
# 数字方式设置权限
chmod 755 deploy.sh     # rwxr-xr-x
chmod 644 config.yaml   # rw-r--r--
chmod 600 id_rsa        # rw------- (SSH 私钥)
chmod 700 .ssh          # rwx------ (SSH 目录)
chmod 777 file          # ❌ 绝不要给 777（安全隐患）

# 符号方式设置权限
chmod u+x script.sh     # 给 User 加执行权限
chmod g-w file          # 去掉 Group 写权限
chmod o=r file          # 设置 Other 为只读
chmod a+r file          # 所有人加读权限
chmod u=rwx,g=rx,o=r file  # 等价于 754

# 递归修改
chmod -R 755 /var/www    # 递归修改目录

# 修改所有者/组
chown zhangsan file              # 改所有者
chown zhangsan:devops file       # 改所有者和组
chown :devops file               # 只改组
chgrp devops file                # 只改组（另一种方式）
chown -R www-data:www-data /var/www  # 递归
```

### 5. SUID、SGID 和 Sticky Bit 深入解析？

**答：**

**SUID (Set User ID)：**
```bash
# 效果：文件执行时以文件所有者身份运行（而非执行者身份）
# 典型例子：passwd 命令

ls -l /usr/bin/passwd
# -rwsr-xr-x 1 root root ... /usr/bin/passwd
#    ^                         虽然普通用户执行，但以 root 身份运行
#    s = SUID + x              因为需要修改 /etc/shadow（只有 root 可写）
#    S = SUID 但没有 x

chmod u+s file         # 设置 SUID
chmod 4755 file        # 数字方式（4 = SUID）

# 安全风险：SUID 的 root 程序如有漏洞，可被提权
# 查找所有 SUID 文件（安全审计）
find / -perm -u=s -type f 2>/dev/null
```

**SGID (Set Group ID)：**
```bash
# 文件：执行时以文件所属组身份运行
# 目录：在目录中新建的文件自动继承目录的组（团队协作用）

# 目录 SGID 示例
mkdir -p /shared/project
chgrp devops /shared/project
chmod 2775 /shared/project      # 2 = SGID

# 此后 devops 组成员在此目录创建的文件都属于 devops 组
# 而不是创建者的主组

chmod g+s dir          # 设置 SGID
```

**Sticky Bit：**
```bash
# 效果：目录中的文件只能由文件所有者或 root 删除
# 典型例子：/tmp 目录

ls -ld /tmp
# drwxrwxrwt 24 root root ... /tmp
#          ^    t = Sticky Bit + x
#               T = Sticky Bit 但没有 x

# 虽然 /tmp 是 777（所有人可写），但 A 不能删除 B 的文件

chmod +t dir           # 设置 Sticky Bit
chmod 1777 dir         # 数字方式（1 = Sticky Bit）
```

### 6. umask 是什么？如何设置？

**答：**

umask 定义新创建文件/目录的默认权限掩码。

```
计算方式:
  新文件权限 = 0666 - umask
  新目录权限 = 0777 - umask

示例 (umask = 0022):
  新文件: 0666 - 0022 = 0644 (rw-r--r--)
  新目录: 0777 - 0022 = 0755 (rwxr-xr-x)

示例 (umask = 0077):
  新文件: 0666 - 0077 = 0600 (rw-------)
  新目录: 0777 - 0077 = 0700 (rwx------)
```

```bash
umask                  # 查看当前 umask
umask 0027             # 临时设置（新文件 640，新目录 750）

# 永久设置
echo "umask 0027" >> ~/.bashrc              # 当前用户
echo "umask 0027" >> /etc/profile.d/umask.sh # 所有用户

# 安全建议
# 普通用户: umask 0022 (默认)
# 高安全系统: umask 0077 (文件只有自己能访问)
```

## 三、ACL 访问控制列表

### 7. ACL 是什么？如何使用？

**答：**

ACL (Access Control List) 提供比 UGO 更细粒度的权限控制，可以为特定用户或组单独设置权限。

```bash
# 查看 ACL
getfacl file.txt
# # file: file.txt
# # owner: zhangsan
# # group: devops
# user::rw-
# user:lisi:r--          ← 给 lisi 用户只读权限
# group::r--
# group:qa:rw-           ← 给 qa 组读写权限
# mask::rw-
# other::---

# 设置 ACL
setfacl -m u:lisi:r file.txt          # 给用户 lisi 读权限
setfacl -m u:wangwu:rw file.txt       # 给用户 wangwu 读写
setfacl -m g:qa:rw file.txt           # 给 qa 组读写
setfacl -m o::--- file.txt            # 其他人无权限
setfacl -m d:g:devops:rw dir/         # 目录默认 ACL（新文件继承）

# 删除 ACL
setfacl -x u:lisi file.txt            # 删除特定 ACL 条目
setfacl -b file.txt                   # 删除所有 ACL

# 递归设置
setfacl -R -m g:devops:rwX dir/       # X = 仅目录和可执行文件加 x

# ACL mask
setfacl -m m::r file.txt              # mask 限制所有 ACL 的最大权限

# 有 ACL 的文件在 ls -l 中权限后面有 + 号
# -rw-r--r--+ 1 zhangsan devops 0 file.txt
```

## 四、sudo 与特权管理

### 8. sudo 配置详解？

**答：**

```bash
# 编辑 sudoers 文件（必须用 visudo！语法错误会锁死 sudo）
visudo

# ===== 语法 =====
# 用户/组  主机=(身份:身份)  [选项] 命令列表

# 基本配置
root          ALL=(ALL:ALL)  ALL          # root 完全权限
%sudo         ALL=(ALL:ALL)  ALL          # sudo 组完全权限
%devops       ALL=(ALL)      NOPASSWD: ALL  # devops 组免密 sudo

# 限制命令（最小权限原则）
deploy        ALL=(root) /usr/bin/systemctl restart nginx, \
                         /usr/bin/systemctl reload nginx, \
                         /usr/bin/docker, \
                         /opt/deploy/scripts/*.sh
# deploy 用户只能以 root 身份执行指定的命令

# 命令别名（分组管理）
Cmnd_Alias  NETWORKING = /sbin/ifconfig, /sbin/route, /sbin/iptables
Cmnd_Alias  SERVICES   = /usr/bin/systemctl, /usr/sbin/service
Cmnd_Alias  DANGEROUS  = /usr/bin/rm, /usr/bin/dd, /usr/sbin/fdisk

# 禁止执行危险命令
%devops       ALL=(ALL) ALL, !DANGEROUS

# 日志记录
Defaults      logfile="/var/log/sudo.log"
Defaults      log_input, log_output       # 记录输入输出

# 安全选项
Defaults      timestamp_timeout=5          # sudo 缓存时间（分钟）
Defaults      passwd_tries=3               # 密码尝试次数
Defaults      requiretty                   # 需要终端
Defaults      secure_path="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin"
```

```bash
# sudoers 片段文件（推荐方式）
# /etc/sudoers.d/deploy
deploy ALL=(root) NOPASSWD: /usr/bin/systemctl restart nginx
deploy ALL=(root) NOPASSWD: /opt/deploy/run.sh

# /etc/sudoers.d/monitoring
nagios ALL=(root) NOPASSWD: /usr/lib/nagios/plugins/*
```

### 9. su 和 sudo 的区别及安全实践？

**答：**

| 特性 | su | sudo |
|------|-----|------|
| 全称 | Switch User / Substitute User | Superuser Do |
| 密码 | 目标用户的密码 | 当前用户自己的密码 |
| 权限范围 | 切换到完整用户环境 | 仅执行单条命令（可配置） |
| 审计日志 | 仅记录切换行为 | 记录每条执行的命令 |
| 安全性 | 需要共享 root 密码 | 无需知道 root 密码 |

```bash
# su 用法
su -              # 切换到 root（login shell，完整环境）
su - zhangsan     # 切换到 zhangsan
su -c "command"   # 以 root 身份执行单条命令

# sudo 用法
sudo command              # 以 root 执行
sudo -u zhangsan command  # 以 zhangsan 身份执行
sudo -i                   # 切换到 root（类似 su -）
sudo -s                   # 获取 root shell（保持当前环境）
sudo -l                   # 查看当前用户的 sudo 权限
sudo -k                   # 清除 sudo 密码缓存
```

**安全最佳实践：**
- 禁用 root SSH 登录：`PermitRootLogin no`
- 使用 sudo 替代 su，所有管理操作走 sudo
- sudoers 遵循最小权限原则
- 启用 sudo 审计日志
- 使用 `sudoers.d/` 片段管理，不直接修改 `/etc/sudoers`

## 五、PAM 认证框架

### 10. PAM 是什么？如何配置？

**答：**

PAM (Pluggable Authentication Modules) 是 Linux 的可插拔认证模块框架，提供统一的认证接口。

```
应用程序:  login, sshd, sudo, su, passwd ...
    ↓
PAM 框架:  /etc/pam.d/ (每个应用一个配置文件)
    ↓
PAM 模块:  /lib/security/ (pam_unix.so, pam_ldap.so ...)
```

```bash
# PAM 配置文件格式
# 类型       控制标记    模块路径          参数
# type       control    module-path      arguments

# /etc/pam.d/sshd 示例
auth       required     pam_sepermit.so         # SELinux 检查
auth       required     pam_env.so              # 设置环境变量
auth       required     pam_faillock.so preauth # 登录失败锁定（前检查）
auth       sufficient   pam_unix.so             # Unix 密码认证
auth       required     pam_faillock.so authfail # 登录失败计数
auth       required     pam_deny.so             # 默认拒绝

account    required     pam_unix.so             # 账户检查
account    required     pam_nologin.so          # 检查 /etc/nologin

password   required     pam_pwquality.so retry=3  # 密码质量检查
password   sufficient   pam_unix.so sha512 shadow # 密码更新

session    required     pam_limits.so           # 资源限制
session    required     pam_unix.so             # 会话日志
```

**PAM 类型：**
| 类型 | 说明 |
|------|------|
| auth | 认证用户身份 |
| account | 账户检查（过期、锁定等） |
| password | 密码修改 |
| session | 会话管理（登录/登出时执行） |

**PAM 控制标记：**
| 标记 | 说明 |
|------|------|
| required | 必须成功，失败后继续检查其他模块再返回失败 |
| requisite | 必须成功，失败立即返回 |
| sufficient | 成功则立即通过（前面的 required 都通过的情况下） |
| optional | 可选，不影响最终结果 |

### 11. 密码策略和账户锁定如何配置？

**答：**

```bash
# ===== 密码复杂度 (pam_pwquality) =====
# /etc/security/pwquality.conf
minlen = 12          # 最小长度
dcredit = -1         # 至少1个数字
ucredit = -1         # 至少1个大写字母
lcredit = -1         # 至少1个小写字母
ocredit = -1         # 至少1个特殊字符
maxrepeat = 3        # 最多连续重复3个字符
reject_username      # 不允许包含用户名
enforce_for_root     # root 也受限制

# ===== 密码有效期 =====
# /etc/login.defs
PASS_MAX_DAYS   90   # 密码最大有效天数
PASS_MIN_DAYS   7    # 密码最短使用天数
PASS_WARN_AGE   14   # 过期前提醒天数
PASS_MIN_LEN    12   # 最小长度

# 对已有用户设置
chage -M 90 -m 7 -W 14 zhangsan

# ===== 登录失败锁定 (pam_faillock) =====
# /etc/security/faillock.conf
deny = 5             # 5次失败后锁定
unlock_time = 600    # 锁定 600 秒（10分钟）
fail_interval = 900  # 在 900 秒窗口内计数

# 查看/解锁
faillock --user zhangsan          # 查看失败记录
faillock --user zhangsan --reset  # 解锁
```

## 六、SSH 安全

### 12. SSH 安全加固配置？

**答：**

```bash
# /etc/ssh/sshd_config

# 基本安全
PermitRootLogin no                   # 禁止 root 登录
PasswordAuthentication no            # 禁止密码登录（只用密钥）
PubkeyAuthentication yes             # 启用公钥认证
PermitEmptyPasswords no              # 禁止空密码
ChallengeResponseAuthentication no   # 禁止交互式认证

# 访问限制
AllowUsers zhangsan lisi             # 只允许特定用户
AllowGroups devops sre               # 只允许特定组
MaxAuthTries 3                       # 最大认证尝试次数
MaxSessions 5                        # 最大并发会话
LoginGraceTime 30                    # 登录超时（秒）

# 网络安全
Port 22222                           # 修改默认端口
ListenAddress 10.0.0.1               # 只监听内网
Protocol 2                           # 仅使用 SSH v2

# 加密算法（禁用弱算法）
KexAlgorithms curve25519-sha256@libssh.org,ecdh-sha2-nistp521
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com

# 超时踢出
ClientAliveInterval 300              # 每 300 秒发心跳
ClientAliveCountMax 2                # 2 次无响应断开

# 日志级别
LogLevel VERBOSE                     # 详细日志

# 转发限制
AllowTcpForwarding no                # 禁止端口转发
X11Forwarding no                     # 禁止 X11 转发
AllowAgentForwarding no              # 禁止 Agent 转发

# 限制 SFTP
Subsystem sftp internal-sftp
Match Group sftponly
    ChrootDirectory /data/sftp/%u
    ForceCommand internal-sftp
    AllowTcpForwarding no
```

### 13. SSH 密钥管理？

**答：**

```bash
# 生成密钥对
ssh-keygen -t ed25519 -C "zhangsan@company.com"  # 推荐 Ed25519
ssh-keygen -t rsa -b 4096 -C "zhangsan@company.com"  # RSA 4096

# 密钥类型对比
# Ed25519: 最推荐，安全性高，密钥短，速度快
# RSA:     兼容性好，需要至少 3072 位
# ECDSA:   中等，某些环境不支持
# DSA:     已弃用

# 复制公钥到远程服务器
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@host

# 等价于
cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys

# 权限要求（必须严格）
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519          # 私钥
chmod 644 ~/.ssh/id_ed25519.pub      # 公钥
chmod 600 ~/.ssh/authorized_keys     # 授权文件
chmod 644 ~/.ssh/config              # 配置文件

# SSH 配置文件 ~/.ssh/config
Host prod-*
    User deploy
    IdentityFile ~/.ssh/id_ed25519_prod
    StrictHostKeyChecking yes

Host jump
    HostName 10.0.0.1
    User admin
    Port 22222

Host internal-*.example.com
    ProxyJump jump              # 通过跳板机连接
    User deploy

# SSH Agent（免重复输入密码）
eval $(ssh-agent -s)
ssh-add ~/.ssh/id_ed25519
ssh-add -l                     # 列出已加载的密钥
```

## 七、SELinux / AppArmor

### 14. SELinux 和 AppArmor 的区别和使用？

**答：**

| 特性 | SELinux | AppArmor |
|------|---------|----------|
| 默认发行版 | RHEL/CentOS/Fedora | Ubuntu/SUSE |
| 控制粒度 | 基于标签（细粒度） | 基于路径（简单） |
| 配置复杂度 | 高 | 低 |
| 学习曲线 | 陡峭 | 平缓 |

```bash
# ===== SELinux =====
# 查看状态
getenforce                    # Enforcing / Permissive / Disabled

# 模式切换
setenforce 0                  # 临时切换为 Permissive

# 永久修改 /etc/selinux/config
SELINUX=enforcing             # enforcing / permissive / disabled

# 查看文件上下文
ls -Z /var/www/html/
# -rw-r--r--. root root system_u:object_r:httpd_sys_content_t:s0 index.html

# 修改上下文
chcon -R -t httpd_sys_content_t /var/www/html/     # 临时
semanage fcontext -a -t httpd_sys_content_t "/data/www(/.*)?"  # 永久
restorecon -Rv /data/www                            # 应用

# 排查 SELinux 阻止的操作
audit2why < /var/log/audit/audit.log
sealert -a /var/log/audit/audit.log

# ===== AppArmor =====
# 查看状态
aa-status                     # 查看所有 profile 状态

# 模式
aa-enforce /etc/apparmor.d/usr.sbin.nginx   # 强制模式
aa-complain /etc/apparmor.d/usr.sbin.nginx  # 投诉模式（只记录）

# 查看日志
journalctl | grep apparmor
```

### 15. iptables 四表五链详解？

**答：**

```
数据包流向:

                  ┌─────────┐
  入站数据包 ──→  │ PREROUTING│ ──→ 路由判断
                  └─────────┘       │
                                    ├── 目标是本机
                                    │      ↓
                                    │  ┌────────┐
                                    │  │ INPUT   │ ──→ 本地进程
                                    │  └────────┘
                                    │
                                    └── 目标是其他主机（转发）
                                           ↓
                                       ┌─────────┐
                                       │ FORWARD  │
                                       └────┬────┘
                                            ↓
  本地进程 ──→ ┌────────┐           ┌──────────┐
               │ OUTPUT  │ ──→      │POSTROUTING│ ──→ 出站数据包
               └────────┘           └──────────┘
```

**四表：**

| 表 | 优先级 | 说明 |
|---|-------|------|
| raw | 最高 | 连接追踪前处理 |
| mangle | 高 | 修改数据包（TTL、TOS等） |
| nat | 中 | 地址转换（SNAT、DNAT、MASQUERADE） |
| filter | 低 | 包过滤（默认表，ACCEPT/DROP/REJECT） |

```bash
# 常用 iptables 命令
# 查看规则
iptables -L -n -v                      # 查看 filter 表
iptables -t nat -L -n -v               # 查看 nat 表

# 添加规则
iptables -A INPUT -p tcp --dport 22 -j ACCEPT    # 允许 SSH
iptables -A INPUT -p tcp --dport 80 -j ACCEPT    # 允许 HTTP
iptables -A INPUT -p tcp --dport 443 -j ACCEPT   # 允许 HTTPS
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT  # 已建立的连接
iptables -A INPUT -i lo -j ACCEPT                # 允许回环
iptables -P INPUT DROP                           # 默认拒绝入站

# 限制连接频率（防暴力破解）
iptables -A INPUT -p tcp --dport 22 -m connlimit --connlimit-above 3 -j DROP
iptables -A INPUT -p tcp --dport 22 -m recent --set --name SSH
iptables -A INPUT -p tcp --dport 22 -m recent --update --seconds 60 --hitcount 4 --name SSH -j DROP

# NAT 转发
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE     # 源 NAT
iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to 192.168.1.10:8080  # 目标 NAT

# 保存和恢复
iptables-save > /etc/iptables/rules.v4
iptables-restore < /etc/iptables/rules.v4
```

### 16. firewalld 的 Zone 和 Service 概念？

**答：**

```bash
# Zone（区域）: 按信任级别划分的网络区域
# 预定义 Zone:
# drop     → 丢弃所有入站，不回复
# block    → 拒绝所有入站，回复 ICMP 拒绝
# public   → 默认，仅允许选定服务
# external → 外网接口，开启 NAT
# dmz      → DMZ 区域
# work     → 工作网络
# home     → 家庭网络
# internal → 内部网络
# trusted  → 信任所有连接

# 查看
firewall-cmd --get-active-zones
firewall-cmd --get-default-zone
firewall-cmd --list-all

# 管理规则
firewall-cmd --zone=public --add-service=http --permanent
firewall-cmd --zone=public --add-service=https --permanent
firewall-cmd --zone=public --add-port=8080/tcp --permanent
firewall-cmd --zone=public --add-source=10.0.0.0/8 --permanent
firewall-cmd --zone=public --add-rich-rule='rule family=ipv4 source address=192.168.1.0/24 port port=3306 protocol=tcp accept' --permanent

# 端口转发
firewall-cmd --zone=public --add-forward-port=port=80:proto=tcp:toport=8080 --permanent

# 应用
firewall-cmd --reload

# 紧急模式
firewall-cmd --panic-on      # 拒绝所有连接
firewall-cmd --panic-off     # 恢复
```
