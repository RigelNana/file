## 一、文件系统基础

### 1. Linux 的目录结构及各目录作用？（FHS 标准）

**答：** Linux 遵循 FHS (Filesystem Hierarchy Standard) 标准：

```
/                     根目录，整个文件系统的起点
├── /bin              基本用户命令（ls, cp, mv, cat, grep 等）
├── /sbin             系统管理命令（fdisk, iptables, mount 等）
├── /etc              系统配置文件
│   ├── /etc/fstab            文件系统挂载表
│   ├── /etc/hosts             主机名解析
│   ├── /etc/passwd            用户信息
│   ├── /etc/shadow            用户密码哈希
│   ├── /etc/group             组信息
│   ├── /etc/sudoers           sudo 配置
│   ├── /etc/ssh/              SSH 配置
│   ├── /etc/nginx/            Nginx 配置
│   ├── /etc/systemd/system/   自定义 systemd unit
│   ├── /etc/sysctl.conf       内核参数配置
│   ├── /etc/resolv.conf       DNS 配置
│   └── /etc/crontab           系统级定时任务
├── /home             普通用户家目录 (/home/username)
├── /root             root 用户家目录
├── /var              可变数据文件
│   ├── /var/log               系统日志
│   ├── /var/lib               应用状态数据 (MySQL, Docker 等)
│   ├── /var/cache             缓存数据
│   ├── /var/spool             队列数据 (邮件、打印、cron)
│   ├── /var/tmp               重启后保留的临时文件
│   └── /var/run → /run        运行时数据（PID 文件等）
├── /tmp              临时文件（所有用户可写，Sticky Bit）
├── /usr              用户程序和只读数据（第二层级目录结构）
│   ├── /usr/bin               用户命令
│   ├── /usr/sbin              系统管理命令
│   ├── /usr/lib               库文件
│   ├── /usr/local/            手动安装的软件（./configure; make install）
│   ├── /usr/share/            架构无关的数据（文档、man pages）
│   └── /usr/include/          C/C++ 头文件
├── /opt              第三方可选软件 (Oracle, Google Chrome)
├── /dev              设备文件
│   ├── /dev/sda               SCSI/SATA 磁盘
│   ├── /dev/nvme0n1           NVMe SSD
│   ├── /dev/null              黑洞（丢弃输出）
│   ├── /dev/zero              零流（用于清空/创建文件）
│   ├── /dev/random            随机数（阻塞式）
│   ├── /dev/urandom           随机数（非阻塞式）
│   ├── /dev/tty               当前终端
│   └── /dev/loop*             回环设备
├── /proc             进程和内核信息的虚拟文件系统
│   ├── /proc/cpuinfo          CPU 信息
│   ├── /proc/meminfo          内存信息
│   ├── /proc/loadavg          系统负载
│   ├── /proc/[PID]/           进程信息
│   ├── /proc/sys/             可调内核参数
│   └── /proc/net/             网络统计
├── /sys              硬件和驱动信息的虚拟文件系统（sysfs）
├── /run              启动后的运行时数据（tmpfs，重启清空）
├── /mnt              临时手动挂载点
├── /media            可移动设备自动挂载点 (USB, CD)
├── /boot             引导文件
│   ├── vmlinuz-*              内核镜像
│   ├── initrd.img-*           初始 RAM 磁盘
│   └── grub/                  GRUB 配置
├── /lib              共享库文件
│   └── /lib/modules/          内核模块
└── /srv              服务数据 (HTTP, FTP 提供的文件)
```

### 2. 软链接和硬链接有什么区别？（深入版）

**答：**

```
硬链接原理:
┌────────────┐     ┌──────────────────────┐
│ 目录项      │     │     inode #12345     │
│ file_a ──────────→│ 权限: -rw-r--r--      │
│            │     │ 大小: 1024 bytes      │
│ file_b ──────────→│ 链接计数: 2           │
│            │     │ 数据块指针 → [实际数据] │
└────────────┘     └──────────────────────┘

软链接原理:
┌────────────┐     ┌─────────────────┐     ┌──────────────────────┐
│ 目录项      │     │ inode #99999    │     │     inode #12345     │
│ link_s ──────────→│ 类型: symlink   │     │ 权限: -rw-r--r--      │
│            │     │ 内容: /path/to/ │     │ 数据块指针 → [实际数据] │
│            │     │   file_a        │──→  │                      │
└────────────┘     └─────────────────┘     └──────────────────────┘
```

| 特性      | 硬链接 (Hard Link)  | 软链接 (Symbolic Link) |
| ------- | ---------------- | ------------------- |
| 创建命令    | `ln source link` | `ln -s source link` |
| inode   | 共享同一个 inode      | 有独立的 inode          |
| 跨文件系统   | ❌ 不能             | ✅ 能                 |
| 链接目录    | ❌ 不能（防循环引用）      | ✅ 能                 |
| 源文件删除   | 仍然可以访问（链接计数>0）   | 变成悬空链接（失效）          |
| 文件大小    | 与源文件相同           | 路径字符串长度             |
| 文件权限    | 与源文件相同（共享 inode） | 自身权限无意义，跟随目标        |
| 效率      | 直接访问，无额外开销       | 需要解析一次路径            |
| 相对/绝对路径 | 不涉及路径            | 支持相对和绝对路径           |

```bash
# 创建示例
echo "hello" > original.txt
ln original.txt hard_link.txt        # 硬链接
ln -s original.txt soft_link.txt     # 软链接

# 验证 inode
ls -li original.txt hard_link.txt soft_link.txt
# 12345 -rw-r--r-- 2 user group 6 original.txt      ← 同一个 inode, 链接数=2
# 12345 -rw-r--r-- 2 user group 6 hard_link.txt      ← 同一个 inode
# 99999 lrwxrwxrwx 1 user group 12 soft_link.txt -> original.txt  ← 不同 inode

# 删除源文件
rm original.txt
cat hard_link.txt     # ✅ 仍可访问
cat soft_link.txt     # ❌ 报错：No such file or directory

# 查找所有硬链接
find / -inum 12345    # 按 inode 号查找
```

### 3. inode 详解？inode 耗尽怎么办？

**答：**

**inode 包含的元数据：**
- 文件类型（普通文件、目录、软链接、设备等）
- 权限（rwxrwxrwx）
- 所有者 UID 和 GID
- 文件大小
- 时间戳：atime（访问时间）、mtime（修改时间）、ctime（属性变更时间）
- 硬链接计数
- 数据块指针（直接指针 × 12 + 一级间接 + 二级间接 + 三级间接）

```bash
# 查看 inode 使用情况
df -i
# Filesystem      Inodes  IUsed   IFree IUse% Mounted on
# /dev/sda1      6553600 235000 6318600    4% /

# inode 耗尽的表现
# "No space left on device" 但 df -h 显示仍有磁盘空间

# 排查：找出 inode 使用最多的目录
find / -xdev -type f | cut -d "/" -f 2-3 | sort | uniq -c | sort -rn | head

# 常见原因：大量小文件（如邮件队列、临时文件、session 文件）

# 解决方案：
# 1. 清理小文件
find /tmp -type f -mtime +7 -delete
find /var/spool/mail -type f -size 0 -delete

# 2. 创建文件系统时指定更多 inode
mkfs.ext4 -N 10000000 /dev/sdb1   # 指定 inode 数量
mkfs.ext4 -i 1024 /dev/sdb1        # 每1KB创建一个inode（更多inode）
```

## 二、文件系统类型

### 4. 常见 Linux 文件系统对比？

**答：**

| 文件系统 | 最大文件 | 最大卷 | 日志 | 快照 | 特点 | 适用场景 |
|---------|---------|-------|------|------|------|---------|
| **ext4** | 16TB | 1EB | ✅ | ❌ | 成熟稳定，向后兼容 | 通用 Linux 系统 |
| **XFS** | 8EB | 8EB | ✅ | ❌ | 高性能大文件 I/O，并行友好 | 数据库、大文件 |
| **Btrfs** | 16EB | 16EB | ✅ | ✅ | COW、快照、透明压缩、子卷 | 需要快照/压缩的场景 |
| **ZFS** | 16EB | 256ZB | ✅ | ✅ | 数据完整性、RAID-Z、去重 | NAS、存储服务器 |
| **tmpfs** | - | 内存大小 | ❌ | ❌ | 基于内存，极快 | /tmp, /run |
| **overlay** | - | - | ❌ | ❌ | 联合挂载，分层叠加 | Docker 容器 |
| **NFS** | 取决于后端 | - | - | - | 网络文件系统 | 共享存储 |

```bash
# 查看当前文件系统类型
df -Th
mount | grep "^/dev"
lsblk -f

# 创建文件系统
mkfs.ext4 /dev/sdb1
mkfs.xfs /dev/sdb1
mkfs.btrfs /dev/sdb1
```

### 5. 什么是日志文件系统（Journaling）？

**答：**

日志文件系统在写入数据前先记录操作日志（Journal），以保证断电或崩溃后的数据一致性。

```
写入流程:
1. 将操作记录到 Journal 区域
2. 将数据写入实际位置
3. 标记 Journal 条目完成

崩溃恢复:
├── Journal 完整 → 重放 Journal，恢复数据
└── Journal 不完整 → 丢弃未完成的事务
```

**ext4 的三种日志模式：**

| 模式 | journal 记录内容 | 安全性 | 性能 |
|------|-----------------|--------|------|
| `journal` | 元数据 + 数据 | 最高 | 最低 |
| `ordered` (默认) | 仅元数据，但保证数据先写入 | 高 | 中 |
| `writeback` | 仅元数据，不保证数据写入顺序 | 低 | 最高 |

```bash
# 查看当前日志模式
tune2fs -l /dev/sda1 | grep "Journal"

# 修改 fstab 中的日志模式
/dev/sda1 / ext4 data=journal 0 1
```

## 三、磁盘管理

### 6. 磁盘分区操作详解？

**答：**

```bash
# 查看磁盘信息
lsblk                         # 列出块设备树
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT  # 自定义列
fdisk -l                      # 列出分区详情
blkid                         # 查看 UUID 和文件系统类型
parted -l                     # GPT 分区查看

# MBR 分区 (fdisk)
fdisk /dev/sdb
# 交互命令: n(新建), d(删除), p(打印), t(改类型), w(写入), q(退出)

# GPT 分区 (gdisk 或 parted)
gdisk /dev/sdb
# 类似 fdisk 的交互命令

# parted（支持 MBR 和 GPT）
parted /dev/sdb
  mklabel gpt                 # 创建 GPT 分区表
  mkpart primary ext4 1MiB 50GiB    # 创建分区
  print                       # 打印分区表
  quit

# 创建文件系统
mkfs.ext4 /dev/sdb1
mkfs.xfs -f /dev/sdb1         # -f 强制

# 获取 UUID
blkid /dev/sdb1
# /dev/sdb1: UUID="abc-123-..." TYPE="ext4"
```

### 7. /etc/fstab 文件详解？

**答：**

```bash
# /etc/fstab 格式（6个字段）
# <设备>     <挂载点>  <文件系统>  <选项>        <dump>  <fsck顺序>

# 使用 UUID（推荐，设备名可能变）
UUID=abc-123  /          ext4    defaults           0       1
UUID=def-456  /home      ext4    defaults           0       2
UUID=ghi-789  /boot/efi  vfat    umask=0077         0       1
UUID=jkl-012  none       swap    sw                 0       0

# 使用 LABEL
LABEL=DATA    /data      xfs     defaults,noatime   0       2

# NFS 挂载
192.168.1.10:/share  /mnt/nfs  nfs  defaults,_netdev  0  0

# tmpfs
tmpfs         /tmp       tmpfs   defaults,noatime,size=2G  0  0
```

**常用挂载选项：**

| 选项 | 说明 |
|------|------|
| `defaults` | rw, suid, dev, exec, auto, nouser, async |
| `noatime` | 不更新访问时间（提升性能） |
| `nodiratime` | 不更新目录访问时间 |
| `noexec` | 不允许执行二进制（安全） |
| `nosuid` | 忽略 SUID/SGID（安全） |
| `nodev` | 不允许设备文件 |
| `ro` | 只读挂载 |
| `_netdev` | 网络设备，网络就绪后才挂载 |
| `discard` | 启用 TRIM（SSD） |
| `nofail` | 设备不存在不影响启动 |

```bash
# 测试 fstab 是否正确（不重启验证）
mount -a                    # 挂载 fstab 中所有未挂载的条目
findmnt --verify            # 验证 fstab 语法
```

## 四、LVM 逻辑卷管理

### 8. LVM 的架构和工作原理？

**答：**

```
物理层:
  /dev/sdb          /dev/sdc          /dev/sdd
     │                 │                 │
     ▼                 ▼                 ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│  PV (物理卷)│    │  PV (物理卷)│    │  PV (物理卷)│
│  100 GB    │    │  200 GB    │    │  150 GB    │
└──────┬─────┘    └─────┬──────┘    └─────┬──────┘
       │                │                  │
       └────────────────┼──────────────────┘
                        ▼
              ┌──────────────────┐
              │    VG (卷组)      │
              │   共 450 GB       │
              │                  │
              │  ┌──────┐ ┌──────┐
              │  │LV_data│ │LV_log│
              │  │200 GB │ │ 50 GB│
              │  │ ext4  │ │ xfs  │
              │  └──────┘ └──────┘
              │  剩余 200GB 未分配  │
              └──────────────────┘
```

**核心概念：**

| 概念 | 全称 | 说明 |
|------|------|------|
| PV | Physical Volume | 物理磁盘或分区 |
| VG | Volume Group | 多个 PV 组成的存储池 |
| LV | Logical Volume | 从 VG 中划分的逻辑分区 |
| PE | Physical Extent | PV 中的最小分配单元（默认 4MB） |
| LE | Logical Extent | LV 中的最小分配单元，映射到 PE |

### 9. LVM 常用操作命令？

**答：**

```bash
# ===== PV 物理卷 =====
pvcreate /dev/sdb /dev/sdc        # 创建 PV
pvs                                # 简要信息
pvdisplay                          # 详细信息
pvremove /dev/sdb                  # 移除 PV

# ===== VG 卷组 =====
vgcreate myvg /dev/sdb /dev/sdc   # 创建 VG
vgs                                # 简要信息
vgdisplay myvg                    # 详细信息
vgextend myvg /dev/sdd            # 添加 PV 到 VG（扩容）
vgreduce myvg /dev/sdc            # 从 VG 移除 PV（需先迁移数据）

# ===== LV 逻辑卷 =====
lvcreate -L 100G -n lv_data myvg     # 创建 100G 的 LV
lvcreate -l 100%FREE -n lv_data myvg # 使用所有空闲空间
lvcreate -L 50G -n lv_data myvg -m 1 # 创建带镜像的 LV
lvs                                   # 简要信息
lvdisplay                             # 详细信息

# ===== 格式化和挂载 =====
mkfs.ext4 /dev/myvg/lv_data
mkdir -p /data
mount /dev/myvg/lv_data /data
echo "/dev/myvg/lv_data /data ext4 defaults 0 2" >> /etc/fstab

# ===== 在线扩容 =====
# 1. 扩展 LV
lvextend -L +50G /dev/myvg/lv_data    # 增加 50G
# 或
lvextend -l +100%FREE /dev/myvg/lv_data  # 使用所有空闲空间

# 2. 扩展文件系统
resize2fs /dev/myvg/lv_data               # ext4（支持在线扩展）
xfs_growfs /data                           # XFS（按挂载点扩展）

# ===== 缩小 LV（ext4，需离线操作！XFS 不支持缩小）=====
umount /data
e2fsck -f /dev/myvg/lv_data               # 先检查文件系统
resize2fs /dev/myvg/lv_data 80G           # 缩小文件系统到 80G
lvreduce -L 80G /dev/myvg/lv_data         # 缩小 LV 到 80G
mount /dev/myvg/lv_data /data

# ===== LVM 快照 =====
lvcreate -L 10G -s -n snap_data /dev/myvg/lv_data  # 创建快照
mount /dev/myvg/snap_data /mnt/snapshot -o ro       # 挂载快照
lvremove /dev/myvg/snap_data                        # 删除快照

# ===== 数据迁移（在线）=====
pvmove /dev/sdb /dev/sdd              # 将 sdb 上的数据迁移到 sdd
```

## 五、RAID 磁盘阵列

### 10. RAID 各级别详解？

**答：**

```
RAID 0 (条带化):
  磁盘A:  [A1][A3][A5]
  磁盘B:  [A2][A4][A6]
  → 数据分散写入，速度翻倍，无冗余

RAID 1 (镜像):
  磁盘A:  [A1][A2][A3]  ← 完全相同
  磁盘B:  [A1][A2][A3]  ← 完全相同
  → 数据完全复制，容量减半，可靠性高

RAID 5 (分布式校验):
  磁盘A:  [A1][A2][Cp]
  磁盘B:  [A3][Bp][A4]
  磁盘C:  [Ap][A5][A6]
  → 校验分布在各盘，允许坏1块盘

RAID 6 (双校验):
  → 类似 RAID 5，但有两份校验
  → 允许同时坏2块盘

RAID 10 (1+0):
  磁盘A:  [A1][A2]  ←镜像→  磁盘B: [A1][A2]
  磁盘C:  [A3][A4]  ←镜像→  磁盘D: [A3][A4]
         └── 条带化 ──────────┘
  → 先镜像再条带，性能和可靠性兼顾
```

| RAID | 最少盘 | 容量 | 读性能 | 写性能 | 容错 | 适用 |
|------|-------|------|--------|--------|------|------|
| 0 | 2 | N | 极高 | 极高 | 无 | 临时数据、缓存 |
| 1 | 2 | N/2 | 高 | 中 | 1块盘 | 系统盘、日志 |
| 5 | 3 | N-1 | 高 | 中-低 | 1块盘 | 通用存储 |
| 6 | 4 | N-2 | 高 | 低 | 2块盘 | 高可靠存储 |
| 10 | 4 | N/2 | 极高 | 高 | 每组1块 | 数据库、高性能 |

```bash
# 软件 RAID (mdadm)
mdadm --create /dev/md0 --level=5 --raid-devices=3 /dev/sd[bcd]1
mdadm --detail /dev/md0         # 查看阵列状态
cat /proc/mdstat                # 查看 RAID 状态
mdadm --manage /dev/md0 --add /dev/sde1  # 添加热备盘

# 模拟磁盘故障
mdadm --manage /dev/md0 --fail /dev/sdb1
mdadm --manage /dev/md0 --remove /dev/sdb1
```

## 六、Swap 交换空间

### 11. Swap 的作用和管理？

**答：**

Swap 是磁盘上用于临时存放内存中不活跃数据的空间，作为内存的补充。

```bash
# 查看 Swap
free -h
swapon --show
cat /proc/swaps

# 创建 Swap 分区
mkswap /dev/sdb2
swapon /dev/sdb2
echo "UUID=xxx swap swap defaults 0 0" >> /etc/fstab

# 创建 Swap 文件
fallocate -l 4G /swapfile          # 创建4G文件
chmod 600 /swapfile                # 设置权限
mkswap /swapfile                   # 格式化为 Swap
swapon /swapfile                   # 启用
echo "/swapfile swap swap defaults 0 0" >> /etc/fstab

# 关闭 Swap（K8s 要求）
swapoff -a                         # 临时关闭所有
sed -i '/swap/d' /etc/fstab        # 永久关闭

# 调整 Swap 使用倾向
sysctl vm.swappiness               # 查看当前值（默认60）
sysctl -w vm.swappiness=10         # 减少 Swap 使用（0-100）
# vm.swappiness=0   尽量不使用 Swap（Linux 3.5+ 仍可能在紧急时使用）
# vm.swappiness=100 积极使用 Swap
```

**Swap 大小建议：**

| 物理内存 | 建议 Swap |
|---------|----------|
| ≤ 2 GB | 2 × 内存 |
| 2-8 GB | 等于内存 |
| 8-64 GB | 至少 4 GB |
| > 64 GB | 至少 4 GB（或不用） |

## 七、特殊文件系统

### 12. /proc 虚拟文件系统详解？

**答：**

```bash
# 系统信息
/proc/cpuinfo           # CPU 详细信息
/proc/meminfo           # 内存详细信息
/proc/loadavg           # 系统负载（1/5/15分钟平均）
/proc/uptime            # 运行时间
```

## 八、磁盘 I/O 与性能

### 14. 如何排查磁盘 I/O 问题？

**答：**

```bash
# 1. 查看磁盘空间
df -h                        # 文件系统使用情况
df -i                        # inode 使用情况

# 2. 找出大文件/目录
du -sh /* 2>/dev/null | sort -rh | head
du -sh /var/log/* | sort -rh | head
find / -xdev -type f -size +100M -exec ls -lh {} \;

# 3. I/O 性能监控
iostat -xz 1                 # 磁盘 I/O 详细统计
# 重点关注: %util(忙碌度), await(等待时间), r/s w/s(IOPS)

iotop -oP                    # 按进程查看 I/O（只显示有 I/O 的）

# 4. I/O 等待分析
vmstat 1
# 关注: wa (I/O wait CPU 百分比)

# 5. 磁盘健康检查
smartctl -a /dev/sda          # SMART 信息
smartctl -t short /dev/sda    # 运行短测试

# 6. 磁盘基准测试
# 顺序写测试
dd if=/dev/zero of=/tmp/test bs=1M count=1024 oflag=direct
# 顺序读测试
dd if=/tmp/test of=/dev/null bs=1M iflag=direct
# 更专业的测试工具
fio --name=randread --ioengine=libaio --rw=randread --bs=4k \
    --numjobs=4 --size=1G --runtime=60 --direct=1 --group_reporting
```
