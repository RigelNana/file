# 进程管理

---

## 1. 进程的状态与生命周期？

**回答：**

```
进程五态模型：
  ┌─────────────────────────────────────────────┐
  │                                              │
  │  新建 ──→ 就绪 ──→ 运行 ──→ 终止            │
  │           ↑  ↓      │                        │
  │           └──┘      ↓                        │
  │           时间片    阻塞 ──→ 就绪             │
  │           到期      (等待IO)  (IO完成)        │
  └─────────────────────────────────────────────┘

Linux 进程状态：
  ┌──────────────┬──────────────────────────────┐
  │ 状态          │ 说明                          │
  ├──────────────┼──────────────────────────────┤
  │ R (Running)  │ 运行/就绪                     │
  │ S (Sleeping) │ 可中断睡眠（等待事件）          │
  │ D (Disk)     │ 不可中断睡眠（等待 IO）        │
  │ T (Stopped)  │ 已停止（SIGSTOP）              │
  │ Z (Zombie)   │ 僵尸进程（已退出，父进程未回收）│
  │ X (Dead)     │ 已销毁                        │
  └──────────────┴──────────────────────────────┘

僵尸进程：
  子进程退出，父进程未调用 wait() 回收
  → 占用 PID 和进程表项
  解决：父进程处理 SIGCHLD / init 进程接管

孤儿进程：
  父进程先退出，子进程被 init(PID 1) 收养
  → 无害，init 会自动回收
```

## 2. 进程和线程的区别？

**回答：**

```
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │ 进程              │ 线程              │
  ├──────────────┼──────────────────┼──────────────────┤
  │ 定义          │ 资源分配基本单位  │ CPU 调度基本单位  │
  │ 地址空间      │ 独立             │ 共享进程地址空间   │
  │ 资源          │ 独立（文件/内存） │ 共享（堆/全局变量）│
  │ 私有          │ 全部             │ 栈/寄存器/PC      │
  │ 创建开销      │ 大               │ 小                │
  │ 上下文切换    │ 慢（需切换页表）  │ 快                │
  │ 通信          │ IPC（开销大）     │ 共享内存（直接）   │
  │ 安全性        │ 进程隔离          │ 一个崩溃全体崩溃   │
  └──────────────┴──────────────────┴──────────────────┘

Linux 实现：
  进程和线程底层都是 task_struct
  线程 = 共享地址空间的轻量级进程
  clone() 系统调用可控制共享标志
```

## 3. 进程间通信（IPC）方式？

**回答：**

```
  ┌──────────────┬──────────────────────────────────┐
  │ 方式          │ 特点                              │
  ├──────────────┼──────────────────────────────────┤
  │ 管道 (pipe)   │ 半双工，父子进程，内核缓冲区      │
  │ 命名管道 FIFO│ 无亲缘关系，文件系统可见          │
  │ 消息队列      │ 有格式消息，异步，内核中转        │
  │ 共享内存      │ 最快，直接读写，需同步机制        │
  │ 信号量        │ 计数器，P/V 操作，进程同步        │
  │ 信号          │ 异步通知（SIGKILL/SIGTERM 等）   │
  │ Socket        │ 跨主机，TCP/UDP/Unix Domain      │
  └──────────────┴──────────────────────────────────┘

共享内存 + 信号量组合最常用：
  共享内存：高性能数据交换
  信号量：控制并发访问

Unix Domain Socket：
  同主机进程通信，比 TCP Socket 快（无网络协议栈）
  Docker、Nginx 进程间通信常用
```

## 4. fork() 的原理？写时复制？

**回答：**

```
fork()：创建子进程，子进程是父进程的副本

  父进程 fork() → 返回子进程 PID
  子进程       → 返回 0

写时复制 (Copy-on-Write, COW)：
  fork() 不立即复制内存
  父子共享相同物理页，标记为只读
  任一方写入时 → 触发缺页异常 → 复制该页

  ┌────────────────────────────────────────┐
  │ fork() 前:                             │
  │   父进程 → [页A] [页B] [页C]           │
  │                                        │
  │ fork() 后 (COW):                       │
  │   父进程 → [页A] [页B] [页C] ← 子进程  │
  │            (共享，标记只读)              │
  │                                        │
  │ 子进程写页B:                            │
  │   父进程 → [页A] [页B] [页C]           │
  │   子进程 → [页A] [页B'] [页C]          │
  │             共享   复制   共享          │
  └────────────────────────────────────────┘

优点：
  fork() 极快（只复制页表）
  exec() 替换进程映像时不浪费复制
```

## 5. 进程调度算法？

**回答：**

```
  ┌──────────────────┬──────────────────────────────┐
  │ 算法              │ 特点                          │
  ├──────────────────┼──────────────────────────────┤
  │ FCFS (先来先服务) │ 简单，护航效应（长进程阻塞短）│
  │ SJF (最短作业)    │ 平均等待最小，难预估时间       │
  │ 优先级调度        │ 高优先级先运行，可能饥饿       │
  │ RR (时间片轮转)   │ 公平，时间片大小影响性能       │
  │ 多级反馈队列      │ 综合方案，动态调整优先级       │
  │ CFS (完全公平)    │ Linux 默认，红黑树按虚拟时间   │
  └──────────────────┴──────────────────────────────┘

Linux CFS 调度器：
  每个进程维护 vruntime（虚拟运行时间）
  vruntime 最小的进程优先执行
  红黑树组织，O(log n) 选取
  权重越高的进程 vruntime 增长越慢 → 获得更多 CPU
```

## 6. 上下文切换的过程和开销？

**回答：**

```
上下文切换：CPU 从一个进程/线程切换到另一个

过程：
  1. 保存当前进程的 CPU 寄存器到 PCB
  2. 更新当前进程状态（运行→就绪/阻塞）
  3. 选择下一个进程（调度算法）
  4. 恢复下一个进程的寄存器
  5. 切换页表（进程切换）/ 刷新 TLB
  6. 跳转到新进程的 PC 继续执行

开销：
  ┌──────────────────┬──────────────────┐
  │ 开销来源          │ 影响              │
  ├──────────────────┼──────────────────┤
  │ 寄存器保存/恢复   │ 直接开销          │
  │ 页表切换          │ 进程切换才有      │
  │ TLB 刷新          │ 后续访存变慢      │
  │ Cache 失效        │ 冷启动效应        │
  │ 内核态切换        │ 用户态→内核态     │
  └──────────────────┴──────────────────┘

  进程切换 ≈ 数微秒
  线程切换 ≈ 数百纳秒（无需切换页表）
  协程切换 ≈ 数十纳秒（用户态，仅切换少量寄存器）
```

## 7. 线程模型（1:1 / M:N）？

**回答：**

```
  ┌──────────┬──────────────────────────────────┐
  │ 模型      │ 说明                              │
  ├──────────┼──────────────────────────────────┤
  │ 1:1      │ 一个用户线程对应一个内核线程       │
  │          │ Linux pthread / Java              │
  │          │ 优：可并行 / 缺：创建开销大        │
  │          │                                   │
  │ M:1      │ 多个用户线程映射一个内核线程       │
  │          │ 优：轻量 / 缺：一个阻塞全部阻塞   │
  │          │                                   │
  │ M:N      │ M 个用户线程映射 N 个内核线程      │
  │          │ Go goroutine (GMP 模型)           │
  │          │ 优：轻量+可并行 / 缺：实现复杂     │
  └──────────┴──────────────────────────────────┘

Go GMP 模型：
  G (Goroutine): 用户态协程，2KB 栈
  M (Machine):   OS 线程
  P (Processor): 逻辑处理器，默认 GOMAXPROCS 个

  P 持有本地队列，M 绑定 P 执行 G
  G 阻塞时 M 释放 P，P 绑定新 M 继续执行
  → 百万级并发
```

## 8. 信号机制？

**回答：**

```
信号：进程间异步通知机制

常用信号：
  ┌──────────┬──────┬──────────────────────────┐
  │ 信号      │ 编号  │ 说明                      │
  ├──────────┼──────┼──────────────────────────┤
  │ SIGHUP   │ 1    │ 终端挂起，常用于重载配置   │
  │ SIGINT   │ 2    │ Ctrl+C 中断              │
  │ SIGQUIT  │ 3    │ Ctrl+\ 退出 + core dump  │
  │ SIGKILL  │ 9    │ 强制终止（不可捕获）       │
  │ SIGSEGV  │ 11   │ 段错误                    │
  │ SIGTERM  │ 15   │ 正常终止（可捕获）         │
  │ SIGCHLD  │ 17   │ 子进程状态变化             │
  │ SIGSTOP  │ 19   │ 停止进程（不可捕获）       │
  └──────────┴──────┴──────────────────────────┘

信号处理方式：
  默认处理：终止/忽略/停止
  自定义处理：signal() / sigaction() 注册处理函数
  SIGKILL 和 SIGSTOP 不可捕获

优雅退出：
  捕获 SIGTERM → 清理资源 → 关闭连接 → 退出
```

## 9. 容器进程隔离原理？

**回答：**

```
Docker 容器 = Namespace 隔离 + Cgroups 限制 + 文件系统

Namespace（隔离）：
  ┌──────────────┬──────────────────────────────┐
  │ Namespace    │ 隔离内容                      │
  ├──────────────┼──────────────────────────────┤
  │ PID          │ 进程 ID（容器内 PID 1）       │
  │ NET          │ 网络栈（独立网卡/IP/端口）    │
  │ MNT          │ 文件系统挂载点                │
  │ UTS          │ 主机名/域名                   │
  │ IPC          │ 进程间通信（信号量/共享内存）  │
  │ USER         │ 用户/组 ID 映射               │
  └──────────────┴──────────────────────────────┘

Cgroups（资源限制）：
  CPU：限制 CPU 使用量/绑核
  内存：限制最大使用且 OOM Kill
  IO：限制磁盘读写速率
  网络：限制带宽

vs 虚拟机：
  容器：共享内核，Namespace 隔离，秒级启动
  虚拟机：独立内核，Hypervisor 隔离，分钟级启动
```

## 10. 进程管理面试速答？

**回答：**

```
Q: 进程和线程区别？
A: 进程是资源分配单位（独立地址空间）
   线程是 CPU 调度单位（共享进程资源）
   线程切换比进程快（无需切换页表）

Q: 僵尸进程和孤儿进程？
A: 僵尸：子进程退出，父进程未 wait() 回收 → 占 PID
   孤儿：父进程先退出，init 收养 → 无害

Q: fork() 做了什么？
A: 创建子进程，写时复制（COW）共享物理页
   父返回子 PID，子返回 0

Q: 进程间通信方式？
A: 管道/消息队列/共享内存/信号量/Signal/Socket
   共享内存最快，Socket 跨网络

Q: Linux 默认调度算法？
A: CFS（完全公平调度）
   红黑树按 vruntime 排序，最小的先执行

Q: 上下文切换开销？
A: 进程切换 ~数微秒（保存寄存器+切换页表+刷TLB）
   线程切换 ~数百纳秒
   协程切换 ~数十纳秒（用户态）

Q: 容器靠什么隔离进程？
A: Namespace 隔离（PID/NET/MNT/UTS/IPC/USER）
   Cgroups 限制资源（CPU/内存/IO）

Q: Go goroutine 为什么高效？
A: M:N 模型，对应 GMP 调度
   2KB 初始栈，用户态切换
   百万级并发
```
