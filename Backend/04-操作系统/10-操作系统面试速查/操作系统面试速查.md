# 操作系统面试速查

---

## 1. 进程与线程核心问题？

**回答：**

```
Q: 进程和线程区别？
A: 进程是资源分配单位，线程是 CPU 调度单位
   线程共享进程地址空间，创建/切换开销小
   进程间隔离，一个崩溃不影响其他

Q: 协程和线程区别？
A: 协程用户态调度，无内核切换开销
   线程内核态调度，可利用多核
   Go goroutine = 协程 + GMP 调度器 + 多核

Q: 进程状态有哪些？
A: 创建→就绪→运行→阻塞→终止
   Linux: R(运行) S(睡眠) D(不可中断) Z(僵尸) T(暂停)

Q: 孤儿进程和僵尸进程？
A: 孤儿：父进程退出，被 init 收养（无害）
   僵尸：子进程退出但父进程未 wait，占PCB（有害）
   解决僵尸：父进程 wait / 杀父进程 / SIGCHLD

Q: 上下文切换开销？
A: 保存/恢复寄存器 + 切换内核栈 + 刷新 TLB
   约 3-5 微秒
   进程切换 > 线程切换 > 协程切换
```

## 2. 调度算法核心问题？

**回答：**

```
Q: 常见调度算法？
A: FCFS（先来先服务）→ 简单但平均等待长
   SJF（短作业优先）→ 最优但可能饥饿
   RR（轮转）→ 公平但上下文切换多
   优先级 → 灵活但可能饥饿
   多级反馈队列 → 综合方案

Q: Linux 用什么调度？
A: CFS（完全公平调度器）
   红黑树按虚拟运行时间排序
   虚拟时间 = 实际时间 / 权重
   优先级低 → 虚拟时间增长快 → 少运行

Q: Go GMP 模型？
A: G=goroutine M=OS线程 P=逻辑处理器
   P 数量 = GOMAXPROCS（默认 CPU 核数）
   工作窃取：空闲 P 从其他 P 偷 G
   hand off：M 阻塞时 P 绑定新 M
```

## 3. 内存管理核心问题？

**回答：**

```
Q: 虚拟内存原理？
A: 每个进程有独立虚拟地址空间
   MMU + 页表 将虚拟地址映射到物理地址
   实现隔离 + 超额分配 + 按需加载

Q: 页面置换算法？
A: OPT（最优）→ 理论最优不可实现
   LRU（最近最少使用）→ 实际常用
   Clock（时钟）→ LRU 近似，Linux 实际用
   FIFO（先进先出）→ 简单但有 Belady 异常

Q: 进程内存布局？
A: 高地址 → 低地址：
   内核空间 | 栈(向下长) | mmap/共享库
   堆(向上长) | BSS(未初始化) | Data(已初始化)
   Text(代码段)

Q: malloc 原理？
A: <128KB：brk() 扩展堆（glibc 分配器管理）
   ≥128KB：mmap() 匿名映射
   glibc ptmalloc2：多 Arena + bins 分级管理

Q: 内存泄漏怎么排查？
A: Go：pprof heap profile
   C：Valgrind / AddressSanitizer
   Linux：/proc/PID/smaps 观察 RSS 增长
```

## 4. IO 模型核心问题？

**回答：**

```
Q: 五种 IO 模型？
A: 阻塞IO：read 阻塞等待数据
   非阻塞IO：轮询检查 → 忙等浪费 CPU
   IO多路复用：select/poll/epoll 监控多个 fd
   信号驱动IO：注册信号，数据就绪通知
   异步IO：aio/io_uring，内核完成后通知

Q: select/poll/epoll 区别？
A: select：1024 fd 限制，每次全量拷贝和遍历
   poll：无数量限制，仍然全量遍历
   epoll：事件驱动，只返回就绪 fd，O(1)
   epoll 用红黑树管理 fd，就绪链表通知

Q: epoll ET vs LT？
A: LT（水平触发）：就绪就通知，简单
   ET（边缘触发）：状态变化才通知，高性能
   ET 必须非阻塞 + 一次读完

Q: 零拷贝？
A: 传统：磁盘→内核buf→用户buf→Socket buf→网卡(4次拷贝)
   sendfile：磁盘→内核buf→(DMA)网卡(2次拷贝)
   mmap+write：减少一次用户态拷贝
   场景：Nginx 静态文件/Kafka 消息推送
```

## 5. 锁与同步核心问题？

**回答：**

```
Q: 互斥锁和自旋锁区别？
A: 互斥锁：获取不到则睡眠，适合长临界区
   自旋锁：获取不到则忙等，适合短临界区
   Go sync.Mutex 先自旋再睡眠（混合策略）

Q: 死锁四个条件？
A: 互斥、持有并等待、不可抢占、循环等待
   打破任一即可避免

Q: CAS 是什么？
A: Compare And Swap 原子操作
   比较旧值一致则替换为新值
   无锁编程基础
   ABA 问题用版本号解决

Q: 读写锁？
A: 读读并发、读写互斥、写写互斥
   适合读多写少场景
   Go: sync.RWMutex

Q: Go 有哪些同步原语？
A: Mutex / RWMutex：互斥/读写锁
   WaitGroup：等待一组 goroutine
   Once：只执行一次
   Cond：条件变量
   atomic：原子操作
   Channel：通信式同步（推荐）
```

## 6. 文件系统核心问题？

**回答：**

```
Q: inode 是什么？
A: 文件元数据结构（权限/大小/时间/块位置）
   文件名在目录项中，不在 inode 中
   硬链接 = 多个目录项指向同一 inode

Q: 硬链接 vs 软链接？
A: 硬链接：同 inode，删原文件不影响
   软链接：新 inode，存路径，原文件删除则失效
   硬链接不能跨分区、不能链接目录

Q: 文件删除磁盘不释放？
A: 文件被其他进程持有 fd
   lsof | grep deleted 找到进程
   重启进程或关闭 fd

Q: ext4 vs xfs？
A: ext4：通用，支持在线扩展
   xfs：大文件高性能，高并发 IO 好
   CentOS7 默认 xfs

Q: Page Cache？
A: 内核用空闲内存缓存磁盘数据
   读文件先查 Page Cache
   free 中 buff/cache 就是这个
   可通过 echo 3 > /proc/sys/vm/drop_caches 清除
```

## 7. 系统调用与中断核心问题？

**回答：**

```
Q: 用户态和内核态？
A: 用户态：应用程序运行，受限权限
   内核态：操作系统运行，全部权限
   切换方式：系统调用/中断/异常

Q: 系统调用过程？
A: 1. libc 封装函数设置调用号到寄存器
   2. 执行 syscall 指令
   3. CPU 切换到内核态
   4. 查 sys_call_table 执行对应函数
   5. 返回用户态

Q: 中断上下半部？
A: 上半部（硬中断）：快速处理，不可睡眠
   下半部（softirq/tasklet）：延迟处理耗时操作
   例：网卡中断上半部读数据→下半部协议栈处理

Q: 如何减少系统调用开销？
A: io_uring（异步批量IO）
   mmap（减少 read/write 调用）
   sendfile（零拷贝）
   vDSO（部分调用用户态完成）
```

## 8. IPC 核心问题？

**回答：**

```
Q: 进程间通信方式？
A: 管道：父子进程，半双工，64KB 缓冲
   消息队列：结构化消息，内核持久
   共享内存：最快，需额外同步
   信号：异步通知（SIGTERM/SIGKILL）
   Socket：最通用，可跨主机
   Unix Socket：本机，比 TCP 快 2-3 倍

Q: 共享内存为什么最快？
A: 两个进程页表映射同一物理内存
   不需要内核拷贝数据
   直接读写内存

Q: Go 进程间通信推荐？
A: goroutine 间 → Channel
   本机进程间 → Unix Socket
   跨机器 → gRPC

Q: SIGTERM vs SIGKILL？
A: SIGTERM(15)：优雅退出，可捕获处理
   SIGKILL(9)：立即杀死，不可捕获
   应先 SIGTERM，超时再 SIGKILL
```

## 9. 容器与虚拟化核心问题？

**回答：**

```
Q: 容器和虚拟机区别？
A: 容器共享宿主内核，秒级启动，MB 级开销
   虚拟机独立内核，分钟级启动，GB 级开销

Q: Docker 依赖哪些 Linux 技术？
A: Namespace：资源隔离（PID/NET/MNT/...）
   Cgroup：资源限制（CPU/内存/IO）
   UnionFS：镜像分层（overlay2）
   pivot_root：切换根文件系统

Q: Namespace 有哪些？
A: PID(进程ID) NET(网络) MNT(挂载)
   UTS(主机名) IPC(进程间通信) USER(用户)
   CGROUP(cgroup视图) TIME(系统时间)

Q: Cgroup 限制什么？
A: cpu（CPU时间） memory（内存上限）
   blkio（磁盘IO） pids（进程数）
   cpuset（CPU绑核）

Q: 容器网络原理？
A: veth pair 连接容器和 docker0 网桥
   容器间通过网桥二层转发
   外部访问通过 iptables DNAT 端口映射
```

## 10. 操作系统高频综合题？

**回答：**

```
Q: 从输入 URL 到页面显示，操作系统做了什么？
A: DNS 解析（UDP/TCP）→ Socket 创建 → TCP 三次握手
   → TLS 握手 → HTTP 请求 → 数据通过协议栈收发
   → epoll 管理连接 → 进程调度处理请求
   → 内存分配/Page Cache → IO 读取资源

Q: 服务器处理高并发，操作系统层面怎么优化？
A: 调大文件描述符限制（ulimit -n）
   调大 TCP 队列（somaxconn/backlog）
   用 epoll + 非阻塞 IO
   TCP 参数调优（tw_reuse/keepalive/buffer）
   CPU 亲和性绑核
   大页内存减少 TLB miss

Q: 线上服务 CPU 100% 怎么排查？
A: 1. top 看哪个进程 CPU 高
   2. top -Hp PID 看哪个线程
   3. strace -p TID 看卡在哪个系统调用
   4. perf top -p PID 看热点函数
   5. Go: pprof /debug/pprof/profile

Q: 线上服务内存 OOM 怎么排查？
A: 1. dmesg 看 OOM killer 日志
   2. /proc/PID/status 看内存占用
   3. Go: pprof /debug/pprof/heap
   4. 检查是否有 goroutine 泄漏
   5. 检查缓存是否无限增长

Q: 磁盘 IO 高怎么排查？
A: 1. iostat -x 1 看 %util 和 await
   2. iotop 看哪个进程 IO 高
   3. strace 看频繁的 read/write
   4. 检查日志写入是否过多
   5. 考虑异步 IO / 缓冲区优化

Q: 大量 TIME_WAIT 怎么处理？
A: 原因：主动关闭方等 2MSL（60s）
   1. ss -s 确认数量
   2. 开启 tcp_tw_reuse
   3. 使用连接池复用连接
   4. 改为长连接（HTTP Keep-Alive）

Q: 进程 D 状态不可中断怎么办？
A: 通常在等待 IO（NFS/磁盘故障）
   kill -9 无效
   检查 IO 子系统（iostat/dmesg）
   修复 IO 问题后通常自动恢复
   极端情况需要重启
```
