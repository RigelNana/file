# 文件系统与IO

---

## 1. Linux 文件系统结构？

**回答：**

```
VFS (Virtual File System)：
  统一接口层，屏蔽不同文件系统差异
  
  应用程序 → VFS → ext4/xfs/btrfs/tmpfs/procfs

文件系统核心概念：
  ┌──────────────┬──────────────────────────────┐
  │ 概念          │ 说明                          │
  ├──────────────┼──────────────────────────────┤
  │ Superblock   │ 文件系统元信息（大小/块数等）  │
  │ Inode        │ 文件元数据（权限/大小/指针）   │
  │ Dentry       │ 目录项（文件名→inode映射）    │
  │ Block        │ 数据块（实际文件内容）         │
  └──────────────┴──────────────────────────────┘

Inode 包含：
  文件类型/权限/所有者/大小/时间戳
  数据块指针（直接+间接+双间接+三间接）
  不包含文件名（文件名在目录的 dentry 中）

硬链接 vs 软链接：
  硬链接：多个文件名指向同一 inode，不能跨文件系统
  软链接：独立 inode，存储目标路径，可以跨文件系统
```

## 2. IO 模型详解？

**回答：**

```
五种 IO 模型：

1. 同步阻塞 IO (Blocking IO)：
   应用 → read() → 阻塞等待 → 数据就绪 → 拷贝到用户空间 → 返回

2. 同步非阻塞 IO (Non-blocking IO)：
   应用 → read() → 未就绪返回 EAGAIN → 轮询 → 就绪 → 拷贝 → 返回

3. IO 多路复用 (select/poll/epoll)：
   应用 → select() 监视多个 fd → 有 fd 就绪 → read()
   一个线程管理多个连接

4. 信号驱动 IO：
   注册 SIGIO → 内核数据就绪后发信号 → 应用 read()

5. 异步 IO (AIO)：
   应用 → aio_read() → 立即返回
   内核完成数据拷贝后通知应用

  ┌──────────────────┬──────────────┬──────────────┐
  │ 模型              │ 等待数据      │ 数据拷贝      │
  ├──────────────────┼──────────────┼──────────────┤
  │ 阻塞 IO          │ 阻塞          │ 阻塞          │
  │ 非阻塞 IO        │ 轮询          │ 阻塞          │
  │ IO 多路复用      │ 阻塞在select  │ 阻塞          │
  │ 信号驱动          │ 非阻塞        │ 阻塞          │
  │ 异步 IO          │ 非阻塞        │ 非阻塞        │
  └──────────────────┴──────────────┴──────────────┘

前四种都是同步 IO（数据拷贝阶段阻塞）
只有异步 IO 是真异步（全程非阻塞）
```

## 3. select/poll/epoll 详解？

**回答：**

```
  ┌──────────────┬──────────────┬──────────────┬──────────────┐
  │              │ select       │ poll         │ epoll        │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ 数据结构      │ fd_set 位图  │ pollfd 数组  │ 红黑树+就绪链表│
  │ fd 上限      │ 1024         │ 无限制       │ 无限制        │
  │ fd 传递      │ 每次全量拷贝  │ 每次全量拷贝 │ epoll_ctl 增量│
  │ 事件检测      │ 线性遍历     │ 线性遍历     │ 回调 O(1)    │
  │ 触发方式      │ 水平触发     │ 水平触发     │ LT/ET        │
  │ 时间复杂度    │ O(n)         │ O(n)         │ O(1)就绪数   │
  └──────────────┴──────────────┴──────────────┴──────────────┘

epoll 三个函数：
  epoll_create()  → 创建 epoll 实例
  epoll_ctl()     → 增删改 fd 监听
  epoll_wait()    → 等待就绪事件

水平触发 (LT) vs 边缘触发 (ET)：
  LT：数据可读就一直通知（默认，安全）
  ET：状态变化只通知一次（高效，需一次读完）

  ET 必须配合非阻塞 fd + 循环读取直到 EAGAIN

Reactor 模式（基于 epoll）：
  单 Reactor 单线程：Redis
  单 Reactor 多线程：
  主从 Reactor：Nginx / Netty
```

## 4. 零拷贝技术？

**回答：**

```
传统文件发送（4 次拷贝 + 4 次上下文切换）：
  磁盘 → 内核缓冲 → 用户缓冲 → Socket 缓冲 → 网卡

  应用: read(fd, buf)  → 内核态→用户态
       write(sock, buf) → 用户态→内核态

零拷贝方案：

1. mmap + write（3 次拷贝）：
   mmap 映射文件 → write 发送
   磁盘 → 内核缓冲 → Socket 缓冲 → 网卡
   减少一次 内核→用户 拷贝

2. sendfile（2 次拷贝 + DMA）：
   sendfile(out_fd, in_fd, offset, count)
   磁盘 →DMA→ 内核缓冲 →DMA→ 网卡
   数据不经过用户态

3. splice（管道中转）：
   在两个 fd 间移动数据，无需用户态拷贝

应用场景：
  ┌──────────────┬──────────────────────────────┐
  │ 工具          │ 零拷贝方式                    │
  ├──────────────┼──────────────────────────────┤
  │ Nginx        │ sendfile                      │
  │ Kafka        │ sendfile + mmap               │
  │ RocketMQ     │ mmap                          │
  │ Netty        │ FileRegion (sendfile)         │
  └──────────────┴──────────────────────────────┘
```

## 5. Page Cache 与缓冲区？

**回答：**

```
Page Cache：内核对磁盘文件的缓存
  读文件：先查 Page Cache → 命中则直接返回 → 未命中则读磁盘并缓存
  写文件：先写 Page Cache（脏页）→ 后台刷盘

脏页刷盘时机：
  定期刷（dirty_writeback_centisecs）
  脏页占比超阈值（dirty_ratio）
  手动 sync/fsync
  内存不足时回收

fsync vs fdatasync：
  fsync：刷数据 + 元数据（inode 信息）
  fdatasync：只刷数据（不更新修改时间等）

Direct IO：
  绕过 Page Cache，直接读写磁盘
  适用：数据库自己管理缓存（MySQL InnoDB）
  O_DIRECT 标志

预读 (Read-Ahead)：
  顺序读时，内核预先读取后续数据到 Page Cache
  readahead() 系统调用
```

## 6. 文件描述符与打开文件？

**回答：**

```
文件描述符 (fd)：进程中打开文件的整数标识

  0: stdin    标准输入
  1: stdout   标准输出
  2: stderr   标准错误

三层结构：
  进程 fd 表 → 系统打开文件表 → inode 表

  ┌────────────┐     ┌──────────────┐     ┌────────┐
  │ 进程 fd 表  │     │ 系统文件表    │     │ inode  │
  │ fd 0 ──────│────→│ 偏移量/权限  │────→│ 元数据 │
  │ fd 1 ──────│────→│ 偏移量/权限  │────→│ 数据块 │
  │ fd 2       │     │ ...          │     │ ...    │
  └────────────┘     └──────────────┘     └────────┘

  fork() 后：子进程复制 fd 表，共享系统文件表项
  dup() 后：同进程两个 fd 指向同一文件表项

fd 上限：
  进程级：ulimit -n（默认 1024，可调大）
  系统级：/proc/sys/fs/file-max

  # 设置进程 fd 上限
  ulimit -n 65536
  
  # 永久设置
  # /etc/security/limits.conf
  # * soft nofile 65536
  # * hard nofile 65536
```

## 7. 磁盘 IO 调度？

**回答：**

```
IO 调度器：优化磁盘 IO 请求的顺序

  ┌──────────────┬──────────────────────────────┐
  │ 调度器        │ 特点                          │
  ├──────────────┼──────────────────────────────┤
  │ noop         │ FIFO，适合 SSD/虚拟化         │
  │ CFQ          │ 完全公平，默认（旧内核）       │
  │ Deadline     │ 保证最大等待时间，适合数据库    │
  │ mq-deadline  │ 多队列版 Deadline（新内核默认）│
  │ BFQ          │ 优化交互体验，按比例分配       │
  │ none/kyber   │ 多队列 SSD 优化               │
  └──────────────┴──────────────────────────────┘

HDD vs SSD IO 特点：
  HDD：随机 IO 慢（需寻道），顺序 IO 尚可
  SSD：随机和顺序 IO 都快，但有写入放大

  # 查看当前调度器
  cat /sys/block/sda/queue/scheduler
  
  # 修改调度器
  echo deadline > /sys/block/sda/queue/scheduler
```

## 8. 标准 IO 与系统 IO？

**回答：**

```
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │ 标准 IO (stdio)   │ 系统 IO (syscall)│
  ├──────────────┼──────────────────┼──────────────────┤
  │ 函数          │ fopen/fread/     │ open/read/       │
  │              │ fwrite/fprintf   │ write/close      │
  │ 缓冲          │ 有用户态缓冲     │ 无缓冲           │
  │ 操作对象      │ FILE *           │ int fd           │
  │ 可移植性      │ ANSI C 标准      │ POSIX            │
  │ 性能          │ 减少系统调用次数 │ 直接系统调用      │
  └──────────────┴──────────────────┴──────────────────┘

标准 IO 缓冲模式：
  全缓冲：缓冲区满才刷出（文件）
  行缓冲：遇到换行符刷出（终端 stdout）
  无缓冲：直接输出（stderr）

  Go io.Writer 类似：bufio.Writer 提供用户态缓冲
```

## 9. 异步 IO 与 io_uring？

**回答：**

```
传统异步 IO (POSIX AIO)：
  aio_read/aio_write
  Linux 内核实现有限，实际用得少

io_uring (Linux 5.1+)：
  新一代异步 IO 框架，高性能

  ┌──────────────────────────────────────────┐
  │ io_uring 架构                             │
  │                                           │
  │  用户空间          内核空间                 │
  │  ┌──────────┐     ┌──────────┐            │
  │  │ 提交队列  │────→│ 提交队列  │ → 执行 IO  │
  │  │ (SQ)     │ mmap│ (SQ)     │            │
  │  └──────────┘     └──────────┘            │
  │  ┌──────────┐     ┌──────────┐            │
  │  │ 完成队列  │←────│ 完成队列  │ ← IO 完成  │
  │  │ (CQ)     │ mmap│ (CQ)     │            │
  │  └──────────┘     └──────────┘            │
  └──────────────────────────────────────────┘

  优势：
    共享内存（无需拷贝）
    批量提交（减少系统调用）
    支持 polling 模式（避免中断）
    比 epoll + 线程池更高效

应用：高性能数据库/文件服务器/网络服务
```

## 10. 文件系统与IO面试速答？

**回答：**

```
Q: select/poll/epoll 区别？
A: select: 1024 fd 限制，每次全量拷贝遍历
   poll: 无限制但仍线性遍历
   epoll: 红黑树管理 fd，回调通知 O(1)

Q: epoll LT 和 ET 模式？
A: LT: 数据可读就通知（安全，默认）
   ET: 状态变化通知一次，需循环读完
   ET 性能更好但编程更复杂

Q: 零拷贝是什么？
A: 数据不经过用户态
   sendfile: 磁盘→内核→网卡（2次DMA拷贝）
   Kafka/Nginx 使用

Q: Page Cache 作用？
A: 内核缓存磁盘文件，减少磁盘IO
   读：先查缓存 / 写：先写缓存再异步刷盘
   fsync 强制刷盘

Q: 文件描述符上限？
A: 进程级：ulimit -n（默认1024）
   系统级：/proc/sys/fs/file-max
   生产环境调到 65536+

Q: inode 是什么？
A: 文件元数据（权限/大小/数据块指针）
   不包含文件名（文件名在目录项中）
   硬链接 = 多个文件名 → 同一 inode

Q: 同步/异步 阻塞/非阻塞？
A: 同步阻塞：read 等待数据
   同步非阻塞：read 立即返回，轮询
   IO 多路复用：select/epoll 管理多fd
   异步 IO：内核完成后通知（io_uring）

Q: Direct IO 什么时候用？
A: 绕过 Page Cache 直接读写磁盘
   数据库自己管理缓存时使用（MySQL InnoDB）
```
