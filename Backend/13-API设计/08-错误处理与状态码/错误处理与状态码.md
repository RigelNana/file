# 错误处理与状态码

---

## 1. HTTP状态码完整指南？

**回答：**

```
  常用状态码：
  ┌──────┬──────────────────┬──────────────────────────┐
  │ 码    │ 名称              │ API场景                  │
  ├──────┼──────────────────┼──────────────────────────┤
  │ 200  │ OK                │ 成功（通用）              │
  │ 201  │ Created           │ 创建成功                  │
  │ 204  │ No Content        │ 删除成功/无返回体         │
  │ 207  │ Multi-Status      │ 批量操作部分成功          │
  │ 301  │ Moved Permanently │ 永久重定向                │
  │ 304  │ Not Modified      │ 缓存未变                  │
  │ 400  │ Bad Request       │ 请求参数错误              │
  │ 401  │ Unauthorized      │ 未认证/Token过期          │
  │ 403  │ Forbidden         │ 无权限                    │
  │ 404  │ Not Found         │ 资源不存在                │
  │ 405  │ Method Not Allowed│ HTTP方法不支持            │
  │ 409  │ Conflict          │ 资源冲突（重复创建）      │
  │ 422  │ Unprocessable     │ 语法正确但语义错误        │
  │ 429  │ Too Many Requests │ 限流                      │
  │ 500  │ Internal Error    │ 服务器内部错误            │
  │ 502  │ Bad Gateway       │ 上游服务错误              │
  │ 503  │ Service Unavailable│ 服务不可用               │
  │ 504  │ Gateway Timeout   │ 上游超时                  │
  └──────┴──────────────────┴──────────────────────────┘

  选择原则：
  4xx = 客户端的问题（参数错/没权限）
  5xx = 服务端的问题（Bug/宕机）
  不要所有错误都返回200
```

## 2. 统一错误响应格式？

**回答：**

```
  错误响应结构：
  {
    "error": {
      "code": "VALIDATION_ERROR",
      "message": "请求参数验证失败",
      "details": [
        {"field": "email", "message": "邮箱格式不正确"},
        {"field": "name", "message": "长度必须在2-32之间"}
      ],
      "request_id": "req-abc-123"
    }
  }

  Go定义：
  type ErrorResponse struct {
      Error ErrorBody `json:"error"`
  }
  
  type ErrorBody struct {
      Code      string        `json:"code"`
      Message   string        `json:"message"`
      Details   []FieldError  `json:"details,omitempty"`
      RequestID string        `json:"request_id,omitempty"`
  }
  
  type FieldError struct {
      Field   string `json:"field"`
      Message string `json:"message"`
  }
  
  func WriteError(w http.ResponseWriter, status int, code, message string) {
      w.Header().Set("Content-Type", "application/json")
      w.WriteHeader(status)
      json.NewEncoder(w).Encode(ErrorResponse{
          Error: ErrorBody{Code: code, Message: message},
      })
  }
  
  func WriteValidationError(w http.ResponseWriter, errors []FieldError) {
      w.Header().Set("Content-Type", "application/json")
      w.WriteHeader(400)
      json.NewEncoder(w).Encode(ErrorResponse{
          Error: ErrorBody{
              Code:    "VALIDATION_ERROR",
              Message: "请求参数验证失败",
              Details: errors,
          },
      })
  }
```

## 3. Go错误处理模式？

**回答：**

```
  自定义业务错误：
  type AppError struct {
      Status  int    `json:"-"`
      Code    string `json:"code"`
      Message string `json:"message"`
      Err     error  `json:"-"` // 内部错误（不暴露给客户端）
  }
  
  func (e *AppError) Error() string { return e.Message }
  func (e *AppError) Unwrap() error { return e.Err }
  
  // 预定义错误
  func ErrNotFound(resource string) *AppError {
      return &AppError{404, "NOT_FOUND", resource + "不存在", nil}
  }
  func ErrBadRequest(msg string) *AppError {
      return &AppError{400, "BAD_REQUEST", msg, nil}
  }
  func ErrInternal(err error) *AppError {
      return &AppError{500, "INTERNAL_ERROR", "服务内部错误", err}
  }

  统一错误处理Handler：
  type HandlerFunc func(w http.ResponseWriter, r *http.Request) error
  
  func Wrap(fn HandlerFunc) http.HandlerFunc {
      return func(w http.ResponseWriter, r *http.Request) {
          err := fn(w, r)
          if err == nil { return }
          
          var appErr *AppError
          if errors.As(err, &appErr) {
              // 业务错误
              WriteError(w, appErr.Status, appErr.Code, appErr.Message)
              if appErr.Err != nil {
                  log.Printf("internal: %v", appErr.Err)
              }
          } else {
              // 未知错误
              WriteError(w, 500, "INTERNAL_ERROR", "服务内部错误")
              log.Printf("unexpected error: %v", err)
          }
      }
  }
  
  // 使用
  mux.HandleFunc("GET /users/{id}", Wrap(GetUser))
  
  func GetUser(w http.ResponseWriter, r *http.Request) error {
      user, err := svc.GetUser(r.Context(), r.PathValue("id"))
      if err != nil {
          return ErrNotFound("用户")
      }
      return json.NewEncoder(w).Encode(user)
  }
```

## 4. 参数校验错误处理？

**回答：**

```
  validator库：
  import "github.com/go-playground/validator/v10"
  
  var validate = validator.New()
  
  type CreateUserInput struct {
      Name  string `json:"name" validate:"required,min=2,max=32"`
      Email string `json:"email" validate:"required,email"`
      Age   int    `json:"age" validate:"gte=0,lte=150"`
      Phone string `json:"phone" validate:"omitempty,len=11"`
  }
  
  func ValidateAndBind(r *http.Request, dst interface{}) []FieldError {
      if err := json.NewDecoder(r.Body).Decode(dst); err != nil {
          return []FieldError{{Field: "body", Message: "JSON格式错误"}}
      }
      
      err := validate.Struct(dst)
      if err == nil { return nil }
      
      var fieldErrors []FieldError
      for _, e := range err.(validator.ValidationErrors) {
          fieldErrors = append(fieldErrors, FieldError{
              Field:   toSnakeCase(e.Field()),
              Message: translateError(e),
          })
      }
      return fieldErrors
  }
  
  func translateError(e validator.FieldError) string {
      switch e.Tag() {
      case "required":
          return "不能为空"
      case "email":
          return "邮箱格式不正确"
      case "min":
          return fmt.Sprintf("最少%s个字符", e.Param())
      case "max":
          return fmt.Sprintf("最多%s个字符", e.Param())
      default:
          return "格式不正确"
      }
  }
  
  // 使用
  func CreateUser(w http.ResponseWriter, r *http.Request) error {
      var input CreateUserInput
      if errs := ValidateAndBind(r, &input); errs != nil {
          WriteValidationError(w, errs)
          return nil
      }
      // ...
  }
```

## 5. 错误日志与监控？

**回答：**

```
  错误分级：
  4xx → 客户端错误 → 记INFO日志 → 不告警
  5xx → 服务端错误 → 记ERROR日志 → 告警

  错误日志格式：
  func logError(r *http.Request, err error, status int) {
      fields := map[string]interface{}{
          "method":     r.Method,
          "path":       r.URL.Path,
          "status":     status,
          "request_id": r.Header.Get("X-Request-ID"),
          "client_ip":  getClientIP(r),
          "error":      err.Error(),
      }
      
      if status >= 500 {
          slog.Error("server error", slog.Any("fields", fields))
      } else {
          slog.Info("client error", slog.Any("fields", fields))
      }
  }

  不要暴露内部错误：
  ✅ 返回给客户端：{"code": "INTERNAL_ERROR", "message": "服务内部错误"}
  ❌ 返回给客户端：{"error": "sql: connection refused to db:5432"}

  原始错误记录在日志中 不返回给客户端

  Prometheus错误指标：
  var httpErrors = promauto.NewCounterVec(prometheus.CounterOpts{
      Name: "http_errors_total",
  }, []string{"method", "path", "status", "error_code"})

  告警规则：
  - alert: HighErrorRate
    expr: rate(http_errors_total{status=~"5.."}[5m]) > 0.1
    for: 2m
    labels:
      severity: critical
```

## 6. 重试与错误恢复？

**回答：**

```
  哪些错误可以重试：
  ✅ 429 Too Many Requests（限流）
  ✅ 503 Service Unavailable（暂时不可用）
  ✅ 504 Gateway Timeout（超时）
  ✅ 网络连接错误
  ❌ 400 Bad Request（参数错永远不会成功）
  ❌ 401/403（认证/授权问题）
  ❌ 404 Not Found

  Retry-After Header：
  HTTP/1.1 429 Too Many Requests
  Retry-After: 30
  X-RateLimit-Remaining: 0
  X-RateLimit-Reset: 1704067200

  Go客户端重试：
  func DoWithRetry(ctx context.Context, fn func() (*http.Response, error)) (*http.Response, error) {
      maxRetries := 3
      backoff := 100 * time.Millisecond
      
      for i := 0; i < maxRetries; i++ {
          resp, err := fn()
          
          if err == nil && !isRetryable(resp.StatusCode) {
              return resp, nil
          }
          
          if i < maxRetries-1 {
              // 指数退避 + 抖动
              jitter := time.Duration(rand.Int63n(int64(backoff / 2)))
              select {
              case <-time.After(backoff + jitter):
              case <-ctx.Done():
                  return nil, ctx.Err()
              }
              backoff *= 2
          }
      }
      return nil, fmt.Errorf("max retries exceeded")
  }
  
  func isRetryable(status int) bool {
      return status == 429 || status == 503 || status == 504
  }
```

## 7. gRPC错误映射？

**回答：**

```
  gRPC状态码 → HTTP状态码映射：
  ┌──────────────────┬──────────┐
  │ gRPC              │ HTTP     │
  ├──────────────────┼──────────┤
  │ OK                │ 200      │
  │ InvalidArgument   │ 400      │
  │ Unauthenticated   │ 401      │
  │ PermissionDenied  │ 403      │
  │ NotFound          │ 404      │
  │ AlreadyExists     │ 409      │
  │ FailedPrecondition│ 412      │
  │ ResourceExhausted │ 429      │
  │ Internal          │ 500      │
  │ Unavailable       │ 503      │
  │ DeadlineExceeded  │ 504      │
  └──────────────────┴──────────┘

  错误转换（gRPC → HTTP）：
  func GRPCToHTTPError(err error) (int, *ErrorBody) {
      st, ok := status.FromError(err)
      if !ok {
          return 500, &ErrorBody{Code: "UNKNOWN", Message: err.Error()}
      }
      
      httpStatus := runtime.HTTPStatusFromCode(st.Code())
      return httpStatus, &ErrorBody{
          Code:    st.Code().String(),
          Message: st.Message(),
      }
  }

  gRPC Gateway自动转换：
  使用grpc-gateway时
  gRPC错误码自动映射为HTTP状态码
  自定义错误处理器可以调整格式
```

## 8. 国际化错误消息？

**回答：**

```
  错误消息分离：
  code → 给程序使用（不变）
  message → 给用户看（可国际化）

  方案：
  {
    "error": {
      "code": "BALANCE_INSUFFICIENT",
      "message": "余额不足"          // 可根据语言变化
    }
  }

  Go i18n错误消息：
  var errorMessages = map[string]map[string]string{
      "en": {
          "NOT_FOUND":       "Resource not found",
          "INVALID_PARAM":   "Invalid parameters",
          "BALANCE_INSUFFICIENT": "Insufficient balance",
      },
      "zh": {
          "NOT_FOUND":       "资源不存在",
          "INVALID_PARAM":   "参数错误",
          "BALANCE_INSUFFICIENT": "余额不足",
      },
  }
  
  func GetErrorMessage(lang, code string) string {
      if msgs, ok := errorMessages[lang]; ok {
          if msg, ok := msgs[code]; ok {
              return msg
          }
      }
      return errorMessages["en"][code] // 默认英文
  }
  
  func WriteLocalizedError(w http.ResponseWriter, r *http.Request, status int, code string) {
      lang := r.Header.Get("Accept-Language")
      if lang == "" { lang = "en" }
      
      message := GetErrorMessage(lang[:2], code)
      WriteError(w, status, code, message)
  }

  客户端优先方案：
  服务端只返回code
  客户端根据code显示本地化消息
  → 更灵活 减少服务端复杂度
```

## 9. 错误处理最佳实践？

**回答：**

```
  DO:
  ✅ 使用正确的HTTP状态码
  ✅ 统一错误响应格式
  ✅ 错误码有意义（NOT_FOUND not err001）
  ✅ 包含request_id方便排查
  ✅ 5xx记录详细日志+告警
  ✅ 4xx记录简要日志
  ✅ 校验错误返回所有字段错误（一次性）
  ✅ 可重试错误返回Retry-After

  DON'T:
  ❌ 所有错误返回200
  ❌ 暴露内部异常堆栈
  ❌ 暴露数据库/SQL错误
  ❌ 只返回"error"没有code
  ❌ 一个字段错就返回 不告诉其他字段也有错
  ❌ 错误消息硬编码在前端

  错误处理检查清单：
  1. 每个API都有错误响应文档
  2. 400错误包含详细字段错误
  3. 401和403区分清楚
  4. 500不暴露内部信息
  5. 有统一的错误处理中间件
  6. 日志包含request_id
  7. 有错误率监控和告警
```

## 10. 错误处理面试速答？

**回答：**

```
Q: 401和403区别？
A: 401=未认证(没登录/Token过期)
   403=已认证但无权限

Q: 400和422区别？
A: 400=请求语法错误(JSON格式错)
   422=语法正确但语义错误(校验失败)

Q: 错误响应怎么设计？
A: 统一格式 code+message+details
   包含request_id方便排查

Q: 5xx错误怎么处理？
A: 不暴露内部信息给客户端
   记ERROR日志+告警+返回通用错误

Q: 哪些错误可以重试？
A: 429/503/504/网络错误
   400/401/403/404不应重试

Q: 校验错误怎么返回？
A: 一次返回所有字段错误
   不要一个个返回让用户逐个修复

Q: Go怎么统一错误处理？
A: 自定义AppError类型
   Handler返回error + Wrap统一处理

Q: 错误消息要国际化吗？
A: 服务端返回code 客户端映射消息
   或根据Accept-Language返回
```
