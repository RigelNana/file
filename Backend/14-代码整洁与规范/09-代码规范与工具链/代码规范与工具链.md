## 1. Go代码规范体系？

**回答：**

```
  Go官方规范文档：
  ┌──────────────────────┬──────────────────────────────┐
  │ 文档                  │ 内容                         │
  ├──────────────────────┼──────────────────────────────┤
  │ Effective Go          │ 官方编程指南                 │
  │ Go Code Review Comments│ Review检查清单              │
  │ Go Style Guide        │ Google Go风格指南(2022)      │
  │ uber-go/guide         │ Uber Go规范                  │
  └──────────────────────┴──────────────────────────────┘

  核心规范要点：
  1. 格式化：gofmt统一 无争论
  2. 命名：驼峰 大小写控制可见性
  3. 注释：导出符号必须文档注释
  4. 错误：显式处理 不忽略
  5. 包：小写单词 不用下划线
  6. 接口：小而专 1-3个方法
  7. 并发：用channel通信 不用共享内存
  8. context：第一个参数 贯穿调用链

  项目规范文档结构：
  docs/
  ├── code-style.md        # 编码规范
  ├── git-workflow.md       # Git工作流
  ├── review-guide.md       # Review指南
  ├── api-convention.md     # API规范
  └── error-code.md         # 错误码规范
```

## 2. golangci-lint配置？

**回答：**

```
  golangci-lint = Go最强静态分析工具
  集成40+检查器 一个工具搞定所有

  .golangci.yml 推荐配置：
  run:
    timeout: 5m
    go: '1.22'
  
  linters:
    enable:
      # 默认启用
      - errcheck        # 未检查的error
      - gosimple        # 代码简化建议
      - govet           # go vet检查
      - ineffassign     # 无效赋值
      - staticcheck     # 高级静态分析
      - unused          # 未使用的代码
      # 推荐启用
      - gofmt           # 格式化检查
      - goimports       # import排序
      - misspell        # 拼写错误
      - gocyclo         # 圈复杂度
      - gosec           # 安全检查
      - prealloc        # 切片预分配
      - unconvert       # 不必要的类型转换
      - unparam         # 未使用的参数
  
  linters-settings:
    gocyclo:
      min-complexity: 15
    errcheck:
      check-type-assertions: true
    govet:
      enable-all: true
  
  issues:
    exclude-rules:
      - path: _test\.go
        linters:
          - gosec    # 测试文件不检查安全
          - errcheck # 测试文件宽松

  运行：
  golangci-lint run ./...
  golangci-lint run --fix ./...  # 自动修复
```

## 3. pre-commit钩子？

**回答：**

```
  提交前自动检查 防止不合规代码入库

  方式1：Git Hooks
  # .git/hooks/pre-commit
  #!/bin/sh
  
  # 格式化检查
  UNFORMATTED=$(gofmt -l .)
  if [ -n "$UNFORMATTED" ]; then
      echo "请先格式化以下文件:"
      echo "$UNFORMATTED"
      exit 1
  fi
  
  # 静态检查
  golangci-lint run ./...
  if [ $? -ne 0 ]; then
      echo "lint检查未通过"
      exit 1
  fi
  
  # 测试
  go test -short ./...

  方式2：pre-commit框架（推荐）
  # .pre-commit-config.yaml
  repos:
  - repo: https://github.com/dnephin/pre-commit-golang
    rev: master
    hooks:
    - id: go-fmt
    - id: go-vet
    - id: go-lint
    - id: go-imports
    - id: go-unit-tests
  
  # 安装
  pip install pre-commit
  pre-commit install

  方式3：Makefile
  .PHONY: check
  check: fmt lint test
  
  fmt:
  	gofmt -w .
  	goimports -w .
  
  lint:
  	golangci-lint run ./...
  
  test:
  	go test -race -cover ./...
  
  # 提交前运行
  make check
```

## 4. CI/CD中的代码质量？

**回答：**

```
  GitHub Actions完整配置：

  name: CI
  on:
    push:
      branches: [main]
    pull_request:
  
  jobs:
    lint:
      runs-on: ubuntu-latest
      steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: '1.22' }
      - uses: golangci/golangci-lint-action@v4
        with: { version: latest }
    
    test:
      runs-on: ubuntu-latest
      steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: '1.22' }
      - run: go test -race -coverprofile=coverage.out ./...
      - run: go tool cover -func=coverage.out
    
    security:
      runs-on: ubuntu-latest
      steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with: { go-version: '1.22' }
      - run: |
          go install golang.org/x/vuln/cmd/govulncheck@latest
          govulncheck ./...
      - run: |
          go install github.com/securego/gosec/v2/cmd/gosec@latest
          gosec ./...

  PR合并条件：
  ✓ lint通过
  ✓ 所有测试通过
  ✓ 覆盖率不下降
  ✓ 无安全漏洞
  ✓ 至少1人Approve
```

## 5. Makefile工程管理？

**回答：**

```
  Go项目标准Makefile：

  .PHONY: all build test lint clean run

  APP_NAME := myapp
  VERSION := $(shell git describe --tags --always)
  BUILD_TIME := $(shell date -u +"%Y-%m-%dT%H:%M:%SZ")
  LDFLAGS := -X main.Version=$(VERSION) -X main.BuildTime=$(BUILD_TIME)
  
  all: lint test build
  
  build:
  	CGO_ENABLED=0 go build -ldflags "$(LDFLAGS)" -o bin/$(APP_NAME) ./cmd/server
  
  run:
  	go run ./cmd/server
  
  test:
  	go test -race -cover -count=1 ./...
  
  test-integration:
  	go test -race -count=1 -tags=integration ./...
  
  lint:
  	golangci-lint run ./...
  
  fmt:
  	gofmt -w .
  	goimports -w .
  
  generate:
  	go generate ./...
  
  mock:
  	mockgen -source=internal/service/interface.go \
  		-destination=internal/service/mock/mock.go
  
  migrate-up:
  	migrate -path ./migrations -database "$(DB_DSN)" up
  
  migrate-down:
  	migrate -path ./migrations -database "$(DB_DSN)" down 1
  
  docker-build:
  	docker build -t $(APP_NAME):$(VERSION) .
  
  clean:
  	rm -rf bin/ coverage.out
  
  help:
  	@grep -E '^[a-zA-Z_-]+:' Makefile | sort

  使用：
  make          # 默认 lint+test+build
  make run      # 本地运行
  make test     # 测试
  make lint     # 检查
```

## 6. 代码生成工具？

**回答：**

```
  Go代码生成（go generate）：
  //go:generate mockgen -source=repo.go -destination=mock_repo.go
  //go:generate stringer -type=Status
  //go:generate protoc --go_out=. --go-grpc_out=. api.proto

  常用生成工具：
  ┌──────────────────────┬──────────────────────┐
  │ 工具                  │ 用途                 │
  ├──────────────────────┼──────────────────────┤
  │ mockgen              │ Mock生成             │
  │ stringer             │ 枚举String方法       │
  │ protoc               │ gRPC代码             │
  │ sqlc                 │ SQL→类型安全Go代码   │
  │ ent                  │ ORM代码              │
  │ oapi-codegen         │ OpenAPI→Go代码       │
  │ wire                 │ 依赖注入             │
  └──────────────────────┴──────────────────────┘

  stringer示例：
  //go:generate stringer -type=OrderStatus
  type OrderStatus int
  const (
      OrderPending   OrderStatus = iota
      OrderConfirmed
      OrderShipped
  )
  // 自动生成 orderstatus_string.go
  // func (i OrderStatus) String() string

  sqlc示例：
  -- query.sql
  -- name: GetUser :one
  SELECT id, name, email FROM users WHERE id = $1;
  
  // 自动生成类型安全的Go函数：
  func (q *Queries) GetUser(ctx context.Context, id int64) (User, error)

  运行所有生成：
  go generate ./...
  
  CI中检查生成代码是否最新：
  go generate ./...
  git diff --exit-code  # 有差异则失败
```

## 8. 版本管理与发布？

**回答：**

```
  语义化版本（SemVer）：
  v主版本.次版本.修订号
  v1.2.3
  
  主版本：不兼容的API修改
  次版本：向下兼容的功能新增
  修订号：向下兼容的Bug修复

  Go Module版本规则：
  v0.x.x → 不稳定 随时可改
  v1.x.x → 稳定 保持兼容
  v2.x.x → 新主版本 import路径加/v2

  Conventional Commits：
  feat: 添加用户注册功能
  fix: 修复订单超时问题
  docs: 更新API文档
  refactor: 重构认证模块
  test: 添加单元测试
  chore: 更新依赖版本
  
  feat!: 修改用户API (BREAKING CHANGE)

  自动化发布：
  # .goreleaser.yml
  builds:
  - env: [CGO_ENABLED=0]
    goos: [linux, darwin, windows]
    goarch: [amd64, arm64]
    ldflags:
    - -s -w
    - -X main.version={{.Version}}
  
  archives:
  - format: tar.gz
    name_template: "{{ .ProjectName }}_{{ .Os }}_{{ .Arch }}"

  发布流程：
  git tag v1.2.3
  git push origin v1.2.3
  # CI自动构建+发布
```

## 10. 工具链面试速答？

**回答：**

```

Q: Go项目标准结构？
A: cmd/(入口) internal/(私有)

Q: Makefile有什么用？
A: 统一构建/测试/检查命令
   make build/test/lint/run

Q: go generate做什么？

Q: 版本号怎么管理？
A: 语义化版本 v主.次.修订

Q: 项目需要哪些配置文件？

Q: 文档怎么管理？
```
