## 2. SQL 注入与防御？

**回答：**

```
SQL 注入示例：
  用户输入: ' OR 1=1 --
  
  拼接后: SELECT * FROM users WHERE name='' OR 1=1 --'
  → 返回所有用户数据

危害：
  数据泄露/数据篡改/删库
  绕过认证登录
  执行系统命令（某些数据库）

防御方法：
  1. 参数化查询（最重要）
  2. ORM 框架（自动参数化）
  3. 输入验证（白名单）
  4. 最小权限原则（数据库账号）
```

```go
// 错误：字符串拼接 → SQL 注入
query := "SELECT * FROM users WHERE name='" + name + "'"

// 正确：参数化查询
db.Query("SELECT * FROM users WHERE name = ?", name)

// 正确：使用 ORM (GORM)
db.Where("name = ?", name).Find(&users)
```

## 3. XSS 攻击与防御？

**回答：**

```
XSS（跨站脚本攻击）：
  恶意脚本注入到网页中执行

类型：
  存储型 XSS：恶意脚本存入数据库（评论/帖子）
  反射型 XSS：恶意脚本在 URL 参数中
  DOM 型 XSS：客户端 JS 直接操作 DOM

  存储型示例：
  用户提交评论: <script>steal(document.cookie)</script>
  其他用户浏览 → 脚本执行 → Cookie 被盗

防御：
  1. 输出编码（HTML Entity 转义）
     < → &lt;   > → &gt;   " → &quot;
  2. CSP（Content Security Policy）
     Content-Security-Policy: script-src 'self'
  3. HttpOnly Cookie（JS 不可访问）
  4. 输入验证和过滤
```

```go
// Go 模板自动转义（html/template）
import "html/template"

tmpl := template.Must(template.New("").Parse(
    `<p>Hello, {{.Name}}</p>`))  // 自动转义
// 如果 Name = "<script>alert(1)</script>"
// 输出: <p>Hello, &lt;script&gt;alert(1)&lt;/script&gt;</p>
```

## 4. CSRF 攻击与防御？

**回答：**

```
CSRF（跨站请求伪造）：
  利用浏览器自动携带 Cookie
  诱导用户在已登录状态下执行恶意操作

  ┌──────────┐    1.已登录银行     ┌──────────┐
  │ 用户浏览器│─────────────────→│ 银行网站  │
  │          │                    │          │
  │ 访问恶意网站                    │          │
  │ 恶意页面自动提交转账表单         │          │
  │ → POST /transfer               │ 执行转账 │
  │ Cookie 自动携带 ──────────────→│          │
  └──────────┘                    └──────────┘

防御方法：
  1. CSRF Token：
     服务端生成随机 Token 放入表单
     提交时验证 Token（攻击者无法获取）

  2. SameSite Cookie：
     SameSite=Strict → 跨站不携带
     SameSite=Lax → GET 跨站携带（默认）

  3. Referer/Origin 检查：
     验证请求来源

  4. 重要操作二次验证：
     短信验证码/密码确认
```

## 5. SSRF 攻击与防御？

**回答：**

```
SSRF（服务端请求伪造）：
  攻击者让服务器发起恶意请求
  访问内网资源或元数据服务

  攻击场景：
  用户输入一个 URL → 服务器去获取内容
  用户输入: http://169.254.169.254/latest/meta-data/
  → 服务器请求 AWS 元数据 → 获取 IAM 凭证

  攻击面：
  http://127.0.0.1:6379/ → 访问内部 Redis
  http://内网IP:端口/ → 扫描内网
  file:///etc/passwd → 读取本地文件
```

```go
// 防御 SSRF
func safeHTTPGet(targetURL string) (*http.Response, error) {
    parsed, err := url.Parse(targetURL)
    if err != nil {
        return nil, err
    }

    // 1. 只允许 http/https
    if parsed.Scheme != "http" && parsed.Scheme != "https" {
        return nil, errors.New("invalid scheme")
    }

    // 2. 解析 IP 并检查是否为内网
    ips, err := net.LookupIP(parsed.Hostname())
    if err != nil {
        return nil, err
    }
    for _, ip := range ips {
        if ip.IsLoopback() || ip.IsPrivate() ||
           ip.IsLinkLocalUnicast() {
            return nil, errors.New("internal IP blocked")
        }
    }

    // 3. 设置超时
    client := &http.Client{Timeout: 5 * time.Second}
    return client.Get(targetURL)
}
```

## 6. DDoS 攻击与防御？

**回答：**

```
DDoS 类型：
  ┌──────────────┬──────────────────────────────┐
  │ 层次          │ 攻击方式                      │
  ├──────────────┼──────────────────────────────┤
  │ 网络层        │ SYN Flood / UDP Flood        │
  │              │ ICMP Flood / 放大攻击         │
  │ 传输层        │ ACK Flood / 连接耗尽          │
  │ 应用层        │ HTTP Flood / Slowloris       │
  │              │ CC 攻击                       │
  └──────────────┴──────────────────────────────┘

SYN Flood 防御：
  SYN Cookie（不存半连接）
  增大 backlog 队列
  SYN 代理（防火墙/LB）

HTTP Flood 防御：
  限流（Rate Limiting）
  WAF（Web 应用防火墙）
  验证码挑战
  CDN 吸收流量

Slowloris 防御：
  攻击方式：缓慢发送请求头，保持连接不完成
  防御：设置请求超时/限制单 IP 并发连接数

综合防御：
  CDN / 高防 IP
  流量清洗
  Anycast 分散流量
  弹性扩容
```

## 7. API 安全？

**回答：**

```
API 安全措施：

  认证（Authentication）：
    JWT / OAuth2 / API Key
    mTLS（微服务间）

  授权（Authorization）：
    RBAC（基于角色）
    ABAC（基于属性）
    最小权限原则

  限流（Rate Limiting）：
    令牌桶 / 漏桶 / 滑动窗口
    按 IP / 用户 / API Key

  输入验证：
    白名单验证
    参数类型和范围检查
    请求体大小限制

  传输安全：
    HTTPS 强制
    HSTS 头
    禁用不安全 TLS 版本

  响应安全头：
    X-Content-Type-Options: nosniff
    X-Frame-Options: DENY
    Strict-Transport-Security: max-age=...

  日志审计：
    记录所有 API 调用
    异常行为告警
    定期安全审计
```

## 8. 密码安全？

**回答：**

```
密码存储（永远不存明文）：

  正确：bcrypt / scrypt / Argon2
  错误：MD5 / SHA1 / SHA256（不加盐）

  bcrypt 特点：
  - 内置盐值
  - 可调慢速因子（cost factor）
  - 防彩虹表和暴力破解
```

```go
import "golang.org/x/crypto/bcrypt"

// 密码哈希
func HashPassword(password string) (string, error) {
    bytes, err := bcrypt.GenerateFromPassword(
        []byte(password), bcrypt.DefaultCost) // cost=10
    return string(bytes), err
}

// 密码验证
func CheckPassword(password, hash string) bool {
    err := bcrypt.CompareHashAndPassword(
        []byte(hash), []byte(password))
    return err == nil
}
```

```
密码策略：
  最小长度 8-12 位
  复杂度要求（字母+数字+特殊字符）
  防暴力破解：失败锁定/延迟
  密码历史：不允许重复使用
  MFA 多因素认证
```

## 9. 网络安全最佳实践？

**回答：**

```
后端安全清单：

  □ HTTPS 全站强制
  □ 参数化查询（防 SQL 注入）
  □ 输出编码（防 XSS）
  □ CSRF Token 或 SameSite Cookie
  □ SSRF 防护（禁止内网请求）
  □ 限流（防 DDoS/暴力破解）
  □ 密码 bcrypt 存储
  □ JWT 正确使用（过期/签名验证）
  □ 安全响应头
  □ 日志记录（不记录敏感数据）
  □ 依赖扫描（CVE 漏洞）
  □ 最小权限原则
  □ 不暴露堆栈信息
  □ 文件上传验证

敏感数据保护：
  - 日志脱敏（手机号/身份证/密码）
  - 传输加密（TLS）
  - 存储加密（AES）
  - 密钥管理（Vault/KMS）
  - 数据分类分级
```

## 10. 网络安全面试速答？

**回答：**

```
Q: SQL 注入怎么防？
A: 参数化查询/ORM
   不要拼接 SQL 字符串

Q: XSS 怎么防？
A: 输出 HTML 编码转义
   CSP 策略/HttpOnly Cookie

Q: CSRF 怎么防？
A: CSRF Token 验证
   SameSite Cookie 属性

Q: SSRF 怎么防？
A: 验证目标 IP 非内网
   白名单域名/协议限制

Q: 密码怎么存？
A: bcrypt/Argon2 哈希（带盐）
   永远不存明文或简单 MD5

Q: HTTPS 中间人攻击防御？
A: 证书验证+HSTS+证书固定

Q: 什么是 JWT 安全风险？
A: 不设过期时间/密钥泄露
   不验证签名/存敏感数据
   正确：短过期+Refresh Token+强密钥

Q: DDoS 怎么防？
A: CDN/高防IP 吸流量
   限流/WAF/验证码
   SYN Cookie 防 SYN Flood
```
