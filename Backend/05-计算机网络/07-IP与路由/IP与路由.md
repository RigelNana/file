# IP与路由

---

## 1. IPv4 报文结构？

**回答：**

```
IPv4 头部（20-60 字节）：
  ┌────┬────┬──────────┬──────────────────┐
  │版本 │头长 │服务类型   │ 总长度            │ 4字节
  ├────┴────┼──────────┼──────────────────┤
  │ 标识    │ 标志+偏移 │                  │ 4字节（分片）
  ├────────┼──────────┼──────────────────┤
  │ TTL    │ 协议      │ 头部校验和        │ 4字节
  ├────────┴──────────┴──────────────────┤
  │ 源 IP 地址                            │ 4字节
  ├──────────────────────────────────────┤
  │ 目标 IP 地址                          │ 4字节
  ├──────────────────────────────────────┤
  │ 选项（可选）                          │ 0-40字节
  └──────────────────────────────────────┘

  TTL：每经过一个路由器减 1，减到 0 则丢弃
       防止数据包在网络中无限循环
       默认 64 或 128

  协议号：TCP=6, UDP=17, ICMP=1

IP 分片：
  数据包 > MTU(1500) → 分片传输
  目标端重组
  问题：一个分片丢失 → 整个包重传
  解决：TCP MSS 协商避免 IP 层分片
```

## 2. IPv6 概述？

**回答：**

```
IPv6 vs IPv4：
  ┌──────────────┬──────────────┬──────────────┐
  │ 对比          │ IPv4         │ IPv6         │
  ├──────────────┼──────────────┼──────────────┤
  │ 地址长度      │ 32 位        │ 128 位       │
  │ 地址数量      │ 43 亿       │ 3.4×10^38   │
  │ 头部          │ 20-60 字节  │ 40 字节固定  │
  │ 分片          │ 路由器/主机  │ 仅发送端     │
  │ NAT          │ 广泛使用     │ 不需要       │
  │ IPSec        │ 可选         │ 内置         │
  │ 广播          │ 支持         │ 改为组播     │
  └──────────────┴──────────────┴──────────────┘

IPv6 地址格式：
  2001:0db8:0000:0000:0000:0000:0000:0001
  简写：2001:db8::1

  特殊地址：
  ::1         → 环回地址（对应 127.0.0.1）
  ::          → 未指定
  fe80::/10   → 链路本地
  fc00::/7    → 唯一本地地址（类似私有地址）

IPv4 → IPv6 过渡：
  双栈：同时支持 IPv4 和 IPv6
  隧道：IPv6 封装在 IPv4 中传输
  NAT64：IPv6 网络访问 IPv4 资源
```

## 3. 子网划分与 CIDR？

**回答：**

```
CIDR（无类别域间路由）：
  IP/前缀长度 表示网络

  192.168.1.0/24
  前 24 位是网络号，后 8 位是主机号
  可用主机：2^8 - 2 = 254 个
  （减去网络地址和广播地址）

子网划分练习：
  10.0.0.0/8
  → 10.0.0.0 ~ 10.255.255.255
  → 16,777,214 个可用 IP

  172.16.0.0/12
  → 172.16.0.0 ~ 172.31.255.255
  → 1,048,574 个可用 IP

  192.168.1.0/26（26位网络号）
  → 子网掩码 255.255.255.192
  → 2^6 = 64 地址/子网
  → 可用主机 62 个
  → 子网1: .0~.63  子网2: .64~.127
     子网3: .128~.191  子网4: .192~.255

VLSM（可变长子网掩码）：
  不同子网用不同掩码
  节省 IP 地址空间
```

## 4. 路由协议？

**回答：**

```
路由协议分类：
  ┌──────────────┬──────────────────────────────┐
  │ 分类          │ 协议                          │
  ├──────────────┼──────────────────────────────┤
  │ 内部网关(IGP) │ RIP / OSPF / IS-IS           │
  │ 外部网关(EGP) │ BGP                          │
  └──────────────┴──────────────────────────────┘

RIP（距离向量）：
  以跳数为度量，最大 15 跳
  简单但收敛慢，小型网络

OSPF（链路状态）：
  基于带宽计算最短路径（Dijkstra）
  收敛快，支持大型网络
  支持区域划分

BGP（路径向量）：
  互联网核心路由协议
  AS（自治系统）间路由
  基于策略的路由选择
  路由表 90 万+ 条

路由表匹配：最长前缀匹配
  10.1.1.0/24 和 10.1.0.0/16 都匹配时
  选 /24（更精确）
```

## 5. NAT 详解？

**回答：**

```
NAT（网络地址转换）：
  内网私有 IP ↔ 公网 IP 转换

类型：
  静态 NAT：一对一映射
  动态 NAT：多对多，从 IP 池分配
  PAT/NAPT：多对一，用端口号区分（最常用）

  ┌──────────┐                ┌──────────┐
  │ 内网主机   │  NAPT 路由器   │ 外网服务器│
  │10.0.0.2:80│               │1.2.3.4:80│
  │           │               │          │
  │ src:10.0.0.2:5000         │          │
  │──────────→│               │          │
  │           │ src:203.0.1.1:30001       │
  │           │──────────────→│          │
  │           │               │          │
  │           │←──────────────│ 响应     │
  │←──────────│ 转换回原IP:端口│          │
  └──────────┘                └──────────┘

  Docker 端口映射 = DNAT：
  iptables -t nat -A PREROUTING \
    -p tcp --dport 8080 \
    -j DNAT --to-destination 172.17.0.2:80

NAT 的问题：
  破坏端到端通信（P2P 需要打洞）
  增加延迟（地址转换开销）
  隐藏真实 IP（可用 X-Forwarded-For）
```

## 6. VPN 技术？

**回答：**

```
VPN（虚拟专用网络）：
  在公网上建立加密隧道

  ┌──────┐    加密隧道    ┌──────┐
  │客户端 │═══════════════│VPN网关│──→ 内网
  │      │   公网传输      │      │
  └──────┘                └──────┘

常见 VPN 协议：
  ┌──────────────┬──────────────────────────────┐
  │ 协议          │ 说明                          │
  ├──────────────┼──────────────────────────────┤
  │ IPSec        │ 网络层加密，安全性高           │
  │ OpenVPN      │ SSL/TLS 基础，跨平台           │
  │ WireGuard    │ 更快更简单，Linux 内核集成     │
  │ L2TP/IPSec   │ 二层隧道 + IPSec 加密         │
  │ PPTP         │ 已不安全，不推荐               │
  └──────────────┴──────────────────────────────┘

WireGuard 优势：
  代码量极少（~4000 行）
  性能优秀（内核态实现）
  现代加密（Curve25519/ChaCha20）
  简单配置
```

## 7. CDN 原理？

**回答：**

```
CDN（内容分发网络）：
  将内容缓存到靠近用户的边缘节点

  用户 → DNS 返回最近 CDN 节点 IP
       → CDN 边缘节点返回缓存内容
       → 缓存未命中则回源服务器获取

  ┌──────┐    ┌─────────┐    ┌──────┐
  │ 用户  │──→│ CDN 节点 │──→│ 源站  │
  │      │←──│ (边缘)   │←──│      │
  └──────┘    └─────────┘    └──────┘
               缓存命中        缓存未命中
               直接返回        回源获取

CDN 调度方式：
  DNS 调度：GeoDNS 返回最近节点 IP
  HTTP 调度：302 重定向到最优节点
  Anycast：同一 IP 路由到最近节点

CDN 适合的内容：
  静态资源（JS/CSS/图片/视频）
  API 加速（动态内容加速）
  边缘计算（CDN 节点运行逻辑）

CDN 缓存控制：
  Cache-Control 头控制缓存策略
  CDN 预热：主动推送内容到边缘
  CDN 刷新：清除边缘缓存
```

## 8. 负载均衡层次？

**回答：**

```
四层负载均衡（传输层）：
  基于 IP + 端口转发
  不解析应用层内容
  性能高（LVS/F5/HAProxy mode tcp）
  
  转发模式：
  NAT：修改目标 IP（最常用）
  DR：修改目标 MAC（最高性能）
  TUN：IP 隧道（跨网段）

七层负载均衡（应用层）：
  解析 HTTP/gRPC 等应用协议
  可基于 URL/Header/Cookie 路由
  Nginx/HAProxy mode http/Envoy

  功能：
  URL 路由：/api → 后端A，/web → 后端B
  主机路由：api.example.com → 后端A
  会话保持：同用户请求到同服务器
  SSL 卸载：在 LB 层解密

负载均衡算法：
  轮询（Round Robin）
  加权轮询（Weighted RR）
  最少连接（Least Connections）
  IP Hash（会话保持）
  一致性哈希
```

## 9. 网络性能指标？

**回答：**

```
关键指标：
  ┌──────────────┬──────────────────────────────┐
  │ 指标          │ 说明                          │
  ├──────────────┼──────────────────────────────┤
  │ 带宽          │ 链路最大传输速率（Mbps/Gbps） │
  │ 吞吐量        │ 实际传输速率                  │
  │ 延迟 (RTT)    │ 数据包往返时间                │
  │ 丢包率        │ 传输中丢失的包占比             │
  │ 抖动 (Jitter)│ 延迟的波动程度                │
  │ PPS          │ 每秒包处理数                   │
  └──────────────┴──────────────────────────────┘

测量工具：
  带宽：iperf3 -c host
  延迟：ping host / ping -c 100 host
  路径：traceroute / mtr
  丢包：ping -c 1000 host（看 loss%）
  抖动：mtr（看 StDev 列）

BDP（带宽延迟积）：
  BDP = 带宽 × RTT
  = 1Gbps × 100ms = 12.5MB
  → TCP 缓冲区至少要这么大才能用满带宽
  → tcp_rmem/tcp_wmem 需要足够大
```

## 10. IP与路由面试速答？

**回答：**

```
Q: IPv4 地址耗尽怎么解决？
A: NAT（地址转换复用公网IP）
   CIDR（无类编址节省IP）
   IPv6（128位地址根本解决）

Q: 路由器和交换机区别？
A: 路由器三层转发（IP），跨网段
   交换机二层转发（MAC），同网段

Q: NAT 怎么工作的？
A: NAPT：多个内网IP共用一个公网IP
   用端口号区分不同连接
   Docker 端口映射就是 DNAT

Q: CDN 怎么工作？
A: DNS 返回最近边缘节点IP
   缓存命中直接返回
   未命中回源获取

Q: 四层 vs 七层负载均衡？
A: 四层：IP+端口转发，高性能（LVS）
   七层：解析HTTP，可按URL路由（Nginx）

Q: BGP 是什么？
A: 互联网核心路由协议
   自治系统间交换路由信息
   路由表90万+条

Q: TTL 作用？
A: 每经路由器减1，减到0丢弃
   防止数据包无限循环
   traceroute 利用 TTL 工作
```
