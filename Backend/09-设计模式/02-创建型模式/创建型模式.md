# 创建型模式

---

## 1. 创建型模式有哪些？解决什么问题？

**回答：**

```
创建型模式：封装对象创建过程，降低耦合

  ┌──────────┬────────────────────────────────┐
  │ 模式      │ 解决问题                       │
  ├──────────┼────────────────────────────────┤
  │ 单例      │ 全局唯一实例                   │
  │ 工厂方法  │ 不指定具体类创建对象           │
  │ 抽象工厂  │ 创建一族相关对象               │
  │ 建造者    │ 分步构建复杂对象               │
  │ 原型      │ 通过复制创建新对象             │
  └──────────┴────────────────────────────────┘
```

---

## 2. 单例模式？

**回答：**

```
保证一个类只有一个实例

  场景：配置管理、数据库连接池、日志器

  实现方式：
  1. 饿汉式（init 时创建）
  2. 懒汉式（首次使用创建）
  3. sync.Once（Go 推荐）
```

```go
// Go 推荐：sync.Once
type Config struct {
    DBHost string
    DBPort int
}

var (
    configInstance *Config
    configOnce     sync.Once
)

func GetConfig() *Config {
    configOnce.Do(func() {
        configInstance = &Config{
            DBHost: os.Getenv("DB_HOST"),
            DBPort: 3306,
        }
    })
    return configInstance
}

// 数据库连接池单例
type DBPool struct {
    db *sql.DB
}

var (
    dbPool   *DBPool
    dbOnce   sync.Once
)

func GetDBPool(dsn string) *DBPool {
    dbOnce.Do(func() {
        db, err := sql.Open("mysql", dsn)
        if err != nil {
            log.Fatal(err)
        }
        db.SetMaxOpenConns(100)
        db.SetMaxIdleConns(20)
        dbPool = &DBPool{db: db}
    })
    return dbPool
}
```

```
注意事项：
  - 单例 = 全局状态 → 不利于测试
  - 并发安全必须保证
  - 能用依赖注入替代就不用单例
  - Go 中 sync.Once 是最安全的实现
```

## 3. 工厂方法模式？

**回答：**

```
定义创建对象的接口，由具体工厂决定创建什么

  ┌──────────┐     ┌──────────┐
  │ Factory  │────→│ Product  │
  │(接口)    │     │(接口)    │
  └────┬─────┘     └────┬─────┘
       │                │
  ┌────┴─────┐     ┌────┴─────┐
  │ MySQLFac │────→│ MySQL    │
  │ PGFac    │────→│ Postgres │
  └──────────┘     └──────────┘
```

```go
// 产品接口
type Storage interface {
    Save(key string, value []byte) error
    Load(key string) ([]byte, error)
}

// 具体产品
type FileStorage struct{ basePath string }
func (s *FileStorage) Save(key string, value []byte) error {
    return os.WriteFile(filepath.Join(s.basePath, key), value, 0644)
}
func (s *FileStorage) Load(key string) ([]byte, error) {
    return os.ReadFile(filepath.Join(s.basePath, key))
}

type S3Storage struct{ bucket string }
func (s *S3Storage) Save(key string, value []byte) error { /* S3 上传 */ return nil }
func (s *S3Storage) Load(key string) ([]byte, error) { /* S3 下载 */ return nil, nil }

// 工厂函数（Go 风格，不需要工厂类）
func NewStorage(storageType string, config map[string]string) (Storage, error) {
    switch storageType {
    case "file":
        return &FileStorage{basePath: config["path"]}, nil
    case "s3":
        return &S3Storage{bucket: config["bucket"]}, nil
    default:
        return nil, fmt.Errorf("unknown storage type: %s", storageType)
    }
}

// 使用：调用方不关心具体实现
storage, _ := NewStorage("s3", map[string]string{"bucket": "my-bucket"})
storage.Save("key", []byte("data"))
```

## 4. 抽象工厂模式？

**回答：**

```
创建一族相关对象

  场景：跨平台 UI（按钮+输入框+弹窗）
        多数据库支持（连接+查询+迁移）

  ┌──────────────┐
  │ DAOFactory   │→ CreateUserDAO()
  │              │→ CreateOrderDAO()
  └──────┬───────┘
     ┌───┴───┐
  ┌──┴──┐ ┌──┴──┐
  │MySQL│ │Mongo│
  │Fac  │ │Fac  │
  └─────┘ └─────┘
```

```go
// 产品族接口
type UserDAO interface {
    GetByID(ctx context.Context, id string) (*User, error)
}

type OrderDAO interface {
    GetByID(ctx context.Context, id string) (*Order, error)
}

// 抽象工厂
type DAOFactory interface {
    CreateUserDAO() UserDAO
    CreateOrderDAO() OrderDAO
}

// MySQL 实现族
type MySQLUserDAO struct{ db *sql.DB }
type MySQLOrderDAO struct{ db *sql.DB }

type MySQLDAOFactory struct{ db *sql.DB }
func (f *MySQLDAOFactory) CreateUserDAO() UserDAO   { return &MySQLUserDAO{db: f.db} }
func (f *MySQLDAOFactory) CreateOrderDAO() OrderDAO { return &MySQLOrderDAO{db: f.db} }

// MongoDB 实现族
type MongoUserDAO struct{ col *mongo.Collection }
type MongoOrderDAO struct{ col *mongo.Collection }

type MongoDAOFactory struct{ db *mongo.Database }
func (f *MongoDAOFactory) CreateUserDAO() UserDAO   { return &MongoUserDAO{col: f.db.Collection("users")} }
func (f *MongoDAOFactory) CreateOrderDAO() OrderDAO { return &MongoOrderDAO{col: f.db.Collection("orders")} }

// 根据配置选择工厂
func NewDAOFactory(dbType string) DAOFactory {
    switch dbType {
    case "mysql":
        return &MySQLDAOFactory{db: mysqlDB}
    case "mongo":
        return &MongoDAOFactory{db: mongoDB}
    }
    return nil
}
```

## 5. 建造者模式？

**回答：**

```
分步构建复杂对象

  场景：
  - 对象有很多可选参数
  - 构造过程复杂需分步
  - 同一构造过程创建不同表示

Go 中两种实现：
  1. 建造者结构体（传统方式）
  2. 函数选项模式（Go 惯用）
```

```go
// 方式1：传造者模式
type ServerBuilder struct {
    host    string
    port    int
    timeout time.Duration
    maxConn int
    tls     bool
}

func NewServerBuilder(host string, port int) *ServerBuilder {
    return &ServerBuilder{host: host, port: port, timeout: 30 * time.Second, maxConn: 100}
}

func (b *ServerBuilder) WithTimeout(t time.Duration) *ServerBuilder {
    b.timeout = t
    return b
}

func (b *ServerBuilder) WithMaxConn(n int) *ServerBuilder {
    b.maxConn = n
    return b
}

func (b *ServerBuilder) WithTLS() *ServerBuilder {
    b.tls = true
    return b
}

func (b *ServerBuilder) Build() *Server {
    return &Server{
        host: b.host, port: b.port,
        timeout: b.timeout, maxConn: b.maxConn, tls: b.tls,
    }
}

// 链式调用
server := NewServerBuilder("0.0.0.0", 8080).
    WithTimeout(60 * time.Second).
    WithMaxConn(500).
    WithTLS().
    Build()
```

```go
// 方式2：函数选项模式（Go 推荐）
type Server struct {
    host    string
    port    int
    timeout time.Duration
    maxConn int
    tls     bool
}

type Option func(*Server)

func WithTimeout(t time.Duration) Option {
    return func(s *Server) { s.timeout = t }
}

func WithMaxConn(n int) Option {
    return func(s *Server) { s.maxConn = n }
}

func WithTLS() Option {
    return func(s *Server) { s.tls = true }
}

func NewServer(host string, port int, opts ...Option) *Server {
    s := &Server{
        host:    host,
        port:    port,
        timeout: 30 * time.Second,
        maxConn: 100,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// 使用
server := NewServer("0.0.0.0", 8080,
    WithTimeout(60*time.Second),
    WithMaxConn(500),
    WithTLS(),
)
```

## 6. 原型模式？

**回答：**

```
通过复制现有对象创建新对象

  场景：
  - 创建成本高（复杂初始化）
  - 需要大量相似对象
  - 运行时决定类型
```

```go
type Prototype interface {
    Clone() Prototype
}

type Config struct {
    Host     string
    Port     int
    Features map[string]bool
    Tags     []string
}

func (c *Config) Clone() *Config {
    // 深拷贝
    newConfig := &Config{
        Host: c.Host,
        Port: c.Port,
    }
    
    // Map 深拷贝
    newConfig.Features = make(map[string]bool, len(c.Features))
    for k, v := range c.Features {
        newConfig.Features[k] = v
    }
    
    // Slice 深拷贝
    newConfig.Tags = make([]string, len(c.Tags))
    copy(newConfig.Tags, c.Tags)
    
    return newConfig
}

// 使用
baseConfig := &Config{
    Host:     "localhost",
    Port:     8080,
    Features: map[string]bool{"auth": true, "log": true},
    Tags:     []string{"web", "api"},
}

// 基于原型创建新配置
devConfig := baseConfig.Clone()
devConfig.Host = "dev.example.com"

prodConfig := baseConfig.Clone()
prodConfig.Host = "prod.example.com"
prodConfig.Port = 443
```

```
Go 深拷贝注意点：
  基本类型：直接赋值
  Slice：make + copy
  Map：新建 + 遍历赋值
  指针：new + 赋值
  嵌套结构体：递归深拷贝
```

## 7. 简单工厂 vs 工厂方法 vs 抽象工厂？

**回答：**

```
  ┌──────────┬──────────────┬──────────────┬──────────────┐
  │          │ 简单工厂     │ 工厂方法     │ 抽象工厂     │
  ├──────────┼──────────────┼──────────────┼──────────────┤
  │ 结构     │ 一个函数     │ 一个接口     │ 一族接口     │
  │ 产品     │ 一种         │ 一种         │ 多种相关     │
  │ 新增产品 │ 修改工厂     │ 新增工厂     │ 新增工厂族   │
  │ 复杂度   │ 低           │ 中           │ 高           │
  │ Go惯用   │ ✅           │ 可以         │ 少用         │
  └──────────┴──────────────┴──────────────┴──────────────┘

  Go 中最常用：简单工厂函数
    func NewXxx(...) *Xxx
    func NewXxx(...) (Interface, error)
```

## 8. 函数选项模式详解？

**回答：**

```
Go 社区最推荐的创建模式

优势：
  - 默认值清晰
  - 可选参数灵活
  - 向后兼容（新增 Option 不破坏已有调用）
  - 自文档化
```

```go
// 实际项目示例：HTTP Client
type Client struct {
    baseURL    string
    timeout    time.Duration
    retries    int
    headers    map[string]string
    httpClient *http.Client
}

type ClientOption func(*Client)

func WithBaseTimeout(t time.Duration) ClientOption {
    return func(c *Client) { c.timeout = t }
}

func WithRetries(n int) ClientOption {
    return func(c *Client) { c.retries = n }
}

func WithHeader(key, value string) ClientOption {
    return func(c *Client) { c.headers[key] = value }
}

func WithHTTPClient(hc *http.Client) ClientOption {
    return func(c *Client) { c.httpClient = hc }
}

func NewClient(baseURL string, opts ...ClientOption) *Client {
    c := &Client{
        baseURL:    baseURL,
        timeout:    10 * time.Second,
        retries:    3,
        headers:    make(map[string]string),
        httpClient: &http.Client{},
    }
    for _, opt := range opts {
        opt(c)
    }
    c.httpClient.Timeout = c.timeout
    return c
}

// 简洁调用
client := NewClient("https://api.example.com")

// 自定义调用
client := NewClient("https://api.example.com",
    WithBaseTimeout(30*time.Second),
    WithRetries(5),
    WithHeader("Authorization", "Bearer token"),
)
```

## 9. 对象池模式？

**回答：**

```
预先创建对象放入池中复用

  场景：
  - 对象创建代价高（DB连接、buffer）
  - 需要频繁创建销毁
  - Go 中 sync.Pool 就是对象池
```

```go
// sync.Pool 使用
var bufPool = sync.Pool{
    New: func() interface{} {
        return new(bytes.Buffer)
    },
}

func ProcessRequest(data []byte) string {
    buf := bufPool.Get().(*bytes.Buffer)
    defer func() {
        buf.Reset()
        bufPool.Put(buf)
    }()
    
    buf.Write(data)
    buf.WriteString(" processed")
    return buf.String()
}

// 连接池模式
type ConnPool struct {
    pool chan *Connection
    factory func() (*Connection, error)
}

func NewConnPool(size int, factory func() (*Connection, error)) *ConnPool {
    p := &ConnPool{
        pool:    make(chan *Connection, size),
        factory: factory,
    }
    return p
}

func (p *ConnPool) Get(ctx context.Context) (*Connection, error) {
    select {
    case conn := <-p.pool:
        if conn.IsAlive() {
            return conn, nil
        }
        conn.Close()
        return p.factory()
    case <-ctx.Done():
        return nil, ctx.Err()
    default:
        return p.factory()
    }
}

func (p *ConnPool) Put(conn *Connection) {
    select {
    case p.pool <- conn:
    default:
        conn.Close() // 池满则关闭
    }
}
```

## 10. 创建型模式面试速答？

**回答：**

```
Q: Go 单例怎么实现？
A: sync.Once + 包级私有变量
   Do 只执行一次 并发安全

Q: 工厂方法和抽象工厂区别？
A: 工厂方法创建一种产品
   抽象工厂创建一族相关产品

Q: Go 中最常用的创建模式？
A: 函数选项模式(Functional Options)
   NewXxx(required, opts ...Option)

Q: 建造者模式什么时候用？
A: 参数多且有可选项
   构造过程复杂需分步

Q: 原型模式注意什么？
A: 必须深拷贝
   Map/Slice/指针要单独处理

Q: 对象池适用场景？
A: 创建代价高+频繁创建销毁
   sync.Pool / 连接池

Q: 简单工厂够用吗？
A: Go中大部分场景够用
   NewXxx函数就是简单工厂
   类型多且经常扩展才考虑工厂方法
```
