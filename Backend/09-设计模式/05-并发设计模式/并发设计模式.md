# 并发设计模式

## 1. Go 中有哪些并发模式？

**回答：**

```
  ┌──────────────────┬──────────────────────────┐
  │ 模式              │ 用途                     │
  ├──────────────────┼──────────────────────────┤
  │ Pipeline         │ 多阶段流水线处理         │
  │ Fan-Out/Fan-In   │ 并行分发 + 聚合结果      │
  │ Worker Pool      │ 固定协程池处理任务       │
  │ Pub/Sub          │ 发布订阅                 │
  │ Or-Done          │ 多任务竞速取最快         │
  │ Semaphore        │ 控制并发数               │
  │ Context 取消     │ 级联取消                 │
  │ errgroup         │ 并发错误收集             │
  └──────────────────┴──────────────────────────┘
```

## 2. Pipeline 管道模式？

**回答：**

```
多阶段流水线，每个阶段通过 channel 连接

  ┌───────┐  ch1  ┌───────┐  ch2  ┌───────┐
  │ 生成  │──────→│ 处理  │──────→│ 输出  │
  └───────┘       └───────┘       └───────┘
```

```go
// 生成阶段
func Generate(ctx context.Context, nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for _, n := range nums {
            select {
            case out <- n:
            case <-ctx.Done():
                return
            }
        }
    }()
    return out
}

// 处理阶段：平方
func Square(ctx context.Context, in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            select {
            case out <- n * n:
            case <-ctx.Done():
                return
            }
        }
    }()
    return out
}

// 处理阶段：过滤
func Filter(ctx context.Context, in <-chan int, predicate func(int) bool) <-chan int {
    out := make(chan int)
    go func() {
        defer close(out)
        for n := range in {
            if predicate(n) {
                select {
                case out <- n:
                case <-ctx.Done():
                    return
                }
            }
        }
    }()
    return out
}

// 组合管道
func main() {
    ctx, cancel := context.WithCancel(context.Background())
    defer cancel()
    
    // Generate → Square → Filter(>10)
    nums := Generate(ctx, 1, 2, 3, 4, 5)
    squared := Square(ctx, nums)
    filtered := Filter(ctx, squared, func(n int) bool { return n > 10 })
    
    for result := range filtered {
        fmt.Println(result) // 16, 25
    }
}
```

## 3. Fan-Out / Fan-In 模式？

**回答：**

```
Fan-Out：一个输入分发给多个 goroutine 并行处理
Fan-In：多个输出合并为一个 channel

  ┌───────┐     ┌────────┐
  │       │────→│Worker 1│────┐
  │ Input │────→│Worker 2│────┼──→ ┌────────┐
  │       │────→│Worker 3│────┘    │ Merge  │
  └───────┘     └────────┘        └────────┘
    Fan-Out                        Fan-In
```

```go
// Fan-Out：启动多个 worker
func FanOut(ctx context.Context, in <-chan Task, workerCount int) []<-chan Result {
    workers := make([]<-chan Result, workerCount)
    for i := 0; i < workerCount; i++ {
        workers[i] = worker(ctx, in)
    }
    return workers
}

func worker(ctx context.Context, in <-chan Task) <-chan Result {
    out := make(chan Result)
    go func() {
        defer close(out)
        for task := range in {
            select {
            case out <- process(task):
            case <-ctx.Done():
                return
            }
        }
    }()
    return out
}

// Fan-In：合并多个 channel
func FanIn(ctx context.Context, channels ...<-chan Result) <-chan Result {
    merged := make(chan Result)
    var wg sync.WaitGroup
    
    for _, ch := range channels {
        wg.Add(1)
        go func(c <-chan Result) {
            defer wg.Done()
            for result := range c {
                select {
                case merged <- result:
                case <-ctx.Done():
                    return
                }
            }
        }(ch)
    }
    
    go func() {
        wg.Wait()
        close(merged)
    }()
    
    return merged
}

// 使用
ctx := context.Background()
tasks := generateTasks(ctx)
workers := FanOut(ctx, tasks, runtime.NumCPU())
results := FanIn(ctx, workers...)

for r := range results {
    fmt.Println(r)
}
```

## 4. Worker Pool 模式？

**回答：**

```
固定数量 goroutine 消费任务队列

  场景：CPU密集任务、限制并发数、批量处理

  ┌──────┐     ┌────────────────┐     ┌──────┐
  │ Task │────→│   Job Queue    │────→│Result│
  │ 提交 │     │  (buffered ch) │     │ 收集 │
  └──────┘     └───┬───┬───┬────┘     └──────┘
                   │   │   │
                 ┌─┴┐ ┌┴─┐┌┴─┐
                 │W1│ │W2││W3│  Workers
                 └──┘ └──┘└──┘
```

```go
type WorkerPool struct {
    workerCount int
    jobs        chan func()
    wg          sync.WaitGroup
}

func NewWorkerPool(workerCount, queueSize int) *WorkerPool {
    p := &WorkerPool{
        workerCount: workerCount,
        jobs:        make(chan func(), queueSize),
    }
    p.start()
    return p
}

func (p *WorkerPool) start() {
    for i := 0; i < p.workerCount; i++ {
        p.wg.Add(1)
        go func() {
            defer p.wg.Done()
            for job := range p.jobs {
                job()
            }
        }()
    }
}

func (p *WorkerPool) Submit(job func()) {
    p.jobs <- job
}

func (p *WorkerPool) Shutdown() {
    close(p.jobs)
    p.wg.Wait()
}

// 使用
pool := NewWorkerPool(10, 100)
defer pool.Shutdown()

var results sync.Map
for i := 0; i < 1000; i++ {
    id := i
    pool.Submit(func() {
        result := heavyComputation(id)
        results.Store(id, result)
    })
}
```

## 5. errgroup 并发错误收集？

**回答：**

```go
import "golang.org/x/sync/errgroup"

func FetchAll(ctx context.Context, urls []string) (map[string][]byte, error) {
    g, ctx := errgroup.WithContext(ctx)
    
    var mu sync.Mutex
    results := make(map[string][]byte)
    
    for _, url := range urls {
        url := url // 捕获变量
        g.Go(func() error {
            req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
            resp, err := http.DefaultClient.Do(req)
            if err != nil {
                return fmt.Errorf("fetch %s: %w", url, err)
            }
            defer resp.Body.Close()
            
            body, err := io.ReadAll(resp.Body)
            if err != nil {
                return fmt.Errorf("read %s: %w", url, err)
            }
            
            mu.Lock()
            results[url] = body
            mu.Unlock()
            return nil
        })
    }
    
    // 等待所有完成，返回第一个错误
    if err := g.Wait(); err != nil {
        return nil, err
    }
    return results, nil
}

// 限制并发数的 errgroup
func ProcessItems(ctx context.Context, items []Item) error {
    g, ctx := errgroup.WithContext(ctx)
    g.SetLimit(10) // 最多 10 个并发
    
    for _, item := range items {
        item := item
        g.Go(func() error {
            return processItem(ctx, item)
        })
    }
    
    return g.Wait()
}
```

## 6. 信号量与并发控制？

**回答：**

```go
// 方式1：带缓冲 channel 做信号量
type Semaphore struct {
    ch chan struct{}
}

func NewSemaphore(n int) *Semaphore {
    return &Semaphore{ch: make(chan struct{}, n)}
}

func (s *Semaphore) Acquire(ctx context.Context) error {
    select {
    case s.ch <- struct{}{}:
        return nil
    case <-ctx.Done():
        return ctx.Err()
    }
}

func (s *Semaphore) Release() {
    <-s.ch
}

// 使用
sem := NewSemaphore(5) // 并发 5

for _, task := range tasks {
    task := task
    go func() {
        sem.Acquire(ctx)
        defer sem.Release()
        process(task)
    }()
}

// 方式2：golang.org/x/sync/semaphore
import "golang.org/x/sync/semaphore"

var sem = semaphore.NewWeighted(10)

func handler(ctx context.Context) error {
    if err := sem.Acquire(ctx, 1); err != nil {
        return err
    }
    defer sem.Release(1)
    
    return doWork(ctx)
}
```

## 7. Or-Done / 竞速模式？

**回答：**

```
多个任务同时执行，取最快的结果

场景：
  多个数据源查询取最快
  超时控制
  冗余请求提高可靠性
```

```go
// 多源竞速
func QueryFastest(ctx context.Context, query string, backends ...Backend) (Result, error) {
    ctx, cancel := context.WithCancel(ctx)
    defer cancel() // 最快的返回后取消其他
    
    resultCh := make(chan Result, len(backends))
    errCh := make(chan error, len(backends))
    
    for _, b := range backends {
        b := b
        go func() {
            result, err := b.Query(ctx, query)
            if err != nil {
                errCh <- err
                return
            }
            resultCh <- result
        }()
    }
    
    // 取第一个成功的
    for i := 0; i < len(backends); i++ {
        select {
        case result := <-resultCh:
            return result, nil
        case <-errCh:
            continue // 尝试下一个
        case <-ctx.Done():
            return Result{}, ctx.Err()
        }
    }
    return Result{}, errors.New("all backends failed")
}

// 超时竞速
func WithTimeout(ctx context.Context, fn func() (interface{}, error), timeout time.Duration) (interface{}, error) {
    ctx, cancel := context.WithTimeout(ctx, timeout)
    defer cancel()
    
    ch := make(chan struct {
        result interface{}
        err    error
    }, 1)
    
    go func() {
        result, err := fn()
        ch <- struct {
            result interface{}
            err    error
        }{result, err}
    }()
    
    select {
    case r := <-ch:
        return r.result, r.err
    case <-ctx.Done():
        return nil, ctx.Err()
    }
}
```

## 8. Context 取消传播？

**回答：**

```
Context 是 Go 并发控制的核心
  
  ┌─────────┐
  │ Root    │ context.Background()
  └────┬────┘
       ├── WithCancel → 手动取消
       ├── WithTimeout → 超时取消
       └── WithValue → 传递值

  取消级联：父 context 取消 → 所有子 context 取消
```

```go
func ProcessOrder(ctx context.Context, orderID string) error {
    // 限定整体超时
    ctx, cancel := context.WithTimeout(ctx, 30*time.Second)
    defer cancel()
    
    // 步骤1：验证库存
    if err := checkInventory(ctx, orderID); err != nil {
        return fmt.Errorf("inventory: %w", err)
    }
    
    // 步骤2：并发执行独立操作
    g, ctx := errgroup.WithContext(ctx)
    
    g.Go(func() error {
        return deductStock(ctx, orderID) // ctx 取消时自动停止
    })
    
    g.Go(func() error {
        return createPayment(ctx, orderID)
    })
    
    return g.Wait()
}

// 长连接中的 Context 使用
func StartWorker(ctx context.Context) {
    ticker := time.NewTicker(time.Second)
    defer ticker.Stop()
    
    for {
        select {
        case <-ctx.Done():
            log.Println("worker stopped:", ctx.Err())
            return
        case <-ticker.C:
            doWork()
        }
    }
}
```

## 9. singleflight 防击穿？

**回答：**

```
多个相同请求只执行一次，共享结果

场景：缓存击穿、热点查询

  请求1 ─┐
  请求2 ─┼──→ 只执行一次 ──→ 共享结果
  请求3 ─┘
```

```go
import "golang.org/x/sync/singleflight"

var group singleflight.Group

func GetUser(ctx context.Context, id string) (*User, error) {
    key := "user:" + id
    
    // 相同 key 的并发调用只执行一次
    result, err, shared := group.Do(key, func() (interface{}, error) {
        // 查缓存
        if user, err := cache.Get(ctx, key); err == nil {
            return user, nil
        }
        
        // 查 DB
        user, err := db.GetUser(ctx, id)
        if err != nil {
            return nil, err
        }
        
        // 写缓存
        cache.Set(ctx, key, user, 5*time.Minute)
        return user, nil
    })
    
    if err != nil {
        return nil, err
    }
    
    log.Printf("shared: %v", shared) // 是否共享了结果
    return result.(*User), nil
}
```

## 10. 并发模式面试速答？

**回答：**

```
Q: Pipeline 模式怎么实现？
A: 每个阶段一个 goroutine
   channel 连接 用 ctx 控制退出

Q: Fan-Out/Fan-In 是什么？
A: Fan-Out:一个输入分发给多个worker
   Fan-In:多个输出合并为一个channel

Q: Worker Pool 怎么实现？
A: buffered chan 作任务队列
   固定数量goroutine消费

Q: errgroup 用来做什么？
A: 并发执行+错误收集
   一个失败可取消其他
   SetLimit 限制并发数

Q: singleflight 解决什么？
A: 相同请求并发去重
   防缓存击穿

Q: 信号量怎么实现？
A: buffered channel
   或 x/sync/semaphore

Q: Context 取消怎么传播？
A: 父ctx取消→所有子ctx取消
   WithCancel/WithTimeout

Q: Or-Done 模式？
A: 多任务竞速取最快结果
   cancel 取消其他
```
