## 1. 常见排序算法的时间和空间复杂度比较？

**回答：**

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ✗ |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 希尔排序 | O(n log n) | O(n^1.3) | O(n²) | O(1) | ✗ |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ✗ |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ✗ |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(k) | ✅ |
| 基数排序 | O(d·n) | O(d·n) | O(d·n) | O(n+k) | ✅ |
| 桶排序 | O(n+k) | O(n+k) | O(n²) | O(n+k) | ✅ |

```
选择决策：
n 较小（< 50）      → 插入排序
基本有序             → 插入排序
要求稳定             → 归并排序
通用排序             → 快速排序（实际最快）
不要最坏 O(n²)       → 堆排序/归并排序
值域有限             → 计数/基数排序
```

## 2. 快速排序的原理和优化？

**回答：**

```go
func quickSort(arr []int, left, right int) {
    if left >= right {
        return
    }
    pivot := partition(arr, left, right)
    quickSort(arr, left, pivot-1)
    quickSort(arr, pivot+1, right)
}

func partition(arr []int, left, right int) int {
    pivot := arr[right]
    i := left
    for j := left; j < right; j++ {
        if arr[j] < pivot {
            arr[i], arr[j] = arr[j], arr[i]
            i++
        }
    }
    arr[i], arr[right] = arr[right], arr[i]
    return i
}
```

**优化策略：**

| 优化 | 方法 | 效果 |
|------|------|------|
| 三数取中 | 首/中/末取中值做 pivot | 避免最坏 O(n²) |
| 随机化 | 随机选 pivot | 概率上避免最坏 |
| 小数组切换 | n < 16 用插入排序 | 减少递归开销 |
| 三路分区 | 把等于 pivot 的聚在中间 | 大量重复元素时高效 |
| 尾递归优化 | 先递归小分区 | 减少栈深度到 O(log n) |

**三路快排：**

```go
// 荷兰国旗问题
func quickSort3Way(arr []int, lo, hi int) {
    if lo >= hi { return }
    lt, gt := lo, hi
    pivot := arr[lo]
    i := lo + 1
    for i <= gt {
        if arr[i] < pivot {
            arr[lt], arr[i] = arr[i], arr[lt]
            lt++; i++
        } else if arr[i] > pivot {
            arr[gt], arr[i] = arr[i], arr[gt]
            gt--
        } else {
            i++
        }
    }
    // arr[lo..lt-1] < pivot = arr[lt..gt] < arr[gt+1..hi]
    quickSort3Way(arr, lo, lt-1)
    quickSort3Way(arr, gt+1, hi)
}
```

## 3. 归并排序的原理？为什么适合外部排序？

**回答：**

```go
func mergeSort(arr []int) []int {
    if len(arr) <= 1 {
        return arr
    }
    mid := len(arr) / 2
    left := mergeSort(arr[:mid])
    right := mergeSort(arr[mid:])
    return merge(left, right)
}

func merge(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i])
            i++
        } else {
            result = append(result, b[j])
            j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```

**适合外部排序的原因：**

```
100GB 文件，内存只有 1GB：

1. 分块：将文件分成 100 个 1GB 的块
2. 内排序：每个块在内存中排序（快排）
3. 多路归并：用最小堆合并 100 个有序块
   - 每块读一小段到内存缓冲区
   - 堆顶弹出最小值写入输出
   - 缓冲区空了从磁盘读下一段

归并排序是顺序读写，对磁盘IO友好
快排是随机访问，不适合外部排序
```

## 4. 堆排序的原理和应用场景？

**回答：**

```go
func heapSort(arr []int) {
    n := len(arr)
    // 建最大堆 O(n)
    for i := n/2 - 1; i >= 0; i-- {
        heapify(arr, n, i)
    }
    // 依次取最大放末尾 O(n log n)
    for i := n - 1; i > 0; i-- {
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)
    }
}

func heapify(arr []int, n, i int) {
    largest := i
    left, right := 2*i+1, 2*i+2
    if left < n && arr[left] > arr[largest] {
        largest = left
    }
    if right < n && arr[right] > arr[largest] {
        largest = right
    }
    if largest != i {
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)
    }
}
```

**特点：**
- 最坏也是 O(n log n)（不像快排有 O(n²)）
- 原地排序 O(1) 空间
- 不稳定
- 实际慢于快排（缓存不友好，比较分散）

**适用场景：** 内存紧张需要原地排序，对最坏复杂度有要求（如实时系统），Top K 问题。

## 5. 计数排序、基数排序、桶排序的区别？

**回答：**

**计数排序 — O(n+k)，k 为值域范围：**

```go
func countingSort(arr []int, maxVal int) []int {
    count := make([]int, maxVal+1)
    for _, v := range arr {
        count[v]++
    }
    result := make([]int, 0, len(arr))
    for i, c := range count {
        for j := 0; j < c; j++ {
            result = append(result, i)
        }
    }
    return result
}
```

**基数排序 — O(d·n)，d 为最大位数：**

```
按位排序（LSD，从最低位到最高位）：
原始:  170, 45, 75, 90, 802, 24, 2, 66
个位:  170, 90, 802, 2, 24, 45, 75, 66
十位:  802, 2, 24, 45, 66, 170, 75, 90
百位:  2, 24, 45, 66, 75, 90, 170, 802
```

**桶排序 — O(n+k)，假设均匀分布：**

```
将数据分到 k 个桶中，每个桶内排序

[0.23, 0.87, 0.12, 0.65, 0.34]
桶0: [0.12] 
桶1: [0.23]
桶2: [0.34]
...
桶6: [0.65]
桶8: [0.87]
按桶顺序合并即可
```

| 算法 | 前提 | 时间 | 适用 |
|------|------|------|------|
| 计数 | 非负整数，值域小 | O(n+k) | 年龄、成绩 |
| 基数 | 整数 | O(d·n) | 电话号码 |
| 桶 | 均匀分布 | O(n) 平均 | 浮点数 |

## 6. Go/Python 中的排序是如何实现的？

**回答：**

**Go `sort.Sort` / `slices.Sort`：**

```
Go 使用 pdqsort（Pattern-Defeating Quicksort）：
1. 小数组（≤ 12）→ 插入排序
2. 一般情况 → 快速排序（三数取中 + Hoare 分区）
3. 递归过深 → 切换堆排序（防止最坏 O(n²)）
4. 检测到部分有序 → 利用已有顺序

Go 1.19+ slices.Sort 使用 pdqsort
Go 1.18- sort.Sort 使用 introsort（快排+堆排+插入排序）
```

**Python `sorted()` / `list.sort()`：**

```
Python 使用 Timsort：
1. 找到已有的有序段（run）
2. 短 run 用插入排序扩展到最小长度（32-64）
3. 归并排序合并 runs
4. 利用 galloping mode 优化归并

特点：
- 最好 O(n)（已有序）
- 平均/最坏 O(n log n)
- 稳定排序
- 空间 O(n)
```

| 语言 | 算法 | 稳定 | 特点 |
|------|------|------|------|
| Go | pdqsort | 否（Sort）/是（SliceStable）| 自适应，结合快排+堆排 |
| Python | Timsort | 是 | 利用已有顺序，实际性能优秀 |
| Java | Timsort（对象）/ DualPivotQuicksort（基本类型）| 对象稳定 | 分类处理 |
| C++ | Introsort | 否 | 快排+堆排+插入排序 |

## 7. 什么是排序的稳定性？有什么实际意义？

**回答：**

**稳定排序：** 相等元素的相对顺序在排序后不变。

```
原始：[(A,3), (B,1), (C,3), (D,2)]

稳定排序（按数字）：
[(B,1), (D,2), (A,3), (C,3)]  ← A 在 C 前面（保持原序）

不稳定排序：
[(B,1), (D,2), (C,3), (A,3)]  ← 可能 C 在 A 前面
```

**稳定排序：** 冒泡、插入、归并、计数、基数、桶、Timsort
**不稳定排序：** 选择、快排、堆排、希尔

**实际意义：**
- 多关键字排序：先按次要关键字排，再按主要关键字稳定排序
- 数据库 ORDER BY 多列
- UI 列表：用户先按名字排序，再按日期排序时保持同日期的名字顺序

## 8. 如何在 O(n) 时间内找到第 K 大的元素？

**回答：**

**快速选择算法（QuickSelect）：**

```go
func findKthLargest(nums []int, k int) int {
    target := len(nums) - k  // 转换为第 target 小
    return quickSelect(nums, 0, len(nums)-1, target)
}

func quickSelect(nums []int, left, right, target int) int {
    if left == right {
        return nums[left]
    }
    // 随机化 pivot 避免最坏情况
    randIdx := left + rand.Intn(right-left+1)
    nums[randIdx], nums[right] = nums[right], nums[randIdx]

    pivot := partition(nums, left, right)
    if pivot == target {
        return nums[pivot]
    } else if pivot < target {
        return quickSelect(nums, pivot+1, right, target)
    }
    return quickSelect(nums, left, pivot-1, target)
}
```

| 方法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 排序 | O(n log n) | O(1)~O(n) | 最简单 |
| 最小堆 | O(n log k) | O(k) | 适合数据流 |
| 快速选择 | O(n) 平均 | O(1) | 最优 |
| BFPRT | O(n) 最坏 | O(n) | 理论最优，常数大 |

## 9. 链表排序和数组排序有什么区别？

**回答：**

| 特性 | 数组排序 | 链表排序 |
|------|---------|---------|
| 随机访问 | O(1) | O(n) |
| 适合快排 | ✅ | 不适合（随机访问慢）|
| 适合归并 | ✅（但需要额外空间）| ✅（天然适合，O(1) 空间）|
| 交换 | O(1) | O(1)（改指针）|

**链表归并排序 — O(n log n)，空间 O(log n)（递归栈）：**

```go
func sortList(head *ListNode) *ListNode {
    if head == nil || head.Next == nil {
        return head
    }
    // 快慢指针找中点
    slow, fast := head, head.Next
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
    }
    mid := slow.Next
    slow.Next = nil // 断开

    left := sortList(head)
    right := sortList(mid)
    return mergeTwoLists(left, right)
}

func mergeTwoLists(l1, l2 *ListNode) *ListNode {
    dummy := &ListNode{}
    curr := dummy
    for l1 != nil && l2 != nil {
        if l1.Val <= l2.Val {
            curr.Next = l1
            l1 = l1.Next
        } else {
            curr.Next = l2
            l2 = l2.Next
        }
        curr = curr.Next
    }
    if l1 != nil { curr.Next = l1 }
    if l2 != nil { curr.Next = l2 }
    return dummy.Next
}
```

## 10. 排序算法面试速答？

**回答：**

```
Q: 快排平均和最坏时间复杂度？
A: 平均 O(n log n)，最坏 O(n²)（已排序+固定取首/尾做 pivot）

Q: 归并排序的空间复杂度？
A: O(n)（数组版），O(log n)（链表版，只有递归栈）

Q: 哪些排序是稳定的？
A: 冒泡、插入、归并、计数、基数、桶、Timsort

Q: 堆排序比快排慢的原因？
A: 缓存不友好，堆的比较和交换是不连续内存访问

Q: 为什么实际工程中快排最常用？
A: 平均性能最好，常数因子小，缓存友好

Q: 一亿个整数排序选什么算法？
A: 内存够→快排/Timsort；内存不够→外部归并排序

Q: 10万个 0-999 的数排序？
A: 计数排序 O(n+k)

Q: 什么是逆序对？怎么统计？
A: (i, j) 满足 i<j 且 a[i]>a[j]。归并排序过程中统计，O(n log n)

Q: 快排的分区(partition)有什么用途？
A: 第K大、荷兰国旗、三向切分

Q: Go 的 sort.Slice 是稳定排序吗？
A: 不是，要用 sort.SliceStable
```
