## 1. 图的基本概念和存储方式？

**回答：**

```
图的分类：
┌──────────────────────────────────────┐
│ 有向图    无向图    加权图    DAG     │
│ A → B    A — B    A—3→B   无环有向图 │
└──────────────────────────────────────┘
```

**两种存储方式：**

```
邻接矩阵：                  邻接表：
  A B C D                   A → [B, C]
A 0 1 1 0                   B → [D]
B 0 0 0 1                   C → [D]
C 0 0 0 1                   D → []
D 0 0 0 0

空间 O(V²)                  空间 O(V+E)
```

| 特性 | 邻接矩阵 | 邻接表 |
|------|---------|--------|
| 空间 | O(V²) | O(V+E) |
| 查边 | O(1) | O(degree) |
| 遍历邻居 | O(V) | O(degree) |
| 适用 | 稠密图 | 稀疏图（大多数场景）|

```go
// Go 邻接表表示
type Graph struct {
    adj map[int][]int
}

func (g *Graph) AddEdge(u, v int) {
    g.adj[u] = append(g.adj[u], v)
    g.adj[v] = append(g.adj[v], u) // 无向图
}
```

## 2. BFS 和 DFS 的实现和应用？

**回答：**

**BFS（广度优先搜索）— 用队列：**

```go
func bfs(graph map[int][]int, start int) []int {
    visited := map[int]bool{start: true}
    queue := []int{start}
    var result []int

    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        result = append(result, node)
        for _, neighbor := range graph[node] {
            if !visited[neighbor] {
                visited[neighbor] = true
                queue = append(queue, neighbor)
            }
        }
    }
    return result
}
```

**DFS（深度优先搜索）— 用栈/递归：**

```go
func dfs(graph map[int][]int, node int, visited map[int]bool) {
    visited[node] = true
    fmt.Println(node)
    for _, neighbor := range graph[node] {
        if !visited[neighbor] {
            dfs(graph, neighbor, visited)
        }
    }
}
```

| 特性 | BFS | DFS |
|------|-----|-----|
| 数据结构 | 队列 | 栈（递归） |
| 空间 | O(宽度) | O(深度) |
| 最短路径 | ✅ 无权图 | ✗ |
| 应用 | 层序遍历、最短路 | 连通性、拓扑排序、回溯 |

## 3. 最短路径算法有哪些？

**回答：**

**Dijkstra — 单源最短路（非负权）：**

```go
func dijkstra(graph [][]Edge, start int) []int {
    n := len(graph)
    dist := make([]int, n)
    for i := range dist {
        dist[i] = math.MaxInt64
    }
    dist[start] = 0

    // 最小堆：(距离, 节点)
    h := &MinHeap{}
    heap.Push(h, [2]int{0, start})

    for h.Len() > 0 {
        curr := heap.Pop(h).([2]int)
        d, u := curr[0], curr[1]
        if d > dist[u] {
            continue // 已有更短路径
        }
        for _, e := range graph[u] {
            if dist[u]+e.weight < dist[e.to] {
                dist[e.to] = dist[u] + e.weight
                heap.Push(h, [2]int{dist[e.to], e.to})
            }
        }
    }
    return dist
}
// 时间 O((V+E) log V)
```

**算法对比：**

| 算法 | 适用 | 时间 | 负权 |
|------|------|------|------|
| Dijkstra | 单源，非负权 | O((V+E)log V) | ✗ |
| Bellman-Ford | 单源，可负权 | O(VE) | ✅ |
| Floyd-Warshall | 全源 | O(V³) | ✅ |
| SPFA | Bellman-Ford 优化 | 平均 O(E) | ✅ |
| A* | 启发式搜索 | 取决于启发函数 | ✗ |

## 4. 拓扑排序的原理和实现？

**回答：**

拓扑排序：将 DAG（有向无环图）的节点按依赖关系排成线性序列。

```
DAG：          拓扑排序结果：
A → B → D     A, C, B, D  或
A → C → D     C, A, B, D  等
```

**Kahn 算法（BFS，入度法）：**

```go
func topologicalSort(n int, edges [][]int) []int {
    inDegree := make([]int, n)
    graph := make([][]int, n)
    for _, e := range edges {
        graph[e[0]] = append(graph[e[0]], e[1])
        inDegree[e[1]]++
    }

    queue := []int{}
    for i := 0; i < n; i++ {
        if inDegree[i] == 0 {
            queue = append(queue, i)
        }
    }

    var result []int
    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        result = append(result, node)
        for _, neighbor := range graph[node] {
            inDegree[neighbor]--
            if inDegree[neighbor] == 0 {
                queue = append(queue, neighbor)
            }
        }
    }

    if len(result) != n {
        return nil // 有环
    }
    return result
}
```

**应用：** 课程安排、任务调度、编译依赖、包管理器依赖解析。

## 5. 如何判断图中是否有环？

**回答：**

**无向图 — 并查集：**

```go
func hasCycleUndirected(n int, edges [][]int) bool {
    parent := make([]int, n)
    for i := range parent {
        parent[i] = i
    }
    var find func(int) int
    find = func(x int) int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }
    for _, e := range edges {
        px, py := find(e[0]), find(e[1])
        if px == py {
            return true // 两端已连通，加边成环
        }
        parent[px] = py
    }
    return false
}
```

**有向图 — DFS 三色标记：**

```go
func hasCycleDirected(graph [][]int, n int) bool {
    // 0: 未访问, 1: 访问中, 2: 已完成
    color := make([]int, n)
    var dfs func(int) bool
    dfs = func(u int) bool {
        color[u] = 1 // 标记为访问中
        for _, v := range graph[u] {
            if color[v] == 1 { return true }  // 后向边 → 有环
            if color[v] == 0 && dfs(v) { return true }
        }
        color[u] = 2 // 标记为已完成
        return false
    }
    for i := 0; i < n; i++ {
        if color[i] == 0 && dfs(i) {
            return true
        }
    }
    return false
}
```

## 6. 并查集（Union-Find）的原理和优化？

**回答：**

```go
type UnionFind struct {
    parent []int
    rank   []int
    count  int // 连通分量数
}

func NewUnionFind(n int) *UnionFind {
    parent := make([]int, n)
    rank := make([]int, n)
    for i := range parent {
        parent[i] = i
    }
    return &UnionFind{parent, rank, n}
}

// 路径压缩
func (uf *UnionFind) Find(x int) int {
    if uf.parent[x] != x {
        uf.parent[x] = uf.Find(uf.parent[x])
    }
    return uf.parent[x]
}

// 按秩合并
func (uf *UnionFind) Union(x, y int) bool {
    px, py := uf.Find(x), uf.Find(y)
    if px == py {
        return false
    }
    if uf.rank[px] < uf.rank[py] {
        px, py = py, px
    }
    uf.parent[py] = px
    if uf.rank[px] == uf.rank[py] {
        uf.rank[px]++
    }
    uf.count--
    return true
}
```

| 优化 | 效果 |
|------|------|
| 路径压缩 | Find 接近 O(1) |
| 按秩合并 | 树高更平衡 |
| 两个都用 | 均摊 O(α(n)) ≈ O(1) |

**应用：** 连通性判断、Kruskal 最小生成树、社交网络关系、等式方程可满足性。

## 7. 最小生成树算法？

**回答：**

**Kruskal 算法（贪心 + 并查集）— O(E log E)：**

```go
func kruskal(n int, edges [][]int) int {
    // edges: [u, v, weight]
    sort.Slice(edges, func(i, j int) bool {
        return edges[i][2] < edges[j][2]
    })

    uf := NewUnionFind(n)
    totalWeight := 0
    edgeCount := 0

    for _, e := range edges {
        if uf.Union(e[0], e[1]) {
            totalWeight += e[2]
            edgeCount++
            if edgeCount == n-1 {
                break
            }
        }
    }
    return totalWeight
}
```

**Prim 算法（贪心 + 最小堆）— O(E log V)：**

```
从任一点出发，每次选连接已选点集和未选点集的最小边。

适用：稠密图（邻接矩阵 O(V²)）
Kruskal 适用：稀疏图
```

| 特性 | Kruskal | Prim |
|------|---------|------|
| 策略 | 边排序 + 并查集 | 点扩展 + 最小堆 |
| 时间 | O(E log E) | O(E log V) |
| 适用 | 稀疏图 | 稠密图 |

## 8. A* 搜索算法的原理？

**回答：**

A* 是带启发函数的 Dijkstra，用于路径搜索：

```
f(n) = g(n) + h(n)
- g(n): 起点到 n 的实际代价
- h(n): n 到终点的估计代价（启发函数）
- f(n): 总估计代价

启发函数要求（可采纳性）：h(n) ≤ 实际代价
常用：曼哈顿距离、欧几里得距离
```

```go
func aStar(grid [][]int, start, end [2]int) int {
    m, n := len(grid), len(grid[0])
    // f = g + h
    gScore := make([][]int, m)
    for i := range gScore {
        gScore[i] = make([]int, n)
        for j := range gScore[i] {
            gScore[i][j] = math.MaxInt64
        }
    }
    gScore[start[0]][start[1]] = 0

    // 优先队列：按 f 值排序
    pq := &PQ{}
    heap.Push(pq, &Node{
        pos:  start,
        f:    heuristic(start, end),
    })

    dirs := [4][2]int{{0,1},{0,-1},{1,0},{-1,0}}
    for pq.Len() > 0 {
        curr := heap.Pop(pq).(*Node)
        if curr.pos == end {
            return gScore[end[0]][end[1]]
        }
        for _, d := range dirs {
            nx, ny := curr.pos[0]+d[0], curr.pos[1]+d[1]
            if nx < 0 || nx >= m || ny < 0 || ny >= n || grid[nx][ny] == 1 {
                continue
            }
            newG := gScore[curr.pos[0]][curr.pos[1]] + 1
            if newG < gScore[nx][ny] {
                gScore[nx][ny] = newG
                heap.Push(pq, &Node{
                    pos: [2]int{nx, ny},
                    f:   newG + heuristic([2]int{nx, ny}, end),
                })
            }
        }
    }
    return -1
}

func heuristic(a, b [2]int) int {
    return abs(a[0]-b[0]) + abs(a[1]-b[1]) // 曼哈顿距离
}
```

**应用：** 游戏寻路、地图导航、机器人路径规划。

## 9. 二分图和网络流的基本概念？

**回答：**

**二分图：** 节点可分为两组，边只在两组之间。

```
二分图：            非二分图（奇环）：
A1 — B1             A — B
A2 — B2              \ |
A1 — B3               C（三角形）
```

**判断二分图 — 染色法：**

```go
func isBipartite(graph [][]int) bool {
    n := len(graph)
    color := make([]int, n) // 0:未染, 1:红, 2:蓝

    for i := 0; i < n; i++ {
        if color[i] != 0 {
            continue
        }
        queue := []int{i}
        color[i] = 1
        for len(queue) > 0 {
            node := queue[0]
            queue = queue[1:]
            for _, neighbor := range graph[node] {
                if color[neighbor] == 0 {
                    color[neighbor] = 3 - color[node]
                    queue = append(queue, neighbor)
                } else if color[neighbor] == color[node] {
                    return false
                }
            }
        }
    }
    return true
}
```

**网络流：** 在有向图中求从源到汇的最大流量。Ford-Fulkerson 方法通过不断寻找增广路径增加流量。应用：最大匹配、最小割、任务分配。

## 10. 图与图算法面试速答？

**回答：**

```
Q: 邻接矩阵和邻接表哪个更常用？
A: 邻接表，因为大多数图是稀疏的，O(V+E) 空间

Q: BFS 能求最短路径吗？
A: 能，但仅限无权图（或所有边权重相同）

Q: Dijkstra 能处理负权边吗？
A: 不能，用 Bellman-Ford

Q: 拓扑排序的前提条件？
A: 有向无环图（DAG），有环则不存在拓扑排序

Q: 并查集的 Find 操作均摊复杂度？
A: O(α(n)) ≈ O(1)，α 是阿克曼函数的反函数

Q: 图的连通分量怎么求？
A: DFS/BFS 遍历标记，或并查集

Q: 什么是强连通分量？
A: 有向图中任意两点互相可达的最大子图，Tarjan/Kosaraju 算法

Q: 欧拉路径和哈密顿路径的区别？
A: 欧拉经过每条边恰好一次，哈密顿经过每个点恰好一次

Q: 最小生成树一定唯一吗？
A: 不一定，边权有重复时可能有多个

Q: A* 和 Dijkstra 的关系？
A: A* = Dijkstra + 启发函数 h(n)。h(n)=0 时 A* 退化为 Dijkstra
```
