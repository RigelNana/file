# 堆与优先队列

---

## 1. 堆的定义和分类？

**回答：**

堆是一种**完全二叉树**，满足堆序性质：

```
最小堆（Min-Heap）：         最大堆（Max-Heap）：
父 ≤ 子                     父 ≥ 子

      1                           9
     / \                         / \
    3   2                       7   8
   / \ / \                     / \ / \
  7  5 4  6                   3  5 4  6

数组表示：                    数组表示：
[1, 3, 2, 7, 5, 4, 6]       [9, 7, 8, 3, 5, 4, 6]
```

| 特性 | 说明 |
|------|------|
| 结构 | 完全二叉树（数组存储）|
| 父节点索引 | (i-1)/2 |
| 左孩子索引 | 2*i+1 |
| 右孩子索引 | 2*i+2 |
| 插入 | O(log n) — 尾部加入后上浮 |
| 删除堆顶 | O(log n) — 尾部换堆顶后下沉 |
| 取堆顶 | O(1) |
| 建堆 | O(n) — 自底向上 |

## 2. 如何实现堆的上浮和下沉操作？

**回答：**

```go
type MinHeap struct {
    data []int
}

// 上浮（插入后调整）
func (h *MinHeap) siftUp(i int) {
    for i > 0 {
        parent := (i - 1) / 2
        if h.data[i] >= h.data[parent] {
            break
        }
        h.data[i], h.data[parent] = h.data[parent], h.data[i]
        i = parent
    }
}

// 下沉（删除堆顶后调整）
func (h *MinHeap) siftDown(i int) {
    n := len(h.data)
    for {
        smallest := i
        left, right := 2*i+1, 2*i+2
        if left < n && h.data[left] < h.data[smallest] {
            smallest = left
        }
        if right < n && h.data[right] < h.data[smallest] {
            smallest = right
        }
        if smallest == i {
            break
        }
        h.data[i], h.data[smallest] = h.data[smallest], h.data[i]
        i = smallest
    }
}

// 插入
func (h *MinHeap) Push(val int) {
    h.data = append(h.data, val)
    h.siftUp(len(h.data) - 1)
}

// 弹出堆顶
func (h *MinHeap) Pop() int {
    n := len(h.data)
    top := h.data[0]
    h.data[0] = h.data[n-1]
    h.data = h.data[:n-1]
    if len(h.data) > 0 {
        h.siftDown(0)
    }
    return top
}
```

```
插入5的过程（上浮）：
[1, 3, 8, 7, 9, _, _]

Step 1: 加到尾部
[1, 3, 8, 7, 9, 5]
                 ↑

Step 2: 5 < 8(父)，交换
[1, 3, 5, 7, 9, 8]
         ↑

Step 3: 5 > 1(父)，停止
```

## 3. 为什么建堆的时间复杂度是 O(n) 而不是 O(n log n)？

**回答：**

```go
// 自底向上建堆 — O(n)
func buildHeap(arr []int) {
    n := len(arr)
    // 从最后一个非叶节点开始下沉
    for i := n/2 - 1; i >= 0; i-- {
        siftDown(arr, i, n)
    }
}
```

**直觉理解：**

```
层次分析（n 个节点的完全二叉树）：
┌─────────────────────────────────────────────┐
│ 层 k │ 节点数    │ 下沉最大次数 │ 总操作    │
│──────│──────────│─────────────│──────────│
│ 0    │ 1        │ h           │ h        │
│ 1    │ 2        │ h-1         │ 2(h-1)   │
│ ...  │ ...      │ ...         │ ...      │
│ h-1  │ 2^(h-1)  │ 1           │ 2^(h-1)  │
│ h    │ 叶子节点  │ 0           │ 0        │
└─────────────────────────────────────────────┘

总操作 = Σ 2^k × (h-k) = O(n)
```

**数学证明（错位相减法）：**

$S = \sum_{k=0}^{h} 2^k(h-k) = 2^{h+1} - h - 2 = O(n)$

而逐个插入建堆：每次插入上浮 O(log n)，共 n 次 → O(n log n)。

## 4. 堆排序的原理和实现？

**回答：**

```
堆排序思路：
1. 建最大堆 O(n)
2. 循环 n-1 次：
   - 堆顶（最大值）与末尾交换
   - 堆大小减1
   - 对堆顶执行下沉 O(log n)
总时间：O(n log n)，空间 O(1)
```

```go
func heapSort(arr []int) {
    n := len(arr)
    // 建最大堆
    for i := n/2 - 1; i >= 0; i-- {
        siftDown(arr, i, n)
    }
    // 排序
    for i := n - 1; i > 0; i-- {
        arr[0], arr[i] = arr[i], arr[0]  // 最大值放到末尾
        siftDown(arr, 0, i)              // 缩小堆范围
    }
}

func siftDown(arr []int, i, n int) {
    for {
        largest := i
        left, right := 2*i+1, 2*i+2
        if left < n && arr[left] > arr[largest] {
            largest = left
        }
        if right < n && arr[right] > arr[largest] {
            largest = right
        }
        if largest == i {
            break
        }
        arr[i], arr[largest] = arr[largest], arr[i]
        i = largest
    }
}
```

| 特性 | 堆排序 |
|------|--------|
| 时间 | O(n log n)（最好/平均/最坏）|
| 空间 | O(1) 原地排序 |
| 稳定性 | 不稳定 |
| 优势 | 最坏情况也是 O(n log n) |
| 劣势 | 缓存不友好，实际性能不如快排 |

## 5. Top-K 问题的多种解法？

**回答：**

**问题：从 n 个元素中找出前 K 个最大的元素**

**方法1：最小堆 — O(n log K)，空间 O(K)**

```go
func topK(nums []int, k int) []int {
    h := &IntHeap{}
    for _, num := range nums {
        heap.Push(h, num)
        if h.Len() > k {
            heap.Pop(h) // 弹出最小的，保留K个最大
        }
    }
    return *h
}
```

**方法2：快速选择 — 平均 O(n)，空间 O(1)**

```go
func findKthLargest(nums []int, k int) int {
    target := len(nums) - k
    left, right := 0, len(nums)-1
    for left < right {
        pivot := partition(nums, left, right)
        if pivot == target {
            return nums[pivot]
        } else if pivot < target {
            left = pivot + 1
        } else {
            right = pivot - 1
        }
    }
    return nums[left]
}
```

| 方法 | 时间 | 空间 | 适用场景 |
|------|------|------|---------|
| 排序 | O(n log n) | O(1) | 简单，n 不大 |
| 最小堆 | O(n log K) | O(K) | K 远小于 n |
| 快速选择 | O(n) 平均 | O(1) | 一次性查找 |
| 计数排序 | O(n+m) | O(m) | 值域有限 |

## 6. 如何用堆实现数据流中位数？

**回答：**

```
思路：维护两个堆
- 最大堆（左半部分）：存较小的一半
- 最小堆（右半部分）：存较大的一半
- 保持大小平衡（差值 ≤ 1）

      maxHeap ≤ minHeap
     [1,2,3]    [4,5,6]
         ↑          ↑
       堆顶3      堆顶4

中位数 = (3 + 4) / 2 = 3.5
```

```go
type MedianFinder struct {
    maxHeap *MaxHeap // 左半部分（较小的）
    minHeap *MinHeap // 右半部分（较大的）
}

func (m *MedianFinder) AddNum(num int) {
    // 先加入 maxHeap
    heap.Push(m.maxHeap, num)
    // 把 maxHeap 最大值移到 minHeap（保证左 ≤ 右）
    heap.Push(m.minHeap, heap.Pop(m.maxHeap))
    // 平衡大小（maxHeap 可以多1个）
    if m.minHeap.Len() > m.maxHeap.Len() {
        heap.Push(m.maxHeap, heap.Pop(m.minHeap))
    }
}

func (m *MedianFinder) FindMedian() float64 {
    if m.maxHeap.Len() > m.minHeap.Len() {
        return float64(m.maxHeap.Peek())
    }
    return float64(m.maxHeap.Peek()+m.minHeap.Peek()) / 2.0
}
```

**时间复杂度：** AddNum O(log n)，FindMedian O(1)

## 7. Go 标准库的 heap 包如何使用？

**回答：**

```go
// 需要实现 heap.Interface
type heap.Interface interface {
    sort.Interface         // Len, Less, Swap
    Push(x interface{})    // 添加元素
    Pop() interface{}      // 移除并返回最后一个元素
}

// 自定义优先队列示例
type Item struct {
    value    string
    priority int
    index    int
}

type PriorityQueue []*Item

func (pq PriorityQueue) Len() int { return len(pq) }
func (pq PriorityQueue) Less(i, j int) bool {
    return pq[i].priority > pq[j].priority // 最大堆
}
func (pq PriorityQueue) Swap(i, j int) {
    pq[i], pq[j] = pq[j], pq[i]
    pq[i].index = i
    pq[j].index = j
}
func (pq *PriorityQueue) Push(x any) {
    item := x.(*Item)
    item.index = len(*pq)
    *pq = append(*pq, item)
}
func (pq *PriorityQueue) Pop() any {
    old := *pq
    n := len(old)
    item := old[n-1]
    old[n-1] = nil
    *pq = old[:n-1]
    return item
}

// 使用
pq := &PriorityQueue{}
heap.Init(pq)
heap.Push(pq, &Item{value: "task1", priority: 3})
heap.Push(pq, &Item{value: "task2", priority: 1})
top := heap.Pop(pq).(*Item) // task1 (priority=3)

// 修改优先级
pq[0].priority = 10
heap.Fix(pq, 0)  // O(log n) 修复堆
```

## 8. 多路归并问题怎么用堆解决？

**回答：**

**场景：合并 K 个有序数组/链表/文件**

```
K 个有序数组：
[1, 5, 9]
[2, 4, 8]
[3, 6, 7]

最小堆（存每个数组当前最小元素）：
初始堆：[1(arr0), 2(arr1), 3(arr2)]

弹出1 → 结果[1]，加入arr0下一个5
堆：[2, 5, 3]

弹出2 → 结果[1,2]，加入arr1下一个4
堆：[3, 5, 4]

... 直到所有数组耗尽
```

```go
type Element struct {
    val   int
    arrIdx int  // 来自哪个数组
    idx    int  // 在数组中的下标
}

func mergeKSorted(arrays [][]int) []int {
    h := &MinHeap{}
    // 每个数组的第一个元素入堆
    for i, arr := range arrays {
        if len(arr) > 0 {
            heap.Push(h, Element{arr[0], i, 0})
        }
    }

    var result []int
    for h.Len() > 0 {
        e := heap.Pop(h).(Element)
        result = append(result, e.val)
        // 同一数组的下一个元素入堆
        if e.idx+1 < len(arrays[e.arrIdx]) {
            heap.Push(h, Element{
                arrays[e.arrIdx][e.idx+1], e.arrIdx, e.idx + 1,
            })
        }
    }
    return result
}
// 时间 O(N log K)，N 是总元素数
```

**应用：外部排序、MapReduce 归并阶段、合并多个日志流**

## 9. 堆和优先队列在实际系统中的应用？

**回答：**

| 应用场景 | 堆类型 | 说明 |
|---------|--------|------|
| 任务调度 | 最小堆（按时间）| 定时任务、Go timer |
| Dijkstra | 最小堆（按距离）| 最短路径算法 |
| Huffman 编码 | 最小堆（按频率）| 数据压缩 |
| 限流器 | 最小堆（按过期时间）| Token bucket |
| K 路归并 | 最小堆 | 外部排序、多流合并 |
| 中位数维护 | 大+小堆 | 数据流统计 |
| 操作系统调度 | 多级反馈队列 | 进程优先级 |

**Go runtime 的 timer 堆：**

```
Go 1.14+ timer 实现：
- 每个 P 一个四叉堆（减少缓存未命中）
- 四叉堆比二叉堆更矮，缓存局部性更好
- 用于：time.After, time.Ticker, context.WithTimeout

Go 网络 poller 的截止时间也用堆管理
```

## 10. 堆与优先队列面试速答？

**回答：**

```
Q: 堆和二叉搜索树的区别？
A: 堆只保证父子关系（父≤子或父≥子），BST 保证左<根<右。堆用数组，BST用指针

Q: 最小堆的堆顶是什么？
A: 最小值

Q: 堆是用什么存储的？
A: 数组。完全二叉树天然适合数组存储，无需额外指针

Q: 建堆为什么 O(n) 不是 O(n log n)？
A: 自底向上建堆，大部分节点在底层（下沉距离短），数学证明级数收敛到 O(n)

Q: 优先队列和堆的关系？
A: 优先队列是抽象数据类型，堆是最常用的实现方式

Q: Top K 用最大堆还是最小堆？
A: 前 K 大用最小堆（堆顶是第K大，小的被淘汰）；前 K 小用最大堆

Q: 堆排序稳定吗？
A: 不稳定

Q: 四叉堆和二叉堆的区别？
A: 四叉堆每个节点4个子节点，更矮，缓存更友好，但比较次数更多

Q: Go 中 heap.Fix 什么时候用？
A: 修改了堆中某个元素的优先级后，调用 Fix 修复堆性质，O(log n)

Q: 为什么 Dijkstra 用最小堆？
A: 每次贪心选距离最小的节点，最小堆 O(log n) 取最小值
```
