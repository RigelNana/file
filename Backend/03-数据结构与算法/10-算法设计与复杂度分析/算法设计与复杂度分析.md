# 算法设计与复杂度分析

---

## 1. 时间复杂度与空间复杂度？

**回答：**

```
时间复杂度：算法执行时间随输入规模 n 的增长趋势
空间复杂度：算法所需额外空间随 n 的增长趋势

常见复杂度排序（从优到劣）：
  O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

  ┌──────────────┬──────────────┬──────────────────────┐
  │ 复杂度        │ 名称          │ 典型算法              │
  ├──────────────┼──────────────┼──────────────────────┤
  │ O(1)         │ 常数          │ 哈希查找/数组下标访问   │
  │ O(log n)     │ 对数          │ 二分查找/平衡树操作    │
  │ O(n)         │ 线性          │ 遍历/线性查找          │
  │ O(n log n)   │ 线性对数       │ 快排/归并排序          │
  │ O(n²)        │ 平方          │ 冒泡/选择排序          │
  │ O(2ⁿ)        │ 指数          │ 暴力子集枚举           │
  │ O(n!)        │ 阶乘          │ 暴力全排列             │
  └──────────────┴──────────────┴──────────────────────┘

分析技巧：
  单层循环 → O(n)
  嵌套循环 → O(n²)
  每次减半 → O(log n)
  递归     → 用主定理 T(n) = aT(n/b) + O(n^d)
  均摊分析 → 如动态数组扩容，单次 O(n)，均摊 O(1)
```

## 2. 递归与迭代的选择？

**回答：**

```
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │ 递归              │ 迭代              │
  ├──────────────┼──────────────────┼──────────────────┤
  │ 代码可读性    │ 通常更直观        │ 可能较复杂         │
  │ 栈空间        │ O(深度)，可能溢出 │ O(1)             │
  │ 性能          │ 函数调用开销      │ 通常更快           │
  │ 适用场景      │ 树/图遍历/分治    │ 简单循环/DP        │
  └──────────────┴──────────────────┴──────────────────┘

递归优化手段：
  1. 尾递归优化：编译器将尾递归转为循环（Go 不支持）
  2. 记忆化：缓存已计算结果，避免重复
  3. 转迭代：用显式栈模拟递归
```

```go
// 递归 → 迭代：二叉树中序遍历
func inorderIterative(root *TreeNode) []int {
    var result []int
    var stack []*TreeNode
    curr := root
    for curr != nil || len(stack) > 0 {
        for curr != nil {
            stack = append(stack, curr)
            curr = curr.Left
        }
        curr = stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        result = append(result, curr.Val)
        curr = curr.Right
    }
    return result
}
```

## 3. 贪心算法思想与证明？

**回答：**

```
贪心：每一步选择当前最优，不回溯

何时能用贪心：
  1. 贪心选择性质：局部最优能导到全局最优
  2. 最优子结构：问题的最优解包含子问题最优解

证明方法：
  交换论证：假设最优解不用贪心选择，交换后不会更差
  数学归纳：证明每步贪心后剩余问题仍满足贪心性质

经典贪心问题：
  ┌──────────────────┬──────────────────────────────┐
  │ 问题              │ 贪心策略                      │
  ├──────────────────┼──────────────────────────────┤
  │ 活动选择          │ 按结束时间排序，选不冲突的     │
  │ 霍夫曼编码        │ 每次合并频率最小的两个         │
  │ 分发饼干          │ 小饼干优先满足小胃口           │
  │ 跳跃游戏          │ 每步更新最远可达位置           │
  │ 最少区间覆盖      │ 按左端排序，贪心选右端最远     │
  └──────────────────┴──────────────────────────────┘
```

```go
// 活动选择问题
func activitySelection(start, end []int) int {
    // 按结束时间排序
    n := len(start)
    // ... 排序逻辑
    count := 1
    lastEnd := end[0]
    for i := 1; i < n; i++ {
        if start[i] >= lastEnd {
            count++
            lastEnd = end[i]
        }
    }
    return count
}
```

## 4. 分治算法思想？

**回答：**

```
分治三步：
  1. 分解 (Divide)：将问题分成若干子问题
  2. 解决 (Conquer)：递归求解子问题
  3. 合并 (Combine)：合并子问题的解

复杂度分析 — 主定理：
  T(n) = a·T(n/b) + O(n^d)

  if d < log_b(a)  → T(n) = O(n^(log_b(a)))
  if d = log_b(a)  → T(n) = O(n^d · log n)
  if d > log_b(a)  → T(n) = O(n^d)

经典分治：
  ┌──────────────────┬────────────────────────┐
  │ 算法              │ 分治方式                │
  ├──────────────────┼────────────────────────┤
  │ 归并排序          │ 分两半→递归→合并        │
  │ 快速排序          │ 选pivot→分区→递归       │
  │ 大整数乘法        │ Karatsuba 分治          │
  │ 最近点对          │ 分半→递归→跨区检查      │
  │ 矩阵乘法          │ Strassen 分块           │
  └──────────────────┴────────────────────────┘
```

```go
// 归并排序 — 经典分治
func mergeSort(arr []int) []int {
    if len(arr) <= 1 { return arr }
    mid := len(arr) / 2
    left := mergeSort(arr[:mid])
    right := mergeSort(arr[mid:])
    return merge(left, right)
}

func merge(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i]); i++
        } else {
            result = append(result, b[j]); j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```

## 5. 回溯算法框架？

**回答：**

```
回溯 = 暴力搜索 + 剪枝

框架：
  func backtrack(路径, 选择列表):
      if 满足结束条件:
          结果.add(路径)
          return
      for 选择 in 选择列表:
          剪枝条件 → continue
          做选择
          backtrack(路径, 选择列表)
          撤销选择

经典问题：
  全排列 / 子集 / 组合 / N皇后 / 数独
```

```go
// 全排列
func permute(nums []int) [][]int {
    var result [][]int
    var path []int
    used := make([]bool, len(nums))

    var backtrack func()
    backtrack = func() {
        if len(path) == len(nums) {
            temp := make([]int, len(path))
            copy(temp, path)
            result = append(result, temp)
            return
        }
        for i := 0; i < len(nums); i++ {
            if used[i] { continue }
            used[i] = true
            path = append(path, nums[i])
            backtrack()
            path = path[:len(path)-1]
            used[i] = false
        }
    }
    backtrack()
    return result
}

// N 皇后
func solveNQueens(n int) int {
    count := 0
    cols := make([]bool, n)
    diag1 := make([]bool, 2*n)  // 主对角线
    diag2 := make([]bool, 2*n)  // 副对角线

    var backtrack func(row int)
    backtrack = func(row int) {
        if row == n { count++; return }
        for col := 0; col < n; col++ {
            if cols[col] || diag1[row-col+n] || diag2[row+col] {
                continue
            }
            cols[col] = true
            diag1[row-col+n] = true
            diag2[row+col] = true
            backtrack(row + 1)
            cols[col] = false
            diag1[row-col+n] = false
            diag2[row+col] = false
        }
    }
    backtrack(0)
    return count
}
```

## 6. 双指针与滑动窗口？

**回答：**

```
双指针类型：
  ┌──────────────┬──────────────────────────────┐
  │ 类型          │ 适用场景                      │
  ├──────────────┼──────────────────────────────┤
  │ 对撞指针      │ 有序数组两数之和/反转          │
  │ 快慢指针      │ 链表环检测/中间节点            │
  │ 滑动窗口      │ 子串/子数组满足条件的极值       │
  └──────────────┴──────────────────────────────┘

滑动窗口模板：
  left, right = 0, 0
  while right < n:
      window.add(arr[right])
      right++
      while 窗口需要收缩:
          window.remove(arr[left])
          left++
      更新答案
```

```go
// 无重复字符的最长子串
func lengthOfLongestSubstring(s string) int {
    window := make(map[byte]int)
    left, ans := 0, 0
    for right := 0; right < len(s); right++ {
        c := s[right]
        window[c]++
        for window[c] > 1 {
            window[s[left]]--
            left++
        }
        if right-left+1 > ans {
            ans = right - left + 1
        }
    }
    return ans
}

// 最小覆盖子串
func minWindow(s string, t string) string {
    need := make(map[byte]int)
    for i := range t { need[t[i]]++ }

    window := make(map[byte]int)
    left, matched := 0, 0
    start, minLen := 0, len(s)+1

    for right := 0; right < len(s); right++ {
        c := s[right]
        window[c]++
        if window[c] == need[c] { matched++ }

        for matched == len(need) {
            if right-left+1 < minLen {
                minLen = right - left + 1
                start = left
            }
            d := s[left]
            if window[d] == need[d] { matched-- }
            window[d]--
            left++
        }
    }
    if minLen > len(s) { return "" }
    return s[start : start+minLen]
}
```

## 7. 位运算技巧？

**回答：**

```
常用位运算：
  ┌────────────────────┬──────────────────────────┐
  │ 操作                │ 含义                      │
  ├────────────────────┼──────────────────────────┤
  │ n & (n-1)          │ 去掉最低位的 1             │
  │ n & (-n)           │ 取最低位的 1               │
  │ n ^ n = 0          │ 相同数异或为 0              │
  │ n >> k & 1         │ 取第 k 位                  │
  │ n | (1 << k)       │ 第 k 位设为 1              │
  │ n & ~(1 << k)      │ 第 k 位设为 0              │
  │ n ^ (1 << k)       │ 第 k 位取反                │
  └────────────────────┴──────────────────────────┘

经典应用：
  判断奇偶:     n & 1
  判断2的幂:    n > 0 && n & (n-1) == 0
  求二进制1数:  循环 n = n & (n-1), 计次数
  两数交换:     a ^= b; b ^= a; a ^= b
```

```go
// 只出现一次的数字（其余出现两次）
func singleNumber(nums []int) int {
    result := 0
    for _, n := range nums {
        result ^= n
    }
    return result
}

// 汉明距离（不同位数）
func hammingDistance(x, y int) int {
    xor := x ^ y
    count := 0
    for xor != 0 {
        count++
        xor &= xor - 1
    }
    return count
}
```

## 8. 前缀和与差分数组？

**回答：**

```
前缀和：快速求区间和 → O(1) 查询

  原数组:   [1, 3, 5, 2, 4]
  前缀和:   [0, 1, 4, 9, 11, 15]
  区间和 [l,r] = prefix[r+1] - prefix[l]

差分数组：区间修改 O(1)

  对区间 [l, r] 加 val:
    diff[l] += val
    diff[r+1] -= val
  最后对 diff 求前缀和还原
```

```go
// 前缀和 — 子数组和等于 K
func subarraySum(nums []int, k int) int {
    count := 0
    prefixSum := 0
    m := map[int]int{0: 1}
    for _, n := range nums {
        prefixSum += n
        if v, ok := m[prefixSum-k]; ok {
            count += v
        }
        m[prefixSum]++
    }
    return count
}

// 差分数组 — 区间加值
func rangeAdd(n int, updates [][]int) []int {
    diff := make([]int, n+1)
    for _, u := range updates {
        l, r, val := u[0], u[1], u[2]
        diff[l] += val
        if r+1 <= n { diff[r+1] -= val }
    }
    result := make([]int, n)
    result[0] = diff[0]
    for i := 1; i < n; i++ {
        result[i] = result[i-1] + diff[i]
    }
    return result
}
```

## 9. 单调栈与单调队列？

**回答：**

```
单调栈：维护递增/递减栈，用于"下一个更大/更小元素"
单调队列：维护滑动窗口最值

  单调栈流程：
    for i := 0; i < n; i++:
        while 栈顶 < nums[i]:   // 递增栈
            弹出栈顶 → 它的"下一个更大"就是 nums[i]
        压入 i

  单调队列流程：
    维护双端队列，队头是窗口最大值的下标
    新元素 > 队尾 → 弹出队尾（维护递减）
    队头超出窗口 → 弹出队头
```

```go
// 每日温度 — 下一个更高温度在几天后
func dailyTemperatures(T []int) []int {
    n := len(T)
    ans := make([]int, n)
    var stack []int // 存下标
    for i := 0; i < n; i++ {
        for len(stack) > 0 && T[i] > T[stack[len(stack)-1]] {
            j := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            ans[j] = i - j
        }
        stack = append(stack, i)
    }
    return ans
}

// 滑动窗口最大值
func maxSlidingWindow(nums []int, k int) []int {
    var deque []int // 递减队列，存下标
    var result []int
    for i := 0; i < len(nums); i++ {
        // 移除超出窗口的队头
        if len(deque) > 0 && deque[0] <= i-k {
            deque = deque[1:]
        }
        // 维护递减
        for len(deque) > 0 && nums[deque[len(deque)-1]] <= nums[i] {
            deque = deque[:len(deque)-1]
        }
        deque = append(deque, i)
        if i >= k-1 {
            result = append(result, nums[deque[0]])
        }
    }
    return result
}
```

## 10. 算法设计面试速答？

**回答：**

```
Q: 时间复杂度怎么分析？
A: 看循环嵌套层数和规模缩减方式
   单循环 O(n), 嵌套 O(n²), 每次减半 O(log n)
   递归用主定理 T(n) = aT(n/b) + O(n^d)

Q: 贪心和 DP 区别？
A: 贪心: 每步取局部最优, 不回溯, 需证明正确性
   DP: 保存所有子问题解, 保证全局最优
   能贪心就贪心 (更高效), 不能才用 DP

Q: 回溯怎么剪枝？
A: 排序后跳过重复元素
   提前判断剩余不可能满足条件
   用 visited 避免重复访问

Q: 滑动窗口适用什么问题？
A: 连续子串/子数组满足某条件的极值
   关键: 窗口扩张和收缩条件明确
   典型: 最长无重复子串, 最小覆盖子串

Q: 前缀和解决什么问题？
A: 频繁查询区间和 → O(1) 查询
   子数组和等于 K → 前缀和 + 哈希
   二维前缀和 → 矩阵区域和

Q: 单调栈解决什么问题？
A: "下一个更大/更小元素" 类问题
   每日温度 / 柱状图最大矩形 / 接雨水

Q: 位运算有什么妙用？
A: n & (n-1) 去掉最低位1 → 判断2的幂/数1个数
   异或: 相同为0 → 找唯一数
   左移右移替代乘除2

Q: 怎么判断用哪种算法？
A: 求最优解 → DP 或贪心
   枚举所有方案 → 回溯
   有序数据 → 二分
   连续区间 → 滑动窗口 / 前缀和
   下一个更大 → 单调栈
```
