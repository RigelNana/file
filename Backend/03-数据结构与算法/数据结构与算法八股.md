# 数据结构与算法八股文

## 一、数组与链表

### 1. 数组和链表的区别？

**答：**

| 特性 | 数组 | 链表 |
|------|------|------|
| 内存 | 连续 | 不连续 |
| 随机访问 | O(1) | O(n) |
| 插入/删除 | O(n) | O(1)（已知位置）|
| 缓存友好 | 是 | 否 |
| 大小 | 固定/动态扩容 | 动态 |

### 2. 如何用链表实现 LRU 缓存？

**答：**

哈希表 + 双向链表：
- 哈希表 O(1) 查找
- 双向链表维护访问顺序
- 访问/插入移到头部，淘汰尾部

## 二、哈希表

### 3. 哈希冲突的解决方案？

**答：**

| 方法 | 原理 | 优缺点 |
|------|------|--------|
| 链地址法 | 每个桶一个链表 | 简单，负载因子可 >1 |
| 开放寻址 | 线性/二次/双重探测 | 缓存友好，删除麻烦 |
| 布谷鸟哈希 | 两个哈希函数+踢出 | 查找 O(1) 最坏 |

### 4. 一致性哈希是什么？

**答：**

将哈希值组织成环，节点和数据都映射到环上。数据存到顺时针方向第一个节点。增删节点只影响相邻数据。虚拟节点解决数据倾斜。

## 三、树

### 5. B+ 树和 B 树的区别？为什么 MySQL 用 B+ 树？

**答：**

| 特性 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储 | 所有节点 | 仅叶子节点 |
| 叶子链表 | 无 | 有（范围查询快）|
| 查询稳定 | 不稳定 | 稳定（总到叶子）|
| 扇出 | 较小 | 更大（内节点更小）|

MySQL 用 B+ 树：范围查询高效、磁盘 IO 更少、查询性能稳定。

### 6. 红黑树的特点和应用？

**答：**

自平衡BST，增删改查 O(log n)。5 条性质保证最长路径 ≤ 2倍最短路径。应用：TreeMap、Linux CFS 调度、epoll 定时器。

## 四、排序

### 7. 常见排序算法比较？

**答：**

| 算法 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|
| 快排 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排 | O(n log n) | O(n log n) | O(1) | 不稳定 |
| 冒泡 | O(n²) | O(n²) | O(1) | 稳定 |
| 插入 | O(n²) | O(n²) | O(1) | 稳定 |

### 8. 快排的优化策略？

**答：**

三数取中选 pivot、小数组切换插入排序、尾递归优化、三路分区（处理重复元素）。

## 五、图

### 9. BFS 和 DFS 的区别和应用？

**答：**

| 特性 | BFS | DFS |
|------|-----|-----|
| 数据结构 | 队列 | 栈/递归 |
| 空间 | O(宽度) | O(深度) |
| 最短路径 | 可找到 | 不保证 |
| 应用 | 最短路、层序遍历 | 连通性、拓扑排序 |

### 10. Dijkstra 算法原理？

**答：**

贪心算法，从源点出发，每次选最短距离的未访问节点，更新邻居距离。优先队列优化后 O((V+E)log V)。不支持负权边。

## 六、动态规划

### 11. 动态规划的核心思想？

**答：**

将问题分解为子问题，保存子问题结果避免重复计算。关键要素：最优子结构、重叠子问题、状态转移方程。

### 12. 背包问题分类？

**答：**

| 类型 | 每件物品 | 状态转移 |
|------|---------|---------|
| 0-1 背包 | 用一次 | dp[j] = max(dp[j], dp[j-w]+v) |
| 完全背包 | 无限次 | 内层正序遍历 |
| 多重背包 | 有限次 | 二进制拆分优化 |

## 七、算法设计

### 13. 常见算法复杂度？

**答：**

O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

### 14. 贪心和动态规划的区别？

**答：**

| 特性 | 贪心 | DP |
|------|------|-----|
| 选择 | 局部最优 | 全局最优 |
| 子问题 | 不回溯 | 保存所有 |
| 正确性 | 需证明 | 最优子结构保证 |

### 15. Top-K 问题的解法？

**答：**

- 堆：维护大小为K的最小堆，O(n log K)
- 快速选择：基于快排分区，平均 O(n)
- 计数排序：值域有限时 O(n)
