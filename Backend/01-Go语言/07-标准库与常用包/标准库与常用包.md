## 1. net/http 包的核心架构？

**回答：**

```
net/http 架构:

  ┌─────────┐    ┌───────────┐    ┌──────────────┐
  │ Client  │───→│ Transport │───→│ Remote Server│
  └─────────┘    └───────────┘    └──────────────┘

  ┌────────────┐    ┌──────────┐    ┌─────────┐
  │ ListenServe│───→│ ServeMux │───→│ Handler │
  └────────────┘    └──────────┘    └─────────┘

核心接口:
  Handler:      ServeHTTP(w ResponseWriter, r *Request)
  ResponseWriter: Write/WriteHeader/Header
  RoundTripper:  RoundTrip(*Request) (*Response, error)
```

```go
// 1. 基本 HTTP Server
http.HandleFunc("/hello", func(w http.ResponseWriter, r *http.Request) {
    w.Write([]byte("Hello"))
})
http.ListenAndServe(":8080", nil)

// 2. 自定义 ServeMux
mux := http.NewServeMux()
mux.HandleFunc("/api/", apiHandler)
server := &http.Server{
    Addr:         ":8080",
    Handler:      mux,
    ReadTimeout:  5 * time.Second,
    WriteTimeout: 10 * time.Second,
    IdleTimeout:  120 * time.Second,
}
server.ListenAndServe()

// 3. 中间件模式
func logging(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("%s %s", r.Method, r.URL.Path)
        next.ServeHTTP(w, r)
    })
}
mux.Handle("/api/", logging(apiHandler))

// 4. HTTP 客户端
client := &http.Client{
    Timeout: 10 * time.Second,
    Transport: &http.Transport{
        MaxIdleConns:        100,
        MaxIdleConnsPerHost: 10,
        IdleConnTimeout:     90 * time.Second,
    },
}
resp, err := client.Get("https://example.com")
defer resp.Body.Close()
```

## 2. context 包详解？

**回答：**

```
context 用途:
  1. 请求级别的取消传播
  2. 超时控制
  3. 传递请求范围的值 (谨慎使用)

  context 树:
    Background()
    ├── WithCancel → cancel()
    │   ├── WithTimeout → 超时自动取消
    │   └── WithValue → 传值
    └── WithDeadline → 截止时间自动取消
```

```go
// 1. WithCancel: 手动取消
ctx, cancel := context.WithCancel(context.Background())
defer cancel()

go func(ctx context.Context) {
    select {
    case <-ctx.Done():
        fmt.Println("cancelled:", ctx.Err())
        return
    case result := <-doWork():
        fmt.Println("result:", result)
    }
}(ctx)

// 2. WithTimeout: 超时控制
ctx, cancel := context.WithTimeout(ctx, 3*time.Second)
defer cancel()

resp, err := http.NewRequestWithContext(ctx, "GET", url, nil)

// 3. WithValue: 传递值 (谨慎使用)
type ctxKey string
const userKey ctxKey = "user"

ctx = context.WithValue(ctx, userKey, currentUser)
user := ctx.Value(userKey).(*User) // 取值

// context 最佳实践:
// 1. 函数第一个参数传 context
// 2. 不要把 context 存在 struct 中
// 3. WithValue 只传请求级数据 (request-id, auth token)
// 4. 不传可选参数 (配置/依赖用依赖注入)
// 5. context 不可变, 每次衍生都是新的
```

## 3. encoding/json 的使用和性能？

**回答：**

```go
type User struct {
    ID        int       `json:"id"`
    Name      string    `json:"name"`
    Email     string    `json:"email,omitempty"`
    CreatedAt time.Time `json:"created_at"`
    Password  string    `json:"-"` // 忽略
}

// Marshal: Go → JSON
data, _ := json.Marshal(user)

// Unmarshal: JSON → Go
var u User
json.Unmarshal(data, &u)

// Encoder/Decoder: 流式处理
encoder := json.NewEncoder(w)
encoder.Encode(user) // 写入 io.Writer

decoder := json.NewDecoder(r.Body)
decoder.Decode(&u) // 从 io.Reader 读取

// 自定义序列化
func (u User) MarshalJSON() ([]byte, error) {
    type Alias User
    return json.Marshal(&struct {
        Alias
        CreatedAt string `json:"created_at"`
    }{
        Alias:     Alias(u),
        CreatedAt: u.CreatedAt.Format("2006-01-02"),
    })
}
```

```
性能:
  标准 json: 使用反射, 较慢
  ┌──────────────┬──────────────┬───────────┐
  │ 库            │ 方式          │ 速度       │
  ├──────────────┼──────────────┼───────────┤
  │ encoding/json │ 反射          │ 基准 1x   │
  │ jsoniter      │ 反射优化+缓存 │ ~3-5x     │
  │ easyjson      │ 代码生成      │ ~5-10x    │
  │ sonic         │ JIT           │ ~5-10x    │
  │ go-json       │ 反射优化      │ ~3-5x     │
  └──────────────┴──────────────┴───────────┘
```

## 4. io 包和 Reader/Writer 设计？

**回答：**

```go
// io 包核心接口 — Go 最优美的设计之一
type Reader interface {
    Read(p []byte) (n int, err error)
}
type Writer interface {
    Write(p []byte) (n int, err error)
}
type Closer interface {
    Close() error
}

// 组合接口
type ReadWriter interface { Reader; Writer }
type ReadCloser interface { Reader; Closer }
type WriteCloser interface { Writer; Closer }
type ReadWriteCloser interface { Reader; Writer; Closer }

// 常用工具函数
io.Copy(dst Writer, src Reader)         // 流式拷贝
io.ReadAll(r Reader) ([]byte, error)    // 读全部
io.LimitReader(r Reader, n int64)       // 限制读取量
io.TeeReader(r Reader, w Writer)        // 读时同时写
io.MultiReader(readers ...Reader)       // 串联多个 Reader
io.MultiWriter(writers ...Writer)       // 同时写多个 Writer
io.Pipe() (*PipeReader, *PipeWriter)    // 同步管道

// 典型使用: 链式处理
r := io.LimitReader(                     // 限制大小
    io.TeeReader(                        // 同时计算 hash
        gzip.NewReader(resp.Body),       // 解压
        hashWriter,
    ),
    maxSize,
)

// bufio: 带缓冲的 I/O
reader := bufio.NewReader(file)    // 默认 4096B 缓冲
scanner := bufio.NewScanner(file)  // 按行扫描
writer := bufio.NewWriter(file)    // 缓冲写入
defer writer.Flush()
```

## 5. sync 包常用工具？

**回答：**

```go
// 1. sync.Mutex: 互斥锁
var mu sync.Mutex
mu.Lock()
defer mu.Unlock()

// 2. sync.RWMutex: 读写锁
var rw sync.RWMutex
rw.RLock()   // 读锁, 允许多个并发读
rw.RUnlock()
rw.Lock()    // 写锁, 独占
rw.Unlock()

// 3. sync.WaitGroup: 等待组
var wg sync.WaitGroup
for i := 0; i < 10; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        doWork()
    }()
}
wg.Wait()

// 4. sync.Once: 单次执行
var once sync.Once
var instance *DB
func GetDB() *DB {
    once.Do(func() {
        instance = connectDB()
    })
    return instance
}

// 5. sync.Map: 并发安全 map
var m sync.Map
m.Store("key", "value")
v, ok := m.Load("key")
m.Delete("key")
m.Range(func(k, v interface{}) bool {
    fmt.Println(k, v)
    return true
})
// 适用: 读多写少, key 稳定 (如缓存)
// 不适用: 频繁写入 → 建议 mutex + map

// 6. sync.Cond: 条件变量
cond := sync.NewCond(&sync.Mutex{})
// 等待方:
cond.L.Lock()
for !condition {
    cond.Wait() // 释放锁 + 等待 + 被唤醒后重新获取锁
}
cond.L.Unlock()
// 通知方:
cond.Signal()    // 唤醒一个
cond.Broadcast() // 唤醒所有
```

## 6. time 包和定时器？

**回答：**

```go
// 时间操作
now := time.Now()
t := time.Date(2024, 1, 1, 0, 0, 0, 0, time.UTC)
d := time.Since(t)       // 从 t 到现在的 Duration
d2 := time.Until(t)      // 从现在到 t 的 Duration
t2 := t.Add(24 * time.Hour)
t.Format("2006-01-02 15:04:05") // Go 特殊的参考时间格式
t, _ = time.Parse("2006-01-02", "2024-06-01")

// 定时器
// 1. time.After (一次性, 注意泄漏)
select {
case result := <-ch:
    process(result)
case <-time.After(5 * time.Second): // 超时
    fmt.Println("timeout")
}

// 2. time.NewTimer (推荐)
timer := time.NewTimer(5 * time.Second)
defer timer.Stop()
select {
case <-timer.C:
    fmt.Println("fired")
}
timer.Reset(3 * time.Second) // 重置

// 3. time.NewTicker (周期性)
ticker := time.NewTicker(1 * time.Second)
defer ticker.Stop()
for range ticker.C {
    fmt.Println("tick")
}

// time.After 泄漏问题:
// for 循环中每次创建新 Timer, 旧的不清理直到触发
// 解决: 用 NewTimer + Reset, 或用 Ticker
```

## 7. strings/bytes/strconv 包？

**回答：**

```go
import (
    "strings"
    "bytes"
    "strconv"
)

// strings 包
strings.Contains("hello", "ell")    // true
strings.HasPrefix("hello", "he")    // true
strings.HasSuffix("hello", "lo")    // true
strings.Split("a,b,c", ",")        // ["a","b","c"]
strings.Join([]string{"a","b"}, ",") // "a,b"
strings.Replace("aaa", "a", "b", 2) // "bba"
strings.TrimSpace(" hello ")        // "hello"
strings.ToUpper("hello")            // "HELLO"
strings.Builder // 高效字符串拼接 ↓

// strings.Builder: 高效拼接
var b strings.Builder
for i := 0; i < 100; i++ {
    b.WriteString("hello")
}
result := b.String()
// 比 += 快得多, 内部维护 []byte, 最后一次性转 string

// bytes 包 (和 strings 类似, 操作 []byte)
bytes.Contains([]byte("hello"), []byte("ell"))
bytes.Buffer // 可读可写的缓冲

// strconv 包
strconv.Itoa(42)              // int → string "42"
strconv.Atoi("42")            // string → int
strconv.FormatFloat(3.14, 'f', 2, 64)   // "3.14"
strconv.ParseFloat("3.14", 64)          // 3.14
strconv.FormatBool(true)      // "true"
strconv.ParseBool("true")     // true

// 性能提示:
// string ↔ []byte 会拷贝
// 频繁操作用 []byte, 最后转 string
// 拼接用 strings.Builder (不是 fmt.Sprintf 或 +=)
```

## 8. sort 包和泛型排序？

**回答：**

```go
// 基本排序
sort.Ints([]int{3, 1, 2})       // [1, 2, 3] 原地排序
sort.Strings([]string{"c","a","b"}) // [a, b, c]
sort.Float64s([]float64{3.1, 1.2, 2.3})

// 自定义排序 (经典方式)
type ByAge []User
func (a ByAge) Len() int           { return len(a) }
func (a ByAge) Less(i, j int) bool { return a[i].Age < a[j].Age }
func (a ByAge) Swap(i, j int)      { a[i], a[j] = a[j], a[i] }
sort.Sort(ByAge(users))

// 简化方式
sort.Slice(users, func(i, j int) bool {
    return users[i].Age < users[j].Age
})
sort.SliceStable(users, func(i, j int) bool { // 稳定排序
    return users[i].Name < users[j].Name
})

// Go 1.21+ slices 包 (泛型)
import "slices"
slices.Sort([]int{3, 1, 2})
slices.SortFunc(users, func(a, b User) int {
    return cmp.Compare(a.Age, b.Age)
})
slices.Contains([]int{1, 2, 3}, 2) // true
slices.Index([]int{1, 2, 3}, 2)    // 1

// 二分查找
i := sort.SearchInts(sorted, target)
// Go 1.21+
i, found := slices.BinarySearch(sorted, target)
```

## 9. 常用第三方库推荐？

**回答：**

```
热门 Go 第三方库:

Web 框架:
  ┌──────────────┬──────────────────────────────┐
  │ 库            │ 特点                          │
```

## 10. 标准库面试速答？

**回答：**

```
Q: net/http 默认 ServeMux 有什么问题?
A: 不支持路径参数/方法匹配/中间件/分组
   Go 1.22 改进了: 支持 GET /user/{id} 模式
   生产一般用 Gin/Chi 等

Q: context 什么时候用?
A: 请求取消传播/超时控制/传递请求级值
   函数第一个参数, 不存 struct
   WithValue 只传 request-id 等元数据

Q: encoding/json 慢怎么办?
A: 代码生成(easyjson), JIT(sonic), 优化库(jsoniter)
   标准 json 基于反射, 慢 3-10 倍

Q: strings.Builder vs bytes.Buffer?
A: Builder 只可写, 惰性分配, 用于拼接 string
   Buffer 可读可写, 更通用
   Builder 不允许拷贝 (有 noCopy 检查)

Q: sync.Map 什么场景适用?
A: 读多写少且 key 稳定 (如缓存)
   频繁写入用 mutex + map 更好
   内部用 read/dirty 两个 map + atomic

Q: sort vs slices 包?
A: sort 基于接口(Sort接口3方法), slices 基于泛型
   slices (Go1.21+) 更简洁: slices.Sort(s)
   slices.SortFunc 自定义比较

Q: time.After 有什么坑?
A: for循环里每次创建新Timer, 旧的不释放直到触发
   解决: NewTimer + Reset, 或用 Ticker
```
