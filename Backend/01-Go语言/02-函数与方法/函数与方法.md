## 1. Go 函数的特点？

**回答：**

```
Go 函数特点:
  1. 多返回值
  2. 命名返回值
  3. 可变参数
  4. 函数是一等公民 (可赋值、传参、返回)
  5. 不支持函数重载
  6. 不支持默认参数
```

```go
// 多返回值
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// 命名返回值 + 裸 return
func calc(a, b int) (sum, diff int) {
    sum = a + b
    diff = a - b
    return // 自动返回 sum, diff
}

// 可变参数
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}
sum(1, 2, 3)
nums := []int{1, 2, 3}
sum(nums...) // 展开 slice

// 函数类型
type Handler func(ctx context.Context, req Request) Response

// 匿名函数
fn := func(x int) int { return x * 2 }
```

## 2. 闭包原理和使用？

**回答：**

```
闭包: 函数 + 捕获的外部变量 (引用捕获)

原理:
  闭包函数会持有外部变量的引用 (指针)
  外部变量逃逸到堆上, 生命周期延长
```

```go
// 基本闭包
func counter() func() int {
    n := 0
    return func() int {
        n++      // 捕获变量 n (引用)
        return n
    }
}
c := counter()
c() // 1
c() // 2
c() // 3

// 陷阱: 循环变量捕获
for i := 0; i < 3; i++ {
    go func() {
        fmt.Println(i) // 都打印 3! 捕获的是同一个 i
    }()
}

// 修复方式1: 参数传递
for i := 0; i < 3; i++ {
    go func(n int) {
        fmt.Println(n) // 0, 1, 2
    }(i)
}

// 修复方式2: 局部变量 (Go 1.22 已修复循环变量问题)
for i := 0; i < 3; i++ {
    i := i // 创建新变量
    go func() {
        fmt.Println(i)
    }()
}

// 常用场景: 中间件、配置函数
func WithTimeout(d time.Duration) Option {
    return func(c *Config) {
        c.Timeout = d
    }
}
```

## 3. 方法和函数的区别？值接收者 vs 指针接收者？

**回答：**

```go
// 函数: 独立的, 不属于任何类型
func Add(a, b int) int { return a + b }

// 方法: 绑定到特定类型的函数
type User struct{ Name string; Age int }

// 值接收者
func (u User) String() string {
    return fmt.Sprintf("%s(%d)", u.Name, u.Age)
}

// 指针接收者
func (u *User) SetAge(age int) {
    u.Age = age // 修改原值
}
```

```
值接收者 vs 指针接收者:
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │ 值接收者          │ 指针接收者        │
  ├──────────────┼──────────────────┼──────────────────┤
  │ 能否修改      │ 不能 (操作副本)  │ 能 (操作原数据)  │
  │ 大型 struct   │ 有拷贝开销       │ 无拷贝开销       │
  │ 值调用        │ ✅               │ ✅ (自动取地址)  │
  │ 指针调用      │ ✅ (自动解引用)  │ ✅               │
  │ 实现接口      │ 值和指针都实现    │ 仅指针实现       │
  │ 并发安全      │ 安全 (副本)      │ 不安全           │
  └──────────────┴──────────────────┴──────────────────┘

选择建议:
  需要修改接收者       → 指针
  接收者是大型 struct  → 指针 (避免拷贝)
  接收者是 map/slice/chan → 值 (本身就是引用)
  不需要修改 + 小 struct → 值
  统一性: 一个类型最好统一用值或指针
```

## 4. defer 详解？

**回答：**

```go
// defer 三大特性:
// 1. 后进先出 (LIFO)
// 2. 参数在注册时求值
// 3. 可以修改命名返回值

// 特性1: LIFO
func f() {
    for i := 0; i < 5; i++ {
        defer fmt.Print(i) // 输出: 4 3 2 1 0
    }
}

// 特性2: 参数立即求值
func f() {
    x := 1
    defer fmt.Println(x) // x=1 此时确定
    x = 2
}
// 输出: 1

// 但闭包捕获的变量是引用
func f() {
    x := 1
    defer func() { fmt.Println(x) }() // 闭包
    x = 2
}
// 输出: 2

// 特性3: 修改命名返回值
func f() (result int) {
    defer func() { result *= 2 }()
    return 5  // result=5 → defer → result=10
}
// 返回 10

// return 语句拆解:
// 1. 给返回值赋值
// 2. 执行 defer
// 3. RET 返回

// 实际场景: 资源清理
func ReadFile(path string) ([]byte, error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, err
    }
    defer f.Close() // 确保关闭

    return io.ReadAll(f)
}
```

## 5. panic 和 recover？

**回答：**

```go
// panic: 立即中断当前函数, 逐层向上传播
// recover: 在 defer 中捕获 panic, 恢复执行

// panic 传播流程:
// funcC panic → funcC 的 defer → funcB 的 defer → funcA 的 defer → 程序崩溃
// 如果某个 defer 中有 recover, 则停止传播

func safeDiv(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panic: %v", r)
        }
    }()
    return a / b, nil // b=0 时 panic
}

// recover 规则:
// 1. 只能在 defer 函数中调用
// 2. 只能捕获当前 goroutine 的 panic
// 3. 返回 panic 的值, 无 panic 返回 nil

// 何时用 panic:
// 1. 不可恢复的错误 (程序初始化失败)
// 2. 库内部严重错误 (数组越界)
// 不应该用 panic 做常规错误处理!

// Web 服务中的 recover 中间件
func RecoveryMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                log.Printf("panic: %v\n%s", err, debug.Stack())
                http.Error(w, "Internal Server Error", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}
```

## 6. init 函数的执行顺序？

**回答：**

```
init 函数特点:
  1. 没有参数和返回值
  2. 不能被手动调用
  3. 一个文件可以有多个 init
  4. 一个包的多个文件都可以有 init

执行顺序:
  1. 递归初始化依赖包
  2. 包级变量初始化
  3. init 函数执行 (按文件名排序, 文件内按顺序)
  4. main 函数执行

  依赖顺序:
    main 依赖 A, A 依赖 B
    → 先初始化 B → 再初始化 A → 最后 main

  同一包: const → var → init()
```

```go
package main

import "fmt"

var x = initVar() // 2. 包级变量

func initVar() int {
    fmt.Println("var init")
    return 1
}

func init() { // 3. init (第一个)
    fmt.Println("init 1")
}

func init() { // 3. init (第二个)
    fmt.Println("init 2")
}

func main() { // 4. main
    fmt.Println("main")
}

// 输出:
// var init
// init 1
// init 2
// main

// 常见用途:
// 1. 注册驱动: import _ "github.com/go-sql-driver/mysql"
// 2. 初始化全局配置
// 3. 检查运行环境
```

## 7. 函数式选项模式 (Functional Options)？

**回答：**

```go
// 问题: Go 没有默认参数和函数重载
// 解决: 函数式选项模式

type Server struct {
    host    string
    port    int
    timeout time.Duration
    maxConn int
}

// Option 类型
type Option func(*Server)

// 选项函数
func WithHost(host string) Option {
    return func(s *Server) { s.host = host }
}
func WithPort(port int) Option {
    return func(s *Server) { s.port = port }
}
func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}
func WithMaxConn(n int) Option {
    return func(s *Server) { s.maxConn = n }
}

// 构造函数
func NewServer(opts ...Option) *Server {
    s := &Server{
        host:    "0.0.0.0",  // 默认值
        port:    8080,
        timeout: 30 * time.Second,
        maxConn: 100,
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// 使用
s := NewServer(
    WithHost("127.0.0.1"),
    WithPort(9090),
    WithTimeout(10 * time.Second),
)
// 优点: 可选参数, 可扩展, 可读性好
```

## 8. 方法集与接口实现？

**回答：**

```
方法集规则:
  ┌──────────────┬──────────────────────────────┐
  │ 类型          │ 方法集                        │
  ├──────────────┼──────────────────────────────┤
  │ T (值类型)    │ 值接收者方法                  │
  │ *T (指针类型) │ 值接收者 + 指针接收者方法      │
  └──────────────┴──────────────────────────────┘
```

```go
type Animal interface {
    Speak() string
    Move()
}

type Dog struct{ Name string }

func (d Dog) Speak() string { return "Woof" }  // 值接收者
func (d *Dog) Move() { fmt.Println("running") } // 指针接收者

// Dog 实现了 Animal 吗？
// var a Animal = Dog{}   // 编译错误! Dog 的方法集只有 Speak
var a Animal = &Dog{}     // OK! *Dog 方法集包含 Speak + Move

// 编译器隐式地址取:
d := Dog{}
d.Move()  // OK, 编译器自动: (&d).Move()
// 但接口赋值时不会自动取地址!

// 嵌入 (Embedding) 方法提升
type GuideDog struct {
    Dog // 嵌入 Dog
}
// GuideDog 自动拥有 Dog 的所有方法
gd := &GuideDog{Dog: Dog{Name: "Buddy"}}
gd.Speak() // "Woof" (提升方法)
```

## 9. 高阶函数和回调？

**回答：**

```go
// 高阶函数: 接收或返回函数的函数

// 1. 函数作为参数 (回调)
func Map(s []int, fn func(int) int) []int {
    result := make([]int, len(s))
    for i, v := range s {
        result[i] = fn(v)
    }
    return result
}
doubled := Map([]int{1, 2, 3}, func(x int) int { return x * 2 })
// [2, 4, 6]

// 2. 函数作为返回值 (工厂)
func Multiplier(factor int) func(int) int {
    return func(x int) int {
        return x * factor
    }
}
triple := Multiplier(3)
triple(5) // 15

// 3. 中间件链
type Middleware func(http.Handler) http.Handler

func Chain(handler http.Handler, middlewares ...Middleware) http.Handler {
    for i := len(middlewares) - 1; i >= 0; i-- {
        handler = middlewares[i](handler)
    }
    return handler
}

// 4. 策略模式
type SortStrategy func(a, b int) bool

func SortWith(data []int, less SortStrategy) {
    sort.Slice(data, func(i, j int) bool {
        return less(data[i], data[j])
    })
}
SortWith(nums, func(a, b int) bool { return a > b }) // 降序
```

## 10. 函数与方法面试速答？

**回答：**

```
Q: Go 支持函数重载吗?
A: 不支持! 一个包内函数名必须唯一
   用可变参数或函数式选项替代

Q: 值接收者和指针接收者怎么选?
A: 需要修改接收者 → 指针
   大 struct → 指针 (避免拷贝)
   小且只读 → 值
   统一: 同类型尽量统一

Q: defer 什么时候执行?
A: 函数 return 之后, 实际返回之前
   LIFO 顺序, 参数注册时求值
   闭包引用的变量是延迟求值的

Q: recover 怎么用?
A: 只能在 defer 中调用
   只捕获当前 goroutine 的 panic
   不要用 panic 做常规错误处理

Q: init 函数执行顺序?
A: 依赖包先执行 → const → var → init → main
   同包多文件按文件名排序

Q: 闭包的陷阱?
A: 循环变量捕获 → 共享同一变量
   修复: 参数传入或局部变量
   Go 1.22 已修复 for 循环变量

Q: 什么是函数式选项模式?
A: 用 func(*Config) 类型实现可选参数
   解决 Go 无默认参数的问题
   WithXxx 函数返回配置闭包
```
