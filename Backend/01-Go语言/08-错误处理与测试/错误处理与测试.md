## 1. Go 错误处理哲学？error vs panic？

**回答：**

```
Go 错误处理哲学:
  "Errors are values" — 错误是普通值, 不是异常

  对比:
  ┌──────────────┬──────────────┬──────────────┐
  │              │ error        │ panic        │
  ├──────────────┼──────────────┼──────────────┤
  │ 用途          │ 预期的错误    │ 不可恢复的程序错误│
  │ 控制流        │ 显式检查返回值│ 中断正常流程   │
  │ 恢复          │ 调用者决定    │ recover 捕获  │
  │ 典型场景      │ 文件不存在    │ 数组越界      │
  │              │ 网络超时      │ nil 指针解引用 │
  │              │ 参数无效      │ 不可能的分支   │
  │ 频率          │ 常用          │ 极少          │
  └──────────────┴──────────────┴──────────────┘

使用规则:
  1. 绝大多数情况用 error
  2. panic 只用于: 程序 Bug / 初始化失败 / 不可能分支
  3. 库代码不应 panic (除非是真正的 Bug)
  4. 如果 panic, 在包边界 recover 并转为 error
```

```go
// error 基本用法
func ReadFile(path string) ([]byte, error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, fmt.Errorf("open file: %w", err) // wrap
    }
    defer f.Close()
    return io.ReadAll(f)
}

// panic + recover (在框架/库边界)
func SafeHandler(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                log.Printf("panic: %v\n%s", err, debug.Stack())
                http.Error(w, "Internal Server Error", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}
```

## 2. error wrapping 与错误链？

**回答：**

```go
// Go 1.13+ 错误链 (Error Wrapping)

// %w 包装错误 (wrap)
func getUser(id int) (*User, error) {
    row := db.QueryRow("SELECT ...", id)
    if err := row.Scan(&u); err != nil {
        return nil, fmt.Errorf("getUser(%d): %w", id, err)
        // 包装了原始错误, 保留错误链
    }
    return &u, nil
}

// errors.Is: 判断错误链中是否包含目标错误
if errors.Is(err, sql.ErrNoRows) {
    // 即使 err 被包装了多层, 也能匹配
}

// errors.As: 从错误链提取特定类型
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println(pathErr.Path)
}

// errors.Unwrap: 解包一层
inner := errors.Unwrap(err)

// 自定义错误实现 Unwrap:
type QueryError struct {
    Query string
    Err   error
}

func (e *QueryError) Error() string {
    return fmt.Sprintf("query %q: %v", e.Query, e.Err)
}

func (e *QueryError) Unwrap() error {
    return e.Err // 支持 errors.Is / errors.As
}
```

```
错误链:
  QueryError → fmt.Errorf %w → sql.ErrNoRows
  
  errors.Is(err, sql.ErrNoRows) → true (遍历链)
  errors.As(err, &queryErr) → true (类型匹配)

%w vs %v:
  %w: 包装, 保留错误链 (可 Is/As)
  %v: 仅格式化为字符串, 断开错误链
  → 需要调用者判断类型时用 %w
  → 不需要暴露内部错误时用 %v
```

## 3. 哨兵错误与自定义错误？

**回答：**

```go
// 哨兵错误 (Sentinel Error)
// 预定义的错误值, 作为包级别变量
var (
    ErrNotFound     = errors.New("not found")
    ErrUnauthorized = errors.New("unauthorized")
    ErrConflict     = errors.New("conflict")
)

// 使用者用 errors.Is 判断:
if errors.Is(err, ErrNotFound) {
    w.WriteHeader(404)
}

// 自定义错误类型 (带额外信息)
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation: %s %s", e.Field, e.Message)
}

// 使用者用 errors.As 提取:
var ve *ValidationError
if errors.As(err, &ve) {
    fmt.Printf("field %s: %s\n", ve.Field, ve.Message)
}

// 业务错误码模式
type BizError struct {
    Code    int    // 业务错误码
    Message string
    Err     error  // 原始错误
}

func (e *BizError) Error() string { return e.Message }
func (e *BizError) Unwrap() error { return e.Err }

var ErrUserNotFound = &BizError{Code: 10001, Message: "用户不存在"}
```

```
错误设计原则:
  ┌──────────────────┬──────────────────────────────┐
  │ 原则              │ 说明                          │
  ├──────────────────┼──────────────────────────────┤
  │ 只处理一次        │ 处理(日志/返回)后不再传播       │
  │ 添加上下文        │ fmt.Errorf("ctx: %w", err)   │
  │ 在边界处处理      │ handler/入口处统一错误处理     │
  │ 错误类型层次      │ 哨兵(简单)/类型(丰富)/wrap(链) │
  │ 不要忽略错误      │ _ = f.Close() → 至少 log     │
  └──────────────────┴──────────────────────────────┘
```

## 4. Go 单元测试基础？

**回答：**

```go
// 文件命名: xxx_test.go (自动被 go test 识别)
// 函数命名: TestXxx(t *testing.T)

// user_test.go
package user

import "testing"

func TestAdd(t *testing.T) {
    got := Add(1, 2)
    if got != 3 {
        t.Errorf("Add(1,2) = %d, want 3", got)
    }
}

// 子测试 (Subtests)
func TestAdd(t *testing.T) {
    tests := []struct {
        name     string
        a, b     int
        expected int
    }{
        {"positive", 1, 2, 3},
        {"zero", 0, 0, 0},
        {"negative", -1, -2, -3},
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.expected {
                t.Errorf("Add(%d,%d) = %d, want %d",
                    tt.a, tt.b, got, tt.expected)
            }
        })
    }
}

// 常用 testing 方法:
// t.Error/Errorf   → 报告错误, 继续执行
// t.Fatal/Fatalf   → 报告错误, 立即停止
// t.Skip/Skipf     → 跳过测试
// t.Parallel()     → 并行执行
// t.Helper()       → 标记为辅助函数 (报错行号正确)
// t.Cleanup(func)  → 注册清理函数
```

```bash
# 常用测试命令
go test ./...                    # 递归测试所有包
go test -v                      # 显示详细输出
go test -run TestAdd             # 只运行匹配的测试
go test -count=1                 # 禁用缓存
go test -race                    # 竞态检测
go test -cover                   # 覆盖率
go test -coverprofile=cover.out  # 覆盖率文件
go tool cover -html=cover.out    # HTML 覆盖率报告
```

```go
// 表驱动测试: Go 推荐的测试模式
func TestParseURL(t *testing.T) {
    tests := map[string]struct {
        input   string
        want    string
        wantErr bool
    }{
        "valid url":     {"https://example.com/path", "example.com", false},
        "with port":     {"https://example.com:8080", "example.com", false},
        "invalid url":   {"not-a-url", "", true},
        "empty":         {"", "", true},
    }
    for name, tt := range tests {
        t.Run(name, func(t *testing.T) {
            got, err := ParseURL(tt.input)
            if (err != nil) != tt.wantErr {
                t.Fatalf("err = %v, wantErr = %v", err, tt.wantErr)
            }
            if got != tt.want {
                t.Errorf("got %q, want %q", got, tt.want)
            }
        })
    }
}

// TestMain: 测试前后的 setup/teardown
func TestMain(m *testing.M) {
    // Setup
    db := setupTestDB()
    
    code := m.Run() // 运行所有测试
    
    // Teardown
    db.Close()

func TestWithDB(t *testing.T) {
```

## 6. Mock 与依赖注入测试？

**回答：**

```go
// 接口 Mock: Go 推荐的方式 (不需要框架)

// 1. 定义接口 (在消费端)
type UserRepository interface {
    GetByID(ctx context.Context, id int64) (*User, error)
    Create(ctx context.Context, u *User) error
}

// 2. 生产实现
type mysqlUserRepo struct{ db *sql.DB }
func (r *mysqlUserRepo) GetByID(ctx context.Context, id int64) (*User, error) {
    // 真实 DB 查询
}

// 3. Mock 实现
type mockUserRepo struct {
    getByIDFunc func(ctx context.Context, id int64) (*User, error)
    createFunc  func(ctx context.Context, u *User) error
}

func (m *mockUserRepo) GetByID(ctx context.Context, id int64) (*User, error) {
    return m.getByIDFunc(ctx, id)
}
func (m *mockUserRepo) Create(ctx context.Context, u *User) error {
    return m.createFunc(ctx, u)
}

// 4. 测试中使用 Mock
func TestUserService_GetUser(t *testing.T) {
    repo := &mockUserRepo{
        getByIDFunc: func(ctx context.Context, id int64) (*User, error) {
            if id == 1 {
                return &User{ID: 1, Name: "Alice"}, nil
            }
            return nil, ErrNotFound
        },
    }
    svc := NewUserService(repo) // 依赖注入
    
    u, err := svc.GetUser(context.Background(), 1)
    if err != nil { t.Fatal(err) }
    if u.Name != "Alice" { t.Errorf("got %s", u.Name) }
}

// 常用 Mock 工具:
// gomock     → Google 官方, 基于接口生成
// testify    → assert/require/mock, 最流行
// sqlmock    → 数据库 Mock
// httptest   → HTTP 测试
// go-mock    → uber 的 Mock 库
```

## 7. Benchmark 基准测试？

**回答：**

```go
// 基准测试: BenchmarkXxx(b *testing.B)

func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ { // b.N 由框架自动调整
        Add(1, 2)
    }

// 避免编译器优化消除
var result int
func BenchmarkFib(b *testing.B) {
    var r int
    for i := 0; i < b.N; i++ {
        r = Fib(20)
    }
    result = r // 防止优化

// 带内存统计
func BenchmarkConcat(b *testing.B) {
    b.ReportAllocs() // 报告内存分配
    for i := 0; i < b.N; i++ {
        s := ""
        for j := 0; j < 100; j++ {
            s += "x"
        }
    }
}

// 子基准测试
func BenchmarkSort(b *testing.B) {
    sizes := []int{100, 1000, 10000}
    for _, size := range sizes {
        b.Run(fmt.Sprintf("size=%d", size), func(b *testing.B) {
            data := generateSlice(size)
            b.ResetTimer() // 排除初始化时间
            for i := 0; i < b.N; i++ {
                sort.Ints(append([]int{}, data...))
            }
        })
    }
```

## 9. Fuzzing 模糊测试 (Go 1.18+)？

**回答：**

```go
// 模糊测试: FuzzXxx(f *testing.F)
// 自动生成随机输入, 发现边界 Bug

func FuzzParseJSON(f *testing.F) {
    // 种子语料 (seed corpus)
    f.Add([]byte(`{"name":"alice"}`))
    f.Add([]byte(`{}`))
    f.Add([]byte(`[]`))
    f.Add([]byte(``))
    
    // 模糊测试函数
    f.Fuzz(func(t *testing.T, data []byte) {
        var result map[string]interface{}
        err := json.Unmarshal(data, &result)
        if err != nil {
            return // 无效输入, 跳过
        }
        // 验证: Marshal 再 Unmarshal 应等价
        encoded, err := json.Marshal(result)
        if err != nil {
            t.Fatalf("Marshal failed: %v", err)
        }
        var result2 map[string]interface{}
        if err := json.Unmarshal(encoded, &result2); err != nil {
            t.Fatalf("round-trip failed: %v", err)
        }
    })
}

func FuzzReverse(f *testing.F) {
    f.Add("hello")
    f.Add("你好世界")
    f.Add("")
    
    f.Fuzz(func(t *testing.T, s string) {
        rev := Reverse(s)
        doubleRev := Reverse(rev)
        if s != doubleRev {
            t.Errorf("Reverse(Reverse(%q)) = %q", s, doubleRev)
        }
        if utf8.ValidString(s) && !utf8.ValidString(rev) {
            t.Errorf("Reverse(%q) produced invalid UTF-8", s)
        }
    })
}
```

```bash
# 运行模糊测试
go test -fuzz=FuzzParseJSON            # 运行指定 fuzz
go test -fuzz=. -fuzztime=30s          # 运行 30 秒
go test -fuzz=FuzzReverse -fuzztime=1m # 运行 1 分钟

# 失败语料保存在 testdata/fuzz/FuzzXxx/ 目录
# 后续 go test 会自动回归测试这些语料
```

```
模糊测试适用场景:
  解析器 (JSON/XML/Protocol)
  编解码 (序列化/反序列化)
  字符串处理函数
  加密/哈希函数
  任何接受外部输入的函数
```

## 10. 错误处理与测试面试速答？

**回答：**

```
Q: Go 错误处理和其他语言有什么不同?
A: 错误是值 (error interface), 显式检查返回值
   不用 try/catch, if err != nil 是核心模式
   panic 只用于不可恢复的 Bug

Q: %w 和 %v 包装错误的区别?
A: %w 保留错误链, 支持 errors.Is/As 判断
   %v 只格式化字符串, 断开错误链
   需要调用者匹配原始错误时用 %w

Q: errors.Is 和 errors.As 区别?
A: Is: 判断错误链中是否包含目标值 (值比较)
   As: 从错误链提取特定类型 (类型断言)
   Is 用于哨兵错误, As 用于自定义错误类型

Q: Go 测试推荐什么模式?
A: 表驱动测试 (table-driven tests)
   map/slice 定义测试用例, t.Run 子测试
   接口 Mock 做依赖注入, 不需要框架

Q: Benchmark 怎么写?
A: BenchmarkXxx(b *testing.B), for i<b.N
   b.ReportAllocs() 报告内存分配
   go test -bench=. -benchmem 运行

Q: Go 1.18 Fuzzing 是什么?
A: 内置模糊测试, 自动生成随机输入
   f.Add 提供种子, f.Fuzz 定义测试逻辑
   适合解析器/编解码/输入处理函数

Q: httptest 怎么测 HTTP?
A: httptest.NewRecorder → 测试 Handler
   httptest.NewServer → 启动假服务测试客户端
   不需要启动真实服务器
```
