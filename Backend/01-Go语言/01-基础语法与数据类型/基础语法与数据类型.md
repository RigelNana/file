## 1. Go 的基本数据类型有哪些？

**回答：**

```
Go 数据类型分类:
  ┌──────────────────────────────────────────────┐
  │ 基础类型                                      │
  │   bool                                        │
  │   整型: int int8 int16 int32 int64            │
  │         uint uint8 uint16 uint32 uint64       │
  │         uintptr byte(=uint8) rune(=int32)     │
  │   浮点: float32 float64                       │
  │   复数: complex64 complex128                  │
  │   字符串: string (不可变, UTF-8 字节序列)      │
  ├──────────────────────────────────────────────┤
  │ 复合类型                                      │
  │   array    [N]T      固定长度                 │
  │   slice    []T       动态数组                 │
  │   map      map[K]V   哈希表                   │
  │   struct   struct{}  结构体                   │
  ├──────────────────────────────────────────────┤
  │ 引用类型                                      │
  │   pointer  *T        指针                     │
  │   channel  chan T    通道                     │
  │   func     func()   函数                     │
  │   interface          接口                     │
  └──────────────────────────────────────────────┘

零值:
  数值 → 0          bool → false
  string → ""       pointer/slice/map/chan/func/interface → nil
  struct → 各字段零值
  array → 各元素零值
```

```go
// int 的大小取决于平台
// 32位系统: int = int32 (4字节)
// 64位系统: int = int64 (8字节)

// string 底层结构
type stringHeader struct {
    Data unsafe.Pointer // 指向字节数组
    Len  int            // 字节长度（不是字符长度）
}

// string 是不可变的
s := "hello"
// s[0] = 'H'  // 编译错误！

// 修改: 需要转为 []byte 或 []rune
bs := []byte(s)
bs[0] = 'H'
s = string(bs) // "Hello"

// rune vs byte
// byte = uint8, 处理 ASCII
// rune = int32, 处理 Unicode (中文等)
s := "你好"
fmt.Println(len(s))         // 6 (字节数)
fmt.Println(len([]rune(s))) // 2 (字符数)
```

## 2. slice 底层原理和扩容机制？

**回答：**

```
slice 底层结构:
  ┌─────────────────────────────────┐
  │ type slice struct {              │
  │     array unsafe.Pointer // 底层数组 │
  │     len   int           // 长度  │
  │     cap   int           // 容量  │
  │ }                                │
  └─────────────────────────────────┘

  s := make([]int, 3, 5)
  
  slice header:  [ptr | len=3 | cap=5]
                   │
                   ↓
  底层数组:  [0] [0] [0] [ ] [ ]
              ← len=3 →
              ←    cap=5     →

切片操作共享底层数组:
  a := []int{1, 2, 3, 4, 5}
  b := a[1:3]  // [2, 3]
  b[0] = 20    // a 也变: [1, 20, 3, 4, 5]
```

```go
// 扩容机制 (Go 1.18+)
// 源码: runtime/slice.go → growslice

// 1. 新容量计算
if newCap < 256 {
    newCap = oldCap * 2           // 小于 256: 翻倍
} else {
    newCap = oldCap + oldCap/4 + 192 // ≥ 256: 约 1.25 倍 + 192
}

// 2. 内存对齐
// 实际容量会按内存分配器的 size class 向上对齐

// 陷阱: append 返回新 slice
s := []int{1, 2, 3}
s2 := append(s, 4)  // 可能返回新底层数组
// 必须接收返回值: s = append(s, 4)

// 预分配避免扩容
s := make([]int, 0, 1000) // 预分配容量
for i := 0; i < 1000; i++ {
    s = append(s, i) // 不会触发扩容
}

// 拷贝 slice (避免共享底层数组)
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src) // 深拷贝
```

## 3. map 底层实现和注意事项？

**回答：**

```
map 底层: 哈希表 (runtime.hmap)
  ┌──────────────────────────────────────┐
  │ type hmap struct {                    │
  │     count     int    // 键值对数量     │
  │     B         uint8  // 桶数 = 2^B    │
  │     hash0     uint32 // 哈希种子      │
  │     buckets   unsafe.Pointer // 桶数组│
  │     oldbuckets unsafe.Pointer // 旧桶 │
  │     nevacuate uintptr // 搬迁进度     │
  │ }                                     │
  └──────────────────────────────────────┘

  bucket (bmap) 结构:
  ┌──────────────────────────────────────┐
  │ tophash [8]uint8  // 高8位哈希, 快速比较│
  │ keys    [8]K      // 8 个 key         │
  │ values  [8]V      // 8 个 value       │
  │ overflow *bmap    // 溢出桶指针        │
  └──────────────────────────────────────┘

查找过程:
  1. 计算 key 的哈希值
  2. 低 B 位确定桶号
  3. 高 8 位 (tophash) 在桶内快速比较
  4. tophash 匹配 → 比较完整 key
  5. 当前桶找不到 → 检查溢出桶

扩容:
  翻倍扩容: 装载因子 > 6.5 (count / 2^B > 6.5)
  等量扩容: 溢出桶太多 (整理碎片, 桶数不变)
  渐进式迁移: 每次读写搬迁 1~2 个桶
```

```go
// 注意事项

// 1. map 不是线程安全的!
// 并发读写会 panic: fatal error: concurrent map read and map write
// 解决: sync.RWMutex 或 sync.Map

// 2. map 遍历顺序随机
for k, v := range m { // 每次顺序可能不同
}

// 3. map 的 value 不可寻址
type User struct{ Age int }
m := map[string]User{"a": {Age: 1}}
// m["a"].Age = 2  // 编译错误!
// 解决: 用指针
m2 := map[string]*User{"a": {Age: 1}}
m2["a"].Age = 2 // OK

// 4. 判断 key 是否存在
if v, ok := m[key]; ok {
    // key 存在
}

// 5. nil map 可读不可写
var m map[string]int
_ = m["key"]      // OK, 返回零值
// m["key"] = 1   // panic!
```

## 4. string 的底层实现和常见操作？

**回答：**

```
string 底层:
  ┌─────────────────────────────┐
  │ type stringHeader struct {   │
  │     Data uintptr  // 字节数组│
  │     Len  int      // 字节长度│
  │ }                            │
  └─────────────────────────────┘

特点:
  1. 不可变 (immutable) — 修改需创建新 string
  2. UTF-8 编码
  3. len() 返回字节数, 不是字符数
  4. 可以包含任意字节 (包括 \x00)
```

```go
// string 与 []byte 转换
s := "hello"
b := []byte(s) // string → []byte (有拷贝)
s2 := string(b) // []byte → string (有拷贝)

// 零拷贝转换 (unsafe, 谨慎使用)
func stringToBytes(s string) []byte {
    return unsafe.Slice(unsafe.StringData(s), len(s))
}

// 高效拼接
// 差: s = s + "a" + "b"  // 每次创建新 string
// 好: strings.Builder
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString("hello")
}
result := builder.String()

// 字符串遍历
s := "你好Go"
// 按字节遍历
for i := 0; i < len(s); i++ {
    fmt.Printf("%x ", s[i]) // UTF-8 字节
}
// 按字符遍历
for i, r := range s {
    fmt.Printf("%d:%c ", i, r) // 0:你 3:好 6:G 7:o
}
```

## 5. struct 内存对齐？

**回答：**

```
内存对齐规则:
  1. 每个字段的偏移量必须是该字段大小的整数倍
  2. struct 总大小必须是最大字段大小的整数倍

示例:
  type Bad struct {
      a bool   // 1 byte + 7 padding
      b int64  // 8 bytes
      c bool   // 1 byte + 7 padding
  } // 总共 24 bytes

  type Good struct {
      b int64  // 8 bytes
      a bool   // 1 byte
      c bool   // 1 byte + 6 padding
  } // 总共 16 bytes

  Bad:  [a|_|_|_|_|_|_|_|b b b b b b b b|c|_|_|_|_|_|_|_]  24B
  Good: [b b b b b b b b|a|c|_|_|_|_|_|_]                   16B
```

```go
// 查看类型大小和对齐
fmt.Println(unsafe.Sizeof(Bad{}))   // 24
fmt.Println(unsafe.Sizeof(Good{}))  // 16
fmt.Println(unsafe.Alignof(Good{}.b)) // 8

// 最佳实践: 按字段大小从大到小排列
type Optimized struct {
    id      int64   // 8
    balance float64 // 8
    age     int32   // 4
    status  int16   // 2
    active  bool    // 1
    flag    bool    // 1
} // 24 bytes (紧凑)

// 空结构体 struct{} 大小为 0
// 常用于: channel 信号, map 模拟 set
done := make(chan struct{})
set := map[string]struct{}{}
```

## 6. 常量 iota 的用法？

**回答：**

```go
// iota: 常量计数器, 每行 +1, 每个 const 块重置

// 基本用法
const (
    A = iota // 0
    B        // 1
    C        // 2
)

// 跳过值
const (
    A = iota // 0
    _        // 跳过 1
    C        // 2
)

// 位运算 (权限/标志位)
const (
    Read    = 1 << iota // 1   (0001)
    Write               // 2   (0010)
    Execute             // 4   (0100)
)
perm := Read | Write // 3 (0011)

// 单位换算
const (
    _  = iota
    KB = 1 << (10 * iota) // 1024
    MB                     // 1048576
    GB                     // 1073741824
    TB                     // 1099511627776
)

// 多个 iota 同行
const (
    a, b = iota, iota + 10 // 0, 10
    c, d                    // 1, 11
)
```

## 7. 值类型和引用类型？make 和 new 的区别？

**回答：**

```
值类型 vs 引用类型:
  ┌──────────────┬──────────────────────────────┐
  │ 值类型        │ int, float, bool, string,    │
  │              │ array, struct                │
  │              │ → 赋值/传参拷贝完整值         │
  ├──────────────┼──────────────────────────────┤
  │ 引用类型      │ slice, map, channel,         │
  │              │ interface, func              │
  │              │ → 赋值/传参共享底层数据       │
  └──────────────┴──────────────────────────────┘
```

```go
// new vs make

// new(T): 分配内存, 返回 *T, 填充零值
// 适用于所有类型
p := new(int)     // *int, 值为 0
s := new(MyStruct) // *MyStruct, 各字段零值

// make(T, args): 初始化并返回 T (不是指针)
// 仅适用于 slice, map, channel
s := make([]int, 5, 10) // []int, len=5, cap=10
m := make(map[string]int) // map[string]int (已初始化)
ch := make(chan int, 3)   // chan int, 缓冲区大小 3

// 区别总结:
// new  → 分配零值内存, 返回指针
// make → 初始化内部结构, 返回值类型本身
```

## 8. 类型转换和类型断言？

**回答：**

```go
// 类型转换: T(v) — 兼容类型间转换
var i int = 42
var f float64 = float64(i) // int → float64
var u uint = uint(f)       // float64 → uint

// string 与数值转换需要 strconv
s := strconv.Itoa(42)          // int → string "42"
i, err := strconv.Atoi("42")  // string → int

// []byte ↔ string
b := []byte("hello") // string → []byte
s := string(b)        // []byte → string

// 类型断言: i.(T) — 从 interface 提取具体类型
var i interface{} = "hello"
s := i.(string)      // 成功: s = "hello"
// n := i.(int)      // panic!

s, ok := i.(string)  // 安全断言
if ok {
    fmt.Println(s)
}

// 类型 switch
switch v := i.(type) {
case string:
    fmt.Println("string:", v)
case int:
    fmt.Println("int:", v)
default:
    fmt.Println("unknown")
}
```

## 9. Go 的比较规则？

**回答：**

```
可比较类型:
  ┌──────────────┬──────────────────────────────┐
  │ 可用 ==      │ bool, int, float, complex,   │
  │              │ string, pointer, channel,    │
  │              │ array (元素可比较), struct    │
  │              │ (字段都可比较), interface     │
  ├──────────────┼──────────────────────────────┤
  │ 不可用 ==    │ slice, map, func             │
  │              │ → 只能与 nil 比较             │
  └──────────────┴──────────────────────────────┘
```

```go
// struct 比较: 所有字段可比较时可用 ==
type Point struct{ X, Y int }
p1, p2 := Point{1, 2}, Point{1, 2}
fmt.Println(p1 == p2) // true

// 包含不可比较字段的 struct 不能用 ==
type Data struct {
    Values []int // slice 不可比较
}
// d1 == d2 // 编译错误!
// 用 reflect.DeepEqual(d1, d2) 但有性能开销

// interface 比较
// 类型和值都相同才相等
var a interface{} = 1
var b interface{} = 1
fmt.Println(a == b) // true

// 注意: 接口底层是不可比较类型时, == 会 panic
var x interface{} = []int{1}
var y interface{} = []int{1}
// fmt.Println(x == y) // panic!
```

## 10. Go 基础语法面试速答？

**回答：**

```
Q: Go 的零值是什么?
A: 变量声明未赋值时的默认值
   数值→0, bool→false, string→"", 引用类型→nil

Q: slice 和 array 区别?
A: array 定长, 值类型, [N]T
   slice 变长, 引用类型, []T, 底层指向 array

Q: map 是线程安全的吗?
A: 不是! 并发读写会 panic
   用 sync.RWMutex 或 sync.Map

Q: string 是可变的吗?
A: 不可变, 底层 (pointer, len)
   修改需转 []byte 再转回

Q: new 和 make 区别?
A: new(T) → 分配零值, 返回 *T, 适用所有类型
   make(T) → 初始化, 返回 T, 仅 slice/map/chan

Q: struct 可以比较吗?
A: 所有字段可比较时可以用 ==
   包含 slice/map/func 字段的不行

Q: iota 是什么?
A: const 块中的行计数器, 从 0 开始
   每行+1, 新 const 块重置

Q: 内存对齐怎么优化?
A: struct 字段按大小从大到小排列
   减少 padding, 节省内存

Q: 值传递还是引用传递?
A: Go 只有值传递! 但 slice/map/chan
   内部含指针, 传值也能修改底层数据
```
