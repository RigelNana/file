# 鉴权与安全 八股文

## 一、认证与授权基础

### 1. 认证(Authentication)和授权(Authorization)的区别？

**答：** 认证是验证"你是谁"（身份识别），授权是判断"你能做什么"（权限控制）。认证在前，授权在后。常见认证方式：用户名密码、Token、证书、生物识别。常见授权模型：RBAC（基于角色）、ABAC（基于属性）、ACL（访问控制列表）。

### 2. Session和Token认证的区别？

**答：** Session是服务端有状态，用户登录后服务端存储Session，客户端通过Cookie携带SessionID。Token是服务端无状态，登录后签发Token（如JWT），客户端每次请求携带Token，服务端验证签名即可。Token适合分布式系统（无需共享Session），Session适合传统Web应用。

### 3. JWT的组成和原理？

**答：** JWT由Header.Payload.Signature三部分组成，Base64编码用`.`连接。Header包含算法类型（HS256/RS256），Payload包含声明（用户ID、过期时间等），Signature是对前两部分的签名。服务端用密钥验证签名，不需要查询数据库。缺点：无法主动失效（需配合黑名单/短过期+RefreshToken）。

### 4. OAuth2.0的四种授权模式？

**答：** 授权码模式（最安全，适合Web应用）、隐式模式（已不推荐）、客户端凭证模式（服务间调用）、密码模式（高信任应用）。推荐使用授权码模式+PKCE。核心流程：用户授权→获取授权码→用授权码换AccessToken→用Token访问资源。

### 5. RBAC权限模型？

**答：** RBAC = Role-Based Access Control。用户→角色→权限的三级结构。RBAC0基础模型，RBAC1支持角色继承，RBAC2支持角色互斥/数量约束，RBAC3 = RBAC1+RBAC2。实现：用户表、角色表、权限表、用户角色关联表、角色权限关联表。

### 6. SSO单点登录原理？

**答：** 一次登录，多个系统共享认证状态。实现方式：1）同域Cookie共享；2）CAS（Central Authentication Service）中心认证服务，各子系统通过ticket验证；3）基于OAuth2/OIDC的SSO，统一认证中心签发Token。核心流程：未登录→跳转认证中心→登录→携带凭证回调→各系统验证凭证。

### 7. HTTPS和TLS的作用？

**答：** HTTPS = HTTP + TLS，提供三个安全保障：1）加密（防窃听，对称+非对称加密）；2）认证（证书验证服务器身份，防冒充）；3）完整性（MAC校验，防篡改）。TLS1.3握手只需1-RTT（1.2需要2-RTT），更安全更快。

### 8. 常见Web安全攻击及防御？

**答：**

| 攻击 | 原理 | 防御 |
|------|------|------|
| XSS | 注入恶意脚本 | 输出转义、CSP |
| CSRF | 伪造用户请求 | Token验证、SameSite Cookie |
| SQL注入 | 拼接恶意SQL | 参数化查询、ORM |
| SSRF | 伪造服务端请求 | 白名单、禁止内网访问 |
| 越权 | 绕过权限检查 | 统一鉴权中间件 |

### 9. 密码安全存储？

**答：** 永远不存明文密码。正确做法：使用bcrypt/scrypt/argon2等慢哈希算法。bcrypt自带盐值，Cost因子控制计算强度（推荐10-12）。不要用MD5/SHA256（太快易暴力破解）。Go中使用`golang.org/x/crypto/bcrypt`，`bcrypt.GenerateFromPassword(password, bcrypt.DefaultCost)`。

### 10. API安全设计？

**答：** 1）认证：JWT/OAuth2；2）授权：RBAC/中间件统一鉴权；3）传输加密：HTTPS；4）输入校验：参数验证+SQL注入防护；5）限流：防暴力破解/DDoS；6）日志审计：记录关键操作；7）敏感数据：加密存储+脱敏返回；8）CORS：严格配置允许域名。

## 二、Token与会话管理

### 11. Access Token和Refresh Token？

**答：** Access Token短有效期（15min-2h），用于访问资源。Refresh Token长有效期（7-30天），用于刷新Access Token。Access Token过期后，用Refresh Token获取新的Access Token，无需重新登录。Refresh Token可单独吊销，安全性更高。

### 12. JWT无法主动失效怎么办？

**答：** 方案：1）短过期时间 + Refresh Token轮换；2）Token黑名单（Redis存已注销的Token，检查时先查黑名单）；3）Token版本号（用户表存token_version，签发时写入JWT，验证时比对）。推荐方案1+2组合。

## 三、加密与安全

### 13. 对称加密和非对称加密？

**答：** 对称加密：加解密用同一密钥（AES），速度快，适合大量数据。非对称加密：公钥加密、私钥解密（RSA/ECDSA），速度慢，适合密钥交换和签名。HTTPS同时用两者：非对称加密协商对称密钥，对称加密传输数据。

### 14. 数据脱敏怎么做？

**答：** 手机号：138****8888；身份证：110***********1234；邮箱：u***@example.com；银行卡：6222 **** **** 1234。实现：1）应用层脱敏（返回前处理）；2）数据库函数脱敏；3）统一脱敏中间件。敏感数据加密存储（AES），查询时解密+脱敏返回。
