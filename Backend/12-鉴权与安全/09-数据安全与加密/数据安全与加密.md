## 1. 对称加密与非对称加密？

**回答：**

```
  ┌──────────────┬───────────────┬───────────────┐
  │ 维度          │ 对称加密       │ 非对称加密     │
  ├──────────────┼───────────────┼───────────────┤
  │ 密钥          │ 加解密同一个   │ 公钥加密私钥解 │
  │ 速度          │ 快 100-1000x  │ 慢             │
  │ 密钥分发      │ 困难          │ 公钥可公开     │
  │ 典型算法      │ AES/ChaCha20  │ RSA/ECDSA/Ed25519│
  │ 用途          │ 数据加密       │ 密钥交换/签名  │
  └──────────────┴───────────────┴───────────────┘

  实际中组合使用（混合加密）：
  TLS握手 = RSA/ECDHE交换会话密钥 + AES加密数据

  推荐算法：
  对称：AES-256-GCM（认证加密 防篡改）
  非对称：ECDSA P-256 / Ed25519
  密钥交换：X25519 / ECDHE

  Go AES-GCM加密：
  func Encrypt(key, plaintext []byte) ([]byte, error) {
      block, err := aes.NewCipher(key)
      if err != nil { return nil, err }
      
      gcm, err := cipher.NewGCM(block)
      if err != nil { return nil, err }
      
      nonce := make([]byte, gcm.NonceSize())
      if _, err := io.ReadFull(rand.Reader, nonce); err != nil {
          return nil, err
      }
      
      // nonce前缀 + 密文（含认证Tag）
      return gcm.Seal(nonce, nonce, plaintext, nil), nil
  }
  
  func Decrypt(key, ciphertext []byte) ([]byte, error) {
      block, _ := aes.NewCipher(key)
      gcm, _ := cipher.NewGCM(block)
      
      nonceSize := gcm.NonceSize()
      nonce, ct := ciphertext[:nonceSize], ciphertext[nonceSize:]
      
      return gcm.Open(nil, nonce, ct, nil)
  }
```

## 2. 哈希与消息认证码？

**回答：**

```
  哈希 = 单向不可逆 固定长度摘要
  ┌──────────────┬──────────┬──────────────────┐
  │ 算法          │ 长度     │ 用途             │
  ├──────────────┼──────────┼──────────────────┤
  │ MD5           │ 128bit   │ 仅文件校验(不安全)│
  │ SHA-1         │ 160bit   │ 已不安全         │
  │ SHA-256       │ 256bit   │ 通用哈希         │
  │ SHA-3         │ 可变     │ 新一代           │
  │ BLAKE2/3      │ 可变     │ 快速安全         │
  └──────────────┴──────────┴──────────────────┘

  HMAC = Hash-based Message Authentication Code
  用途：验证消息完整性 + 来源认证

  Go HMAC：
  func SignHMAC(key []byte, message string) string {
      mac := hmac.New(sha256.New, key)
      mac.Write([]byte(message))
      return hex.EncodeToString(mac.Sum(nil))
  }
  
  func VerifyHMAC(key []byte, message, signature string) bool {
      expected := SignHMAC(key, message)
      return hmac.Equal([]byte(expected), []byte(signature))
  }

  密码哈希 ≠ 普通哈希：
  bcrypt/scrypt/argon2 专为密码设计
  1. 加盐（每个密码独立盐）
  2. 计算慢（防暴力破解）
  3. 可调参数（工作因子）

  Go bcrypt：
  hash, _ := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
  err := bcrypt.CompareHashAndPassword(hash, []byte(password))
```

## 3. TLS/HTTPS详解？

**回答：**

```
  TLS 1.3 握手流程（1-RTT）：
  Client                              Server
  │──── ClientHello ────────────────→│
  │     支持的密码套件                │
  │     Key Share（ECDHE公钥）       │
  │                                   │
  │←─── ServerHello ────────────────│
  │     选定密码套件                  │
  │     Key Share（ECDHE公钥）       │
  │     证书 + 签名                   │
  │     Finished                      │
  │                                   │
  │──── Finished ───────────────────→│
  │     开始加密通信                  │

  TLS 1.3 改进：
  ┌──────────────────────┬──────────────────┐
  │ 改进                  │ 说明             │
  ├──────────────────────┼──────────────────┤
  │ 1-RTT握手             │ 减少延迟         │
  │ 0-RTT（可选）         │ 恢复时0延迟      │
  │ 移除不安全算法         │ 仅AEAD(GCM/ChaCha)│
  │ 前向保密（必须）       │ ECDHE密钥交换    │
  │ 加密的握手消息         │ 证书加密传输     │
  └──────────────────────┴──────────────────┘

  Go HTTPS服务：
  // 自动Let's Encrypt
  import "golang.org/x/crypto/acme/autocert"
  
  m := &autocert.Manager{
      Cache:      autocert.DirCache("certs"),
      Prompt:     autocert.AcceptTOS,
      HostPolicy: autocert.HostWhitelist("example.com"),
  }
  
  srv := &http.Server{
      Addr:      ":443",
      TLSConfig: m.TLSConfig(),
  }
  srv.ListenAndServeTLS("", "")

  证书链：根CA → 中间CA → 服务器证书
  浏览器信任根CA → 验证整个链 → 确认服务器身份
```

## 4. 密钥管理？

**回答：**

```
  密钥管理原则：
  1. 不在代码中硬编码密钥
  2. 密钥和数据分开存储
  3. 定期轮换
  4. 最小权限访问
  5. 审计密钥使用

  密钥管理方案：
  ┌──────────────┬──────────────────────────────┐
  │ 方案          │ 适用场景                     │
  ├──────────────┼──────────────────────────────┤
  │ HashiCorp Vault│ 自建 功能全面               │
  │ AWS KMS       │ AWS生态                      │
  │ 阿里云KMS     │ 阿里云生态                   │
  │ K8s Secrets   │ 简单场景（需加密etcd）       │
  │ 环境变量      │ 开发/简单部署                │
  └──────────────┴──────────────────────────────┘

  信封加密（Envelope Encryption）：
  数据 ← DEK加密（数据加密密钥）
  DEK ← KEK加密（密钥加密密钥 存KMS）
  优势：轮换KEK不需要重加密数据

  K8s Secret + 外部密钥管理：
  # External Secrets Operator
  apiVersion: external-secrets.io/v1beta1
  kind: ExternalSecret
  metadata:
    name: db-credentials
  spec:
    refreshInterval: 1h
    secretStoreRef:
      name: vault-backend
      kind: SecretStore
    target:
      name: db-credentials
    data:
    - secretKey: password
      remoteRef:
        key: secret/data/db
        property: password

  密钥轮换：
  双密钥窗口期 → 新旧密钥同时有效
  解密时先尝试新密钥 失败再用旧密钥
  过渡期结束后移除旧密钥
```

## 5. 数据脱敏？

**回答：**

```
  脱敏 = 隐藏敏感信息 保留数据格式

  脱敏规则：
  ┌──────────────┬───────────────┬───────────────┐
  │ 数据类型      │ 原始           │ 脱敏后        │
  ├──────────────┼───────────────┼───────────────┤
  │ 手机号        │ 13812345678   │ 138****5678   │
  │ 身份证        │ 110105199001011234│ 110***********34│
  │ 邮箱          │ user@test.com │ u***@test.com │
  │ 银行卡        │ 6222021234561234│ 6222****1234  │
  │ 姓名          │ 张三           │ 张*           │
  └──────────────┴───────────────┴───────────────┘

  Go脱敏函数：
  func MaskPhone(phone string) string {
      if len(phone) != 11 { return phone }
      return phone[:3] + "****" + phone[7:]
  }
  
  func MaskIDCard(id string) string {
      if len(id) < 6 { return id }
      return id[:3] + strings.Repeat("*", len(id)-5) + id[len(id)-2:]
  }
  
  func MaskEmail(email string) string {
      parts := strings.SplitN(email, "@", 2)
      if len(parts) != 2 { return email }
      name := parts[0]
      if len(name) <= 1 { return email }
      return name[:1] + "***@" + parts[1]
  }

  脱敏策略：
  1. 响应层脱敏（Middleware/序列化时）
  2. 日志脱敏（Log中自动替换）
  3. 数据库层脱敏（视图/权限控制）

  JSON序列化脱敏：
  type User struct {
      Name  string `json:"name"`
      Phone string `json:"phone"`
  }
  
  func (u User) MarshalJSON() ([]byte, error) {
      type Alias User
      return json.Marshal(&struct {
          Alias
          Phone string `json:"phone"`
      }{
          Alias: Alias(u),
          Phone: MaskPhone(u.Phone),
      })
  }
```

## 6. 数据分级与合规？

**回答：**

```
  数据分级：
  ┌──────────────┬──────────────────────────────┐
  │ 级别          │ 示例                         │
  ├──────────────┼──────────────────────────────┤
  │ 公开          │ 产品介绍/新闻               │
  │ 内部          │ 内部文档/员工通讯           │
  │ 机密          │ 用户PII/财务报表            │
  │ 绝密          │ 密钥/核心算法/密码          │
  └──────────────┴──────────────────────────────┘

  PII（个人可识别信息）：
  姓名/身份证/手机/邮箱/地址/生物信息
  → 存储加密 + 传输加密 + 脱敏展示

  合规要求：
  GDPR（欧盟）：
    用户有权被遗忘（删除数据）
    数据最小收集
    跨境传输限制
    72小时数据泄露通知

  CCPA（加州）/中国个保法：
    类似GDPR 侧重点不同

  技术对应：
  合规要求         → 技术实现
  数据最小收集     → API只返回必要字段
  存储加密         → AES + KMS
  传输加密         → TLS 1.3
  访问控制         → RBAC + 审计日志
  数据删除         → 软删除 + 定时清理
  数据可迁移       → 导出API
```

## 7. 数据库加密？

**回答：**

```
  三个层次：
  1. 传输加密（TLS连接）
     连接串加 sslmode=require
  
  2. 存储加密（TDE 透明数据加密）
     磁盘级加密 对应用透明
     MySQL: CREATE TABLE t ENGINE=InnoDB ENCRYPTION='Y'
  
  3. 字段级加密（应用层）
     敏感字段单独加密存储

  Go字段级加密：
  type EncryptedField struct {
      Ciphertext []byte
  }
  
  // GORM自定义类型
  func (e *EncryptedField) Scan(value interface{}) error {
      bytes, ok := value.([]byte)
      if !ok { return fmt.Errorf("invalid type") }
      e.Ciphertext = bytes
      return nil
  }
  
  func (e EncryptedField) Value() (driver.Value, error) {
      return e.Ciphertext, nil
  }
  
  type User struct {
      ID    uint
      Name  string
      Phone EncryptedField // 加密存储
  }
  
  // 写入时加密
  ct, _ := Encrypt(key, []byte("13812345678"))
  user.Phone = EncryptedField{Ciphertext: ct}
  
  // 读取时解密
  plaintext, _ := Decrypt(key, user.Phone.Ciphertext)

  检索加密数据：
  方案1：存储哈希索引（精确匹配）
  方案2：确定性加密（Same input → Same output）
  方案3：盲索引（前N位哈希）
  注意：加密后无法模糊查询
```

## 8. 数字签名与证书？

**回答：**

```
  数字签名 = 私钥签名 + 公钥验证
  作用：身份认证 + 完整性 + 不可否认

  流程：
  发送方                              接收方
  数据 → Hash → 摘要                  数据 → Hash → 摘要A
  摘要 → 私钥加密 → 签名              签名 → 公钥解密 → 摘要B
  发送 [数据 + 签名]                   比较 摘要A == 摘要B

  Go Ed25519签名：
  // 生成密钥对
  pub, priv, _ := ed25519.GenerateKey(rand.Reader)
  
  // 签名
  message := []byte("important data")
  signature := ed25519.Sign(priv, message)
  
  // 验证
  valid := ed25519.Verify(pub, message, signature)

  X.509证书：
  包含：公钥 + 持有者信息 + 颁发者 + 有效期 + 签名
  证书链：Root CA → Intermediate CA → 服务器证书

  mTLS（双向TLS）：
  客户端和服务端都验证对方证书
  微服务间通信/API网关常用

  Go mTLS Server：
  caCert, _ := os.ReadFile("ca.crt")
  pool := x509.NewCertPool()
  pool.AppendCertsFromPEM(caCert)
  
  tlsConfig := &tls.Config{
      ClientCAs:  pool,
      ClientAuth: tls.RequireAndVerifyClientCert,
  }
  srv := &http.Server{TLSConfig: tlsConfig}

  JWT中的签名：
  RS256 = RSA-SHA256（私钥签名 公钥验证）
  适合微服务：Auth服务私钥签发 各服务公钥验证
```

## 9. 安全审计与监控？

**回答：**

```
  审计日志记录什么：
  1. 认证事件：登录/登出/失败/MFA
  2. 授权事件：权限拒绝/越权尝试
  3. 数据操作：敏感数据CRUD
  4. 系统操作：配置变更/用户管理
  5. API调用：关键接口调用记录

  审计日志结构：
  type AuditLog struct {
      Timestamp  time.Time `json:"timestamp"`
      UserID     string    `json:"user_id"`
      Action     string    `json:"action"`
      Resource   string    `json:"resource"`
      ResourceID string    `json:"resource_id"`
      IP         string    `json:"ip"`
      UserAgent  string    `json:"user_agent"`
      Result     string    `json:"result"` // success/denied/error
      Detail     string    `json:"detail"`
  }
  
  func RecordAudit(ctx context.Context, log AuditLog) {
      log.Timestamp = time.Now()
      log.UserID = getUserID(ctx)
      log.IP = getClientIP(ctx)
      // 写入独立审计存储（不可篡改）
      auditStore.Write(log)
  }

  安全监控告警：
  ┌──────────────────────┬──────────────────────┐
  │ 指标                  │ 阈值                 │
  ├──────────────────────┼──────────────────────┤
  │ 登录失败次数          │ 同IP 5次/分钟        │
  │ 403响应激增           │ 同用户 10次/分钟     │
  │ 异常API调用           │ 新IP访问敏感接口     │
  │ 大量数据导出          │ 单次>1000条          │
  │ 非工作时间操作        │ admin夜间操作        │
  └──────────────────────┴──────────────────────┘

  审计日志要求：
  1. 独立存储（攻击者无法删除）
  2. 不可篡改（追加写入/WORM）
  3. 保留时间满足合规（6个月~7年）
  4. 不记录敏感数据（密码/Token）
```

## 10. 数据安全面试速答？

**回答：**

```
Q: 对称加密和非对称加密区别？
A: 对称加密一个密钥(AES 快)
   非对称公钥加密私钥解(RSA 慢)
   实际TLS组合使用

Q: 推荐什么加密算法？
A: 对称AES-256-GCM 非对称Ed25519/ECDSA
   密码bcrypt/argon2 哈希SHA-256

Q: 密钥怎么管理？
A: 不硬编码 用Vault/KMS
   信封加密(DEK加密数据 KEK加密DEK)
   定期轮换

Q: 怎么做数据脱敏？
A: 手机138****5678 邮箱u***@test.com
   响应层统一脱敏 日志也要脱敏

Q: GDPR要求什么？
A: 最小收集+存储加密+传输加密
   数据删除权+72h泄露通知

Q: 数据库怎么加密？
A: 传输TLS+存储TDE+敏感字段应用层加密
   加密后无法模糊查询 用盲索引解决

Q: 什么是数字签名？
A: 私钥签名公钥验证
   身份认证+完整性+不可否认

Q: 审计日志怎么做？
A: 记录认证/授权/敏感操作
   独立存储+不可篡改+不记录密码
```
