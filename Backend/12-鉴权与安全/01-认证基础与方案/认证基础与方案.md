## 1. 认证方案全景？

**回答：**

```
  ┌──────────────────┬──────────────────────────────┐
  │ 认证方式          │ 适用场景                     │
  ├──────────────────┼──────────────────────────────┤
  │ 用户名+密码       │ 传统Web登录                  │
  │ Session+Cookie    │ 服务端渲染应用               │
  │ JWT Token         │ 前后端分离/微服务            │
  │ OAuth2            │ 第三方登录                   │
  │ API Key           │ 开放API/服务间调用           │
  │ mTLS证书          │ 服务间安全通信               │
  │ LDAP/AD           │ 企业内部认证                 │
  │ 生物识别          │ 移动端（指纹/面部）          │
  └──────────────────┴──────────────────────────────┘

  现代认证推荐：
  ToC应用：JWT + OAuth2（第三方登录）
  ToB应用：JWT + RBAC + SSO
  微服务间：mTLS / JWT
  开放API：API Key + OAuth2
```

## 2. 基于用户名密码的认证？

**回答：**

```
  流程：
  1. 用户提交用户名+密码
  2. 服务端查询用户 验证密码Hash
  3. 验证通过 → 签发Token/创建Session

  密码安全存储（bcrypt）：
  import "golang.org/x/crypto/bcrypt"
  
  // 注册时 Hash密码
  func HashPassword(password string) (string, error) {
      bytes, err := bcrypt.GenerateFromPassword(
          []byte(password), bcrypt.DefaultCost) // Cost=10
      return string(bytes), err
  }
  
  // 登录时 验证密码
  func CheckPassword(password, hash string) bool {
      err := bcrypt.CompareHashAndPassword(
          []byte(hash), []byte(password))
      return err == nil
  }

  密码策略：
  最少8位 包含大小写+数字+特殊字符
  限制登录尝试次数（5次锁定）
  不允许使用常见弱密码
  定期要求修改密码（可选 NIST已不推荐）

  防暴力破解：
  登录限流（IP+账号维度）
  验证码（连续失败后）
  账号锁定（临时锁定30分钟）
```

## 3. 多因素认证(MFA)？

**回答：**

```
  MFA = Multi-Factor Authentication

  三种因素：
  你知道的：密码/PIN
  你拥有的：手机/硬件Key/Authenticator
  你的特征：指纹/面部/虹膜

  MFA至少使用两种不同因素

  TOTP（推荐）：
  Time-based One-Time Password
  RFC 6238 标准
  Google Authenticator/Authy

  TOTP原理：
  1. 服务端生成Secret → 用户绑定Authenticator
  2. 双方用 Secret + 当前时间窗口 生成6位码
  3. 用户输入验证码 服务端比对

Go TOTP实现：
  import "github.com/pquerna/otp/totp"
  
  // 生成密钥
  key, _ := totp.Generate(totp.GenerateOpts{
      Issuer:      "MyApp",
      AccountName: "user@example.com",
  })
  // key.Secret() → 存数据库
  // key.URL()    → 生成二维码

  // 验证
  valid := totp.Validate(userInputCode, secret)

  其他MFA方式：
  短信验证码（不推荐 SIM卡劫持风险）
  WebAuthn/FIDO2（硬件安全Key 最安全）
  邮箱验证码
```

## 4. HTTP认证机制？

**回答：**

```
  1. Basic Auth
     Authorization: Basic base64(user:pass)
     简单但不安全（Base64非加密）
     必须配合HTTPS使用
     适合内部工具/测试

  2. Bearer Token
     Authorization: Bearer <token>
     JWT/OAuth2常用
     无状态 适合API

  3. Digest Auth
     基于质询-响应（Challenge-Response）
     比Basic安全但使用少

  4. API Key
     Header: X-API-Key: xxx
     Query: ?api_key=xxx
     简单 适合开放API

Go中间件统一认证：
  func AuthMiddleware(next http.Handler) http.Handler {
      return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
          token := r.Header.Get("Authorization")
          if token == "" {
              http.Error(w, "unauthorized", 401)
              return
          }
          
          // 去掉 "Bearer " 前缀
          token = strings.TrimPrefix(token, "Bearer ")
          
          claims, err := validateJWT(token)
          if err != nil {
              http.Error(w, "invalid token", 401)
              return
          }
          
          // 将用户信息放入context
          ctx := context.WithValue(r.Context(), "user", claims)
          next.ServeHTTP(w, r.WithContext(ctx))
      })
  }
```

## 5. 认证与网关？

**回答：**

```
  API网关统一认证 → 后端服务不需要重复验证

  ┌────────┐     ┌──────────────┐     ┌──────────┐
  │ Client │────→│  API Gateway │────→│ Service  │
  │  JWT   │     │  验证JWT     │     │ 信任内部 │
  │        │     │  注入用户信息 │     │ X-User-ID│
  └────────┘     └──────────────┘     └──────────┘

  网关验证JWT后：
  通过Header传递用户信息给后端
  X-User-ID: 12345
  X-User-Role: admin

  后端服务只读Header 不需要再验证JWT

  Kong/APISIX JWT插件：
  自动验证签名
  自动检查过期
  可配置白名单路由（不需认证的API）

  内部服务间调用：
  mTLS（证书双向认证）
  或 内部JWT（服务账号签发）
  K8s内部通信 → Service Mesh(Istio) mTLS自动管理
```

## 6. 无密码认证？

**回答：**

```
  Passwordless = 不使用密码的认证方式

  方式：
  1. Magic Link
     输入邮箱 → 发送登录链接 → 点击登录
     链接含一次性Token 有效期短（10分钟）

  2. WebAuthn/FIDO2
     硬件安全Key（YubiKey）或平台认证器（TouchID）
     基于公钥加密 最安全
     注册：生成密钥对 公钥存服务端
     登录：私钥签名Challenge 服务端验证

  3. OTP（一次性密码）
     手机/邮箱收验证码
     TOTP（Authenticator App更安全）

  4. Passkey（Apple/Google推广）
     WebAuthn的用户友好封装
     跨设备同步 替代密码

  Magic Link Go示例：
  func SendMagicLink(email string) error {
      token := generateSecureToken(32)
      // 存Redis 10分钟过期
      rdb.Set(ctx, "magic:"+token, email, 10*time.Minute)
      
      link := fmt.Sprintf("https://app.com/auth/verify?token=%s", token)
      return sendEmail(email, "Login", "Click: "+link)
  }
  
  func VerifyMagicLink(token string) (string, error) {
      email, err := rdb.GetDel(ctx, "magic:"+token).Result()
      if err != nil {
          return "", errors.New("invalid or expired token")
      }
      return email, nil // 签发JWT
  }
```

## 7. 第三方登录流程？

**回答：**

```
  基于OAuth2授权码模式

  ┌────────┐  1.点击登录   ┌──────────┐
  │ 用户    │─────────────→│ 应用后端  │
  └────────┘              └─────┬────┘
       │                        │ 2.重定向
       │  3.用户授权             ▼
       ├──────────────→ ┌──────────────┐
       │                │ GitHub/Google │ 第三方
       │  4.回调+code   │  OAuth Server│
       ├←──────────────│              │
       │                └──────┬───────┘
       │                       │
  ┌────▼───┐  5.code换token   │
  │ 应用    │─────────────────→│
  │ 后端    │←──access_token───│
  │        │  6.获取用户信息    │
  │        │─────────────────→│
  │        │←──user info───────│
  └────────┘

Go GitHub OAuth示例：
  // 1. 跳转授权
  func handleLogin(w http.ResponseWriter, r *http.Request) {
      state := generateState() // CSRF防护
      url := fmt.Sprintf(
          "https://github.com/login/oauth/authorize?client_id=%s&state=%s&scope=user:email",
          clientID, state)
      http.Redirect(w, r, url, http.StatusTemporaryRedirect)
  }
  
  // 2. 回调处理
  func handleCallback(w http.ResponseWriter, r *http.Request) {
      code := r.URL.Query().Get("code")
      // 验证state防CSRF
      
      // 3. code换token
      token := exchangeCodeForToken(code)
      
      // 4. 获取用户信息
      user := getUserInfo(token)
      
      // 5. 查找或创建用户 签发JWT
      jwt := issueJWT(user)
  }
```

## 8. 认证安全防护？

**回答：**

```
  常见认证攻击及防御：

  1. 暴力破解
     防御：登录限流+验证码+账号锁定
  
  2. 凭证填充(Credential Stuffing)
     用泄露的密码库尝试登录
     防御：监测异常登录+强制MFA
  
  3. 会话劫持
     窃取Session/Token
     防御：HTTPS+HttpOnly Cookie+Token绑定IP
  
  4. 钓鱼攻击
     伪造登录页面
     防御：WebAuthn(域名绑定)+用户教育
  
  5. Token盗取
     XSS注入窃取Token
     防御：HttpOnly Cookie存储+CSP+输出转义

  安全配置清单：
  □ HTTPS强制（HSTS头）
  □ Cookie: Secure, HttpOnly, SameSite=Lax
  □ 密码bcrypt Hash（Cost>=10）
  □ 登录失败限流
  □ JWT短过期（15min）+ RefreshToken
  □ 敏感操作二次验证
  □ 登录日志审计
```

## 9. 认证方案选型？

**回答：**

```
  ┌──────────────────┬──────────────────────────────┐
  │ 场景              │ 推荐方案                     │
  ├──────────────────┼──────────────────────────────┤
  │ 单体Web应用       │ Session + Cookie             │
  │ 前后端分离        │ JWT (Access+Refresh)         │
  │ 微服务            │ JWT + API Gateway统一验证    │
  │ 第三方登录        │ OAuth2 授权码模式+PKCE       │
  │ 企业SSO          │ OIDC + SAML                  │
  │ 服务间调用        │ mTLS / 内部JWT               │
  │ 开放API          │ API Key + OAuth2             │
  │ 高安全场景        │ MFA + WebAuthn               │
  └──────────────────┴──────────────────────────────┘

  JWT vs Session 选择：
  分布式/无状态 → JWT
  需要主动失效 → Session或JWT+黑名单
  移动端 → JWT（Cookie在移动端不便）
  SSR应用 → Session+Cookie更自然
```

## 10. 认证基础面试速答？

**回答：**

```
Q: 认证和授权的区别？
A: 认证=验证你是谁(Authentication)
   授权=判断你能做什么(Authorization)

Q: JWT和Session区别？
A: JWT无状态 服务端不存储 适合分布式
   Session有状态 服务端存储 需要共享

Q: 密码怎么安全存储？
A: bcrypt/argon2慢Hash+自动加盐
   永远不存明文 不用MD5/SHA

Q: MFA是什么？
A: 多因素认证 至少两种因素
   推荐TOTP(Authenticator)+密码

Q: OAuth2授权码流程？
A: 跳转授权→用户同意→回调带code
   →code换token→token访问资源

Q: 怎么防暴力破解？
A: 登录限流+验证码+账号临时锁定
   IP+账号双维度限制

Q: Token存哪里安全？
A: HttpOnly Cookie最安全(防XSS)
   配合Secure+SameSite属性

Q: 服务间认证怎么做？
A: mTLS证书双向认证(最安全)
   或内部JWT 通过Service Mesh管理
```
