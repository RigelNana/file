## 1. cProfile如何进行性能分析？

**回答：**

`cProfile` 是 Python 内置的确定性性能分析器，统计每个函数的调用次数和耗时。

```python
import cProfile
import pstats

# 方式1：命令行
# python -m cProfile -s cumulative script.py

# 方式2：代码内分析
def main():
    data = [i**2 for i in range(100000)]
    result = sorted(data, reverse=True)
    return sum(result[:1000])

cProfile.run('main()', 'output.prof')

# 分析结果
stats = pstats.Stats('output.prof')
stats.sort_stats('cumulative')
stats.print_stats(20)  # 前20个最耗时的函数

# 方式3：上下文管理器
import cProfile
import io

def profile_func(func):
    def wrapper(*args, **kwargs):
        pr = cProfile.Profile()
        pr.enable()
        result = func(*args, **kwargs)
        pr.disable()
        s = io.StringIO()
        ps = pstats.Stats(pr, stream=s).sort_stats('cumulative')
        ps.print_stats(15)
        print(s.getvalue())
        return result
    return wrapper
```

```
┌──────── cProfile 输出解读 ────────┐
│                                    │
│  ncalls   : 调用次数               │
│  tottime  : 函数本身耗时(不含子调用) │
│  percall  : tottime / ncalls       │
│  cumtime  : 函数总耗时(含子调用)    │
│  percall  : cumtime / ncalls       │

```

## 2. timeit如何进行微基准测试？

**回答：**

`timeit` 用于精确测量小段代码的执行时间，自动处理预热和多次运行。

```python
import timeit

# 命令行使用
# python -m timeit "'-'.join(str(n) for n in range(100))"
# python -m timeit "'-'.join(map(str, range(100)))"

# 代码内使用
t1 = timeit.timeit(
    stmt="'-'.join(str(n) for n in range(100))",
    number=10000
)

t2 = timeit.timeit(
    stmt="'-'.join(map(str, range(100)))",
    number=10000
)

print(f"生成器: {t1:.4f}s")
print(f"map:    {t2:.4f}s")

# 使用setup参数
t = timeit.timeit(
    stmt="sorted(data)",
    setup="import random; data = [random.randint(0, 1000) for _ in range(1000)]",
    number=1000
)

# repeat 多次测量取最小值（减少波动）
results = timeit.repeat(
    stmt="sum(range(1000))",
    repeat=5,
    number=10000
)
print(f"最快: {min(results):.4f}s")

# IPython/Jupyter 中
# %timeit sum(range(1000))      # 自动选择次数
# %%timeit                      # 整个cell计时
```

```
┌──────── 基准测试注意事项 ────────┐
│                                  │
│ ✅ 多次运行取最小值               │
│ ✅ 隔离测试，避免外部干扰         │
│ ✅ 使用相同数据集对比             │
│ ✅ 关注相对差异而非绝对时间       │
│ ❌ 不要在测试中包含I/O操作       │
│ ❌ 不要测量太小的代码差异         │
```

## 4. Python常见性能优化技巧？

**回答：**

```python
# 1. 选择合适的数据结构
# 查找操作：list O(n) → set/dict O(1)
items = [1, 2, 3, ..., 100000]
item_set = set(items)

x in items      # O(n) 慢
x in item_set   # O(1) 快

# 2. 列表推导代替 for 循环
# 慢
result = []
for x in range(10000):
    result.append(x**2)

# 快（快 ~30%）
result = [x**2 for x in range(10000)]

# 3. 使用局部变量
# 局部变量查找比全局变量快
def fast_func():
    local_len = len  # 缓存到局部
    result = [local_len(s) for s in strings]

# 4. 字符串拼接
# 慢: s += "a"（每次创建新字符串）
# 快: ''.join(parts)
parts = []
for chunk in data:
    parts.append(chunk)
result = ''.join(parts)

# 5. 使用 lru_cache 缓存
from functools import lru_cache

@lru_cache(maxsize=256)
def expensive(n):
    return heavy_compute(n)

# 6. 使用 __slots__ 减少内存
class Point:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

# 7. 使用生成器处理大数据
# 不要: data = [process(x) for x in huge_list]
# 应该: data = (process(x) for x in huge_list)

# 8. 避免不必要的函数调用
# 在热循环中避免频繁的属性查找和函数调用
# 慢
for item in big_list:
    result.append(item.strip().lower())

# 快
strip = str.strip
lower = str.lower
append = result.append
for item in big_list:
    append(lower(strip(item)))
```

```python
# Cython 示例 (compute.pyx)
# cython: language_level=3

def primes_python(nb_primes):
    """纯Python版本"""
    primes = []
    n = 2
    while len(primes) < nb_primes:
        for p in primes:
            if n % p == 0:
                break
        else:
            primes.append(n)
        n += 1
    return primes

# Cython类型声明版本（快10-100倍）
def primes_cython(int nb_primes):
    cdef int n, i
    cdef list primes = []
    n = 2
    while len(primes) < nb_primes:
        for p in primes:
            if n % p == 0:
                break
        else:
            primes.append(n)
        n += 1
    return primes
```

```python
import numpy as np

def dot_python(a, b):

        total += a[i] * b[i]
```

## 7. 如何优化Python程序的I/O性能？

```python
# 1. 批量I/O代替逐条操作
# 慢：逐行写入
with open('output.txt', 'w') as f:
    for line in data:
        f.write(line + '\n')

# 快：批量写入
with open('output.txt', 'w') as f:
    f.writelines(line + '\n' for line in data)

# 更快：一次写入
with open('output.txt', 'w') as f:
    f.write('\n'.join(data))

# 2. 缓冲区大小
with open('huge.bin', 'rb', buffering=1024*1024) as f:  # 1MB缓冲
    while chunk := f.read(8192):
        process(chunk)

# 3. 内存映射大文件
import mmap

with open('huge.bin', 'r+b') as f:
    mm = mmap.mmap(f.fileno(), 0)
    # mm 可以像 bytes 一样使用，但不占全部内存
    print(mm[100:200])
    mm.close()

# 4. 异步I/O
import asyncio
import aiofiles

async def read_files(paths):
    tasks = []
    for path in paths:
        tasks.append(read_file(path))
    return await asyncio.gather(*tasks)

async def read_file(path):
    async with aiofiles.open(path, 'r') as f:
        return await f.read()

# 5. 数据库批量操作
# 慢：逐条插入
for item in items:
    cursor.execute("INSERT INTO t VALUES (%s)", (item,))

# 快：批量插入
cursor.executemany("INSERT INTO t VALUES (%s)", [(i,) for i in items])

# 更快：COPY命令（PostgreSQL）
# copy_from / copy_expert

# 6. 连接池
from sqlalchemy.pool import QueuePool
```

**与 cProfile 的区别**：
- `cProfile`：函数级别分析，哪个函数最慢
- `line_profiler`：行级别分析，函数内哪行最慢
- 通常先用 cProfile 定位慢函数，再用 line_profiler 定位慢行

## 9. Python程序的并发性能优化策略？

**回答：**

```python
# 根据任务类型选择并发策略

# 1. I/O密集型 → asyncio（最高效）
import asyncio
import aiohttp

async def fetch_all(urls):
    sem = asyncio.Semaphore(50)  # 限制并发数

    async def fetch(url):
        async with sem:
            async with aiohttp.ClientSession() as session:
                async with session.get(url) as resp:
                    return await resp.text()

    return await asyncio.gather(*[fetch(url) for url in urls])

# 2. CPU密集型 → multiprocessing
from multiprocessing import Pool

def cpu_task(data):
    return heavy_compute(data)

with Pool() as pool:  # 默认使用全部CPU核心
    results = pool.map(cpu_task, data_chunks)

# 3. 混合型 → ProcessPoolExecutor + asyncio
from concurrent.futures import ProcessPoolExecutor

async def hybrid_processing(data_list):
    loop = asyncio.get_running_loop()
    with ProcessPoolExecutor() as pool:
        # CPU密集部分放进程池
        futures = [
            loop.run_in_executor(pool, cpu_task, data)
            for data in data_list
        ]
        return await asyncio.gather(*futures)

# 4. 并行数值计算 → NumPy + 多进程
import numpy as np

# NumPy 内部已使用多线程（BLAS/LAPACK）
large_matrix = np.random.rand(10000, 10000)
result = np.dot(large_matrix, large_matrix)  # 自动多核

# 5. 缓存优化
from functools import lru_cache

@lru_cache(maxsize=10000)
def cached_query(key):
    return db.query(key)
```

## 10. 性能优化与调试面试速答？

**回答：**

```
Q: Python性能分析的第一步应该做什么？
A: 先用cProfile找出热点函数（占用时间最多的函数），而不是凭感觉优化。

Q: timeit和time.time()测性能的区别？
A: timeit自动多次运行取平均、禁用GC、更精确。time.time()受系统负载影响大。

Q: 如何减少Python的内存使用？
A: 使用生成器代替列表、__slots__减少对象开销、及时del无用对象、使用array代替list存数值。

Q: 列表推导为什么比for循环快？
A: 列表推导在C层面优化了循环和append，减少了Python字节码的执行开销。

Q: 什么时候用Cython？
A: 计算密集型热点函数，保持Python语法的同时获得C级性能。适合科学计算和数据处理。

Q: PyPy和CPython的区别？
A: PyPy使用JIT编译，长时间运行的纯Python代码可快5-100倍。但对C扩展兼容性不如CPython。

Q: 如何优化数据库查询性能？
A: 使用索引、批量操作、连接池、N+1问题用预加载解决、避免SELECT *。

Q: str的+拼接为什么慢？
A: 字符串不可变，每次+都创建新对象并复制，n次拼接是O(n²)。应该用join()是O(n)。

Q: 什么是热点函数？
A: 消耗运行时间最多的函数。80%的时间通常花在20%的代码上（二八法则）。

Q: Python有哪些替代方案可以获得更好的性能？
A: Cython编译热点函数、NumPy/Pandas向量化、PyPy JIT编译、C扩展、Rust绑定(pyo3)。
```
