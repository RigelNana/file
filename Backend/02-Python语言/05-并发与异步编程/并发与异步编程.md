## 1. GIL是什么？为什么CPython需要GIL？

**回答：**

GIL（Global Interpreter Lock，全局解释器锁）是 CPython 中的一把互斥锁，**保证同一时刻只有一个线程执行 Python 字节码**。

```
┌──────────────── GIL 工作原理 ────────────────┐
│                                              │
│  线程1: ████░░░░████░░░░████                 │
│  线程2: ░░░░████░░░░████░░░░                 │
│         ─────────────────────→ 时间           │
│                                              │
│  █ = 持有GIL执行   ░ = 等待GIL              │
│                                              │
│  即使在多核CPU上，同一时刻也只有一个线程运行    │
└──────────────────────────────────────────────┘
```

**为什么需要 GIL**：
1. CPython 使用引用计数管理内存，引用计数的操作不是原子的
2. GIL 避免了多线程并发修改引用计数导致的内存错误
3. 简化了 CPython 解释器和 C 扩展的实现

**GIL 的释放时机**：
- I/O 操作（文件读写、网络请求）
- `time.sleep()`
- 执行 C 扩展代码（可以主动释放 GIL）
- 每执行一定数量的字节码指令后（Python 3.2+ 基于时间间隔，默认 5ms）

```python
# GIL 对不同任务的影响
# CPU密集型：多线程无法并行 → 用多进程
# I/O密集型：线程等待I/O时释放GIL → 多线程有效

import sys
print(sys.getswitchinterval())  # 0.005 (5ms)
```

## 2. threading模块的使用和线程同步机制？

**回答：**

```python
import threading

# 创建线程的两种方式
# 方式1：传入目标函数
def worker(name, count):
    for i in range(count):
        print(f"{name}: {i}")

t = threading.Thread(target=worker, args=("T1", 5), daemon=True)
t.start()
t.join(timeout=10)  # 等待线程结束

# 方式2：继承 Thread
class MyThread(threading.Thread):
    def run(self):
        print(f"线程 {self.name} 运行中")
```

**线程同步原语**：

```python
# 1. Lock - 互斥锁
lock = threading.Lock()
with lock:          # 自动 acquire/release
    shared_data += 1

# 2. RLock - 可重入锁（同一线程可多次获取）
rlock = threading.RLock()

# 3. Semaphore - 信号量（限制并发数）
sem = threading.Semaphore(5)  # 最多5个线程同时访问
with sem:
    access_resource()

# 4. Event - 事件通知
event = threading.Event()
event.wait()    # 阻塞等待
event.set()     # 唤醒所有等待线程
event.clear()   # 重置

# 5. Condition - 条件变量
cond = threading.Condition()
with cond:
    cond.wait()       # 等待通知
    cond.notify_all() # 通知所有等待线程

# 6. Barrier - 屏障（所有线程到齐后一起继续）
barrier = threading.Barrier(3)
barrier.wait()
```

## 3. multiprocessing模块如何实现真正的并行？

**回答：**

`multiprocessing` 通过创建**独立进程**绕过 GIL，每个进程有自己的 Python 解释器和内存空间。

```python
from multiprocessing import Process, Pool, Queue, Manager

# 创建进程
def cpu_work(n):
    return sum(i*i for i in range(n))

p = Process(target=cpu_work, args=(10**7,))
p.start()
p.join()

# 进程池
with Pool(processes=4) as pool:
    # map: 并行处理
    results = pool.map(cpu_work, [10**6]*8)

    # apply_async: 异步提交
    future = pool.apply_async(cpu_work, (10**7,))
    result = future.get(timeout=30)

    # imap: 惰性版 map
    for result in pool.imap(cpu_work, range(100)):
        print(result)
```

**进程间通信（IPC）**：

```python
# 1. Queue - 进程安全队列
from multiprocessing import Queue
q = Queue()
q.put("data")
data = q.get()

# 2. Pipe - 管道（两个进程间）
from multiprocessing import Pipe
parent_conn, child_conn = Pipe()
parent_conn.send("hello")
msg = child_conn.recv()

# 3. Manager - 共享状态（代理对象）
with Manager() as manager:
    shared_list = manager.list()
    shared_dict = manager.dict()
    # 多个进程可以安全操作这些对象

# 4. Value/Array - 共享内存
from multiprocessing import Value, Array
counter = Value('i', 0)       # 共享整数
arr = Array('d', [1.0, 2.0])  # 共享double数组
```

```
┌────── 多线程 vs 多进程 ──────┐
│                              │
│  多线程:                      │
│  ✅ 共享内存，通信方便          │
│  ✅ 创建开销小                 │
│  ❌ 受GIL限制                 │
│  → 适合I/O密集型              │
│                              │
│  多进程:                      │
│  ✅ 真正并行，利用多核          │
│  ❌ 内存开销大（独立内存空间）   │
│  ❌ IPC复杂                   │
│  → 适合CPU密集型              │
└──────────────────────────────┘
```

## 4. concurrent.futures的使用方法和优势？

**回答：**

`concurrent.futures` 提供统一的高层接口，封装了线程池和进程池。

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
from concurrent.futures import as_completed, wait

# ThreadPoolExecutor - I/O密集型
def fetch_url(url):
    import urllib.request
    with urllib.request.urlopen(url) as resp:
        return resp.read()

urls = ["http://example.com"] * 10

with ThreadPoolExecutor(max_workers=5) as executor:
    # 方式1：map（有序返回）
    results = executor.map(fetch_url, urls)
    for result in results:
        print(len(result))

    # 方式2：submit + as_completed（谁先完成先处理）
    futures = {executor.submit(fetch_url, url): url for url in urls}
    for future in as_completed(futures):
        url = futures[future]
        try:
            data = future.result(timeout=10)
            print(f"{url}: {len(data)} bytes")
        except Exception as e:
            print(f"{url} 失败: {e}")

# ProcessPoolExecutor - CPU密集型
def heavy_compute(n):
    return sum(i*i for i in range(n))

with ProcessPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(heavy_compute, [10**7]*8))
```

```
┌──────── Future 对象方法 ────────┐
│                                │
│ future.result(timeout=None)    │
│   → 获取结果（阻塞）            │
│                                │
│ future.done()                  │
│   → 是否完成                   │
│                                │
│ future.cancel()                │
│   → 尝试取消                   │
│                                │
│ future.add_done_callback(fn)   │
│   → 完成时回调                  │
│                                │
│ future.exception()             │
│   → 获取异常                    │
└────────────────────────────────┘
```

## 5. asyncio事件循环的工作原理？

**回答：**

`asyncio` 基于**事件循环（Event Loop）+ 协程（Coroutine）**实现单线程并发。

```
┌──────────── 事件循环原理 ────────────┐
│                                      │
│  事件循环 (Event Loop):               │
│  ┌──────────────────────────────┐    │
│  │  while True:                 │    │
│  │    就绪任务 = 检查I/O和定时器  │    │
│  │    for task in 就绪任务:      │    │
│  │        执行 task (直到await)  │    │
│  └──────────────────────────────┘    │
│                                      │
│  协程遇到 await 时:                   │
│  1. 挂起当前协程                      │
│  2. 注册I/O回调                      │
│  3. 执行其他就绪协程                  │
│  4. I/O完成后恢复被挂起的协程          │
└──────────────────────────────────────┘
```

```python
import asyncio

async def fetch_data(name, delay):
    print(f"{name}: 开始请求")
    await asyncio.sleep(delay)  # 模拟I/O，不阻塞事件循环
    print(f"{name}: 请求完成")
    return f"{name}_data"

async def main():
    # 并发执行多个协程
    results = await asyncio.gather(
        fetch_data("A", 2),
        fetch_data("B", 1),
        fetch_data("C", 3),
    )
    # 总耗时约3秒（非2+1+3=6秒）
    print(results)

asyncio.run(main())

# TaskGroup（Python 3.11+，更安全的并发）
async def main_v2():
    async with asyncio.TaskGroup() as tg:
        task1 = tg.create_task(fetch_data("A", 2))
        task2 = tg.create_task(fetch_data("B", 1))
    # 任一任务异常会取消其他任务
    print(task1.result(), task2.result())
```

**重要原则**：`await` 只能在 `async def` 中使用，不要在协程中调用阻塞函数（如 `time.sleep`、同步 I/O），否则会阻塞整个事件循环。

## 6. async/await的语法和使用规范？

**回答：**

```python
import asyncio

# async def 定义协程函数
async def async_function():
    return 42

# await 等待协程完成
async def main():
    result = await async_function()

# 异步上下文管理器
class AsyncDB:
    async def __aenter__(self):
        self.conn = await connect_db()
        return self.conn

    async def __aexit__(self, *args):
        await self.conn.close()

async def use_db():
    async with AsyncDB() as conn:
        await conn.execute("SELECT 1")

# 异步迭代器
class AsyncRange:
    def __init__(self, n):
        self.n = n
        self.i = 0

    def __aiter__(self):
        return self

    async def __anext__(self):
        if self.i >= self.n:
            raise StopAsyncIteration
        await asyncio.sleep(0.1)
        self.i += 1
        return self.i

async def iterate():
    async for i in AsyncRange(5):
        print(i)

# 异步生成器
async def async_gen():
    for i in range(5):
        await asyncio.sleep(0.1)
        yield i

async def consume():
    async for value in async_gen():
        print(value)

# 异步推导式
async def get_all():
    return [i async for i in async_gen()]
```

```
┌──────── 异步编程要点 ────────┐
│                              │
│ ✅ I/O操作用 await            │
│ ✅ 并发用 gather/TaskGroup    │
│ ✅ 用 asyncio.run() 启动     │
│                              │
│ ❌ 不要在协程中调用阻塞函数    │
│ ❌ 不要在协程外用 await       │
│ ❌ 不要手动管理事件循环        │
│                              │
│ 阻塞操作→用run_in_executor:  │
│ await loop.run_in_executor(  │
│     None, blocking_func      │
│ )                            │
└──────────────────────────────┘
```

## 7. 异步编程中的常见陷阱和最佳实践？

**回答：**

```python
import asyncio

# 陷阱1：在协程中调用阻塞函数
async def bad_example():
    import time
    time.sleep(5)  # ❌ 阻塞整个事件循环！

async def good_example():
    await asyncio.sleep(5)  # ✅ 非阻塞

# 如果必须调用同步阻塞函数
async def with_blocking():
    loop = asyncio.get_running_loop()
    result = await loop.run_in_executor(None, blocking_function)

# 陷阱2：忘记 await
async def oops():
    result = async_function()  # ❌ 没有 await，result 是协程对象
    result = await async_function()  # ✅

# 陷阱3：创建了Task但不等待
async def fire_and_forget():
    task = asyncio.create_task(some_coroutine())
    # 如果不 await task，异常会被静默吞掉
    # 且可能被GC回收
    await task  # ✅ 确保等待完成

# 最佳实践：异常处理
async def safe_fetch(url):
    try:
        async with aiohttp.ClientSession() as session:
            async with session.get(url) as resp:
                return await resp.json()
    except aiohttp.ClientError as e:
        print(f"请求失败: {e}")
        return None

# 最佳实践：超时控制
async def with_timeout():
    try:
        result = await asyncio.wait_for(
            slow_operation(),
            timeout=10.0
        )
    except asyncio.TimeoutError:
        print("操作超时")

# 最佳实践：信号量限制并发
async def limited_concurrency():
    sem = asyncio.Semaphore(10)  # 最多10个并发

    async def limited_fetch(url):
        async with sem:
            return await fetch(url)

    tasks = [limited_fetch(url) for url in urls]
    return await asyncio.gather(*tasks)
```

## 8. 线程安全和进程安全的区别？如何确保数据安全？

```python
import threading
from queue import Queue

    global counter
    for _ in range(100000):
        counter += 1  # ❌ 非原子操作，count读取-修改-写入

threads = [threading.Thread(target=increment) for _ in range(10)]

lock = threading.Lock()

# 解决方案2：线程安全的数据结构
q = Queue()

# 解决方案3：threading.local（线程局部变量）

# 解决方案4：原子操作（某些操作在GIL下天然安全）
```

## 10. 并发与异步编程面试速答？

**回答：**

```
Q: GIL能被移除吗？
A: Python 3.13引入了实验性的free-threaded模式(--disable-gil)。Jython和IronPython没有GIL。

Q: 协程和线程的区别？
A: 协程是用户态调度、协作式切换、单线程内执行，开销极小。线程是OS调度、抢占式切换、有上下文切换成本。

Q: asyncio.gather和asyncio.wait的区别？
A: gather返回有序结果列表，wait返回(done, pending)两个集合，支持FIRST_COMPLETED等策略。

Q: 什么是线程池？为什么要用？
A: 预创建一组线程复用，避免频繁创建销毁线程的开销，并限制并发数。

Q: daemon线程是什么？
A: 守护线程在所有非守护线程结束后自动终止，适合后台任务。主线程结束时守护线程被杀掉。

Q: 如何在异步代码中调用同步函数？
A: 使用loop.run_in_executor()将阻塞操作放到线程池中执行，返回awaitable。

Q: asyncio.create_task和await的区别？
A: create_task立即调度协程在后台运行并返回Task对象，await等待协程完成。

Q: 什么是事件循环？
A: 一个循环检查I/O事件和定时器，调度就绪协程执行。是asyncio的核心调度器。

Q: multiprocessing的start方法有哪些？
A: fork(Linux默认，快但不安全)、spawn(Windows默认，安全但慢)、forkserver(折中方案)。

Q: 如何调试异步代码？
A: asyncio.run(debug=True)开启调试模式，会检测未await的协程和慢回调。
```
