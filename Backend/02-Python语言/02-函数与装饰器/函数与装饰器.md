## 1. Python函数参数的类型和传递顺序是什么？

**回答：**

Python 函数参数分为五种类型，定义时必须按以下顺序排列：

```
┌──────────────── 参数定义顺序 ────────────────┐
│                                              │
│  def func(                                   │
│      a, b,           # 1. 位置参数            │
│      c=10,           # 2. 默认值参数          │
│      *args,          # 3. 可变位置参数         │
│      key,            # 4. keyword-only参数    │
│      **kwargs         # 5. 可变关键字参数      │
│  ):                                          │
│                                              │
│  调用: func(1, 2, 3, 4, 5, key='x', z=99)   │
│  a=1, b=2, c=3, args=(4,5), key='x'         │
│  kwargs={'z': 99}                            │
└──────────────────────────────────────────────┘
```

```python
# 仅位置参数 (3.8+, PEP 570)
def func(a, b, /, c, d):
    pass
# a, b 只能按位置传递，不能用关键字

# 仅关键字参数
def func(*, key1, key2):
    pass
# key1, key2 只能用关键字传递

# 完整示例
def func(pos_only, /, normal, *, kw_only):
    pass
```

**参数传递机制**：Python 使用**传对象引用（call by object reference）**：
- 传递的是对象的引用（指针的副本）
- 可变对象在函数内修改会影响外部
- 不可变对象在函数内"修改"会创建新对象，不影响外部

```python
def modify(lst, num):
    lst.append(4)   # 修改了外部的list
    num += 1         # 创建了新的int对象，不影响外部

my_list = [1, 2, 3]
my_num = 10
modify(my_list, my_num)
print(my_list)  # [1, 2, 3, 4]  ← 被修改
print(my_num)   # 10            ← 未被修改
```

## 2. 什么是闭包(closure)？闭包的作用和使用场景？

**回答：**

闭包是指一个内部函数**引用了外部函数的变量**，并且外部函数已经返回。闭包使得外部函数的局部变量在函数结束后仍然存活。

```python
def make_counter(initial=0):
    count = initial

    def counter():
        nonlocal count   # 声明使用外层作用域变量
        count += 1
        return count

    return counter

c = make_counter(10)
print(c())  # 11
print(c())  # 12

# 查看闭包变量
print(c.__closure__[0].cell_contents)  # 12
```

**闭包的三个条件**：
1. 存在嵌套函数
2. 内部函数引用了外部函数的变量
3. 外部函数返回内部函数

```python
# 经典陷阱：循环中的闭包
funcs = []
for i in range(5):
    funcs.append(lambda: i)  # 所有lambda都引用同一个i

print([f() for f in funcs])  # [4, 4, 4, 4, 4] ← 全是4！

# 解决方案1：默认参数绑定
funcs = [lambda i=i: i for i in range(5)]
print([f() for f in funcs])  # [0, 1, 2, 3, 4]

# 解决方案2：使用闭包工厂
def make_func(i):
    return lambda: i
funcs = [make_func(i) for i in range(5)]
```

**应用场景**：装饰器、回调函数、延迟计算、数据隐藏。

## 3. 装饰器的本质和实现原理是什么？

**回答：**

装饰器是一个接受函数作为参数并返回新函数的可调用对象。`@decorator` 是 `func = decorator(func)` 的语法糖。

```python
import functools

# 基本装饰器
def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        import time
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__} 耗时 {elapsed:.4f}秒")
        return result
    return wrapper

@timer
def slow_func():
    import time
    time.sleep(1)

# 等价于：slow_func = timer(slow_func)
```

```
┌──────── 装饰器调用链 ────────┐
│                              │
│  @decorator                  │
│  def func(): ...             │
│                              │
│  执行过程：                   │
│  1. 定义 func                │
│  2. 调用 decorator(func)     │
│  3. 返回 wrapper             │
│  4. func = wrapper           │
│  5. 调用 func() 实际执行      │
│     wrapper() → func()       │
└──────────────────────────────┘
```

**`functools.wraps` 的重要性**：保留原函数的 `__name__`、`__doc__`、`__module__` 等元信息，不加的话调试和文档生成会出问题。

## 4. 如何实现带参数的装饰器和类装饰器？

**回答：**

```python
import functools

# 带参数的装饰器 = 三层嵌套
def retry(max_retries=3, exceptions=(Exception,)):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_retries):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_retries - 1:
                        raise
                    print(f"重试 {attempt + 1}/{max_retries}")
        return wrapper
    return decorator

@retry(max_retries=5, exceptions=(ConnectionError,))
def fetch_data(url):
    ...


# 类装饰器：实现 __call__ 方法
class CacheDecorator:
    def __init__(self, func):
        self.func = func
        self.cache = {}
        functools.update_wrapper(self, func)

    def __call__(self, *args):
        if args not in self.cache:
            self.cache[args] = self.func(*args)
        return self.cache[args]

@CacheDecorator
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)


# 用类装饰类（类装饰器装饰类）
def singleton(cls):
    instances = {}
    @functools.wraps(cls)
    def get_instance(*args, **kwargs):
        if cls not in instances:
            instances[cls] = cls(*args, **kwargs)
        return instances[cls]
    return get_instance

@singleton
class Database:
    pass
```

## 5. 多个装饰器的执行顺序是怎样的？

**回答：**

多个装饰器**从下往上**包装（装饰），**从上往下**执行（调用）。

```python
def decorator_a(func):
    print("装饰器A: 装饰")
    def wrapper(*args, **kwargs):
        print("装饰器A: 执行前")
        result = func(*args, **kwargs)
        print("装饰器A: 执行后")
        return result
    return wrapper

def decorator_b(func):
    print("装饰器B: 装饰")
    def wrapper(*args, **kwargs):
        print("装饰器B: 执行前")
        result = func(*args, **kwargs)
        print("装饰器B: 执行后")
        return result
    return wrapper

@decorator_a
@decorator_b
def greet():
    print("Hello!")

# 装饰阶段输出（从下往上）：
# 装饰器B: 装饰
# 装饰器A: 装饰

greet()
# 调用阶段输出（从上往下，像洋葱模型）：
# 装饰器A: 执行前
# 装饰器B: 执行前
# Hello!
# 装饰器B: 执行后
# 装饰器A: 执行后
```

```
┌──────────── 洋葱模型 ────────────┐
│                                  │
│  @decorator_a   ←─ 装饰顺序      │
│  @decorator_b                    │
│  def func()                      │
│                                  │
│  等价于: decorator_a(decorator_b(func))  │
│                                  │
│  执行: A前 → B前 → func → B后 → A后     │
└──────────────────────────────────┘
```

## 6. lambda表达式的用法和限制？

**回答：**

`lambda` 是创建匿名函数的表达式，只能包含**单个表达式**，不能有语句。

```python
# 基本用法
square = lambda x: x ** 2
add = lambda a, b: a + b

# 排序中使用
students = [('Alice', 90), ('Bob', 85), ('Charlie', 92)]
students.sort(key=lambda s: s[1], reverse=True)

# 与 map/filter/reduce 搭配
nums = [1, 2, 3, 4, 5]
squares = list(map(lambda x: x**2, nums))
evens = list(filter(lambda x: x % 2 == 0, nums))

from functools import reduce
total = reduce(lambda a, b: a + b, nums)

# 条件表达式
classify = lambda x: "正" if x > 0 else ("零" if x == 0 else "负")
```

**lambda 的限制**：
- 只能写一个表达式，不能用赋值、循环、异常处理等语句
- 不支持类型注解
- 没有名字，调试困难
- 复杂逻辑应使用 `def` 定义具名函数

**PEP 8 建议**：不要把 lambda 赋值给变量（`f = lambda x: x`），应直接用 `def`。

## 7. 高阶函数 map、filter、reduce 的用法？

**回答：**

高阶函数是接受函数作为参数或返回函数的函数。

```python
# map: 对每个元素应用函数
nums = [1, 2, 3, 4, 5]
squares = list(map(lambda x: x**2, nums))       # [1, 4, 9, 16, 25]
strs = list(map(str, nums))                       # ['1','2','3','4','5']

# 多个可迭代对象
sums = list(map(lambda a, b: a + b, [1,2,3], [4,5,6]))  # [5,7,9]

# filter: 过滤元素
evens = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]
# 去除None和空值
clean = list(filter(None, [0, 1, '', 'a', None]))  # [1, 'a']

# reduce: 累积计算
from functools import reduce
product = reduce(lambda a, b: a * b, nums)         # 120
# 带初始值
total = reduce(lambda a, b: a + b, nums, 100)      # 115
```

**Pythonic 的替代方案**：

```python
# 优先使用推导式代替 map/filter
squares = [x**2 for x in nums]                  # 代替 map
evens = [x for x in nums if x % 2 == 0]         # 代替 filter

# 使用内置函数代替 reduce
sum(nums)          # 代替 reduce 求和
max(nums)          # 代替 reduce 求最大
all(conditions)    # 代替 reduce 全部满足
any(conditions)    # 代替 reduce 任意满足
```

## 8. functools模块中有哪些常用工具？

**回答：**

```python
import functools

# 1. lru_cache - 最近最少使用缓存
@functools.lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

fibonacci.cache_info()   # 查看缓存命中率
fibonacci.cache_clear()  # 清空缓存

# Python 3.9+ 更简洁
@functools.cache  # 无限缓存
def factorial(n):
    return n * factorial(n-1) if n else 1

# 2. partial - 偏函数（固定部分参数）
from functools import partial
int_from_binary = partial(int, base=2)
int_from_binary('1010')  # 10

# 3. wraps - 保留被装饰函数的元信息
# （见装饰器部分）

# 4. total_ordering - 自动补全比较方法
@functools.total_ordering
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade
    def __eq__(self, other):
        return self.grade == other.grade
    def __lt__(self, other):
        return self.grade < other.grade
    # 自动生成 __le__, __gt__, __ge__

# 5. singledispatch - 单分派泛型函数
@functools.singledispatch
def process(data):
    raise NotImplementedError

@process.register(str)
def _(data):
    return data.upper()

@process.register(list)
def _(data):
    return [x * 2 for x in data]
```

## 9. Python中的作用域规则(LEGB)是什么？

**回答：**

Python 变量查找遵循 **LEGB 规则**，按以下顺序查找：

```
┌──────────────── LEGB 作用域 ────────────────┐
│                                             │
│  ┌─── B (Built-in) 内置作用域 ──────────┐   │
│  │  print, len, int, True, None...       │   │
│  │  ┌─── G (Global) 全局/模块作用域 ──┐  │   │
│  │  │  模块级变量和函数                  │  │   │
│  │  │  ┌─── E (Enclosing) 闭包作用域 ┐│  │   │
│  │  │  │  外层函数的局部变量           ││  │   │
│  │  │  │  ┌─── L (Local) 局部作用域 ┐││  │   │
│  │  │  │  │  当前函数的局部变量       │││  │   │
│  │  │  │  └─────────────────────────┘││  │   │
│  │  │  └─────────────────────────────┘│  │   │
│  │  └─────────────────────────────────┘  │   │
│  └───────────────────────────────────────┘   │
│  查找顺序: L → E → G → B                     │
└─────────────────────────────────────────────┘
```

```python
x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)      # "local"  ← L

    inner()
    print(x)          # "enclosing"  ← E

outer()
print(x)              # "global"  ← G

# 修改外部作用域变量
def outer():
    count = 0
    def inner():
        nonlocal count    # 声明使用 Enclosing 作用域
        count += 1
    inner()
    print(count)  # 1

# global 声明
y = 10
def func():
    global y             # 声明使用 Global 作用域
    y = 20
func()
print(y)  # 20
```

**注意**：`global` 和 `nonlocal` 不创建变量，只是声明引用的作用域。避免滥用 `global`，会破坏封装性。

## 10. 函数与装饰器面试速答？

**回答：**

```
Q: Python中函数是一等公民是什么意思？
A: 函数可以赋值给变量、作为参数传递、作为返回值、存储在数据结构中。

Q: 可变默认参数有什么陷阱？
A: def f(lst=[]): 默认列表在多次调用间共享。应改为 def f(lst=None): lst = lst or []。

Q: 装饰器和闭包的关系？
A: 装饰器通常通过闭包实现，内部wrapper函数引用了外部参数func形成闭包。

Q: @staticmethod和@classmethod的区别？
A: staticmethod不接收隐式参数，classmethod第一个参数是cls（类本身）。

Q: yield和return的区别？
A: return终止函数并返回值，yield暂停函数并产出值，下次调用从yield处恢复。

Q: 什么是偏函数(partial)？
A: functools.partial固定函数的部分参数，生成新函数。如 int2 = partial(int, base=2)。

Q: 为什么要用functools.wraps？
A: 保留被装饰函数的__name__、__doc__等元信息，否则调试和文档生成会出问题。

Q: 递归的深度限制是多少？
A: 默认1000层(sys.getrecursionlimit())，可用sys.setrecursionlimit()修改，但建议改用迭代。

Q: Python中的尾递归优化？
A: CPython不支持尾递归优化(TCO)，深度递归会栈溢出。应改用迭代或使用蹦床函数。

Q: 什么是回调函数？
A: 作为参数传递给另一个函数、在特定事件发生时被调用的函数。常用于异步和事件驱动编程。
```
